## Failed Scenarios

### defi-08: Kamino Lending (USDC Supply)
- **Failed Step**: Step 3 (Kamino USDC Supply dryRun)
- **1차**: RPC 429 (rate limit) → 5초 대기 후 재시도
- **2차**: `Missing accounts for Solana contract call`
- **Root Cause**: KaminoSdkWrapper.convertInstructions() (kamino-sdk-wrapper.ts:326)에서 SDK가 반환한 instruction의 `keys` 배열이 비어있음 → SolanaAdapter.buildContractCall() (adapter.ts:686) accounts 빈 배열 검증 실패
- **이전 대비**: #413 동일 에러, 미수정

### defi-09: Pendle Yield Trading (PT)
- **Failed Step**: Step 3 (PT buy dryRun)
- **Error**: `Expected array, received object` (Zod union 검증)
- **Root Cause**: Pendle API v2 응답 형식이 PendleConvertResponseSchema (schemas.ts:60-74)와 불일치. 스키마는 object|array 2가지만 수용하나 실제 API가 다른 wrapper 형식 반환
- **이전 대비**: #373→#398→#403→#407→#414 → **6회차 동일 패턴**
- **권장**: 실제 Pendle API 응답을 raw 캡처 후 스키마 재정의 필요

### defi-10: Drift Perpetual Trading
- **Failed Step**: Step 3 (Drift USDC deposit dryRun)
- **Error**: `429 Too Many Requests` (3회 재시도: 3초, 10초, 30초 간격)
- **Root Cause**: DriftSdkWrapper.getClient().subscribe()가 다수 RPC 호출 → 공용 Solana RPC rate limit 초과. SDK에 재시도/backoff 없고 RpcPool 미연동
- **이전 대비**: #415 동일 에러, 미수정

### defi-14: DCent 2-hop Auto-Routing (ETH→LINK)
- **dryRun**: PASS — DCent API가 2-hop 경로(ETH→WETH→LINK) 반환, 트랜잭션 빌드 성공
- **시뮬레이션**: FAIL — 잔액 부족 (need 0.236 ETH, have 0.015 ETH). DCent 프로토콜 수수료 포함 value
- **get_quotes**: `{"id":"...","status":"success"}` — quotes 데이터 누락, 빈 응답
- **이전 대비**: #409 부분 해결 — dryRun은 이제 성공, get_quotes 응답 형식 개선 필요

### defi-15: DCent 크로스체인 (EVM→Solana)
- **Failed Step**: Step 2 (crosschain dryRun)
- **Error**: `DCent API returned empty txdata`
- **시도**: 0.002 ETH → empty txdata, 0.005 ETH → empty txdata
- **Root Cause**: DCent API가 EVM→Solana 크로스체인 경로를 지원하지 않거나, CAIP-19 Solana asset ID 매핑 미지원
- **권장**: DCent API가 실제 크로스체인을 지원하는지 문서 확인 필요. 미지원 시 시나리오 폐기

### defi-16: DCent Solana 스왑 (SOL→USDC)
- **Failed Step**: Step 2 (DCent Solana dryRun)
- **Error**: `D'CENT Swap only supports EVM (ethereum) chains. Got: solana`
- **Root Cause**: dcent-swap/index.ts:150-155에 EVM 전용 체인 가드 존재. DCent Swap 프로바이더가 Solana 체인을 명시적으로 거부
- **이전 대비**: #410(txdata.from/to 스키마 회귀)과 다른 에러 — 체인 가드가 추가됨
- **Issue**: #416 (신규) DCent Swap Solana 지원 미구현
- **권장**: DCent API가 Solana 스왑을 지원하는지 확인 후, 미지원 시 defi-16 시나리오 폐기
