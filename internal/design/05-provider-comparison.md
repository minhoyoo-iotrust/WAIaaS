> 따라서 이 문서의 목적은 프로바이더 선택이 아닌:
> - 업계 표준 기능 벤치마킹
> - 직접 구축 시 참고할 기능 목록 확인
> - 직접 구축 vs 프로바이더 비용/기능 비교

## 1. 개요

### 1.1 비교 목적

본 문서는 주요 WaaS(Wallet as a Service) 프로바이더의 기능을 비교 분석한다. 분석 목적은 다음과 같다:

| 목적 | 설명 |
|------|------|
| **벤치마킹** | 업계 표준 기능 및 보안 모델 파악 |
| **기능 참조** | 직접 구축 시 필수 기능 목록 도출 |
| **비용 비교** | 직접 구축 vs 프로바이더 비용 분석 |
| **차별화 포인트** | WAIaaS만의 차별화 요소 발굴 |

### 1.2 비교 대상 프로바이더

| 프로바이더 | 본사 | 설립 | 주요 고객 |
|------------|------|------|----------|
| **Turnkey** | 미국 | 2022 | Coinbase, Stripe |
| **Dfns** | 프랑스 | 2020 | Deutsche Bank, Fidelity |
| **Crossmint** | 미국 | 2021 | Mastercard, Adidas |
| **Privy** | 미국 | 2022 | Uniswap, Zora |
| **Dynamic** | 미국 | 2021 | Dapper, ENS |
| **Capsule** | 미국 | 2023 | 스타트업 중심 |

## 2. 프로바이더 개별 분석

### 2.1 Turnkey

#### 2.1.1 개요

Turnkey는 AWS Nitro Enclaves 기반 TEE(Trusted Execution Environment)를 핵심으로 하는 WaaS 프로바이더다. Coinbase, Stripe 등 대형 고객사를 보유하고 있으며, 기관급 보안을 강조한다.

#### 2.1.2 보안 모델

```
┌─────────────────────────────────────────────────────────┐
│                  Turnkey 보안 아키텍처                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │           AWS Nitro Enclaves (TEE)              │   │
│  │  ┌─────────────────────────────────────────┐    │   │
│  │  │      격리된 키 생성 및 서명 환경        │    │   │
│  │  │  - 호스트 접근 차단                     │    │   │
│  │  │  - 메모리 암호화                        │    │   │
│  │  │  - Attestation 검증                     │    │   │
│  │  └─────────────────────────────────────────┘    │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  추가 보안 기능:                                        │
│  - 정책 엔진 (Policy Engine)                           │
│  - 웹훅 기반 승인 플로우                               │
│  - 다중 서명 지원                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 2.1.3 Solana 지원

| 기능 | 지원 여부 | 상세 |
|------|----------|------|
| ED25519 서명 | O | 네이티브 지원 |
| 트랜잭션 서명 | O | VersionedTransaction 포함 |
| 메시지 서명 | O | 임의 메시지 서명 가능 |
| SPL 토큰 | O | 전체 지원 |

#### 2.1.4 정책 엔진

```typescript
// Turnkey 정책 예시
{
  "policyId": "spend-limit-policy",
  "effect": "EFFECT_ALLOW",
  "consensus": "1-of-1",
  "condition": {
    "activityType": "ACTIVITY_TYPE_SIGN_RAW_PAYLOAD",
    "spendLimit": {
      "amount": "1000000000", // 1 SOL in lamports
      "period": "PERIOD_DAY"
    }
  }
}
```

#### 2.1.5 가격 모델

| 플랜 | 월 비용 | 포함 사항 |
|------|---------|----------|
| **Starter** | 무료 | 1,000 지갑, 10,000 서명/월 |
| **Growth** | $599/월 | 50,000 지갑, 무제한 서명 |
| **Enterprise** | 협의 | 무제한, 전용 지원 |

#### 2.1.6 장점 및 단점

| 장점 | 단점 |
|------|------|
| TEE 기반 최고 수준 보안 | 비용이 상대적으로 높음 |
| 정책 엔진 강력 | 벤더 락인 위험 |
| Coinbase 검증 | 자체 커스터마이징 제한 |
| 우수한 문서화 | 특정 리전 제한 |

### 2.2 Dfns

#### 2.2.1 개요

Dfns는 MPC(Multi-Party Computation) 기반 WaaS로, CGGMP21 프로토콜을 채택하여 최신 보안 표준을 따른다. Deutsche Bank, Fidelity 등 전통 금융기관 고객을 보유한다.

#### 2.2.2 보안 모델

```
┌─────────────────────────────────────────────────────────┐
│                   Dfns MPC 아키텍처                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────┐   ┌─────────┐   ┌─────────┐             │
│   │ Dfns    │   │ Client  │   │ AWS/GCP │             │
│   │ Server  │   │ Device  │   │ KMS     │             │
│   │ Share   │   │ Share   │   │ Share   │             │
│   └────┬────┘   └────┬────┘   └────┬────┘             │
│        │             │             │                   │
│        └──────┬──────┴──────┬──────┘                   │
│               │             │                          │
│               ▼             ▼                          │
│        ┌─────────────────────────┐                     │
│        │   CGGMP21 프로토콜      │                     │
│        │   (4라운드 서명)        │                     │
│        └─────────────────────────┘                     │
│                                                         │
│  특징:                                                  │
│  - 2-of-3 threshold 기본                               │
│  - 키가 단일 위치에 존재하지 않음                       │
│  - Kudelski Security 감사 완료                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 2.2.3 Solana 지원

| 기능 | 지원 여부 | 상세 |
|------|----------|------|
| ED25519 서명 | O | MPC로 분산 서명 |
| 트랜잭션 서명 | O | 모든 타입 지원 |
| 다중 체인 | O | 50+ 체인 지원 |
| 프로그래머블 | O | 커스텀 정책 API |

#### 2.2.4 가격 모델

| 플랜 | 월 비용 | 특징 |
|------|---------|------|
| **Standard** | $999/월~ | 기본 MPC 기능 |
| **Enterprise** | 협의 | 전용 인프라, SLA |

> **주의**: Dfns는 프리미엄 포지셔닝으로 비용이 높은 편이다.

#### 2.2.5 장점 및 단점

| 장점 | 단점 |
|------|------|
| CGGMP21 최신 MPC | 높은 비용 |
| 금융기관 검증 | 서명 지연 (2-5초) |
| 다중 체인 지원 | 실시간 거래에 부적합 |
| 자체 호스팅 옵션 | 복잡한 통합 |

### 2.3 Crossmint

#### 2.3.1 개요

Crossmint은 AI 에이전트 지갑에 특화된 기능을 제공하는 프로바이더로, **Agent Wallets** 제품을 통해 AI 에이전트의 자율적 거래를 지원한다.

#### 2.3.2 보안 모델

```
┌─────────────────────────────────────────────────────────┐
│              Crossmint Dual Key 아키텍처                 │
├─────────────────────────────────────────────────────────┤
│                                                         │
│    ┌───────────────────┐    ┌───────────────────┐      │
│    │     Owner Key     │    │     Agent Key     │      │
│    │   (소유자 통제)   │    │   (에이전트 운영) │      │
│    └─────────┬─────────┘    └─────────┬─────────┘      │
│              │                        │                 │
│              └──────────┬─────────────┘                 │
│                         │                               │
│              ┌──────────▼──────────┐                    │
│              │   Smart Wallet      │                    │
│              │   (스마트 컨트랙트)  │                    │
│              │                      │                    │
│              │  - 금액 한도         │                    │
│              │  - 화이트리스트      │                    │
│              │  - 시간 제어         │                    │
│              └─────────────────────┘                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 2.3.3 AI 에이전트 특화 기능

| 기능 | 설명 |
|------|------|
| **Dual Key Architecture** | Owner와 Agent 키 분리 |
| **Spending Limits** | 트랜잭션/일/주 단위 한도 |
| **Whitelist** | 허용된 주소/프로그램만 거래 |
| **Time Controls** | 시간대별 거래 허용/차단 |
| **Escalation** | 고액 거래 Owner 승인 요구 |

#### 2.3.4 Solana 지원

| 기능 | 지원 여부 | 상세 |
|------|----------|------|
| ED25519 서명 | O | 네이티브 지원 |
| Smart Wallet | O | Solana 프로그램 기반 |
| SPL 토큰 | O | 전체 지원 |
| NFT | O | Metaplex 표준 지원 |

#### 2.3.5 가격 모델

| 플랜 | 비용 | 특징 |
|------|------|------|
| **Free** | 무료 | 100 API 호출/일 |
| **Developer** | $0.01/호출 | 종량제 |
| **Pro** | $299/월 | 50,000 호출 포함 |
| **Enterprise** | 협의 | 무제한, 전용 지원 |

#### 2.3.6 장점 및 단점

| 장점 | 단점 |
|------|------|
| AI 에이전트 특화 | Smart Wallet 의존 |

### 2.4 Privy

#### 2.4.1 개요

Privy는 Shamir's Secret Sharing(SSS)과 Enclaves를 결합한 하이브리드 보안 모델을 제공한다. 주로 임베디드 지갑 용도로 Uniswap, Zora 등에서 사용된다.

#### 2.4.2 보안 모델

```
┌─────────────────────────────────────────────────────────┐
│                Privy 보안 아키텍처                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  키 분산 (Shamir's Secret Sharing):                     │
│                                                         │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐              │
│  │ Device  │   │ Privy   │   │ Recovery│              │
│  │ Share   │   │ Enclave │   │ Share   │              │
│  │  (1)    │   │  (2)    │   │  (3)    │              │
│  └─────────┘   └─────────┘   └─────────┘              │
│                                                         │
│  복구: 2-of-3 threshold                                │
│  - 디바이스 + Privy: 일반 사용                         │
│  - Privy + Recovery: 디바이스 분실 복구                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 2.4.3 Solana 지원

| 기능 | 지원 여부 | 상세 |
|------|----------|------|
| ED25519 서명 | O | 지원 |
| 임베디드 지갑 | O | 주력 기능 |
| 소셜 로그인 | O | Google, Twitter, Discord |
| 이메일 로그인 | O | OTP 기반 |

#### 2.4.4 가격 모델

| 플랜 | 비용 | MAU (월간 활성 사용자) |
|------|------|------------------------|
| **Free** | 무료 | 100 MAU |
| **Starter** | $99/월 | 500 MAU |
| **Growth** | $299/월 | 2,000 MAU |
| **Enterprise** | 협의 | 무제한 |

#### 2.4.5 장점 및 단점

| 장점 | 단점 |
|------|------|
| 소셜 로그인 통합 | MAU 기반 비용 증가 |
| 임베디드 지갑 | AI 에이전트 특화 기능 부족 |
| 우수한 UX | B2B API 제한적 |
| 빠른 통합 | 정책 엔진 없음 |

│                                                         │
│  ┌─────────┐   ┌─────────┐                             │
│  │ Dynamic │   │ Client  │                             │
│  │ Server  │   │ Device  │                             │
│  │ Share   │   │ Share   │                             │
│  └────┬────┘   └────┬────┘                             │
│       │             │                                   │
│       └──────┬──────┘                                   │
│              │                                          │

│       │   서명      │                                   │

│                                                         │

### 2.6 Capsule

#### 2.6.1 개요

Capsule은 비교적 신규 프로바이더로, MPC 기반 보안과 소셜 로그인 통합을 제공한다. 스타트업 중심으로 사용된다.

#### 2.6.2 보안 모델

- MPC 기반 키 분산
- 2-of-2 threshold
- 소셜 로그인 복구

| 플랜 | 비용 | 특징 |
|------|------|------|
| **Free** | 무료 | 기본 기능 |
| **Pro** | $199/월 | 확장 기능 |
| **Enterprise** | 협의 | 전용 지원 |

#### 2.6.4 장점 및 단점

| 장점 | 단점 |
|------|------|
| 저렴한 가격 | 검증 부족 |
| 간단한 통합 | 기능 제한적 |

## 5. 직접 구축 권장 근거

### 5.1 결정 배경

02-CONTEXT.md의 결정에 따라 WAIaaS는 직접 구축 방향을 선택했다. 이 결정의 근거는 다음과 같다:

### 5.2 직접 구축 권장 이유

```
┌─────────────────────────────────────────────────────────────┐
│                   직접 구축 권장 근거                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 보안 통제                                               │
│     - 키가 자사 인프라에만 존재                             │
│     - 제3자 접근 완전 차단                                  │
│     - 감사 로그 완전 소유                                   │
│                                                             │
│  2. 기술 차별화                                             │
│     - AI 에이전트 특화 기능 자유 구현                       │
│     - 경쟁 프로바이더와 동일 기술 아닌 자체 IP              │
│     - 성능 최적화 가능                                      │
│                                                             │
│  3. 장기 비용 효율                                          │
│     - 5년 기준 36% 비용 절감                                │
│     - 트래픽 증가에 따른 비용 선형 증가 방지                │
│     - 프로바이더 가격 인상 위험 없음                        │
│                                                             │
│  4. 데이터 주권                                             │
│     - 규제 변경에 자체 대응 가능                            │
│     - 고객 데이터 제3자 노출 없음                           │
│     - GDPR, SOC2 자체 인증 가능                             │
│                                                             │
│  5. 유연성                                                  │
│     - 새로운 체인 추가 자유                                 │
│     - API 설계 최적화                                       │
│     - 정책 엔진 완전 커스텀                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
