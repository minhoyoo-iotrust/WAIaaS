# #223 프리셋 지갑의 Approval Method 변경 시 경고 미표시

- **유형:** ENHANCEMENT
- **심각도:** LOW
- **마일스톤:** —
- **상태:** OPEN

## 증상

Admin UI에서 D'CENT 같은 빌트인 프리셋 지갑을 선택한 상태에서 Approval Method를 변경할 때 아무런 경고 없이 변경됨. 프리셋 기본값(`sdk_ntfy`)에서 다른 값으로 바꾸면 서명 요청이 정상 전달되지 않을 수 있으나 사용자가 이를 인지할 수 없음.

## 기대 동작

1. 프리셋 지갑 선택 시 `approvalMethod` 자동 세팅 (현재도 동작)
2. 사용자가 프리셋 기본값과 다른 Approval Method로 변경하려 할 때 **경고 표시**:
   - "D'CENT 프리셋 기본값은 Wallet App (ntfy)입니다. 변경 시 서명 요청이 정상적으로 전달되지 않을 수 있습니다."
   - 기본값 유지 / 변경 선택지 제공
3. 변경 자체는 차단하지 않음 — 경고 확인 후 변경 허용

## 수정 범위

- `packages/admin/src/pages/wallets.tsx` (Owner 설정 섹션)
- 빌트인 프리셋의 `approvalMethod`와 현재 선택값 비교 → 불일치 시 경고 UI 표시

## 테스트 항목

- [ ] 경고에서 "기본값 유지" 선택 시 원래 값으로 복원 확인
- [ ] 경고에서 "변경" 선택 시 변경 허용 확인
- [ ] 프리셋이 아닌 커스텀 지갑에서는 경고 미표시 확인
