# WAIaaS ERC-8004 Trustless Agents

On-chain agent identity, reputation, and validation via the ERC-8004 standard. Register AI agents on the Identity Registry (ERC-721 NFT), manage reputation through the Reputation Registry, and request third-party validation through the Validation Registry. All three registries are deployed on Ethereum mainnet.

> AI agents must NEVER request the master password. Use only your session token.

## Base URL / Authentication

```
http://localhost:3100
```

- **Read endpoints (GET):** Require `Authorization: Bearer <token>` (sessionAuth).
- **Write actions:** Executed via `POST /v1/actions/erc8004_agent/{action}` with sessionAuth. Subject to policy evaluation (6-stage pipeline).
- **Admin configuration:** See docs/admin-manual/erc8004-setup.md for provider and registry setup.

## Permissions

### Agent (sessionAuth)
- Query agent identity, reputation, validation status (4 GET endpoints)
- Execute write actions: register, set wallet, set URI, set metadata, give/revoke feedback, request validation

## 1. REST API Endpoints (Read-Only)

### GET /v1/erc8004/agent/{agentId} -- Agent Identity Info

Returns on-chain identity data from the Identity Registry plus local DB metadata.

```bash
curl -s http://localhost:3100/v1/erc8004/agent/42 \
  -H 'Authorization: Bearer wai_sess_eyJ...'
```

**Parameters:**

| Parameter | Type   | Required | Description                         |
| --------- | ------ | -------- | ----------------------------------- |
| `agentId` | string | Yes      | On-chain agent ID (uint256 string). |

**Response (200):**
```json
{
  "agentId": "42",
  "wallet": "0x1234567890abcdef1234567890abcdef12345678",
  "uri": "https://agent.example.com/v1/erc8004/registration-file/wallet-uuid",
  "metadata": {"status": "REGISTERED"},
  "registryAddress": "0x8004A169FB4a3325136EB29fA0ceB6D2e539a432",
  "chainId": 1
}
```

| Field             | Type   | Description                                    |
| ----------------- | ------ | ---------------------------------------------- |
| `agentId`         | string | On-chain agent ID.                             |
| `wallet`          | string | Linked wallet address (from getAgentWallet).   |
| `uri`             | string | Registration file URI (from tokenURI).         |
| `metadata`        | object | Local DB metadata (status, etc.).              |
| `registryAddress` | string | Identity Registry contract address.            |
| `chainId`         | number | Chain ID where the registry is deployed.       |

**Errors:** `CHAIN_ERROR` (RPC failure), `ADAPTER_NOT_AVAILABLE` (registry not configured).

### GET /v1/erc8004/agent/{agentId}/reputation -- Reputation Summary

Returns aggregated reputation score from the Reputation Registry, optionally filtered by tags.

```bash
curl -s 'http://localhost:3100/v1/erc8004/agent/42/reputation?tag1=reliability&tag2=' \
  -H 'Authorization: Bearer wai_sess_eyJ...'
```

**Parameters:**

| Parameter | Type   | Required | Description                              |
| --------- | ------ | -------- | ---------------------------------------- |
| `agentId` | string | Yes      | On-chain agent ID (path param).          |
| `tag1`    | string | No       | Category filter tag 1 (query param).     |
| `tag2`    | string | No       | Category filter tag 2 (query param).     |

**Response (200):**
```json
{
  "agentId": "42",
  "count": 15,
  "score": "7500",
  "decimals": 2,
  "tag1": "reliability",
  "tag2": ""
}
```

| Field      | Type   | Description                                                     |
| ---------- | ------ | --------------------------------------------------------------- |
| `agentId`  | string | On-chain agent ID.                                              |
| `count`    | number | Total number of feedback entries matching the tag filter.       |
| `score`    | string | Aggregated reputation score (numeric string, scaled by decimals).|
| `decimals` | number | Decimal precision for interpreting the score.                   |
| `tag1`     | string | Applied tag1 filter.                                            |
| `tag2`     | string | Applied tag2 filter.                                            |

**Errors:** `CHAIN_ERROR` (RPC failure), `ADAPTER_NOT_AVAILABLE` (registry not configured).

  "agentId": "42",
  "response": 1,

| Field          | Type   | Description                                          |

## 2. Write Actions (Action Provider)

All write actions are executed via the Action Provider framework. Use `POST /v1/actions/erc8004_agent/{action}` with sessionAuth. Each action resolves to a CONTRACT_CALL transaction that goes through the 6-stage pipeline with full policy evaluation.

**Note:** The provider is enabled by default since v30.11. To disable: set `actions.erc8004_agent_enabled=false` via Admin UI > Agent Identity.

### a. register_agent -- Register Agent Identity

Mints an ERC-721 identity NFT on the Identity Registry. Auto-generates a registration file and sets the tokenURI.

| Field         | Type     | Required | Description                                    |
| ------------- | -------- | -------- | ---------------------------------------------- |
| `name`        | string   | Yes      | Agent display name (1-100 chars).              |
| `description` | string   | No       | Agent description (max 500 chars).             |
| `services`    | array    | No       | Service endpoints: `{name, endpoint, version?}`.|
| `metadata`    | object   | No       | Key-value metadata (string -> string).         |

**Risk:** high | **Default tier:** APPROVAL

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/register_agent \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{
    "params": {
      "name": "My AI Agent",
      "description": "Autonomous trading agent",
      "services": [{"name": "REST API", "endpoint": "https://agent.example.com/v1"}]
    },
    "network": "ethereum"
  }'
```

### b. set_agent_wallet -- Link Wallet via EIP-712

Links an EVM wallet address to the agent identity via EIP-712 typed data signature. Requires owner approval through WalletConnect or Admin UI.

| Field      | Type   | Required | Description                                      |
| ---------- | ------ | -------- | ------------------------------------------------ |
| `agentId`  | string | Yes      | On-chain agent ID (numeric string).              |
| `deadline` | string | No       | Signature deadline timestamp (default: 1h from now). |

**Risk:** high | **Default tier:** APPROVAL

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/set_agent_wallet \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"agentId": "42"}, "network": "ethereum"}'
```

### c. unset_agent_wallet -- Unlink Wallet

Removes the wallet link from the agent identity.

| Field     | Type   | Required | Description                         |
| --------- | ------ | -------- | ----------------------------------- |
| `agentId` | string | Yes      | On-chain agent ID (numeric string). |

**Risk:** high | **Default tier:** APPROVAL

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/unset_agent_wallet \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"agentId": "42"}, "network": "ethereum"}'
```

### d. set_agent_uri -- Set Registration File URI

Updates the tokenURI pointing to the agent's registration file.

| Field     | Type   | Required | Description                         |
| --------- | ------ | -------- | ----------------------------------- |
| `agentId` | string | Yes      | On-chain agent ID (numeric string). |
| `uri`     | string | Yes      | Registration file URL.              |

**Risk:** medium | **Default tier:** DELAY

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/set_agent_uri \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"agentId": "42", "uri": "https://agent.example.com/.well-known/agent.json"}, "network": "ethereum"}'
```

### e. set_metadata -- Set Agent Metadata

Sets a key-value metadata entry for the agent on the Identity Registry.

| Field     | Type   | Required | Description                                   |
| --------- | ------ | -------- | --------------------------------------------- |
| `agentId` | string | Yes      | On-chain agent ID (numeric string).           |
| `key`     | string | Yes      | Metadata key (1-64 chars).                    |
| `value`   | string | Yes      | Metadata value (encoded to bytes on-chain).   |

**Risk:** low | **Default tier:** NOTIFY

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/set_metadata \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"agentId": "42", "key": "version", "value": "2.0"}, "network": "ethereum"}'
```

### f. give_feedback -- Submit Reputation Feedback

Submits a reputation feedback entry for another agent on the Reputation Registry.

| Field           | Type   | Required | Description                                    |
| --------------- | ------ | -------- | ---------------------------------------------- |
| `agentId`       | string | Yes      | Target agent ID (numeric string).              |
| `value`         | number | Yes      | Feedback value (-100 to 100, integer).         |
| `valueDecimals` | number | No       | Value decimal precision (0-8, default 0).      |
| `tag1`          | string | No       | Category tag 1 (max 32 chars).                 |
| `tag2`          | string | No       | Category tag 2 (max 32 chars).                 |
| `endpoint`      | string | No       | Service endpoint being rated.                  |
| `feedbackURI`   | string | No       | URI to detailed feedback document.             |

**Risk:** low | **Default tier:** NOTIFY

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/give_feedback \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"agentId": "42", "value": 85, "tag1": "reliability"}, "network": "ethereum"}'
```

### g. revoke_feedback -- Revoke Feedback

Revokes a previously submitted feedback entry by index.

| Field           | Type   | Required | Description                              |
| --------------- | ------ | -------- | ---------------------------------------- |
| `agentId`       | string | Yes      | Target agent ID (numeric string).        |
| `feedbackIndex` | number | Yes      | Index of feedback to revoke (0-based).   |

**Risk:** low | **Default tier:** INSTANT

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/revoke_feedback \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"agentId": "42", "feedbackIndex": 0}, "network": "ethereum"}'
```

### h. request_validation -- Request Third-Party Validation

Submits a validation request to the Validation Registry. Feature-gated: requires `actions.erc8004_validation_registry_address` to be configured.

| Field                | Type   | Required | Description                                          |
| -------------------- | ------ | -------- | ---------------------------------------------------- |
| `agentId`            | string | Yes      | Agent ID requesting validation (numeric string).     |
| `validatorAddress`   | string | Yes      | Validator EVM address (0x-prefixed, 40 hex chars).   |
| `requestDescription` | string | Yes      | Description of the validation request (1-1000 chars).|
| `tag`                | string | No       | Validation method tag (max 32 chars).                |

**Risk:** medium | **Default tier:** DELAY

```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/request_validation \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{
    "params": {
      "agentId": "42",
      "validatorAddress": "0x9876543210fedcba9876543210fedcba98765432",
      "requestDescription": "Verify agent identity and capabilities",
      "tag": "kyc"
    },
    "network": "ethereum"
  }'
```

## 3. SDK Methods

### TypeScript SDK

```typescript
import { WAIaaSClient } from '@waiaas/sdk';

const client = new WAIaaSClient({
  baseUrl: 'http://localhost:3100',
  sessionToken: 'wai_sess_eyJ...',
});
```

**8 Write Methods:**

```typescript
// Identity Registry
const reg = await client.registerAgent({ name: 'My Agent', description: 'AI trading agent' });
const link = await client.setAgentWallet({ agentId: '42' });
const unlink = await client.unsetAgentWallet({ agentId: '42' });
const uri = await client.setAgentUri({ agentId: '42', uri: 'https://agent.example.com/reg.json' });
const meta = await client.setAgentMetadata({ agentId: '42', key: 'version', value: '2.0' });

// Reputation Registry
const fb = await client.giveFeedback({ agentId: '42', value: 85, tag1: 'reliability' });
const revoke = await client.revokeFeedback({ agentId: '42', feedbackIndex: 0 });

// Validation Registry
const val = await client.requestValidation({
  agentId: '42',
  validatorAddress: '0x9876...',
  requestDescription: 'Verify identity',
  tag: 'kyc',
});
```

**3 Read Methods:**

```typescript
// Get agent identity info
const info = await client.getAgentInfo('42');
// -> { agentId, wallet, uri, metadata, registryAddress, chainId }

// Get agent reputation
const rep = await client.getAgentReputation('42', { tag1: 'reliability' });
// -> { agentId, count, score, decimals, tag1, tag2 }

// Get validation status
const status = await client.getValidationStatus('0xabcdef...');
// -> { requestHash, validator, agentId, response, responseHash, tag, lastUpdate }
```

## 5. Common Workflows

### Register an Agent and Set Up Registration File

1. Register the agent (requires owner approval):
```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/register_agent \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"name": "My Agent", "description": "Autonomous trading"}, "network": "ethereum"}'
```

3. After approval and on-chain confirmation, view the registration file:
```bash
curl -s http://localhost:3100/v1/erc8004/registration-file/<walletId> \
  -H 'Authorization: Bearer wai_sess_eyJ...'
```

### Give Feedback and Query Reputation

1. Submit feedback for another agent:
```bash
curl -s -X POST http://localhost:3100/v1/actions/erc8004_agent/give_feedback \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer wai_sess_eyJ...' \
  -d '{"params": {"agentId": "99", "value": 75, "tag1": "reliability"}, "network": "ethereum"}'
```

2. Query the agent's reputation:
```bash
curl -s 'http://localhost:3100/v1/erc8004/agent/99/reputation?tag1=reliability' \
  -H 'Authorization: Bearer wai_sess_eyJ...'
```

## 6. Error Reference

| Code                     | HTTP | Description                                    | Recovery                                   |
| ------------------------ | ---- | ---------------------------------------------- | ------------------------------------------ |
| `CHAIN_ERROR`            | 502  | On-chain RPC call failed.                      | Check RPC endpoint connectivity.           |
| `ADAPTER_NOT_AVAILABLE`  | 503  | Registry address not configured.               | Configure the registry in Admin Settings.  |
| `WALLET_NOT_FOUND`       | 404  | Wallet UUID does not exist.                    | Verify the walletId.                       |
| `ACTION_NOT_FOUND`       | 404  | Action provider or action not registered.      | Enable provider in Admin Settings.         |
| `ACTION_VALIDATION_FAILED`| 400 | Input parameters failed Zod validation.        | Check parameter types and constraints.     |
| `POLICY_VIOLATION`       | 403  | Transaction blocked by active policy.          | Check policies with GET /v1/policies.      |
| `API_KEY_REQUIRED`       | 403  | Provider requires config not yet set.          | Set required settings in Admin UI.         |
