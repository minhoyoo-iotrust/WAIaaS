### Issues Found

**ISSUE 1 (High): DASH-04 — Dashboard txHash missing from API**
- `GET /admin/status` handler (`admin.ts:841`) does not select `transactions.txHash`
- `AdminStatusResponseSchema` (`openapi-schemas.ts:591`) omits `txHash`
- Dashboard `ExplorerLink` always receives `txHash={undefined}`
- **Fix:** Add `txHash` to DB select, mapper, and Zod schema

**ISSUE 2 (High): TXN-03 — Date filter type mismatch**
- `transactions.tsx:148-149` sends ISO date strings (`"2026-02-23"`)
- Backend `z.coerce.number()` produces `NaN` → filter is non-functional
- `notifications.tsx` correctly converts dates to epoch — pattern exists, just not applied
- **Fix:** Convert dates to epoch seconds before sending (copy notifications.tsx pattern)

**ISSUE 3 (Low): WDET-02 — Client-side filter on paginated data**
- Wallet Transactions tab filters apply client-side on 20-item page only
- Filters don't trigger server re-fetch across all pages
- **Acceptable as design limitation** or fix by forwarding filters to server
