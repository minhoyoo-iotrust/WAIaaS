- [x] **Phase 306: Encrypted Backup & Restore 설계** - 암호화 아카이브 포맷, BackupService 확장, CLI 커맨드, config.toml 키 정의

## Phase Details

### Phase 304: Transaction Dry-Run 설계
**Goal**: AI 에이전트가 트랜잭션 실행 전 정책 평가 결과와 예상 수수료를 사전 확인할 수 있는 시뮬레이션 설계가 완성된다
**Depends on**: Nothing (first phase)
**Requirements**: SIM-01, SIM-02, SIM-03, SIM-04
**Success Criteria** (what must be TRUE):
  1. SimulationResult Zod 스키마가 정의되어 정책 tier, 예상 수수료, 잔액 변화, 경고 목록을 표현할 수 있다
  2. PipelineContext dryRun 플래그에 의한 Stage 분기가 명확히 설계되어 Stage 1-2-3-5b만 실행하고 부수 효과가 없음이 보장된다
  3. POST /v1/transactions/simulate 엔드포인트의 입력/출력/에러 코드가 OpenAPI 수준으로 정의된다
  4. SDK simulate() 메서드와 MCP waiaas_simulate_transaction tool 확장 스펙이 기존 인터페이스와 일관되게 정의된다
**Plans**: TBD

Plans:
- [x] 304-01: SimulationResult 스키마 + PipelineContext dryRun 분기 설계
- [x] 304-02: REST API + SDK/MCP 확장 스펙 + 설계 문서 갱신

### Phase 305: Audit Log Query API 설계
**Goal**: 감사 이벤트 커버리지가 12개+로 확대되고, 외부에서 필터/페이지네이션으로 감사 로그를 조회할 수 있는 API가 설계된다
**Depends on**: Nothing (independent)
**Requirements**: AUDIT-01, AUDIT-02, AUDIT-03, AUDIT-04
**Success Criteria** (what must be TRUE):
  1. AuditEventType enum이 기존 2개에서 12개+로 확대되고, 각 이벤트의 발생 서비스/스테이지가 매핑된다
  2. AuditLogQuerySchema와 AuditLogResponseSchema가 Zod로 정의되어 wallet_id, event_type, from/to, tx_id, severity 필터를 지원한다
  3. cursor pagination 설계가 id AUTOINCREMENT 기반으로 정의되어 기본 50건, 최대 200건 제한이 명시된다
  4. GET /v1/audit-logs 엔드포인트 스펙이 masterAuth 인증, 에러 코드(INVALID_CURSOR 등)와 함께 완성된다
**Plans**: TBD

Plans:
- [x] 305-01: AuditEventType 확대 + 삽입 지점 매핑 + 쿼리/응답 스키마 설계
- [x] 305-02: cursor pagination + REST API 엔드포인트 스펙 + 설계 문서 갱신

### Phase 306: Encrypted Backup & Restore 설계
**Goal**: 마스터 비밀번호 기반 암호화 백업/복원이 독립 CLI 커맨드로 사용 가능하도록 설계된다
**Depends on**: Nothing (independent)
**Requirements**: BKUP-01, BKUP-02, BKUP-03, BKUP-04
**Success Criteria** (what must be TRUE):
  1. 백업 아카이브 바이너리 포맷(헤더 + 메타데이터 + AES-256-GCM 암호화 페이로드)이 바이트 수준으로 사양화된다
  2. BackupService 암호화 확장이 기존 인터페이스를 유지하면서 Argon2id KDF + VACUUM INTO 원자적 스냅샷을 포함하도록 설계된다
  3. waiaas backup / waiaas restore CLI 인터페이스가 안전 장치(확인 프롬프트, 데몬 정지 확인, 기존 데이터 보존)와 함께 설계된다
  4. config.toml [backup] 섹션 키(backup_dir, backup_interval, backup_retention_count)가 평탄 키 원칙을 준수하여 정의된다

### Phase 307: Webhook Outbound 설계
**Goal**: 외부 시스템이 WAIaaS 이벤트를 프로그래밍 방식으로 구독하고 HMAC 서명으로 검증할 수 있는 webhook 메커니즘이 설계된다
**Depends on**: Phase 305 (감사 이벤트 확대가 webhook 필터링 대상 이벤트 체계를 확립)
**Requirements**: HOOK-01, HOOK-02, HOOK-03, HOOK-04, HOOK-05
**Success Criteria** (what must be TRUE):
  1. webhooks, webhook_logs 테이블 스키마가 Drizzle + Zod SSoT로 정의되어 기존 18 테이블 체계와 일관된다
  2. HMAC-SHA256 서명 생성/검증 프로토콜(X-WAIaaS-Signature 헤더)이 수신 측 검증 가이드와 함께 정의된다
  3. 비동기 재시도 큐(최대 3회, 지수 백오프 1s-2s-4s)가 메인 파이프라인 비블로킹으로 설계된다
  4. REST API 4 엔드포인트(POST/GET/DELETE webhooks, GET logs)가 masterAuth 인증으로 스펙화된다
  5. EventBus 연동 및 이벤트 필터링 메커니즘이 기존 NotificationChannel과의 관계와 함께 설계된다

- [x] 307-02: REST API 스펙 + EventBus 연동 + 설계 문서 갱신

### Phase 308: Admin Stats + AutoStop Plugin 설계
**Goal**: 운영 통계를 단일 JSON 엔드포인트로 확인할 수 있고, AutoStop 규칙이 플러그인 구조로 확장 가능하도록 설계된다
**Depends on**: Phase 305 (감사 이벤트 체계 활용), Phase 304 (TX 통계 참조)
**Requirements**: STAT-01, STAT-02, STAT-03, STAT-04, PLUG-01, PLUG-02, PLUG-03, PLUG-04
**Success Criteria** (what must be TRUE):
  1. AdminStatsResponseSchema가 transactions/sessions/wallets/rpc/autostop/system 7 카테고리를 Zod로 정의한다
  2. 인메모리 카운터 인터페이스(RPC 호출, 세션 카운트, TX 카운터)와 DB 집계 쿼리 + 1분 TTL 캐시가 설계된다
  3. IAutoStopRule 인터페이스와 RuleResult 타입이 정의되어 기존 3개 규칙 클래스가 구현체로 리팩토링 가능하다
  4. RuleRegistry(런타임 등록/해제/조회)가 설계되어 새 규칙 추가 시 하드코딩 없이 플러그인 등록만으로 확장된다
  5. GET /v1/admin/stats + GET /v1/admin/autostop/rules 엔드포인트 스펙과 규칙별 enable/disable Admin Settings 토글이 설계된다
