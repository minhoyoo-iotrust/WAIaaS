## Phase Details

### Phase 330: UI 재구성 + Feature Gate 정비
**Goal**: 운영자가 직관적인 메뉴 이름으로 DeFi 액션과 Agent Identity를 관리하고, 모든 프로바이더가 기본 활성화된 상태에서 시작할 수 있는 상태
**Depends on**: Nothing (first phase)
**Requirements**: MENU-01, MENU-02, MENU-03, MENU-04, TOGL-01, TOGL-02, TOGL-03, TOGL-04, TOGL-05, GATE-01, GATE-02, GATE-03, GATE-04
**Success Criteria** (what must be TRUE):
  1. Admin UI 사이드바에 "DeFi"와 "Agent Identity" 메뉴가 표시되고, Agent Identity가 DeFi 바로 다음에 위치한다
  2. DeFi 페이지(`#/defi`)에 ERC-8004 프로바이더 카드가 없고, Agent Identity 페이지(`#/agent-identity`) 상단에 활성화/비활성화 토글이 있다
  3. 토글을 끄면 Registered Actions 테이블이 읽기 전용으로 표시되고 관리 탭이 숨겨지며, 토글을 켜면 전체 UI가 렌더링된다
  4. 신규 설치 시 10개 액션 프로바이더가 모두 활성화 상태이고, 기존 운영자의 수동 설정은 DB v42 마이그레이션에서 존중된다
  5. 기존 테스트가 기본값 변경과 통합된 settings 파싱 방식에 맞게 업데이트되어 통과한다

### Phase 331: 액션 메타데이터 + Tier 오버라이드
**Goal**: 운영자가 각 액션의 설명을 확인하고, 액션별 보안 Tier를 드롭다운으로 조정할 수 있는 상태
**Depends on**: Phase 330
**Requirements**: DESC-01, DESC-02, TIER-01, TIER-02, TIER-03, TIER-04, TIER-05, TIER-06, TIER-07, TIER-08
**Success Criteria** (what must be TRUE):
  1. DeFi와 Agent Identity 페이지의 Registered Actions 테이블에 각 액션의 영문 설명이 Description 컬럼으로 표시된다
  2. Registered Actions 테이블의 Tier 셀이 드롭다운이며, INSTANT/NOTIFY/DELAY/APPROVAL 중 선택하면 즉시 Settings에 반영된다
  3. 오버라이드된 tier는 시각적으로 구분되고(뱃지 색상 또는 "customized" 라벨), Reset to default로 프로바이더 기본값 복원이 가능하다
  4. 파이프라인에서 트랜잭션 실행 시 Settings override가 프로바이더 하드코딩 기본값보다 우선 적용된다
