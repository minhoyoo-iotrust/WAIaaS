## Phase Details

### Phase 239: Foundation -- Shared Components + Admin API
**Goal**: 후속 페이지가 의존하는 공용 UI 컴포넌트와 크로스 지갑 Admin API가 준비되어 있다
**Depends on**: Nothing (first phase of v27.4)
**Requirements**: COMP-01, COMP-02, COMP-03, API-01, API-02
**Success Criteria** (what must be TRUE):
  1. ExplorerLink 컴포넌트에 13개 네트워크의 txHash를 전달하면 각각 올바른 블록 익스플로러 URL이 렌더링된다
  2. FilterBar 컴포넌트가 다중 필드(드롭다운, 날짜 범위 등)를 받아 URL query params로 동기화하며, 필터 변경 시 콜백이 호출된다
  3. SearchInput 컴포넌트에 텍스트를 입력하면 debounce 후 필터링 콜백이 호출된다
  4. GET /v1/admin/transactions API가 masterAuth로 보호되며 지갑/타입/상태/네트워크/날짜 범위 필터 + offset/limit 페이지네이션을 지원한다
  5. GET /v1/admin/incoming API가 masterAuth로 보호되며 지갑/체인/상태/suspicious 필터 + offset/limit 페이지네이션을 지원한다

### Phase 240: Transactions Page + Dashboard
**Goal**: 운영자가 전체 지갑의 트랜잭션을 단일 페이지에서 조회/필터/검색/페이지네이션할 수 있고, 대시보드에서 승인 대기/실패 건을 즉시 파악하여 트랜잭션 페이지로 이동할 수 있다
**Depends on**: Phase 239 (ExplorerLink, FilterBar, SearchInput, GET /v1/admin/transactions)
**Requirements**: TXN-01, TXN-02, TXN-03, TXN-04, TXN-05, TXN-06, DASH-01, DASH-02, DASH-03, DASH-04
**Success Criteria** (what must be TRUE):
  1. /transactions 라우트에서 전체 지갑의 트랜잭션이 시간/지갑명/타입 배지/수신자/금액(+USD)/네트워크/상태 배지/txHash 익스플로러 링크 컬럼으로 표시된다
  2. 지갑/타입/상태/네트워크/날짜 범위 필터와 txHash/수신자 주소 검색으로 트랜잭션을 좁힐 수 있다
  3. 서버사이드 페이지네이션으로 페이지를 이동하면 새 데이터가 로드되며, 트랜잭션 행을 클릭하면 전체 필드가 확장 표시된다
  4. 대시보드에서 APPROVAL 대기 건수 카드가 표시되고, Failed Txns/Recent Activity 카드 클릭 시 /transactions로 해당 필터가 적용된 채 이동한다
  5. 대시보드 Recent Activity 테이블에 네트워크 컬럼과 txHash 익스플로러 링크가 표시된다
**Plans**: TBD

Plans:
- [ ] 240-01: Transactions page (table, filters, pagination, search, row expand)
- [ ] 240-02: Dashboard improvements (approval card, clickable cards, network/txHash columns)
