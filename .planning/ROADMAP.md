## Phase Details

### Phase 258: GasCondition 코어 파이프라인
**Goal**: 가스 조건이 지정된 트랜잭션이 조건 충족까지 안전하게 대기하고, 충족 시 자동으로 파이프라인을 재개하며, 타임아웃 시 취소되는 상태
**Depends on**: Nothing (first phase of v28.5)
**Requirements**: PIPE-01, PIPE-02, PIPE-03, PIPE-04, PIPE-05, PIPE-06, EVAL-01, EVAL-02, EVAL-03, EVAL-04, WRKR-01, WRKR-02, WRKR-03, WRKR-04, WRKR-05, WRKR-06, NOTF-01, NOTF-02, NOTF-03
**Success Criteria** (what must be TRUE):
  1. gasCondition(maxGasPrice/maxPriorityFee/timeout)이 지정된 트랜잭션이 정책 평가 통과 후 GAS_WAITING 상태로 진입하고, gasCondition 미지정 트랜잭션은 기존 동작 그대로 즉시 실행된다
  2. EVM에서 eth_gasPrice/eth_maxPriorityFeePerGas, Solana에서 computeUnitPrice 기반으로 가스 조건이 평가되어, 조건 충족 시 GAS_WAITING에서 Stage 4로 파이프라인이 재개된다
  3. 타임아웃 초과 시 트랜잭션이 CANCELLED로 전이되고, GAS_WAITING 진입/조건 충족/타임아웃 취소 시 각각 TX_GAS_WAITING/TX_GAS_CONDITION_MET/TX_CANCELLED 알림이 발송된다
  4. 데몬 재시작 후 DB의 GAS_WAITING 트랜잭션이 자동 복원되어 재평가가 재개되고, max_pending_count 초과 시 새 gasCondition 요청이 에러로 거부된다

- [x] 258-02: GasConditionTracker (EVM + Solana) + AsyncPolling + Settings 5키 + 데몬 라이프사이클 -- completed 2026-02-25

### Phase 259: 외부 인터페이스 통합
**Goal**: 에이전트가 REST API, MCP, SDK 등 모든 인터페이스에서 gasCondition을 지정할 수 있고, Admin이 Settings UI에서 운영 파라미터를 런타임 조정할 수 있는 상태
**Depends on**: Phase 258
**Requirements**: INTF-01, INTF-02, INTF-03, INTF-04, INTF-05, INTF-06, INTF-07
**Success Criteria** (what must be TRUE):
  1. REST API 트랜잭션 제출 엔드포인트에서 gasCondition 옵션 필드를 지정하면 가스 조건부 실행이 동작한다
  2. Admin Settings에서 gas_condition 5개 키(enabled/poll_interval_sec/default_timeout_sec/max_timeout_sec/max_pending_count)를 런타임 변경할 수 있고, Admin UI System > Settings에 Gas Condition 설정 섹션이 표시된다
  3. MCP send_token/call_contract 등 도구와 TS/Python SDK에서 gasCondition 파라미터가 노출되어 에이전트가 사용할 수 있다
  4. ActionProvider resolve 후 gasCondition이 적용되어 DeFi 트랜잭션에서도 가스 조건부 실행이 동작하고, transactions.skill.md에 가스 조건부 실행 가이드가 포함된다
**Plans**:
