#### Missing Connections (Integration Gap)

**CRITICAL -- E2E tests not updated for paginated response format**

| Issue | From | To | Detail |
|-------|------|----|--------|
| Paginated response breaking e2e tests | Phase 436 (API pagination) | @waiaas/e2e-tests | 7 test assertions call `body.find()` or `Array.isArray(body)` on paginated wrapper |

**Affected e2e test files:**
- `packages/e2e-tests/src/__tests__/core-auth-wallet-session.e2e.test.ts` -- line 48: `Array.isArray(body)` fails
- `packages/e2e-tests/src/__tests__/core-policy.e2e.test.ts` -- lines 81, 113, 134: `body.find()` fails ("body.find is not a function")
- `packages/e2e-tests/src/__tests__/advanced-x402-erc8004-erc8128.e2e.test.ts` -- lines 106, 127: `body.find()` fails
- `packages/e2e-tests/src/__tests__/core-wallet-lifecycle.e2e.test.ts` -- indirect failure (session creation depends on session list assertion)

**Root cause:** Phase 436 changed GET /v1/sessions and GET /v1/policies response from raw array to `{data: [...], total, limit, offset}`. Daemon unit tests, Admin UI, and Admin UI tests were updated, but e2e tests in `@waiaas/e2e-tests` were not.

**Fix:** Update e2e tests to use `body.data` instead of `body` for array operations, or parse `body.data ?? body` for backward compat.
