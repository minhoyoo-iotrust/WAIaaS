## 4. PUT /v1/admin/autostop/rules/:id 엔드포인트

### 4.1 엔드포인트 사양

| 항목 | 값 |
|------|-----|
| 메서드 | PUT |
| 경로 | `/v1/admin/autostop/rules/:id` |
| 인증 | masterAuth |
| 요청 바디 | `{ enabled?: boolean, config?: Record<string, unknown> }` |
| 응답 (성공) | 200 AutoStopRuleInfo |
| 응답 (규칙 없음) | 404 RULE_NOT_FOUND |
| 응답 (인증 실패) | 401 INVALID_MASTER_PASSWORD |

### 4.2 요청 스키마

```typescript
export const AutoStopRuleUpdateRequestSchema = z.object({
  enabled: z.boolean().optional(),
  config: z.record(z.unknown()).optional(),
});
```

### 4.3 동작

1. `registry.getRule(id)` → 없으면 404
2. `enabled`가 있으면 `registry.setEnabled(id, enabled)` → Admin Settings에도 반영
3. `config`가 있으면 `rule.updateConfig(config)` → Admin Settings에도 반영
4. 갱신된 `rule.getStatus()` 반환

### 4.4 요청/응답 예시

```
PUT /v1/admin/autostop/rules/consecutive_failures
{ "enabled": false }
→ 200: { "id": "consecutive_failures", "enabled": false, ... }

PUT /v1/admin/autostop/rules/unusual_activity
{ "config": { "threshold": 30, "windowSec": 600 } }
→ 200: { "id": "unusual_activity", "config": { "threshold": 30, "windowSec": 600 }, ... }
```

## 5. Admin Settings 규칙 토글 (PLUG-04)

### 5.1 신규 Setting 키

기존 `autostop` 카테고리에 규칙별 enable/disable 키를 추가한다.

| 키 | category | 기본값 | 설명 |
|----|----------|--------|------|
| `autostop.rule.consecutive_failures.enabled` | autostop | `true` | ConsecutiveFailures 규칙 활성화 |
| `autostop.rule.unusual_activity.enabled` | autostop | `true` | UnusualActivity 규칙 활성화 |
| `autostop.rule.idle_timeout.enabled` | autostop | `true` | IdleTimeout 규칙 활성화 |

### 5.2 기존 키와의 관계

| 기존 키 | 역할 | 신규 키와의 관계 |
|---------|------|----------------|
| `autostop.enabled` | 전체 AutoStop 활성화/비활성화 | 유지 (글로벌 마스터 스위치) |
| `autostop.consecutive_failures_threshold` | 임계값 | 유지 (설정 변경 시 `rule.updateConfig` 호출) |
| `autostop.unusual_activity_threshold` | 임계값 | 유지 |
| `autostop.unusual_activity_window_sec` | 윈도우 | 유지 |
| `autostop.idle_timeout_sec` | 타임아웃 | 유지 |
| `autostop.idle_check_interval_sec` | 검사 주기 | 유지 |

**관계**: `autostop.enabled = false`이면 모든 규칙 비활성. `autostop.enabled = true`일 때 개별 규칙의 `autostop.rule.{id}.enabled` 토글이 적용됨.

### 5.3 Hot Reload 연동

기존 `hot-reload.ts`의 autostop 섹션 처리에 규칙별 토글 로직 추가:

```typescript
// hot-reload.ts (기존 autostop 핸들러 확장)
if (key.startsWith('autostop.rule.') && key.endsWith('.enabled')) {
  const ruleId = key.replace('autostop.rule.', '').replace('.enabled', '');
  const enabled = value === 'true';
  autoStopService.registry.setEnabled(ruleId, enabled);
}
```

### 5.4 SETTING_DEFINITIONS 추가 항목

```typescript

{ key: 'autostop.rule.unusual_activity.enabled', category: 'autostop', configPath: 'security.autostop_rule_unusual_activity_enabled', defaultValue: 'true', isCredential: false },
{ key: 'autostop.rule.idle_timeout.enabled', category: 'autostop', configPath: 'security.autostop_rule_idle_timeout_enabled', defaultValue: 'true', isCredential: false },
```

### 7.3 API 호출 패턴

```typescript
// dashboard.tsx 내 stats 섹션
const stats = useSignal<AdminStatsResponse | null>(null);

useEffect(() => {
  const fetchStats = async () => {
    const data = await apiGet<AdminStatsResponse>(API.ADMIN_STATS);
    stats.value = data;
  };
  fetchStats();

  // 30초마다 자동 새로고침
  const interval = setInterval(fetchStats, 30_000);
  return () => clearInterval(interval);
}, []);
```
