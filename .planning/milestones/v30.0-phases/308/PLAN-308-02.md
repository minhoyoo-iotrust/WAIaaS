**Phase:** 308

## 1. 개요

기존 AutoStopService의 3개 구체 규칙 클래스(ConsecutiveFailuresRule, UnusualActivityRule, IdleTimeoutRule)를 공통 인터페이스 `IAutoStopRule`로 추출하고, `RuleRegistry`를 통해 런타임 등록/해제/조회가 가능한 플러그인 구조로 리팩토링한다.

### 1.1 현재 구조의 문제점

```
AutoStopService (하드코딩)
├── consecutiveFailuresRule: ConsecutiveFailuresRule  ← 직접 인스턴스화
├── unusualActivityRule: UnusualActivityRule           ← 직접 인스턴스화
├── idleTimeoutRule: IdleTimeoutRule                   ← 직접 인스턴스화
│
├── start() → 각 규칙의 고유 메서드를 EventBus에 개별 바인딩
├── updateConfig() → 각 규칙의 고유 update 메서드를 개별 호출
└── getStatus() → 각 규칙의 고유 get 메서드를 개별 호출
```

문제:
1. **공통 인터페이스 없음** -- 3개 클래스가 각자 다른 시그니처 (`onTransactionFailed`, `onWalletActivity`, `checkIdle`)
2. **하드코딩된 규칙 목록** -- 새 규칙 추가 시 AutoStopService 소스 수정 필수
3. **개별 update 메서드** -- 규칙마다 다른 설정 변경 API

### 1.2 목표 구조

```
AutoStopService (플러그인 기반)
├── registry: RuleRegistry
│   ├── register(rule: IAutoStopRule): void
│   ├── unregister(ruleId: string): void
│   ├── getRules(): IAutoStopRule[]
│   └── getRule(id: string): IAutoStopRule | undefined
│
├── start() → registry.getRules()를 순회하며 규칙별 이벤트 구독
├── updateConfig() → registry에 위임
└── getStatus() → registry에 위임
```

## 2. IAutoStopRule 인터페이스 (PLUG-01)

### 2.1 인터페이스 정의

```typescript
// packages/daemon/src/services/autostop/types.ts

/**
 * AutoStop 규칙의 공통 인터페이스.
 *
 * 모든 규칙은 이벤트 기반으로 동작하며,
 * AutoStopService가 EventBus 이벤트를 수신하여 해당 규칙에 전달한다.
 */
export interface IAutoStopRule {
  /** 규칙 고유 식별자 (e.g., 'consecutive_failures', 'unusual_activity') */
  readonly id: string;

  /** 사람이 읽을 수 있는 규칙 이름 */
  readonly displayName: string;

  /** 규칙 설명 */
  readonly description: string;

  /** 규칙이 구독하는 이벤트 타입 목록 */
  readonly subscribedEvents: AutoStopEventType[];

  /** 규칙 활성화 여부 */
  enabled: boolean;

  /**
   * 이벤트 수신 시 규칙 평가.
   * AutoStopService가 EventBus에서 이벤트를 받아 해당 규칙에 전달한다.
   *
   * @param event - 수신된 이벤트
   * @returns RuleResult - 규칙 트리거 여부 + 대상 월렛
   */
  evaluate(event: AutoStopEvent): RuleResult;

  /**
   * 주기적 검사 (선택적).
   * IdleTimeoutRule처럼 타이머 기반 검사가 필요한 규칙용.
   * null을 반환하면 주기적 검사가 불필요한 규칙.
   */
  tick?(nowSec: number): RuleTickResult[];

  /**
   * 현재 규칙 상태 조회 (모니터링용).
   */
  getStatus(): RuleStatus;

  /**
   * 런타임 설정 변경.
   * Admin Settings에서 규칙별 설정 변경 시 호출.
   */
  updateConfig(config: Record<string, unknown>): void;

  /**
   * 규칙 상태 초기화 (테스트용 또는 강제 리셋).
   */
  reset(): void;
}
```

### 2.2 관련 타입 정의

```typescript
// packages/daemon/src/services/autostop/types.ts

/**
 * AutoStop 이벤트 타입.
 * EventBus 이벤트 이름과 1:1 매핑.
 */
export type AutoStopEventType =
  | 'transaction:failed'
  | 'transaction:completed'
  | 'wallet:activity';

/**
 * AutoStop 이벤트 데이터.
 * EventBus에서 수신한 이벤트 + 메타데이터.
 */
export interface AutoStopEvent {
  type: AutoStopEventType;
  walletId: string;
  timestamp: number;
  details?: Record<string, unknown>;
}

/**
 * 규칙 평가 결과.
 */
export interface RuleResult {
  /** 규칙이 트리거되었는지 여부 */
  triggered: boolean;
  /** 대상 월렛 ID */
  walletId: string;
  /** 트리거 시 실행할 액션 */
  action?: RuleAction;
}

/**
 * 규칙이 트리거될 때 실행할 액션.
 */
export type RuleAction =
  | 'SUSPEND_WALLET'       // 월렛 정지 (기존 ConsecutiveFailures, UnusualActivity)
  | 'NOTIFY_IDLE'          // 유휴 알림 (기존 IdleTimeout)
  | 'KILL_SWITCH_CASCADE'; // Kill Switch 발동 (미래 확장용)

/**
 * tick() 결과 (주기적 검사).
 * 단일 tick에서 여러 월렛/세션이 감지될 수 있음.
 */
export interface RuleTickResult {
  triggered: boolean;
  walletId: string;
  sessionId?: string;
  action: RuleAction;
}

/**
 * 규칙 상태 (모니터링용).
 */
export interface RuleStatus {
  id: string;
  displayName: string;
  enabled: boolean;
  config: Record<string, unknown>;
  state: Record<string, unknown>;  // 규칙별 내부 상태 (trackedWallets, trackedSessions 등)
}
```

### 2.3 기존 RuleResult와의 관계

현재 `autostop-rules.ts`에 정의된 `RuleResult`는 `{ triggered: boolean; walletId: string }`이다. 새 `RuleResult`는 이를 확장하여 `action` 필드를 추가한다.

| 필드 | 기존 | 신규 | 비고 |
|------|------|------|------|
| triggered | O | O | 유지 |
| walletId | O | O | 유지 |
| action | X | O | 신규 -- 규칙별 다른 액션 가능 |

기존 코드 하위 호환: `action`이 없으면 기본 `SUSPEND_WALLET`로 처리.

## 3. RuleRegistry (PLUG-02)

### 3.1 인터페이스

```typescript
// packages/daemon/src/services/autostop/rule-registry.ts

export interface IRuleRegistry {
  /** 규칙 등록. 동일 id가 이미 존재하면 교체. */
  register(rule: IAutoStopRule): void;

  /** 규칙 해제. 해당 id가 없으면 무시. */
  unregister(ruleId: string): void;

  /** 등록된 모든 규칙 조회 (등록 순서 유지). */
  getRules(): IAutoStopRule[];

  /** 활성화된 규칙만 조회. */
  getEnabledRules(): IAutoStopRule[];

  /** 특정 규칙 조회. */
  getRule(id: string): IAutoStopRule | undefined;

  /** 특정 이벤트를 구독하는 규칙 조회. */
  getRulesForEvent(eventType: AutoStopEventType): IAutoStopRule[];

  /** tick()이 구현된 규칙 조회 (주기적 검사 대상). */
  getTickableRules(): IAutoStopRule[];

  /** 규칙 활성화/비활성화 토글. */
  setEnabled(ruleId: string, enabled: boolean): void;

  /** 등록된 규칙 수. */
  readonly size: number;
}
```

### 3.2 구현 설계

```typescript
export class RuleRegistry implements IRuleRegistry {
  private rules = new Map<string, IAutoStopRule>();

  register(rule: IAutoStopRule): void {
    this.rules.set(rule.id, rule);
  }

  unregister(ruleId: string): void {
    this.rules.delete(ruleId);
  }

  getRules(): IAutoStopRule[] {
    return [...this.rules.values()];
  }

  getEnabledRules(): IAutoStopRule[] {
    return this.getRules().filter(r => r.enabled);
  }

  getRule(id: string): IAutoStopRule | undefined {
    return this.rules.get(id);
  }

  getRulesForEvent(eventType: AutoStopEventType): IAutoStopRule[] {
    return this.getEnabledRules().filter(r => r.subscribedEvents.includes(eventType));
  }

  getTickableRules(): IAutoStopRule[] {
    return this.getEnabledRules().filter(r => typeof r.tick === 'function');
  }

  setEnabled(ruleId: string, enabled: boolean): void {
    const rule = this.rules.get(ruleId);
    if (rule) rule.enabled = enabled;
  }

  get size(): number {
    return this.rules.size;
  }
}
```

### 3.3 등록 순서 보장

`Map`은 삽입 순서를 보장한다. 규칙 평가 순서:

### 4.1 ConsecutiveFailuresRule → IAutoStopRule

```typescript
// packages/daemon/src/services/autostop/rules/consecutive-failures.rule.ts

export class ConsecutiveFailuresRule implements IAutoStopRule {
  readonly id = 'consecutive_failures';
  readonly displayName = 'Consecutive Failures';
  readonly description = 'Suspends wallet after N consecutive transaction failures';
  readonly subscribedEvents: AutoStopEventType[] = ['transaction:failed', 'transaction:completed'];
  enabled = true;

  private failureCounts = new Map<string, number>();
  private threshold: number;

  constructor(threshold = 5) {
    this.threshold = threshold;
  }

  evaluate(event: AutoStopEvent): RuleResult {
    if (event.type === 'transaction:completed') {
      // 성공 시 카운터 리셋
      this.failureCounts.set(event.walletId, 0);
      return { triggered: false, walletId: event.walletId };
    }

    // transaction:failed
    const count = (this.failureCounts.get(event.walletId) ?? 0) + 1;
    this.failureCounts.set(event.walletId, count);

    return {
      triggered: count >= this.threshold,
      walletId: event.walletId,
      action: 'SUSPEND_WALLET',
    };
  }

  // tick 불필요 (이벤트 기반만)
  tick = undefined;

  getStatus(): RuleStatus {
    return {
      id: this.id,
      displayName: this.displayName,
      enabled: this.enabled,
      config: { threshold: this.threshold },
      state: { trackedWallets: this.failureCounts.size },
    };
  }

  updateConfig(config: Record<string, unknown>): void {
    if (typeof config.threshold === 'number') {
      this.threshold = config.threshold;
    }
  }

  reset(): void {
    this.failureCounts.clear();
  }

  /** 기존 resetWallet 유지 (하위 호환) */
  resetWallet(walletId: string): void {
    this.failureCounts.delete(walletId);
  }
}
```

```typescript
// packages/daemon/src/services/autostop/rules/unusual-activity.rule.ts

export class UnusualActivityRule implements IAutoStopRule {
  readonly id = 'unusual_activity';
  readonly displayName = 'Unusual Activity';
  readonly description = 'Suspends wallet when activity frequency exceeds threshold within time window';
  readonly subscribedEvents: AutoStopEventType[] = ['wallet:activity'];
  enabled = true;

  private activityTimestamps = new Map<string, number[]>();
  private threshold: number;
  private windowSec: number;

  constructor(threshold = 20, windowSec = 300) {
    this.threshold = threshold;
    this.windowSec = windowSec;
  }

  evaluate(event: AutoStopEvent): RuleResult {
    const cutoff = event.timestamp - this.windowSec;
    let timestamps = this.activityTimestamps.get(event.walletId) ?? [];
    timestamps = timestamps.filter(t => t > cutoff);
    timestamps.push(event.timestamp);
    this.activityTimestamps.set(event.walletId, timestamps);

    return {
      triggered: timestamps.length >= this.threshold,
      walletId: event.walletId,
      action: 'SUSPEND_WALLET',
    };
  }

  tick = undefined;

  getStatus(): RuleStatus {
    return {
      id: this.id,
      displayName: this.displayName,
      enabled: this.enabled,
      config: { threshold: this.threshold, windowSec: this.windowSec },
      state: { trackedWallets: this.activityTimestamps.size },
    };
  }

  updateConfig(config: Record<string, unknown>): void {
    if (typeof config.threshold === 'number') this.threshold = config.threshold;
    if (typeof config.windowSec === 'number') this.windowSec = config.windowSec;
  }

  reset(): void {
    this.activityTimestamps.clear();
  }
}
```

### 4.3 IdleTimeoutRule → IAutoStopRule

```typescript
// packages/daemon/src/services/autostop/rules/idle-timeout.rule.ts

export class IdleTimeoutRule implements IAutoStopRule {
  readonly id = 'idle_timeout';
  readonly displayName = 'Idle Timeout';
  readonly description = 'Sends notification when session is idle beyond configured timeout';
  readonly subscribedEvents: AutoStopEventType[] = ['wallet:activity'];
  enabled = true;

  // Map<walletId, Map<sessionId, lastActivitySec>>
  private lastActivity = new Map<string, Map<string, number>>();
  private idleTimeoutSec: number;

  constructor(idleTimeoutSec = 3600) {
    this.idleTimeoutSec = idleTimeoutSec;
  }

  evaluate(event: AutoStopEvent): RuleResult {
    const sessionId = event.details?.sessionId as string | undefined;
    const activity = event.details?.activity as string | undefined;

    if (activity === 'SESSION_CREATED' && sessionId) {
      this.registerSession(event.walletId, sessionId, event.timestamp);
    } else if (sessionId) {
      this.onActivity(event.walletId, event.timestamp, sessionId);
    } else {
      this.onActivity(event.walletId, event.timestamp);
    }

    // evaluate는 즉시 트리거하지 않음 -- tick()에서 검사
    return { triggered: false, walletId: event.walletId };
  }

  tick(nowSec: number): RuleTickResult[] {
    const results: RuleTickResult[] = [];

    for (const [walletId, sessions] of this.lastActivity) {
      for (const [sessionId, lastTime] of sessions) {
        if (nowSec - lastTime > this.idleTimeoutSec) {
          results.push({
            triggered: true,
            walletId,
            sessionId,
            action: 'NOTIFY_IDLE',
          });
        }
      }
    }

    // tick 후 감지된 세션 제거 (중복 알림 방지)
    for (const result of results) {
      if (result.sessionId) {
        this.removeSession(result.walletId, result.sessionId);
      }
    }

    return results;
  }

  getStatus(): RuleStatus {
    let sessionCount = 0;
    for (const sessions of this.lastActivity.values()) {
      sessionCount += sessions.size;
    }

    return {
      id: this.id,
      displayName: this.displayName,
      enabled: this.enabled,
      config: { idleTimeoutSec: this.idleTimeoutSec },
      state: { trackedSessions: sessionCount },
    };
  }

  updateConfig(config: Record<string, unknown>): void {
    if (typeof config.idleTimeoutSec === 'number') {
      this.idleTimeoutSec = config.idleTimeoutSec;
    }
  }

  reset(): void {
    this.lastActivity.clear();
  }

  // --- 내부 헬퍼 ---

  private registerSession(walletId: string, sessionId: string, nowSec: number): void {
    let walletSessions = this.lastActivity.get(walletId);
    if (!walletSessions) {
      walletSessions = new Map();
      this.lastActivity.set(walletId, walletSessions);
    }
    walletSessions.set(sessionId, nowSec);
  }

  private onActivity(walletId: string, nowSec: number, sessionId?: string): void {
    const walletSessions = this.lastActivity.get(walletId);
    if (!walletSessions) return;

    if (sessionId && walletSessions.has(sessionId)) {
      walletSessions.set(sessionId, nowSec);
    } else if (!sessionId) {
      for (const sid of walletSessions.keys()) {
        walletSessions.set(sid, nowSec);
      }
    }
  }

  private removeSession(walletId: string, sessionId: string): void {
    const walletSessions = this.lastActivity.get(walletId);
    if (walletSessions) {
      walletSessions.delete(sessionId);
      if (walletSessions.size === 0) this.lastActivity.delete(walletId);
    }
  }
}
```

### 4.4 리팩토링 요약 매트릭스

| 기존 클래스 | 신규 클래스 | id | subscribedEvents | tick | action |
|------------|------------|-----|-----------------|------|--------|
| ConsecutiveFailuresRule | ConsecutiveFailuresRule (implements IAutoStopRule) | `consecutive_failures` | transaction:failed, transaction:completed | 없음 | SUSPEND_WALLET |
| UnusualActivityRule | UnusualActivityRule (implements IAutoStopRule) | `unusual_activity` | wallet:activity | 없음 | SUSPEND_WALLET |
| IdleTimeoutRule | IdleTimeoutRule (implements IAutoStopRule) | `idle_timeout` | wallet:activity | checkIdle | NOTIFY_IDLE |

### 4.5 기존 동작 보존 보장

| 기존 동작 | 신규에서의 보존 방법 |
|----------|-------------------|
| `onTransactionFailed()` → threshold 도달 시 wallet SUSPENDED | `evaluate(event)` 내부에서 동일 로직, `action: 'SUSPEND_WALLET'` |
| `onTransactionCompleted()` → 카운터 리셋 | `evaluate(event)` 내부에서 `transaction:completed` 처리 |
| 슬라이딩 윈도우 빈도 감지 | `evaluate(event)` 내부에서 동일 Map 기반 로직 |
| setInterval idle 체크 | `tick()` 메서드 → AutoStopService가 기존 타이머로 호출 |
| idle 세션 알림 후 추적 제거 | `tick()` 내부에서 결과 반환 후 제거 |
| `updateThreshold()` 등 개별 메서드 | `updateConfig(config)` 통합 메서드 |
| `getTrackedCount()` 등 | `getStatus().state` 에서 통합 반환 |

## 5. AutoStopService 리팩토링 설계

### 5.1 리팩토링된 AutoStopService

```typescript
// packages/daemon/src/services/autostop/autostop-service.ts

export class AutoStopService {
  private readonly registry: IRuleRegistry;
  private readonly sqlite: Database;
  private readonly eventBus: EventBus;
  private readonly killSwitchService: KillSwitchService;
  private readonly notificationService?: NotificationService;
  private readonly metricsCounter?: IMetricsCounter;
  private readonly config: AutoStopConfig;
  private idleCheckTimer: ReturnType<typeof setInterval> | null = null;

  constructor(opts: {
    sqlite: Database;
    eventBus: EventBus;
    killSwitchService: KillSwitchService;
    notificationService?: NotificationService;
    metricsCounter?: IMetricsCounter;
    config?: Partial<AutoStopConfig>;
    registry?: IRuleRegistry;
  }) {
    this.registry = opts.registry ?? new RuleRegistry();
    // ... 기존 초기화

    // 기존 3규칙 자동 등록
    this.registry.register(
      new ConsecutiveFailuresRule(this.config.consecutiveFailuresThreshold)
    );
    this.registry.register(
      new UnusualActivityRule(this.config.unusualActivityThreshold, this.config.unusualActivityWindowSec)
    );
    this.registry.register(
      new IdleTimeoutRule(this.config.idleTimeoutSec)
    );
  }

  start(): void {
    if (!this.config.enabled) return;

    // 이벤트 구독: 모든 규칙의 subscribedEvents를 수집하여 통합 구독
    const eventTypes = new Set<AutoStopEventType>();
    for (const rule of this.registry.getRules()) {
      for (const evt of rule.subscribedEvents) {
        eventTypes.add(evt);
      }
    }

    for (const eventType of eventTypes) {
      this.eventBus.on(eventType, (data) => {
        this.handleEvent(eventType, data);
      });
    }

    // 주기적 검사 타이머 (tickable 규칙이 있는 경우만)
    if (this.registry.getTickableRules().length > 0) {
      this.idleCheckTimer = setInterval(() => {
        this.runTick();
      }, this.config.idleCheckIntervalSec * 1000);
    }
  }

  private handleEvent(type: AutoStopEventType, data: any): void {
    const event: AutoStopEvent = {
      type,
      walletId: data.walletId,
      timestamp: data.timestamp ?? Math.floor(Date.now() / 1000),
      details: data.details ?? data,
    };

    const rules = this.registry.getRulesForEvent(type);
    for (const rule of rules) {
      const result = rule.evaluate(event);
      if (result.triggered) {
        this.executeAction(result);
      }
    }
  }

  private runTick(): void {
    const nowSec = Math.floor(Date.now() / 1000);
    for (const rule of this.registry.getTickableRules()) {
      const results = rule.tick!(nowSec);
      for (const result of results) {
        if (result.triggered) {
          this.executeTickAction(result);
        }
      }
    }
  }

  private executeAction(result: RuleResult): void {
    switch (result.action ?? 'SUSPEND_WALLET') {
      case 'SUSPEND_WALLET':
        this.suspendWallet(result.walletId, /* ruleId */ 'auto');
        break;
      case 'KILL_SWITCH_CASCADE':
        this.killSwitchService.activateWithCascade('autostop');
        break;
    }
  }

  private executeTickAction(result: RuleTickResult): void {
    switch (result.action) {
      case 'NOTIFY_IDLE':
        this.notifyIdleSession(result.walletId, result.sessionId ?? '');
        break;
      case 'SUSPEND_WALLET':
        this.suspendWallet(result.walletId, 'idle_timeout');
        break;
    }
  }

  // suspendWallet, notifyIdleSession, manualTrigger, stop, updateConfig, getStatus
  // → 기존 로직 유지, registry 위임 추가
}
```

### 6.1 신규 규칙 추가 예시

```typescript
// 향후: LargeTransactionRule
export class LargeTransactionRule implements IAutoStopRule {
  readonly id = 'large_transaction';
  readonly displayName = 'Large Transaction Alert';
  readonly description = 'Alerts when single transaction exceeds threshold USD amount';
  readonly subscribedEvents: AutoStopEventType[] = ['wallet:activity'];
  enabled = false; // 기본 비활성

  evaluate(event: AutoStopEvent): RuleResult {

    if (amountUsd && amountUsd > this.threshold) {
      return { triggered: true, walletId: event.walletId, action: 'NOTIFY_IDLE' };
    }
    return { triggered: false, walletId: event.walletId };
  }
  // ...
}
```
