# Plan 304-01: SimulationResult 스키마 + PipelineContext dryRun 분기 설계

**Phase:** 304 (Transaction Dry-Run 설계)
**Requirements:** SIM-01, SIM-02
**설계 문서 영향:** doc 32 (TX-PIPE), doc 33 (LOCK-MECH)

---

## 1. 설계 목적

AI 에이전트가 트랜잭션을 실행하기 전에 다음을 사전 확인할 수 있는 시뮬레이션 기능을 설계한다:

1. **정책 평가 결과** -- 어떤 tier로 분류되는지 (INSTANT/NOTIFY/DELAY/APPROVAL)
2. **예상 수수료** -- 온체인 시뮬레이션 기반 가스비 추정
3. **잔액 변화** -- 전송 후 예상 잔액
4. **경고 목록** -- 정책 위반, 잔액 부족, 가격 오라클 불가 등

핵심 원칙: **부수 효과 없음(side-effect free)**. DB 삽입 없음, 온체인 서명/제출 없음, 알림 없음.

## 2. SimulationResult Zod 스키마 (SIM-01)

### 2.1 스키마 정의

```typescript
// packages/core/src/schemas/simulation.schema.ts

import { z } from 'zod';
import { PolicyTierEnum } from '../enums/policy.js';
import { ChainTypeEnum, NetworkTypeEnum } from '../enums/chain.js';

/**
 * 시뮬레이션 경고 항목.
 * 트랜잭션 실행 전 에이전트가 인지해야 할 주의사항.
 */
export const SimulationWarningSchema = z.object({
  /** 경고 코드 (머신 소비용). */
  code: z.enum([
    'INSUFFICIENT_BALANCE',          // 잔액 부족
    'INSUFFICIENT_BALANCE_WITH_FEE', // 잔액 + 수수료 합산 부족
    'ORACLE_PRICE_UNAVAILABLE',      // USD 가격 조회 불가 (정책 평가 제한)
    'SIMULATION_FAILED',             // 온체인 시뮬레이션 실패
    'HIGH_FEE_RATIO',                // 수수료가 전송 금액 대비 높음 (>10%)
    'APPROVAL_REQUIRED',             // APPROVAL tier -- Owner 승인 필요
    'DELAY_REQUIRED',                // DELAY tier -- 시간 지연 적용
    'CUMULATIVE_LIMIT_WARNING',      // 누적 한도 80%+ 도달
    'TOKEN_NOT_IN_ALLOWED_LIST',     // ALLOWED_TOKENS 정책에 미포함
    'CONTRACT_NOT_WHITELISTED',      // CONTRACT_WHITELIST 미포함
    'NETWORK_NOT_ALLOWED',           // ALLOWED_NETWORKS 미포함
    'DOWNGRADED_NO_OWNER',           // APPROVAL→DELAY 다운그레이드 (Owner 미등록)
  ]),
  /** 사람이 읽을 수 있는 경고 메시지 (영어). */
  message: z.string(),
  /** 경고 심각도. */
  severity: z.enum(['info', 'warning', 'error']),
});
export type SimulationWarning = z.infer<typeof SimulationWarningSchema>;

/**
 * 잔액 변화 예측.
 * 시뮬레이션 결과로 예상되는 잔액 변화.
 */
export const BalanceChangeSchema = z.object({
  /** 자산 유형: 'native' (SOL/ETH) 또는 토큰 주소. */
  asset: z.string(),
  /** 자산 심볼 (SOL, ETH, USDC 등). */
  symbol: z.string(),
  /** 소수점 자릿수. */
  decimals: z.number().int().min(0),
  /** 현재 잔액 (smallest unit, digit string). */
  currentBalance: z.string(),
  /** 예상 잔액 변화량 (digit string, 음수 = 감소). 수수료 포함. */
  changeAmount: z.string(),
  /** 트랜잭션 후 예상 잔액 (digit string). */
  afterBalance: z.string(),
});
export type BalanceChange = z.infer<typeof BalanceChangeSchema>;

/**
 * 수수료 추정 정보.
 */
export const FeeEstimateResultSchema = z.object({
  /** 예상 수수료 (smallest unit, digit string). */
  estimatedFee: z.string(),
  /** 네이티브 토큰 심볼 (SOL/ETH). */
  feeSymbol: z.string(),
  /** 소수점 자릿수 (SOL=9, ETH=18). */
  feeDecimals: z.number().int(),
  /** USD 환산 수수료 (오라클 사용 가능 시). */
  feeUsd: z.number().nullable(),
  /** ATA 생성 필요 여부 (Solana SPL 토큰 전송 시). */
  needsAtaCreation: z.boolean().optional(),
  /** ATA 생성 렌트 비용 (Solana, digit string). */
  ataRentCost: z.string().optional(),
});
export type FeeEstimateResult = z.infer<typeof FeeEstimateResultSchema>;

/**
 * 정책 평가 결과 요약.
 */
export const PolicyResultSchema = z.object({
  /** 정책 평가 tier. */
  tier: PolicyTierEnum,
  /** 정책 허용 여부. */
  allowed: z.boolean(),
  /** 거부 사유 (allowed=false 시). */
  reason: z.string().optional(),
  /** DELAY tier 대기 시간 (초). */
  delaySeconds: z.number().int().optional(),
  /** APPROVAL 격상 사유. */
  approvalReason: z.enum(['per_tx', 'cumulative_daily', 'cumulative_monthly']).optional(),
  /** Owner 미등록으로 인한 APPROVAL→DELAY 다운그레이드 여부. */
  downgraded: z.boolean().optional(),
  /** 누적 지출 경고 (80%+ 도달 시). */
  cumulativeWarning: z.object({
    type: z.enum(['daily', 'monthly']),
    ratio: z.number(),
    spent: z.number(),
    limit: z.number(),
  }).optional(),
});
export type PolicyResult = z.infer<typeof PolicyResultSchema>;

/**
 * Transaction Dry-Run 시뮬레이션 결과.
 * POST /v1/transactions/simulate 응답 바디.
 */
export const DryRunSimulationResultSchema = z.object({
  /** 시뮬레이션 전체 성공 여부. */
  success: z.boolean(),
  /** 정책 평가 결과. */
  policy: PolicyResultSchema,
  /** 수수료 추정 (온체인 시뮬레이션 성공 시). */
  fee: FeeEstimateResultSchema.nullable(),
  /** 잔액 변화 예측 (native + token). */
  balanceChanges: z.array(BalanceChangeSchema),
  /** 경고 목록. */
  warnings: z.array(SimulationWarningSchema),
  /** 온체인 시뮬레이션 상세 (adapter.simulateTransaction 결과). */
  simulation: z.object({
    /** 온체인 시뮬레이션 성공 여부. */
    success: z.boolean(),
    /** 시뮬레이션 로그. */
    logs: z.array(z.string()),
    /** 소비 유닛 (Solana compute units / EVM gas used). */
    unitsConsumed: z.string().nullable(),
    /** 시뮬레이션 에러 메시지. */
    error: z.string().nullable(),
  }),
  /** 요청 메타데이터. */
  meta: z.object({
    /** 체인 타입. */
    chain: ChainTypeEnum,
    /** 네트워크. */
    network: NetworkTypeEnum,
    /** 트랜잭션 타입 (TRANSFER/TOKEN_TRANSFER/CONTRACT_CALL/APPROVE/BATCH). */
    transactionType: z.string(),
    /** 시뮬레이션 실행 시간 (ms). */
    durationMs: z.number().int(),
  }),
});
export type DryRunSimulationResult = z.infer<typeof DryRunSimulationResultSchema>;
```

### 2.2 스키마 설계 근거

| 필드 | 근거 |
|------|------|
| `policy` | IPolicyEngine.evaluate() 결과를 그대로 노출. 에이전트가 tier에 따라 사전 판단 가능. |
| `fee` | IChainAdapter.simulateTransaction()의 estimatedFee + getBalance() 기반 계산. null = 시뮬레이션 실패 시. |
| `balanceChanges` | 에이전트가 전송 후 잔액을 예측하여 연쇄 트랜잭션 계획 수립 가능. native + token 모두 포함. |
| `warnings` | 기계 소비용 code + 사람용 message. severity로 에이전트가 심각도 판단. |
| `simulation` | IChainAdapter.simulateTransaction() 원본 결과 투명 노출. 디버깅/로깅 용도. |
| `meta` | 응답 컨텍스트 -- 어떤 체인/네트워크에서, 어떤 타입으로, 얼마나 걸렸는지. |

### 2.3 기존 SimulationResult와의 관계

`@waiaas/core`의 기존 `SimulationResult` 인터페이스(chain-adapter.types.ts)는 **IChainAdapter 내부 타입**이다:

```typescript
// 기존: chain-adapter 4단계 파이프라인 내부용
interface SimulationResult {
  success: boolean;
  logs: string[];
  unitsConsumed?: bigint;
  error?: string;
}
```

새로운 `DryRunSimulationResultSchema`는 **REST API 응답용**으로, 기존 타입을 래핑하면서 정책/수수료/잔액/경고를 추가한다. 기존 타입은 변경하지 않는다.

### 3.3 Stage 분기 설계

dry-run 모드에서는 **Stage 1 → 2 → 3 → 5b**만 실행하고 나머지는 건너뛴다.

```
일반 모드:   Stage 1 → 2 → 3 → 3.5 → 4 → 5(a→b→c→d) → 6
dry-run:     Stage 1' → 2' → 3' → 5'(a→b)
```

#### 3.3.1 각 Stage 동작 비교

| Stage | 일반 모드 | dry-run 모드 | 차이점 |
|-------|-----------|-------------|--------|
| **1 Validate** | Zod 검증 + DB INSERT PENDING | Zod 검증만 (txId 생성 없음, DB 삽입 없음) | DB INSERT 건너뜀 |
| **2 Auth** | sessionId 검증 + 세션 제약 확인 | sessionId 검증만 (usage stats 갱신 없음) | usageStats 업데이트 없음 |
| **3 Policy** | evaluateAndReserve + DB tier UPDATE | evaluate만 (reserve 없음, DB 업데이트 없음) | TOCTOU reserved amount 제외, DB 미갱신 |
| **3.5 Gas** | 가스 조건 확인 + GAS_WAITING | 건너뜀 (dry-run에서 가스 조건 의미 없음) | 완전 스킵 |
| **4 Wait** | DELAY/APPROVAL 큐잉 + PIPELINE_HALTED | 건너뜀 (tier 정보만 응답에 포함) | 완전 스킵 |
| **5a Build** | buildByType() 호출 | buildByType() 호출 (동일) | 동일 |
| **5b Simulate** | adapter.simulateTransaction() | adapter.simulateTransaction() (동일) | 동일 (읽기 전용 RPC 호출) |
| **5c Sign** | decryptPrivateKey + signTransaction | 건너뜀 (개인 키 접근 없음) | 완전 스킵 |
| **5d Submit** | submitTransaction + DB UPDATE | 건너뜀 (온체인 제출 없음) | 완전 스킵 |
| **6 Confirm** | waitForConfirmation | 건너뜀 | 완전 스킵 |

#### 3.3.2 Stage 분기 구현 전략

기존 stage 함수에 `if (ctx.dryRun)` 분기를 추가하는 대신, **별도의 `executeDryRun()` 메서드**를 TransactionPipeline에 추가한다. 이유:

1. 기존 `executeSend()` 경로의 복잡성을 증가시키지 않음
2. dry-run은 동기적 응답 (fire-and-forget 아님) -- `executeSend()`의 비동기 패턴과 다름
3. 테스트 격리 용이

```typescript
// packages/daemon/src/pipeline/pipeline.ts 확장

export class TransactionPipeline {
  // ... 기존 메서드 유지 ...

  /**
   * Dry-run: 트랜잭션 시뮬레이션 (부수 효과 없음).
   *
   * Stage 1' (validate only) → 2' (auth check only) → 3' (policy evaluate only)
   * → 5a (build) → 5b (simulate)
   *
   * DB 삽입/갱신 없음, 온체인 서명/제출 없음, 알림 없음.
   *
   * @returns DryRunSimulationResult
   */
  async executeDryRun(
    walletId: string,
    request: SendTransactionRequest | TransactionRequest,
    sessionId?: string,
  ): Promise<DryRunSimulationResult> {
    const startTime = performance.now();
    const collector = createDryRunCollector(startTime);

    // 1. Wallet 조회
    const wallet = await this.getWallet(walletId);
    if (!wallet) throw new WAIaaSError('WALLET_NOT_FOUND', { ... });

    // 2. 네트워크 해결
    const resolvedNetwork = resolveNetwork(...);

    // 3. Stage 1': Zod 검증만 (DB 삽입 없음)
    stage1ValidateDryRun(request, collector);

    // 4. Stage 2': 세션 검증만 (usage stats 갱신 없음)
    // sessionId가 있으면 세션 유효성 확인
    stage2AuthDryRun(sessionId, collector);

    // 5. Stage 3': 정책 평가만 (reserve 없음, DB 갱신 없음)
    await stage3PolicyDryRun(walletId, request, wallet, collector);

    // 6. 현재 잔액 조회 (잔액 변화 계산용)
    await fetchCurrentBalances(adapter, wallet.publicKey, request, collector);

    // 7. Stage 5a: 빌드 + Stage 5b: 시뮬레이션
    await stage5BuildAndSimulateDryRun(adapter, request, wallet.publicKey, collector);

    // 8. 결과 조합
    return assembleDryRunResult(collector, wallet, resolvedNetwork, request);
  }
}
```

### 3.4 부수 효과 격리 보장 (SIM-02 핵심)

dry-run 모드에서 **절대 발생하지 않아야 하는** 부수 효과 목록:

| 부수 효과 | 보장 방법 |
|-----------|----------|
| DB INSERT (transactions 테이블) | txId 생성 안 함, INSERT 호출 안 함 |
| DB UPDATE (transactions.tier 등) | DB 참조 없음 (collector에 결과 저장) |
| DB evaluateAndReserve | evaluate()만 호출 (evaluateAndReserve 아님) |
| 개인 키 복호화 | Stage 5c 완전 스킵 |
| 온체인 서명 | Stage 5c 완전 스킵 |
| 온체인 제출 | Stage 5d 완전 스킵 |
| 알림 발송 | notificationService 호출 없음 |
| EventBus emit | eventBus.emit 호출 없음 |
| 감사 로그 | auditLog INSERT 없음 |
| DelayQueue 큐잉 | Stage 4 완전 스킵 |
| ApprovalWorkflow 생성 | Stage 4 완전 스킵 |
| GasConditionTracker 등록 | Stage 3.5 완전 스킵 |

**검증 전략**: 구현 시 dry-run 경로에서 `ctx.db.insert()`, `ctx.db.update()`, `notificationService`, `eventBus`, `keyStore.decryptPrivateKey()` 호출이 없음을 단위 테스트로 검증.

### 3.5 정책 평가 경로 (doc 33 읽기 전용)

기존 `DatabasePolicyEngine.evaluate()`는 이미 읽기 전용이다. `evaluateAndReserve()`만 DB 쓰기를 수행한다.

```
// 기존 IPolicyEngine 인터페이스
evaluate(walletId, transaction) → PolicyEvaluation  // 읽기 전용 ✓
evaluateAndReserve(walletId, transaction) → PolicyEvaluation  // DB 쓰기 ✓
```

dry-run에서는 `evaluate()`만 호출하므로 자연스럽게 부수 효과 없음이 보장된다.

단, Stage 3의 USD 금액 환산(resolveEffectiveAmountUsd)은 **가격 오라클 읽기 호출**이므로 dry-run에서도 수행한다. 이는 읽기 전용 외부 API 호출이라 부수 효과 없음.

### 3.6 잔액 변화 계산 로직

```typescript
async function computeBalanceChanges(
  adapter: IChainAdapter,
  walletAddress: string,
  request: TransactionRequest,
  unsignedTx: UnsignedTransaction | null,
  collector: DryRunCollector,
): BalanceChange[] {
  const changes: BalanceChange[] = [];

  // 1. 네이티브 잔액 변화 (전송 금액 + 수수료)
  const nativeBalance = await adapter.getBalance(walletAddress);
  const transferAmount = getRequestAmount(request);
  const fee = unsignedTx?.estimatedFee ?? 0n;

  const nativeChange = -(BigInt(transferAmount) + fee);
  // TOKEN_TRANSFER인 경우 네이티브 변화는 수수료만
  if ('type' in request && request.type === 'TOKEN_TRANSFER') {
    nativeChange = -fee;
  }

  changes.push({
    asset: 'native',
    symbol: NATIVE_SYMBOLS[adapter.chain] ?? 'UNKNOWN',
    decimals: NATIVE_DECIMALS[adapter.chain] ?? 18,
    currentBalance: nativeBalance.balance.toString(),
    changeAmount: nativeChange.toString(),
    afterBalance: (nativeBalance.balance + nativeChange).toString(),
  });

  // 2. 토큰 잔액 변화 (TOKEN_TRANSFER/APPROVE)
  if ('type' in request && request.type === 'TOKEN_TRANSFER') {
    // 토큰 잔액 조회 후 변화 계산
    // ... (getAssets에서 해당 토큰 잔액 추출)
  }

  // 3. 잔액 부족 경고 추가
  if (nativeBalance.balance + nativeChange < 0n) {
    collector.warnings.push({
      code: 'INSUFFICIENT_BALANCE_WITH_FEE',
      message: `Insufficient balance after fee: need ${(-nativeChange).toString()}, have ${nativeBalance.balance.toString()}`,
      severity: 'error',
    });
  }

  return changes;
}
```

## 4. 에러 처리 설계

dry-run은 가능한 한 **에러를 경고로 변환**하여 부분 결과를 반환한다.

| 상황 | 일반 모드 | dry-run 모드 |
|------|-----------|-------------|
| Zod 검증 실패 | 400 에러 throw | 400 에러 throw (동일) |
| 세션 만료 | 401 에러 throw | 401 에러 throw (동일) |
| 지갑 미존재 | WALLET_NOT_FOUND throw | WALLET_NOT_FOUND throw (동일) |
| 정책 거부 (allowed=false) | CANCELLED + DB UPDATE | success=false, policy.allowed=false 반환 |
| 온체인 시뮬레이션 실패 | FAILED + DB UPDATE | success=true, simulation.success=false, warning 추가 |
| RPC 연결 실패 | CHAIN_ERROR throw | fee=null, simulation 빈 결과, warning 추가 |
| 가격 오라클 불가 | NOTIFY 격상 | warning(ORACLE_PRICE_UNAVAILABLE) 추가, 정책은 네이티브 기준 |

**핵심 차이**: 일반 모드에서 에러로 처리되는 시뮬레이션 실패가 dry-run에서는 **부분 성공(success=true) + 경고**로 반환된다. 에이전트는 경고를 보고 판단할 수 있다. 단, 정책이 `allowed=false`이면 `success=false`이다.
