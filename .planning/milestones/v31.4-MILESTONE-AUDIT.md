#### E2E Flow Verification

##### Flow 1: Perp Trading (Open Position)

```
User (MCP/SDK) -> executeAction('hyperliquid_perp', 'hl_open_position', params)
  -> Stage 1 (Validate): Zod schema HlOpenPositionInputSchema
  -> Stage 2 (Auth): session auth
  -> Stage 3 (Policy): getSpendingAmount(margin = size*price/leverage)
  -> Stage 4 (Delay/Approval): riskLevel=high, defaultTier=APPROVAL
  -> Pre-Stage 5: requiresSigningKey=true -> key decrypt
  -> Provider.resolve(): Signer.signL1Action -> ExchangeClient.exchange()
  -> Stage 5: isApiDirectResult=true -> skip on-chain, CONFIRMED status
  -> DB: transactions + hyperliquid_orders INSERT
  -> Stage 6: Notification (TX_CONFIRMED)
```
**Status: COMPLETE** -- all steps wired

##### Flow 2: Spot Trading (Buy)

```
User -> hl_spot_buy -> same pipeline as perp
  -> getSpendingAmount: size*price (no leverage)
  -> Signer.signL1Action (spot asset index = 10000+)
  -> ExchangeClient.exchange()
```
**Status: COMPLETE** -- reuses perp pipeline

##### Flow 3: Sub-account Create

```
User -> hl_create_sub_account -> pipeline
  -> SubAccountProvider.resolve() -> SubAccountService.createSubAccount()
  -> Signer.signUserSignedAction('CreateSubAccount')
  -> ExchangeClient.exchange()
  -> DB: hyperliquid_sub_accounts INSERT
```
**Status: COMPLETE** -- User-Signed Action path wired

##### Flow 4: Query Data (Positions)

```
User (SDK hlGetPositions / MCP waiaas_hl_get_positions)
  -> GET /v1/wallets/:id/hyperliquid/positions
  -> HyperliquidMarketData.getPositions() -> ExchangeClient.info()
  -> Return JSON (no pipeline)
```
**Status: COMPLETE** -- read-only bypass pipeline

##### Flow 5: Admin UI Display

```
Admin UI /hyperliquid page
  -> 5 tabs: Overview, Orders, Spot, Sub-accounts, Settings
  -> Each tab fetches via REST GET endpoints (auto-refresh 10s)
  -> AccountSummary, PositionsTable, OpenOrdersTable, SpotBalancesTable, SpotOrdersTable, SubAccountList, SubAccountDetail, SettingsPanel
```
**Status: COMPLETE** -- all 8 components created, all REST routes mounted

##### Flow 6: connect-info Capability

```
GET /v1/connect-info
  -> Check settings: actions.hyperliquid_enabled
  -> capabilities.push('hyperliquid')
  -> Prompt hints for Perp + Spot + Sub-account
```
**Status: COMPLETE** -- settings-driven capability detection
