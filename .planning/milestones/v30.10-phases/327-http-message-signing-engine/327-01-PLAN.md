    - Derived component @request-target resolves to path + query
  </behavior>
  <action>
    Create `signature-input-builder.ts` implementing two functions:

    1. `buildSignatureInput(params: { coveredComponents: string[], keyid: string, algorithm: string, created: number, expires: number, nonce?: string }): string`
       - Construct RFC 9421 Structured Fields format for Signature-Input header.
       - Format: `sig1=("@method" "@target-uri" ...);created=TIMESTAMP;keyid="erc8128:...";alg="eip191-secp256k1"`
       - Add `;nonce="UUID"` only when nonce is provided (not false/undefined).
       - Add `;expires=TIMESTAMP` always.
       - Use `SIGNATURE_LABEL` from constants.ts for the label name.
       - Quote component identifiers with double quotes inside parentheses.
       - Quote string parameter values (keyid, alg, nonce).
       - Integer parameters (created, expires) are unquoted.

    2. `buildSignatureBase(params: { method: string, url: string, headers: Record<string, string>, coveredComponents: string[], signatureInput: string }): string`
       - Construct RFC 9421 Signature Base (section 2.5) as a multi-line string.
       - For each covered component, emit a line: `"component-id": value\n`
       - Derived components (prefixed with @):
         - `@method` -> uppercase HTTP method
         - `@target-uri` -> full URL string
         - `@authority` -> hostname from URL (lowercase), append :port if non-default (not 80/443)
         - `@request-target` -> path + query string from URL
       - Regular components (no @ prefix): look up value from headers (case-insensitive).
       - Final line: `"@signature-params": <inner-list-portion-of-signature-input>\n`
         where inner-list-portion is everything after `sig1=` in the Signature-Input.
       - Lines are separated by `\n` (single newline, no trailing newline after last line).

    Key implementation details:
    - Do NOT use the `structured-headers` npm package for building (avoid dependency for simple construction). Use string formatting. The `structured-headers` dependency noted in STATE.md can be reconsidered later if parsing is needed.
    - Header lookups must be case-insensitive (lowercase all header keys before lookup).
    - The Signature Base is what gets signed with EIP-191 signMessage.
  </action>
