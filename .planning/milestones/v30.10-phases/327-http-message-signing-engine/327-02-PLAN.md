    Create `http-message-signer.ts` with:

    ```typescript
    interface SignHttpMessageParams {
      method: string;
      url: string;
      headers: Record<string, string>;
      body?: string;
      privateKey: `0x${string}`;  // Hex private key (decrypted by caller)
      chainId: number;
      address: string;  // Wallet address
      coveredComponents?: string[];
      preset?: 'minimal' | 'standard' | 'strict';
      ttlSec?: number;  // default 300
      nonce?: string | false;  // false to disable, undefined for auto-generate
    }

    interface SignHttpMessageResult {
      headers: {
        'Signature-Input': string;
        'Signature': string;
        'Content-Digest'?: string;
      };
      keyid: string;
      algorithm: string;
      created: number;
      expires: number;
      coveredComponents: string[];
    }

    async function signHttpMessage(params: SignHttpMessageParams): Promise<SignHttpMessageResult>
    ```

    Implementation flow:
    1. Resolve coveredComponents: if custom provided use it, else lookup preset from DEFAULT_COVERED_COMPONENTS (default 'standard').
    2. If body exists AND coveredComponents includes 'content-digest': compute Content-Digest via buildContentDigest(body). Add to headers copy.
    3. If coveredComponents includes 'content-digest' but body is undefined/empty: remove 'content-digest' from coveredComponents (auto-adjust for bodyless requests).
    4. Generate nonce: if nonce === false, no nonce. If nonce is a string, use it. If nonce is undefined, generate crypto.randomUUID().
    5. Compute timestamps: created = Math.floor(Date.now() / 1000), expires = created + (ttlSec ?? 300).
    6. Build keyid via buildKeyId(chainId, address).
    7. Build Signature-Input via buildSignatureInput({ coveredComponents, keyid, algorithm: DEFAULT_ALGORITHM, created, expires, nonce }).
    8. Build Signature Base via buildSignatureBase({ method, url, headers (with Content-Digest added), coveredComponents, signatureInput }).
    9. Sign: use viem's `privateKeyToAccount(privateKey)` then `account.signMessage({ message: signatureBase })` for EIP-191. This produces a hex signature.
    10. Encode signature: convert hex to bytes, then base64 encode, wrap as `sig1=:base64:`.
    11. Return headers object + metadata.

    Note: The function takes privateKey directly (the daemon's route handler will decrypt from keystore before calling). This keeps the signer pure -- no keystore dependency.

    Use `import { privateKeyToAccount } from 'viem/accounts'` and `import { Hex } from 'viem'` for signing.

    ```typescript
    interface VerifyHttpSignatureParams {
      method: string;
      url: string;
      headers: Record<string, string>;  // Must include Signature-Input, Signature, and optionally Content-Digest
      body?: string;
    }

    async function verifyHttpSignature(params: VerifyHttpSignatureParams): Promise<VerifyResult>
    ```

    Implementation:
    a. Extract Signature-Input header (case-insensitive lookup). If missing, return { valid: false, error: 'Missing Signature-Input header', recoveredAddress: null, keyid: '' }.
    b. Extract Signature header. If missing, return { valid: false, error: 'Missing Signature header', recoveredAddress: null, keyid: '' }.
    c. Parse Signature-Input to extract covered components and parameters:
       - Parse the inner list: split on `;` to get component list and params.
       - Extract: coveredComponents (from parenthesized list), keyid, alg, created, expires, nonce.
       - This is string parsing (no structured-headers library needed for this format).
    d. Check expiry: if `expires` exists and `expires < Math.floor(Date.now() / 1000)`, return { valid: false, error: 'Signature expired' }.
    e. If 'content-digest' is in coveredComponents and body is provided:
       - Recompute Content-Digest via buildContentDigest(body).
       - Compare with the Content-Digest header value. If mismatch, return { valid: false, error: 'Content-Digest mismatch' }.
    f. Reconstruct Signature Base using buildSignatureBase({ method, url, headers, coveredComponents, signatureInput: <inner-list> }).
    g. Extract signature bytes from Signature header: parse `sig1=:base64:`, decode base64 to hex.
    h. Recover address: use viem's `recoverMessageAddress({ message: signatureBase, signature: hexSignature })`.
    i. Parse keyid via parseKeyId(keyid). Compare recovered address with keyid address (case-insensitive).
    j. If match: return { valid: true, recoveredAddress, keyid }.
    k. If no match: return { valid: false, error: 'Recovered address does not match keyid', recoveredAddress, keyid }.
