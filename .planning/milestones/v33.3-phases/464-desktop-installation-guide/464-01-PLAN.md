문서 구조:

1. **개요** — Desktop App 소개 (Tauri 2 기반, SEA 바이너리 내장, Admin Web UI 통합)

2. **다운로드** — GitHub Releases 페이지(`https://github.com/minhoyoo-iotrust/WAIaaS/releases`)에서 `desktop-v*` 태그 릴리스 찾기. OS별 아티팩트 테이블:
   - macOS: `.dmg` (aarch64 Apple Silicon, x86_64 Intel)
   - Windows: `.msi` (x86_64)
   - Linux: `.AppImage`, `.deb` (x86_64)

3. **macOS 설치** (per IG-01, IG-05)
   - `.dmg` 다운로드 → 마운트 → Applications 폴더로 드래그
   - **Gatekeeper 경고 해제 (macOS 14 이하)**: 시스템 환경설정 → 보안 및 개인정보 보호 → 일반 → "확인 없이 열기" 클릭. 또는 터미널에서 `xattr -cr /Applications/WAIaaS\ Desktop.app`
   - **Gatekeeper 경고 해제 (macOS 15 Sequoia+)**: System Settings → Privacy & Security → 스크롤 다운하여 "WAIaaS Desktop" 항목 찾기 → "Open Anyway" 클릭. Sequoia에서는 처음 실행 시 "열 수 없습니다" 대화상자가 표시되며, Security 설정에서 허용해야 함을 명시. `xattr -cr` 명령도 여전히 유효.

4. **Windows 설치** (per IG-02, IG-05)
   - `.msi` 다운로드 → 실행
   - **SmartScreen 경고**: "Windows에서 PC를 보호함" 대화상자 → "추가 정보" 클릭 → "실행" 클릭
   - 설치 위치: `C:\Program Files\WAIaaS Desktop\`
   - 시작 메뉴에서 "WAIaaS Desktop" 검색하여 실행

5. **Linux 설치** (per IG-03, IG-05)
   - **AppImage**: 다운로드 → `chmod +x WAIaaS-Desktop_*.AppImage` → 실행. FUSE 필요 시 `sudo apt install libfuse2` (Ubuntu 22.04+)
   - **deb**: `sudo dpkg -i waiaas-desktop_*.deb` → `sudo apt install -f` (의존성 해결)
   - 데스크톱 환경에서 애플리케이션 메뉴에 자동 등록

6. **Setup Wizard** (per IG-04)
   - 5단계 안내:
     1. **마스터 비밀번호 설정**: 데몬 관리용 비밀번호 생성 (Argon2id 해시)
     2. **네트워크 선택**: EVM/Solana 네트워크 선택 및 RPC URL 설정
     3. **지갑 생성**: 첫 번째 지갑 생성 (HD 키 자동 생성)
     4. **Owner 설정**: Owner 등록 방법 선택 (WalletConnect, 건너뛰기 가능)
     5. **완료**: 대시보드로 이동, 시스템 트레이 아이콘 확인
   - Setup Wizard는 최초 실행 시에만 표시됨
   - 완료 후 시스템 트레이에 3색 아이콘(초록=실행 중, 노랑=시작 중, 빨강=오류)이 표시됨

7. **자동 업데이트** (per IG-06)
   - Ed25519 서명 검증 방식 설명: Tauri updater가 GitHub Releases의 `latest.json` 엔드포인트 확인
   - 업데이트 알림 시 자동 다운로드 + 서명 검증 + 설치
   - 수동 업그레이드: GitHub Releases에서 최신 버전 다운로드 → 기존 앱 덮어쓰기 설치
   - 업데이트 엔드포인트: `https://github.com/minhoyoo-iotrust/WAIaaS/releases/latest/download/latest.json`

8. **트러블슈팅**
   - macOS: "손상된 파일" 오류 → `xattr -cr` 명령
   - Windows: MSI 설치 실패 → 관리자 권한으로 실행
   - Linux: AppImage "permission denied" → `chmod +x` 확인
   - 데몬 시작 실패 → 포트 3100 사용 중 여부 확인 (`lsof -i :3100` / `netstat -ano | findstr 3100`)
   - 로그 위치 안내
