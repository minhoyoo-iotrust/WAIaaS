2. In `stage5ExecuteSmartAccount` (stages.ts), implement explicit 120% gas safety margin per project CLAUDE.md rule `(estimatedGas * 120n) / 100n`:

   Before calling `bundlerClient.sendUserOperation()`, use `bundlerClient.prepareUserOperation()` to get the Bundler's gas estimates, then apply the 120% safety margin:

```typescript
// Step 1: Prepare the UserOperation to get Bundler gas estimates
const prepared = await bundlerClient.prepareUserOperation({ calls });

// Step 2: Apply 120% safety margin per project rule (CLAUDE.md)
const safeCallGasLimit = (prepared.callGasLimit * 120n) / 100n;
const safeVerificationGasLimit = (prepared.verificationGasLimit * 120n) / 100n;
const safePreVerificationGas = (prepared.preVerificationGas * 120n) / 100n;

// Step 3: Submit with overridden gas limits
const userOpHash = await bundlerClient.sendUserOperation({
  calls,
  userOperation: {
    callGasLimit: safeCallGasLimit,
    verificationGasLimit: safeVerificationGasLimit,
    preVerificationGas: safePreVerificationGas,
  },
});
```

   This ensures the project-wide gas safety margin rule is applied explicitly, regardless of whether the Bundler service includes its own padding. The `prepareUserOperation` call triggers `eth_estimateUserOperationGas` under the hood, and we apply our `(estimated * 120n) / 100n` on top.
