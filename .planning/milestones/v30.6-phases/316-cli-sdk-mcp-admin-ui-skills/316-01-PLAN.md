     ```typescript
     export interface CreateWalletResponse {
       id: string;

       chain: string;

       publicKey: string;

       accountType: string;
     ```

   - Add to existing `WalletInfoResponse` interface: `accountType?: string`, `signerKey?: string | null`, `deployed?: boolean`.
   - Add to existing `ConnectInfoWallet` interface: `accountType?: string`.

2. In `packages/sdk/src/client.ts`:
   - Import new types: `CreateWalletParams`, `CreateWalletResponse`.
   - Add `createWallet()` method to `WAIaaSClient`. This method requires masterAuth (X-Master-Password header), which the SDK does not currently support via session token. Instead, add a `masterPassword` option:
     - Add `private masterPassword?: string` to the class.
     - Add `masterPassword?: string` to `WAIaaSClientOptions`.
     - In constructor, store `this.masterPassword = options.masterPassword`.
     - `createWallet()` method:
       ```typescript
       async createWallet(params: CreateWalletParams): Promise<CreateWalletResponse> {
         if (!this.masterPassword) {
           throw new WAIaaSError('MASTER_PASSWORD_REQUIRED', 'createWallet requires masterPassword in client options');
         }
         const body: Record<string, unknown> = {
           name: params.name,
         };
         if (params.chain) body.chain = params.chain;
         if (params.environment) body.environment = params.environment;
         if (params.accountType) body.accountType = params.accountType;
         if (params.createSession !== undefined) body.createSession = params.createSession;
         return this.http.request<CreateWalletResponse>('POST', '/v1/wallets', body, {
           'X-Master-Password': this.masterPassword,
         });
       }
       ```
     - NOTE: The http.request method needs to accept optional extra headers. Check if HttpClient already supports this. If not, add an optional `headers` parameter to the `request` method in `internal/http.ts`.
