    **Strategy decision (per WCON-05):** Use Plan B approach (daemon REST API WC pair) as the primary implementation. The spike (Phase 459) was PENDING manual verification and @reown/appkit adds significant bundle size + CSP complexity. The daemon already has full WalletConnect support via `/v1/wallets/{id}/wc/pair` REST API. This approach:
    - Zero additional npm dependencies (no @reown/appkit in admin bundle)
    - Reuses existing walletconnect.tsx page patterns (proven working)
    - No CSP additions needed (all WC traffic handled server-side by daemon)
    - Works identically in Desktop and browser (but we guard with isDesktop() per WCON-04)

    **wc-types.ts**:
    ```typescript
    export interface WcPairingState {
      status: 'idle' | 'pairing' | 'waiting' | 'connected' | 'expired' | 'error';
      qrCodeDataUrl: string | null;  // base64 QR image from API
      uri: string | null;            // WC URI for deep-link
      expiresAt: number | null;
      error: string | null;
      ownerAddress: string | null;
    }

    export interface WcConnectionResult {
      success: boolean;
      ownerAddress?: string;
      chain?: string;
      error?: string;
    }
    ```

    **wc-connector.ts**:
    - Export `connectViaWalletConnect(walletId: string, masterPassword: string): Promise<WcConnectionResult>`
    - Implementation flow:
      1. Call `POST /v1/wallets/{id}/wc/pair` with masterAuth header -> get `{ uri, qrCode, expiresAt }`
      2. Expose QR data via a Preact signal `pairingState` (type `Signal<WcPairingState>`) so the modal can reactively display it
      3. Poll `GET /v1/wallets/{id}/wc/pair/status` every 3 seconds (same as walletconnect.tsx)
      4. When status === 'connected', return `{ success: true, ownerAddress: session.ownerAddress, chain: session.chainId }`
      5. When status === 'expired', return `{ success: false, error: 'Pairing expired' }`
      6. Timeout after 5 minutes (100 polls)
    - Export `pairingState` signal for reactive UI
    - Export `cancelPairing()` to clear interval and reset state
    - All fetch calls use `X-Master-Password` header from the provided masterPassword parameter
    - IMPORTANT: This file is in desktop/ directory but does NOT import @tauri-apps. It uses standard fetch API. The isDesktop() guard is applied at the consumer level (wizard/app.tsx), not here.

    **wc-qr-modal.tsx**:
    - Preact component: `WcQrModal({ open, onClose, onConnected })`
    - Props: `open: boolean`, `onClose: () => void`, `onConnected: (result: WcConnectionResult) => void`
    - Uses `Modal` component from `../../components/modal`
    - Reads `pairingState` signal to show:
      - 'pairing': spinner "Starting pairing..."
      - 'waiting': QR code image (280x280) + "Scan with Phantom, MetaMask, or WalletConnect-compatible wallet" + "Waiting for connection..." subtext
      - 'connected': success checkmark + owner address display, auto-close after 2s
      - 'expired': error message + "Try Again" button
      - 'error': error message + "Try Again" button
    - On close, calls `cancelPairing()` + `onClose()`
    - Style: centered QR, project CSS variables, consistent with existing Modal usage in walletconnect.tsx
