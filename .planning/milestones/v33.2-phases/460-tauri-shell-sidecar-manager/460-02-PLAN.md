1. `packages/daemon/scripts/build-sea.mjs` 생성 (ESM 스크립트):
   - esbuild로 데몬 entry point를 단일 JS 파일로 번들:
     * entryPoint: `src/index.ts` (또는 실제 데몬 진입점 확인 필요)
     * format: "cjs" (SEA는 CJS만 지원)
     * platform: "node"
     * target: "node22"
     * bundle: true
     * external: ["sodium-native", "better-sqlite3", "argon2"] (native addon은 번들 제외)
     * outfile: "dist/daemon-bundle.cjs"
   - 번들 후 native addon .node 파일을 SEA assets로 복사:
     * sodium-native: `node_modules/sodium-native/prebuilds/{platform}-{arch}/node.napi.node` (경로 확인 필요)
     * better-sqlite3: `node_modules/better-sqlite3/prebuilds/{platform}-{arch}/node.napi.node`
     * argon2: `node_modules/argon2/lib/binding/napi-v3-{platform}-{arch}/argon2.node`
   - Node.js SEA blob 생성: `node --experimental-sea-config sea-config.json`
   - 호스트 node 바이너리 복사 + blob 주입:
     * macOS: `cp $(which node) dist/waiaas-daemon && codesign --remove-signature dist/waiaas-daemon && npx postject dist/waiaas-daemon NODE_SEA_BLOB dist/sea-prep.blob --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6e072b8b5df1996b2 --macho-segment-name NODE_SEA && codesign --sign - dist/waiaas-daemon`
     * Linux: `cp $(which node) dist/waiaas-daemon && npx postject dist/waiaas-daemon NODE_SEA_BLOB dist/sea-prep.blob --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6e072b8b5df1996b2`
     * Windows: `node -e "require('fs').copyFileSync(process.execPath, 'dist/waiaas-daemon.exe')" && npx postject dist/waiaas-daemon.exe NODE_SEA_BLOB dist/sea-prep.blob --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6e072b8b5df1996b2`
   - 최종 바이너리를 Tauri externalBin 위치로 복사:
     * `apps/desktop/src-tauri/binaries/waiaas-daemon-{target_triple}`
     * target_triple은 현재 플랫폼 자동 감지: `aarch64-apple-darwin`, `x86_64-unknown-linux-gnu`, etc.
   - 스크립트에 `--target` 옵션 지원 (크로스 컴파일용, 기본값: 현재 플랫폼)

2. `packages/daemon/sea-config.json` 생성 (설계 문서 39 섹션 4.1.1 준수):
   ```json
   {
     "main": "dist/daemon-bundle.cjs",
     "output": "dist/sea-prep.blob",
     "disableExperimentalSEAWarning": true,
     "useCodeCache": true,
     "assets": {
       "sodium-native.node": "native-addons/sodium-native.node",
       "better_sqlite3.node": "native-addons/better_sqlite3.node",
       "argon2.node": "native-addons/argon2.node"
     }
   }
   ```
   assets 경로는 build-sea.mjs가 prebuild에서 native-addons/ 디렉토리로 복사한 후의 상대 경로.

3. `packages/daemon/src/infrastructure/native-loader.ts` 생성 (설계 문서 39 섹션 4.1.1 준수):
   - `loadNativeAddon(assetName, fallbackRequire)` 함수 구현:
     * `sea.isSea()` 체크: true면 SEA assets에서 .node 파일 추출 -> tmpdir에 기록 -> `process.dlopen()` 로딩
     * false면 fallbackRequire() 호출 (일반 개발 환경)
     * 추출 파일 경로: `{tmpdir}/waiaas-{assetName}-{pid}` (PID 포함으로 동시 실행 충돌 방지)
     * finally 블록에서 추출 파일 정리 (rmSync, 에러 무시)
   - 주의: `node:sea` import는 조건부로 해야 함. 비-SEA 환경에서는 `node:sea` 모듈이 없을 수 있으므로 dynamic import + try/catch로 감싸기:
     ```typescript
     let seaModule: { isSea: () => boolean; getRawAsset: (name: string) => ArrayBuffer } | null = null;
     try {
       seaModule = await import('node:sea');
     } catch {
       // Not running as SEA — use normal require
     }
     ```
   - 내보내는 객체: `{ sodiumNative, betterSqlite3, argon2Native }`
   - 기존 코드에서 `sodium-native`, `better-sqlite3`, `argon2`를 직접 import하는 부분은 이번 플랜에서 수정하지 않음 (native-loader.ts를 참조하도록 변경하는 것은 별도 작업). 이번에는 native-loader.ts 파일 생성만 수행.

4. `packages/daemon/src/lifecycle/daemon-startup.ts` 수정:
   - 서버 시작 직후 (`@hono/node-server`의 `serve()` 반환 후) 실제 할당된 포트를 stdout에 출력하는 코드 추가:
     ```typescript
     const actualPort = (server.address() as AddressInfo).port;
     // Sidecar Manager가 파싱하는 포트 프로토콜 (설계 문서 39 섹션 4.2.1)
     console.log(`WAIAAS_PORT=${actualPort}`);
     // Fallback: 파일에도 기록
     writeFileSync(join(dataDir, 'daemon.port'), String(actualPort));
     ```
   - `port = 0` 지원을 확인: 기존 config에서 port를 읽어 서버에 전달하는 부분에서 0이 허용되는지 확인하고, 필요시 수정.
   - 주의: 기존 startup 로직의 다른 부분은 변경하지 않음. 포트 출력 코드만 추가.

5. `packages/daemon/package.json` 수정:
   - scripts에 "build:sea" 추가: `"build:sea": "node scripts/build-sea.mjs"`
   - devDependencies에 esbuild, postject 추가 (이미 있으면 스킵)

주의사항:
- SEA 바이너리 생성은 현재 플랫폼에서만 가능 (크로스 컴파일은 CI에서 각 플랫폼별로 실행)
- native addon prebuild 경로는 패키지 버전에 따라 다를 수 있음. build-sea.mjs에서 glob으로 찾는 패턴 사용
- argon2의 prebuild 경로가 sodium-native/better-sqlite3와 다름 (lib/binding/ vs prebuilds/) -- 설계 문서 39 주의사항 참조
- `node:sea` 모듈은 Node.js 21+에서만 사용 가능. package.json engines에 "node": ">=22" 확인
