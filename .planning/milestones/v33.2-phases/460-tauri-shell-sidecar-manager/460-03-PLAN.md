   - 스플래시 화면에 상태 진행 표시:
     * "Starting WAIaaS daemon..." (초기)
     * "Waiting for port assignment..." (spawn 후)
     * "Connecting to daemon..." (포트 발견 후, health check 중)
     * "Loading Admin Dashboard..." (health OK 후)
   - 에러 상태 표시:
     * "Failed to start daemon: {error}" (빨간 텍스트)
     * "Retry" 버튼 (클릭 시 Tauri IPC invoke('start_daemon') 호출)
   - Tauri event listener로 Rust에서 상태 업데이트 수신:
     ```javascript
     // Tauri 2.x event API -- splash.html은 로컬 파일이므로 window.__TAURI_INTERNALS__ 직접 사용
     if (window.__TAURI_INTERNALS__) {
       const { listen } = window.__TAURI_INTERNALS__;
       listen('sidecar-status', (event) => { updateStatus(event.payload); });
     }
     ```
   - 주의: splash.html은 로컬 파일(tauri:// 프로토콜)이므로 Tauri IPC가 기본 동작. Remote WebView와 달리 별도 capability 불필요.
   - 스타일: 중앙 정렬, WAIaaS 로고(텍스트), CSS-only 스피너, 다크 배경(#1a1a2e)

2. `apps/desktop/src-tauri/src/sidecar.rs` 수정:
   - `start()` 메서드에 AppHandle 매개변수 추가 (이미 있으면 확인)
   - 각 단계에서 Tauri event emit으로 스플래시에 상태 전달:
     ```rust
     app.emit("sidecar-status", json!({ "stage": "spawning" })).ok();
     // ... spawn ...
     app.emit("sidecar-status", json!({ "stage": "port_discovery", "port": port })).ok();
     // ... health check ...
     app.emit("sidecar-status", json!({ "stage": "ready", "port": port })).ok();
     // ... 에러 시 ...
     app.emit("sidecar-status", json!({ "stage": "error", "message": err_msg })).ok();
     ```

3. `apps/desktop/src-tauri/src/main.rs` 수정:
   - setup 콜백에서 데몬 시작 + navigate 시퀀스를 비동기로 실행:
     ```rust
     let app_handle = app.handle().clone();
     tauri::async_runtime::spawn(async move {
         let manager = app_handle.state::<SidecarManager>();
         match manager.start(&app_handle).await {
             Ok(status) => {
                 let url = format!("http://127.0.0.1:{}/admin", status.port);
                 if let Some(window) = app_handle.get_webview_window("main") {
                     window.navigate(url.parse().unwrap());
                 }
             }
             Err(e) => {
                 app_handle.emit("sidecar-status", json!({
                     "stage": "error",
                     "message": e
                 })).ok();
             }
         }
     });
     ```
   - on_window_event에서 CloseRequested 시:
     * SidecarManager.stop(false) 호출 (graceful shutdown)
     * 종료 전 대기 (최대 40초, 설계 문서 39 기준)
   - 주의: `window.navigate()`는 `tauri::Url`을 받음. `url.parse::<tauri::Url>()` 사용.

4. 에러 복구 플로우:
   - 스플래시 화면에서 "Retry" 버튼 클릭 시 `start_daemon` IPC 호출
   - 성공 시 스플래시에서 Admin Web UI로 전환
