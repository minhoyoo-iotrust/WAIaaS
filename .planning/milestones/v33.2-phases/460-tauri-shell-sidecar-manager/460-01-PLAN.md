Rust types (src-tauri/src/types.rs):
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DaemonStatus {
    pub running: bool,
    pub pid: Option<u32>,
    pub port: u16,
    pub uptime_secs: Option<u64>,
    pub health: Option<HealthResponse>,
    pub restart_count: u32,
    pub state: DaemonState,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum DaemonState { Stopped, Starting, Running, Stopping, Crashed, Error }

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct HealthResponse {
    pub status: String,      // "ok" | "degraded" | "error"
    pub version: String,
    pub uptime: u64,
    pub kill_switch: String, // "NORMAL" | "ACTIVATED" | "RECOVERING"
}
```

IPC 커맨드 6개:
- start_daemon -> Result<DaemonStatus, String>
- stop_daemon(StopDaemonArgs) -> Result<(), String>
- restart_daemon -> Result<DaemonStatus, String>
- get_daemon_status -> Result<DaemonStatus, String>
- get_daemon_logs(GetLogsArgs) -> Result<Vec<String>, String>
- send_notification(NotificationArgs) -> Result<(), String>

포트 발견 프로토콜:
- stdout 파싱: /^WAIAAS_PORT=(\d+)$/ (primary, 10초 타임아웃)
- 파일 폴백: {data_dir}/daemon.port (fallback)

   - [dependencies]: tauri (features = ["devtools"]), tauri-plugin-notification, tauri-plugin-process, tauri-plugin-shell, serde, serde_json, tokio (features = ["full"]), reqwest (features = ["json"]), log

   - identifier: "dev.waiaas.desktop"
   - app.windows[0].url: "splash.html" (스플래시 페이지, SidecarManager가 준비 완료 후 동적 URL로 전환)
   - app.windows[0].title: "WAIaaS Desktop"
   - app.windows[0].width: 1280, height: 800

   - bundle.identifier: "dev.waiaas.desktop"

   - remote.urls: ["http://127.0.0.1:*/*", "http://localhost:*/*"]
   - permissions: core:default, waiaas:allow-* 6개, shell:allow-execute (sidecar만), notification:*, updater:*, process:*

6. `apps/desktop/src-tauri/src/types.rs` 생성:
   - DaemonStatus, DaemonState, HealthResponse, StartDaemonArgs, StopDaemonArgs, GetLogsArgs, NotificationArgs 구조체/enum 정의
   - 모두 Serialize + Deserialize derive

7. `apps/desktop/src-tauri/src/lockfile.rs` 생성:
   - PID lockfile 관리: create_lockfile(data_dir) -> Result<()>, remove_lockfile(data_dir), check_lockfile(data_dir) -> Option<u32>
   - lockfile 경로: {data_dir}/daemon.pid
   - 기존 PID가 있으면 프로세스 alive 체크(kill(pid, 0)), alive이면 Err("AlreadyRunning"), dead이면 stale lockfile 제거 후 새로 생성

8. `apps/desktop/src-tauri/src/sidecar.rs` 생성 (설계 문서 39 섹션 4.2-4.3 준수):
   - `SidecarManager` 구조체: child(Mutex<Option<CommandChild>>), status(Mutex<DaemonStatus>), log_buffer(Mutex<VecDeque<String>>), data_dir(String)
   - `start()`: (SIDE-01, SIDE-02)
     * PID lockfile 체크 (SIDE-05)
     * tauri::api::process::Command::new_sidecar("waiaas-daemon") 또는 shell plugin의 Command::sidecar()로 spawn
     * args: ["--port=0", "--data-dir={data_dir}"]
     * stdout 이벤트 리스너에서 `WAIAAS_PORT=(\d+)` 정규식 파싱 (10초 타임아웃)
     * 파싱 실패 시 {data_dir}/daemon.port 파일에서 폴백 읽기
     * /health 폴링으로 준비 완료 대기 (최대 30초, 2초 간격)
     * log_buffer에 stdout/stderr 라인 저장 (최대 1000줄, VecDeque 사용)
   - `stop(force: bool)`: (SIDE-04)
     * force=false: POST http://127.0.0.1:{port}/v1/admin/shutdown -> 35초 대기 -> SIGTERM -> 5초 -> SIGKILL
     * force=true: 즉시 SIGKILL
     * PID lockfile 제거
   - `check_health()`: GET /health (2초 타임아웃), 2회 연속 실패 시 Crashed 판정
   - `handle_crash()`: (SIDE-03) restart_count < 3이면 5초 대기 후 자동 재시작, 3회 초과 시 Error 상태
   - `get_logs(lines: u32)`: log_buffer에서 최근 N줄 반환
   - Windows Job Object (SIDE-06): `#[cfg(windows)]` 블록에서 windows-rs crate로 Job Object 생성, JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE 플래그 설정, 자식 프로세스를 Job에 할당. Non-Windows에서는 no-op.

9. `apps/desktop/src-tauri/src/commands.rs` 생성 (설계 문서 39 섹션 3.6, 4.4 준수):
   - 6개 #[tauri::command] async fn 정의
   - start_daemon: SidecarManager.start() 호출
   - stop_daemon(args: StopDaemonArgs): SidecarManager.stop(args.force) 호출
   - restart_daemon: stop(false) -> 1초 대기 -> start()
   - get_daemon_status: SidecarManager.check_health() 호출
   - get_daemon_logs(args: GetLogsArgs): SidecarManager.get_logs(args.lines) 호출
   - send_notification(args: NotificationArgs, app: AppHandle): tauri-plugin-notification으로 OS 알림 전송

10. `apps/desktop/src-tauri/src/main.rs` 생성:
    - SidecarManager 인스턴스를 Tauri managed state로 등록
    - invoke_handler에 6개 커맨드 등록
    - plugin 등록: tauri-plugin-notification, tauri-plugin-process, tauri-plugin-shell
    - setup 콜백에서:
      * data_dir 결정 (app.path().app_data_dir())
      * Remote WebView capability 설정 (CapabilityBuilder, #[cfg(not(dev))] 가드) -- 설계 문서 39 섹션 3.7
      * 자동 시작: SidecarManager.start() 호출
      * 포트 발견 후 main window navigate: window.navigate(format!("http://127.0.0.1:{port}/admin"))
    - on_window_event에서 CloseRequested 시 SidecarManager.stop() 호출

    - 심플한 스플래시 화면 ("WAIaaS Desktop - Starting daemon...")
