1. **apps/desktop/scripts/build-sea-ci.sh** 작성:
   CI에서 SEA 바이너리를 빌드하고 올바른 target triple 이름으로 복사하는 스크립트.
   ```bash
   #!/bin/bash
   set -euo pipefail
   # Usage: ./build-sea-ci.sh <target-triple>
   # Example: ./build-sea-ci.sh aarch64-apple-darwin
   TARGET_TRIPLE="${1:?Usage: build-sea-ci.sh <target-triple>}"

   # Step 1: Build SEA binary using existing build-sea.mjs
   cd packages/daemon
   node scripts/build-sea.mjs

   # Step 2: Copy to Tauri externalBin with correct target triple name
   DEST="../../apps/desktop/src-tauri/binaries/waiaas-daemon-${TARGET_TRIPLE}"
   if [[ "$TARGET_TRIPLE" == *"windows"* ]]; then
     cp dist/waiaas-daemon.exe "${DEST}.exe"
   else
     cp dist/waiaas-daemon "${DEST}"
     chmod +x "${DEST}"
   fi
   echo "SEA binary ready: ${DEST}"
   ```
   `chmod +x` 부여.

2. **.github/workflows/desktop-release.yml** 작성:

   **트리거**: `desktop-v*` 태그 push + workflow_dispatch (수동)

   **Jobs 구조:**

   **Job 1: create-release**
   - `runs-on: ubuntu-latest`
   - `actions/github-script`로 GitHub Release 생성 (draft: true)
   - 태그에서 버전 추출 (`desktop-v0.1.0` -> `0.1.0`)
   - outputs: `release_id`

   **Job 2: build-tauri (matrix)**
   - `needs: create-release`
   - Matrix 설정:
     ```yaml
     matrix:
       include:
         - platform: macos-latest
           target: aarch64-apple-darwin
           rust_target: aarch64-apple-darwin
           args: --target aarch64-apple-darwin
         - platform: macos-latest
           target: x86_64-apple-darwin
           rust_target: x86_64-apple-darwin
           args: --target x86_64-apple-darwin
         - platform: ubuntu-22.04
           target: x86_64-unknown-linux-gnu
           rust_target: x86_64-unknown-linux-gnu
           args: ""
         - platform: windows-latest
           target: x86_64-pc-windows-msvc
           rust_target: x86_64-pc-windows-msvc
           args: ""
     ```
   - Steps:
     a. `actions/checkout@v4`
     b. `actions/setup-node@v4` (Node 22)
     c. `pnpm install` (for daemon build)
     d. `dtolnay/rust-toolchain@stable` + target 추가
     e. Linux만: `sudo apt-get update && sudo apt-get install -y libwebkit2gtk-4.1-dev libgtk-3-dev libayatana-appindicator3-dev librsvg2-dev`
     f. SEA 바이너리 빌드: `bash apps/desktop/scripts/build-sea-ci.sh ${{ matrix.target }}`
     g. `tauri-apps/tauri-action@v0` 호출:
        ```yaml
        with:
          projectPath: apps/desktop
          tauriScript: pnpm tauri
          args: ${{ matrix.args }}
          releaseId: ${{ needs.create-release.outputs.release_id }}
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
          TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
          # macOS code signing (optional, skipped if not set)
          APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
          APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
          APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
          APPLE_ID: ${{ secrets.APPLE_ID }}
          APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
          APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
        ```

   **Job 3: publish-release**
   - `needs: build-tauri`
   - `runs-on: ubuntu-latest`
   - `actions/github-script`로 Release를 draft: false로 전환

   중요: tauri-action이 자동으로 latest.json + `.sig` 서명 파일을 생성하여 Release에 업로드. `createUpdaterArtifacts: "v1Compatible"`이 tauri.conf.json에 있으면 자동 동작.

3. DIST-03 (macOS 코드 사이닝) 관련: tauri-action이 `APPLE_CERTIFICATE`, `APPLE_SIGNING_IDENTITY`, `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID` 환경변수가 설정되면 자동으로 코드 사이닝 + notarization을 수행한다. 시크릿이 없으면 서명 없이 빌드만 진행 (개발 단계에서는 이것이 정상).
