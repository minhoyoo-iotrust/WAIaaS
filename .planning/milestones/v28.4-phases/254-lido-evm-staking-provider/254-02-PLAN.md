**Update `packages/actions/src/index.ts`:**

1. Add import and re-export for LidoStakingActionProvider:
   ```typescript
   export { LidoStakingActionProvider } from './providers/lido-staking/index.js';
   export { LIDO_STAKING_DEFAULTS, LIDO_MAINNET_ADDRESSES, LIDO_TESTNET_ADDRESSES, getLidoAddresses } from './providers/lido-staking/config.js';
   export type { LidoStakingConfig } from './providers/lido-staking/config.js';
   ```

2. Add lido_staking entry in the `providers` array inside `registerBuiltInProviders()`:
   ```typescript
   {
     key: 'lido_staking',
     enabledKey: 'actions.lido_staking_enabled',
     factory: () => {
       // Resolve environment from SettingsService: mainnet addresses by default,
       // testnet (Holesky) addresses when environment is 'testnet'
       const envSetting = settingsReader.get('environment') ?? 'mainnet';
       const isTestnet = envSetting === 'testnet';
       const { getLidoAddresses } = require_dynamic_lido();

       const addresses = getLidoAddresses(isTestnet ? 'testnet' : 'mainnet');
       const config: LidoStakingConfig = {
         enabled: true,
         stethAddress: settingsReader.get('actions.lido_staking_steth_address') || addresses.stethAddress,
         withdrawalQueueAddress: settingsReader.get('actions.lido_staking_withdrawal_queue_address') || addresses.withdrawalQueueAddress,
       };
       return new LidoStakingActionProvider(config);
     },
   },
   ```

   Note: Since registerBuiltInProviders already imports from the same package, use direct import at top of file (not dynamic). The environment resolution follows the existing pattern: SettingsReader.get() returns string for environment setting. If the Admin Settings override an address, use that; otherwise use the environment-derived default.

   IMPORTANT: Use the same factory pattern as lifi/zerox_swap/jupiter_swap. Read steth_address and withdrawal_queue_address from SettingsReader — if the value is empty string (default), fall back to getLidoAddresses() based on environment. This allows Admin to override individual addresses while defaulting to environment-appropriate ones.
