# Requirements Archive: v26.3 Push Relay Server

**Archived:** 2026-02-20

## v1 Requirements (All Complete)

### ENCODE — 데몬 SignRequest 인코딩 통일

- [x] **ENCODE-01**: NtfySigningChannel이 SignRequest를 base64url 인코딩하여 ntfy message 필드에 publish한다
- [x] **ENCODE-02**: NtfySigningChannel이 displayMessage를 ntfy title 필드에 설정한다
- [x] **ENCODE-03**: wallet-sdk subscribeToRequests()가 변경된 인코딩으로 정상 파싱한다

### SUB — ntfy SSE 구독

- [x] **SUB-01**: Relay가 설정된 wallet_names의 서명+알림 토픽을 SSE로 구독한다
- [x] **SUB-02**: Relay가 SSE 연결 끊김 시 지수 백오프로 자동 재연결한다
- [x] **SUB-03**: Relay가 ntfy message를 base64url 디코딩 + Zod 검증으로 파싱한다
- [x] **SUB-04**: Relay가 토픽 패턴으로 서명 요청과 알림을 구분한다

### PUSH — Push Provider

- [x] **PUSH-01**: PushwooshProvider가 createMessage API로 푸시를 전송한다
- [x] **PUSH-02**: FcmProvider가 FCM v1 API로 푸시를 전송한다
- [x] **PUSH-03**: Push data 필드에 전체 SignRequest/NotificationMessage JSON이 포함된다
- [x] **PUSH-04**: ntfy priority 5 → Push high, priority 3 → Push normal로 매핑된다
- [x] **PUSH-05**: 5xx 응답 시 지수 백오프 3회 재시도한다
- [x] **PUSH-06**: 인증 실패(401/403) 시 재시도 없이 에러 로그한다
- [x] **PUSH-07**: invalidTokens를 DB에서 자동 삭제한다

### REG — Device Token Registry

- [x] **REG-01**: POST /devices로 walletName+pushToken+platform 디바이스를 등록한다
- [x] **REG-02**: DELETE /devices/:token으로 디바이스를 해제한다
- [x] **REG-03**: 동일 pushToken 재등록 시 upsert로 처리한다
- [x] **REG-04**: X-API-Key 없는 요청은 401로 거부한다
- [x] **REG-05**: GET /health로 ntfy 연결 상태와 Push 프로바이더 상태를 반환한다

### INFRA — 배포 인프라

- [x] **INFRA-01**: @waiaas/push-relay npm 패키지로 빌드/발행한다
- [x] **INFRA-02**: Docker 이미지(waiaas/push-relay)를 빌드한다
- [x] **INFRA-03**: release-please-config.json에 push-relay 패키지를 등록한다
- [x] **INFRA-04**: release.yml에 push-relay npm publish + Docker push job을 추가한다
- [x] **INFRA-05**: config.toml 로딩 + Zod 검증으로 설정을 관리한다
- [x] **INFRA-06**: SIGTERM/SIGINT 시 graceful shutdown을 수행한다
