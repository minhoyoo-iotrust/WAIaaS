1. **smart-account-clients.ts**: Major refactor.

   a. Define `WalletProviderData` interface:
   ```typescript
   export interface WalletProviderData {
     aaProvider: AaProviderName | null;
     aaProviderApiKey: string | null;  // decrypted
     aaBundlerUrl: string | null;      // for custom
     aaPaymasterUrl: string | null;    // for custom
   }
   ```

   b. Replace `resolveBundlerUrl(settingsService, networkId)` with `resolveWalletBundlerUrl(wallet: WalletProviderData, networkId: string): string`:
   - If `wallet.aaProvider` is null, throw CHAIN_ERROR 'Provider not configured for this wallet'.
   - If `wallet.aaProvider` is 'custom', return `wallet.aaBundlerUrl` (throw if null).
   - If `wallet.aaProvider` is 'pimlico'|'alchemy': call `resolveProviderChainId(provider, networkId)`. If null, throw CHAIN_ERROR with `Provider ${provider} does not support network ${networkId}`. Then call `buildProviderBundlerUrl(provider, chainId, apiKey)`.

   c. Replace `resolvePaymasterUrl(settingsService, networkId)` with `resolveWalletPaymasterUrl(wallet: WalletProviderData, networkId: string): string | null`:
   - If `wallet.aaProvider` is null, return null.
   - If `wallet.aaProvider` is 'custom', return `wallet.aaPaymasterUrl` (nullable).
   - If `wallet.aaProvider` is 'pimlico'|'alchemy': same URL as bundler (unified endpoint).

   d. Update `BundlerClientOptions` interface: replace `settingsService: SettingsService` with `walletProvider: WalletProviderData`. Keep client, account, networkId.

   e. Update `createSmartAccountBundlerClient`: use `resolveWalletBundlerUrl(opts.walletProvider, opts.networkId)` and `resolveWalletPaymasterUrl(opts.walletProvider, opts.networkId)`.

   f. Keep old `resolveBundlerUrl`/`resolvePaymasterUrl` as deprecated re-exports temporarily (they will be removed when settings keys are deleted, but may be referenced in tests). Actually, remove them completely since we're removing the global settings in this plan.

   g. Remove `createSmartAccountPaymasterClient` standalone function (no longer needed, paymaster is always bundled with bundler client).

2. **index.ts**: Update exports: remove old resolvers, export `resolveWalletBundlerUrl`, `resolveWalletPaymasterUrl`, `WalletProviderData`.

3. **smart-account-clients.test.ts**: Rewrite tests for new wallet-based resolver. Test all behavior cases above. Mock `resolveProviderChainId` and `buildProviderBundlerUrl` indirectly by providing known networkId/apiKey combos and asserting URL output.
