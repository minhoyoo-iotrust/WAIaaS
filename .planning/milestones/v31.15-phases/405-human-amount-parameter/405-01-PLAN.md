  **Step 1: Zod 스키마 수정 (packages/core/src/schemas/transaction.schema.ts)**

  TransferRequestSchema, TokenTransferRequestSchema, ApproveRequestSchema에 humanAmount 옵션 필드를 추가하고 superRefine으로 XOR 검증을 구현한다.

  패턴:
  ```typescript
  // amount를 optional로 변경하고 humanAmount 추가
  const TransferRequestBaseSchema = z.object({
    type: z.literal('TRANSFER'),
    to: z.string().min(1),
    amount: z.string().regex(numericStringPattern, 'amount must be a numeric string (lamports/wei)').optional(),
    humanAmount: z.string().min(1).optional()
      .describe('Human-readable amount (e.g., "1.5" for 1.5 ETH). Mutually exclusive with amount.'),
    memo: z.string().max(256).optional(),
    network: NetworkTypeEnumWithLegacy.optional(),
    ...gasConditionField,
  });

  export const TransferRequestSchema = TransferRequestBaseSchema.superRefine((data, ctx) => {
    if (data.amount && data.humanAmount) {
      ctx.addIssue({
        code: z.ZodIssueCode.custom,
        message: 'amount and humanAmount are mutually exclusive. Provide exactly one.',
        path: ['humanAmount'],
      });
    }
    if (!data.amount && !data.humanAmount) {
      ctx.addIssue({
        code: z.ZodIssueCode.custom,
        message: 'Either amount or humanAmount must be provided.',
        path: ['amount'],
      });
    }
  });
  ```

  동일 패턴을 TokenTransferRequestSchema, ApproveRequestSchema에 적용.

  중요: `TransactionRequestSchema = z.discriminatedUnion('type', [...])` 은 superRefine이 적용된 스키마도 수용한다 (기존 SignMessageRequestSchema에서 검증됨). 그러나 discriminatedUnion은 `type` 리터럴이 최상위에 있어야 하므로, superRefine을 `refine` 대신 사용하되 base schema의 `type` 필드는 유지한다. 만약 discriminatedUnion이 superRefine 스키마를 거부하면, `TransactionRequestSchema`에서 `z.union`으로 전환하거나, 각 스키마를 `.refine()` 대신 `.superRefine()`으로 래핑한다.

  **대안 전략 (discriminatedUnion 호환성 이슈 시):** amount를 required로 유지하고, 별도의 `humanAmount` 필드만 추가한 뒤 API 라우트 레벨에서 XOR 검증을 수행한다. 이 경우 amount의 regex를 `/^\d+$/`에서 `/^\d*$/`로 완화하거나, API 레벨에서 humanAmount가 있으면 amount를 '0'으로 대체한 뒤 parse 한다. 어느 접근이든 실행 가능하므로 빌드 통과하는 쪽을 선택한다.

  **Step 2: resolveHumanAmount 헬퍼 (packages/daemon/src/api/routes/transactions.ts)**

  ```typescript
  import { parseAmount } from '@waiaas/core';

  /**
   * If humanAmount is provided, convert to smallest-unit string using decimals.
   * Returns the original amount if humanAmount is not present.
   */
  export function resolveHumanAmount(
    request: { amount?: string; humanAmount?: string },
    decimals: number,
  ): string {
    if (request.humanAmount) {
      return parseAmount(request.humanAmount, decimals).toString();
    }
    return request.amount!;
  }
  ```
