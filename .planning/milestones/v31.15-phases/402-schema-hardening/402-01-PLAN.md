    **Non-CLOB providers (smallest-unit 사용, 10개):**
    현재 단위가 이미 smallest-unit인 provider (6개):
    1. `jupiter-swap/index.ts`: SwapInputSchema의 `amount` → `.describe('Amount in smallest units (lamports). Example: "1000000000" = 1 SOL')`
    2. `zerox-swap/index.ts`: SwapInputSchema의 `sellAmount` → `.describe('Sell amount in smallest units (wei). Example: "1000000000000000000" = 1 ETH')`
    3. `lifi/index.ts`: LiFiCrossSwapInputSchema의 `fromAmount` → `.describe('Amount in smallest units of source token (wei/lamports). Example: "1000000" = 1 USDC')`
    4. `across/index.ts`: AcrossQuoteInputSchema와 AcrossExecuteInputSchema의 `amount` → `.describe('Amount in smallest units (wei). Example: "1000000" = 1 USDC')`
    5. `dcent-swap/index.ts`: GetQuotesInputSchema와 DexSwapInputSchema의 `amount` → `.describe('Amount in smallest units. Example: "1000000000000000000" = 1 token with 18 decimals')`
    6. `pendle/input-schemas.ts`: 모든 `amountIn`과 `amount` 필드에 → `.describe('Amount in smallest units (wei). Example: "1000000000000000000" = 1 token with 18 decimals')`

    현재 human-readable 단위를 사용하는 provider (4개 -- Phase 403에서 마이그레이션 예정이지만 현재 상태 그대로 description 추가):
    7. `aave-v3/schemas.ts`: 이미 JSDoc 주석에 human-readable 명시. Zod `.describe()` 추가: `.describe('Amount in human-readable format (e.g., "100.5"). Note: will migrate to smallest units in future. Use "max" for full repay/withdraw.')`
    8. `kamino/schemas.ts`: 동일 패턴. `.describe('Amount in human-readable format (e.g., "100.5"). Note: will migrate to smallest units in future. Use "max" for full repay/withdraw.')`
    9. `lido-staking/index.ts`: LidoStakeInputSchema `amount` → `.describe('Amount in human-readable ETH (e.g., "1.0"). Note: will migrate to smallest units (wei) in future.')`, LidoUnstakeInputSchema도 동일하게 stETH 기준
    10. `jito-staking/index.ts`: JitoStakeInputSchema `amount` → `.describe('Amount in human-readable SOL (e.g., "1.0"). Note: will migrate to smallest units (lamports) in future.')`, JitoUnstakeInputSchema도 동일하게 JitoSOL 기준

    **CLOB providers (exchange-native 단위, 3개):**
    11. `hyperliquid/schemas.ts`:
        - HlOpenPositionInputSchema `size` → `.describe('Position size in exchange-native decimal units (e.g., "1.0" = 1 contract). NOT smallest units -- Hyperliquid uses human-readable decimals.')`
        - HlTransferUsdcInputSchema `amount` → `.describe('USDC amount in exchange-native decimal units (e.g., "1000.50"). NOT smallest units.')`
        - HlSubTransferInputSchema `amount` → `.describe('USDC amount in exchange-native decimal units (e.g., "1000.50"). NOT smallest units.')`
        - HlPlaceOrderInputSchema `size`, `triggerPrice`, `price` 도 적절한 description 추가
        - HlClosePositionInputSchema `size` → `.describe('Partial close size in exchange-native decimal units. Omit for full close.')`
    12. `drift/schemas.ts`:
        - OpenPositionInputSchema `size` → `.describe('Base asset size in exchange-native decimal units (e.g., "1.0" SOL-PERP). NOT smallest units -- Drift uses human-readable decimals.')`
        - AddMarginInputSchema `amount` → `.describe('Collateral amount in exchange-native decimal units (e.g., "100" USDC). NOT smallest units.')`
        - WithdrawMarginInputSchema `amount` → `.describe('Withdrawal amount in exchange-native decimal units. NOT smallest units.')`
        - ClosePositionInputSchema `size` → `.describe('Partial close size in exchange-native units. Omit for full close.')`
        - ModifyPositionInputSchema `newSize` → `.describe('New position size in exchange-native units.')`
    13. `polymarket/schemas.ts`: 이미 .describe() 있음. `size` description에 exchange-native 명시 추가:
        - PmBuySchema `size` → `.describe('Number of outcome tokens to buy (exchange-native, e.g., "100"). NOT smallest units.')`
        - PmSellSchema `size` → `.describe('Number of outcome tokens to sell (exchange-native). NOT smallest units.')`
        - PmBuySchema `price` → `.describe('Limit price in 0-1 range (e.g., "0.65" = 65 cents per token). Exchange-native decimal.')`

    **주의사항:**
    - .min() 체인 뒤에 .describe()를 추가한다 (예: `z.string().min(1, 'msg').describe('...')`)
    - 기존 JSDoc 주석은 유지하되, Zod .describe()가 OpenAPI/MCP 자동 노출에 사용된다
    - .or() 체인이 있는 경우(aave/kamino의 max 리터럴), .or() 앞의 z.string()에 .describe()를 추가하고 union 전체를 감싸지 않는다. 예: `z.string().min(1, 'msg').describe('...').or(z.literal('max'))`
    - erc8004는 amount 필드가 없으므로 건너뛴다
