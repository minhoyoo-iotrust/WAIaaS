    **2. 테스트 파일 (onchain-human-amount.e2e.test.ts):**
    onchain-transfer.e2e.test.ts 패턴을 따른다:

    - `beforeAll`: EVM/SOL 지갑 조회, 세션 생성
    - `afterAll`: 세션 정리

    **Test 1: ETH humanAmount transfer**
    - shouldSkipNetwork('ethereum-sepolia') 체크
    - POST /v1/transactions/send: `{ to: self-address, humanAmount: "0.000000000000000001", type: "TRANSFER", network: "ethereum-sepolia" }`
    - 응답 status 200/201, id 존재
    - pollTxStatus로 CONFIRMED/COMPLETED 확인

    **Test 2: SOL humanAmount transfer**
    - shouldSkipNetwork('solana-devnet') 체크
    - POST /v1/transactions/send: `{ to: self-address, humanAmount: "0.000000001" }`
    - 응답 status 200/201, id 존재
    - pollTxStatus로 CONFIRMED/COMPLETED 확인

    **Test 3: humanAmount XOR error test (offchain)**
    - POST /v1/transactions/send: `{ to: self-address, amount: "1", humanAmount: "0.000000001", type: "TRANSFER", network: "ethereum-sepolia" }`
    - 응답 status 400, error code 확인

    **Test 4: Action provider humanAmount (graceful)**
    - shouldSkipNetwork('ethereum-sepolia') 체크
    - GET /v1/actions/providers로 Jupiter 또는 다른 swap provider 확인
    - provider가 없으면 graceful skip
    - provider가 있으면 POST /v1/actions/{provider}/swap: `{ params: { humanAmount: "0.001", decimals: 18, ... }, network: "ethereum-sepolia" }`
    - 200 응답이면 성공, 400 (no liquidity/token error)이면 graceful skip
    - 핵심: humanAmount 파라미터가 서버에서 수용되는 것을 검증

    pollTxStatus 헬퍼는 onchain-transfer.e2e.test.ts에서 사용하는 것과 동일 패턴으로 인라인 구현.
