admin.ts를 thin aggregator로 재구성한다. 기존 3,107줄에서 ~150줄로 축소.

**admin.ts 새 구조:**

```
// 1. imports: OpenAPIHono, openApiValidationHook
// 2. type exports: KillSwitchState, AdminRouteDeps (기존 그대로 유지 — 외부에서 import함)
// 3. adminRoutes() 함수: router 생성 후 5개 register 함수 호출

import { OpenAPIHono } from '@hono/zod-openapi';
import { openApiValidationHook } from './openapi-schemas.js';
import { registerAdminAuthRoutes } from './admin-auth.js';
import { registerAdminSettingsRoutes } from './admin-settings.js';
import { registerAdminNotificationRoutes } from './admin-notifications.js';
import { registerAdminWalletRoutes } from './admin-wallets.js';
import { registerAdminMonitoringRoutes } from './admin-monitoring.js';

// (AdminRouteDeps, KillSwitchState 타입 정의 유지)

export function adminRoutes(deps: AdminRouteDeps): OpenAPIHono {
  const router = new OpenAPIHono({ defaultHook: openApiValidationHook });

  registerAdminAuthRoutes(router, deps);
  registerAdminNotificationRoutes(router, deps);
  registerAdminSettingsRoutes(router, deps);
  registerAdminWalletRoutes(router, deps);
  registerAdminMonitoringRoutes(router, deps);

  return router;
}
```

**핵심 규칙:**
- AdminRouteDeps 인터페이스는 admin.ts에 유지 (외부 소비자: server.ts, index.ts)
- KillSwitchState 인터페이스도 admin.ts에 유지
- adminRoutes 함수 시그니처 변경 없음
- 모든 createRoute 정의, 핸들러 코드, 헬퍼 함수 제거
- import는 5개 register 함수 + OpenAPIHono + openApiValidationHook + AdminRouteDeps에 필요한 타입만

**index.ts:** 변경 없음 확인. `export { adminRoutes, type AdminRouteDeps, type KillSwitchState } from './admin.js'`가 그대로 동작하는지 확인.

**AdminRouteDeps import 정리:** AdminRouteDeps에서 참조하는 타입들 (BetterSQLite3Database, schema, JwtSecretManager, MasterPasswordRef, NotificationService, DaemonConfig, SettingsService, AdapterPool, IPriceOracle, IForexRateService, ActionProviderRegistry, KillSwitchService, VersionCheckService, DelayQueue, ApprovalWorkflow, RpcPool, EncryptedBackupService, AdminStatsService, AutoStopService, SQLiteDatabase)의 import type 문은 admin.ts에 유지해야 한다.
