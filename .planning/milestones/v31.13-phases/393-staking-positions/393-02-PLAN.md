**Add helper to jito-stake-pool.ts:**

Add `getJitoSolExchangeRate(rpcUrl: string, stakePoolAddress: string): Promise<number>` that:
1. Calls Solana RPC `getAccountInfo` for the stake pool address with `encoding: 'base64'`.
2. Parses the SPL Stake Pool account data to extract `total_lamports` (u64 at offset 258) and `pool_token_supply` (u64 at offset 266). See SPL Stake Pool layout.
3. Returns `total_lamports / pool_token_supply` as a floating-point exchange rate.
4. Alternatively, simpler approach: call `getAccountInfo` for the stake pool, decode the account data, and calculate the rate. If this is too complex, use a simpler heuristic: fetch SOL price and jitoSOL price from the price oracle and derive the ratio. Per D2 decision, provider-internal oracle is preferred.

**Simpler alternative (recommended):** Add `getJitoSolBalance(rpcUrl: string, walletAddress: string, jitosolMint: string): Promise<{ amount: bigint; uiAmount: number } | null>` that calls `getTokenAccountsByOwner` with `jsonParsed` encoding and extracts the balance. The `jsonParsed` response provides `uiAmount` directly.

For the exchange rate, add `getStakePoolExchangeRate(rpcUrl: string, stakePoolAddress: string): Promise<number>` that:
- Calls `getAccountInfo` for the stake pool
- Decodes base64 account data as Buffer
- Reads `total_lamports` (LE u64 at byte offset 258) and `pool_token_supply` (LE u64 at byte offset 266)
- Returns `Number(total_lamports) / Number(pool_token_supply)` as float
