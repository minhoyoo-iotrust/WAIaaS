Create `packages/daemon/src/rpc-proxy/tx-adapter.ts` with:

1. **EthTransactionParams interface**: `{ from?: string, to?: string | null, value?: string, data?: string, gas?: string, gasPrice?: string, maxFeePerGas?: string, maxPriorityFeePerGas?: string, nonce?: string }`

2. **RpcTransactionAdapter class** with `convert(params: EthTransactionParams, network: string): TransactionRequest`:
   - `to` is null/undefined: → `{ type: 'CONTRACT_DEPLOY', bytecode: params.data!, value: hexToDecimal(params.value) || '0', network }`
   - `to` + data starts with `0xa9059cbb` (ERC-20 transfer): → decode recipient (bytes 12-32) and amount (bytes 32-64), return `{ type: 'TOKEN_TRANSFER', tokenAddress: params.to, to: recipient, amount: decodedAmount, network }`
   - `to` + data starts with `0x095ea7b3` (ERC-20 approve): → decode spender and amount, return `{ type: 'APPROVE', tokenAddress: params.to, spenderAddress: spender, amount: decodedAmount, network }`
   - `to` + data (other selector): → `{ type: 'CONTRACT_CALL', to: params.to, data: params.data, value: hexToDecimal(params.value) || '0', network }`
   - `to` + no data (or '0x'): → `{ type: 'TRANSFER', to: params.to, amount: hexToDecimal(params.value) || '0', network }`

3. **toHexChainId(chainId: number): string**: `"0x" + chainId.toString(16)` for eth_chainId responses (RPC-04)

4. **hexToDecimal(hex?: string): string**: Converts hex value strings to decimal. Returns '0' if undefined.

Use the same selector constants as tx-parser.ts (0xa9059cbb, 0x095ea7b3). Follow Pitfall 14: 0x23b872dd (transferFrom) falls through to CONTRACT_CALL. Do NOT try to classify NFT transfers at this layer.
