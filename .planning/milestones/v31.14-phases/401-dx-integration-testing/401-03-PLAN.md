**TEST-01: JSON-RPC 2.0 protocol compliance** (~5 tests)
- Valid JSON-RPC 2.0 request with string id returns valid response
- Valid JSON-RPC 2.0 request with number id returns valid response
- Request with missing jsonrpc field is rejected with INVALID_REQUEST
- Request with missing method field is rejected with INVALID_REQUEST
- Notification (id omitted) for read methods returns no response

**TEST-02: Signing method intercept** (~4 tests)
- eth_sendTransaction with valid params creates a pipeline execution context with correct TransactionRequest
- eth_signTransaction returns signed hex without broadcasting
- eth_accounts returns session wallet address in array
- eth_chainId returns hex chainId derived from URL parameter

**TEST-03: Passthrough method proxy** (~3 tests)
- eth_call is forwarded to RPC pool and result returned
- eth_getBalance is forwarded to RPC pool
- Unsupported method (debug_traceTransaction) returns -32601

**TEST-04: CONTRACT_DEPLOY classification** (~3 tests)
- eth_sendTransaction with to=null classifies as CONTRACT_DEPLOY in tx-parser
- eth_sendTransaction with to=undefined classifies as CONTRACT_DEPLOY
- CONTRACT_DEPLOY with oversized bytecode is rejected (bytecodeSize limit)

**TEST-05: Async approval long-poll** (~3 tests)
- IMMEDIATE tier transaction returns result synchronously
- DELAY tier transaction uses SyncPipelineExecutor with configured timeout
- APPROVAL tier uses CompletionWaiter; timeout returns -32000 with txId

**TEST-06: Batch request handling** (~3 tests)
- Array of mixed read+write requests returns array of responses
- Empty array returns INVALID_REQUEST
- Batch with notifications filters out notification responses

**TEST-07: Auth/security** (~4 tests)
- Request without session auth is rejected (verify route requires sessionAuth)
- from field mismatch with session wallet returns error
- from field auto-filled when omitted
- eth_sendRawTransaction explicitly rejected with -32602

Use `vi.fn()` and `vi.mock()` for pipeline, EventBus, CompletionWaiter, RpcPassthrough dependencies. Create a helper `createTestDispatcher()` factory that wires up mocked deps for reuse across tests.
