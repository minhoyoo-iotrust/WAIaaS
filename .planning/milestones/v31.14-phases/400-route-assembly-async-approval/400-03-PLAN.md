**1. Bytecode size limit** (SEC-05):

In the route handler, after parsing the JSON-RPC body and before dispatching, for `eth_sendTransaction` requests:

Add a pre-dispatch check:
```typescript
const MAX_BYTECODE_SIZE = 48 * 1024; // 48KB in bytes (96KB hex chars since 2 hex chars = 1 byte)

// Check for CONTRACT_DEPLOY (to is null/undefined) with oversized bytecode
if (method === 'eth_sendTransaction') {
  const txParams = (params?.[0] ?? {}) as EthTransactionParams;
  if (!txParams.to && txParams.data) {
    // Remove '0x' prefix for size calculation
    const bytecodeHex = txParams.data.startsWith('0x') ? txParams.data.slice(2) : txParams.data;
    const bytecodeSize = bytecodeHex.length / 2; // 2 hex chars = 1 byte
    if (bytecodeSize > MAX_BYTECODE_SIZE) {
      return c.json(
        jsonRpcError(
          requestId,
          JSON_RPC_ERRORS.INVALID_PARAMS,
          `Bytecode size ${bytecodeSize} bytes exceeds limit of ${MAX_BYTECODE_SIZE} bytes`,
        ),
      );
    }
  }
}
```

Read the limit from SettingsService if available: `deps.settingsService?.get('rpc_proxy.max_bytecode_size')` parsed as integer, falling back to `48 * 1024`.
