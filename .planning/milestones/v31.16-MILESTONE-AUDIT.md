```
  -> All downstream code receives canonical "ethereum-mainnet"

```

#### Flow 2: CAIP-19 assetId-only Token Transfer

```
Input: { type: "TOKEN_TRANSFER", token: { assetId: "eip155:1/erc20:0xa0b8..." } }
  -> TokenInfoBaseSchema .superRefine: assetId present -> address/decimals/symbol optional [408-01]
  -> transactions.ts route: resolveTokenFromAssetId() called before humanAmount [408-02]
  -> parseAssetId("eip155:1/erc20:0xa0b8...") -> { chainId, namespace, address, network }
  -> TokenRegistryService lookup -> fills address/decimals/symbol
  -> Network inferred from assetId if not provided (AST-06)
  -> Pipeline receives fully resolved TokenInfo
Status: COMPLETE (11 daemon tests + 13 core tests)
```

#### Flow 3: Response CAIP Enrichment

```
DB query returns: { network: "ethereum-mainnet", balance: "1000000", ... }
  -> enrichBalance() at c.json() boundary [wallet.ts:332]
  -> networkToCaip2("ethereum-mainnet") -> "eip155:1"
  -> nativeAssetId("ethereum-mainnet") -> "eip155:1/slip44:60"
  -> Response: { ...original, chainId: "eip155:1", assetId: "eip155:1/slip44:60" }
  -> Applied to: balance, assets, transactions, incoming, NFTs, DeFi positions, staking, tokens
  -> connect-info: supportedChainIds = CAIP-2 array of all 15 networks
Status: COMPLETE (15 integration tests + 20 unit tests + 1 connect-info test)
```

#### Flow 4: MCP resolve_asset Tool

```
Input: { assetId: "eip155:1/erc20:0xa0b8..." }
  -> resolve_asset tool [mcp/resolve-asset.ts]
  -> Local CAIP-19 parser (no @waiaas/core dependency)
  -> API client GET /v1/tokens?network=...&search=...
  -> Response: { address, decimals, symbol, name, network, chainId, isNative, isRegistered }
  -> Unregistered: isRegistered=false, decimals/symbol=null
  -> Native (slip44): isNative=true, address=null
Status: COMPLETE (5 MCP tests)
```
