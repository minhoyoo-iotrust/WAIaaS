## E2E Flow Verification

### Flow 1: Admin UI Preset Selection → Auto-Setup → DB
```
wallets.tsx WALLET_PRESETS → walletTypeSelect signal → handleSaveOwner()
→ PUT /v1/wallets/{id}/owner { wallet_type: 'dcent' }
→ SetOwnerRequestSchema validation → BUILTIN_PRESETS lookup
→ sqlite.transaction: UPDATE wallets + PresetAutoSetupService.apply()
→ 4-step: signing SDK enable → WalletLinkRegistry → preferred_wallet → preferred_channel
→ Response: { walletType, approvalMethod, warning }
→ Admin UI refreshes wallet detail + walletType badge
```
**Status:** COMPLETE

### Flow 2: Auto-Setup Failure → Full Rollback
```
PresetAutoSetupService.apply() throws
→ Settings snapshot restoration (each key individually)
→ Error re-thrown → sqlite.transaction() rollback
→ DB: wallet_type and owner_approval_method remain NULL
→ API: 500 response
```
**Status:** COMPLETE

### Flow 3: Push Relay Payload Transform Pipeline
```
config.toml [relay.push.payload] → ConfigSchema.parse()
→ ConfigurablePayloadTransformer(config)
→ NtfySubscriber({ transformer })
→ SSE message → buildPushPayload() → transformer.transform()
→ merge: { ...static_fields, ...category_map[category], ...original_data }
→ IPushProvider.send()

Bypass: no [relay.push.payload] → transformer=undefined → raw payload passes through
```
**Status:** COMPLETE
