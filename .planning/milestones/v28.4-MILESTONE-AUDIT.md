## Integration Gaps

### GAP-1: Staking unstake pipeline does not enroll transactions for async polling

**Affects:** ASYNC-01, ASYNC-02, ASYNC-03, ASYNC-04
**Root cause:** Pipeline Stage 6 sets `status=CONFIRMED` but never writes `bridge_status=PENDING` + `bridge_metadata={tracker:'lido-withdrawal'|'jito-epoch'}` for staking unstake transactions. `AsyncPollingService.pollAll()` queries `bridge_status IN ('PENDING', 'BRIDGE_MONITORING')` — finds nothing.
**Fix:** After staking unstake pipeline completes, write `bridge_status=PENDING` + `bridge_metadata` with tracker name and notificationEvent to enable async polling pickup.

### GAP-2: Action provider name not persisted in transaction metadata

**Affects:** SAPI-01, SAPI-02, INTF-03
**Root cause:** `actionProvider` is an in-memory field on `ContractCallRequest` used only for policy bypass — never persisted to `transactions.metadata` column. `GET /v1/wallet/staking` queries `metadata LIKE '%lido_staking%'` which never matches.
**Fix:** Persist `{provider, action}` into `transactions.metadata` column when executing action-provider transactions, either in Stage 1 insert or in the actions.ts post-pipeline handler.

## Broken E2E Flows

2. **Jito unstake → async tracking → notification**: Same break point
