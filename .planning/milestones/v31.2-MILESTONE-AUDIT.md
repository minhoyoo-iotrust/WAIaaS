### Issue 2: Sign Route Network Resolution Heuristic (Minor Design Concern)

**Severity:** Low
**Details:** The sign route (userop.ts line 426-436) resolves network by iterating RPC config keys and falling back to `ethereum-sepolia`. The build record does not store the network field (D11). This means if multiple EVM networks are configured, the sign route may pick the wrong one.
**Impact:** In practice, the callData was built against a specific network's RPC, so signing with a different network's chain ID could produce an invalid signature. However, since the SmartAccount address and callData are validated against the build record, the actual risk is limited -- the signature is computed locally and the EntryPoint address is constant.
**Recommendation:** Consider storing network in userop_builds table in a future iteration for correctness.
**Affected Requirements:** SIGN-01, SIGN-07
