**1. site/build.mjs에 목록 페이지 생성 로직 추가:**

빌드 Phase 2 이후에 listing page 생성 단계를 추가한다.

`generateListingPage(section, pages, template)` 함수를 만든다:
- `pages`는 해당 section의 front-matter 배열 (title, description, date, slug, category)
- Blog 목록: 날짜 역순 정렬, 각 항목에 title (링크), date, description 표시
- Docs 목록: category별로 그룹화 (Technical 카테고리), 각 항목에 title (링크), description 표시
- 목록 페이지 HTML은 template.html의 nav/footer를 재사용하되, article 영역에 목록 콘텐츠를 삽입
- Blog 목록 제목: "Blog", 설명: "Insights on AI agent wallet security, architecture, and integration guides."
- Docs 목록 제목: "Documentation", 설명: "Technical references for WAIaaS architecture, API, security, and deployment."
- 출력: site/blog/index.html, site/docs/index.html

목록 항목 HTML 구조 (CRT 테마에 맞는 최소한 스타일):
```html
<ul class="listing">
  <li>
    <a href="/blog/{slug}/">{title}</a>
    <time>{date}</time>
    <p>{description}</p>
  </li>
</ul>
```

목록 페이지용 CSS를 article.css에 추가하지 말고, 목록 페이지 HTML 내에 `<style>` 블록으로 인라인한다 (article.css는 개별 글용이므로). 목록 스타일:
- `.listing` — list-style: none, padding: 0
- `.listing li` — border-bottom: 1px solid var(--border), padding: 16px 0
- `.listing a` — font-size: 1.1rem, color: var(--green), font-weight: 500
- `.listing time` — font-size: 0.8rem, color: var(--text-dim), margin-left: 12px
- `.listing p` — color: var(--text), margin-top: 4px, font-size: 0.85rem

빌드 Phase 2에서 각 파일 처리 시 front-matter 데이터를 수집하고, Phase 2 완료 후 listing 페이지를 생성한다.
