<task type="auto">
  <name>Task 1: Build script + HTML template</name>
  <files>site/build.mjs, site/template.html, package.json</files>
  <action>
1. Install dev dependencies (no new runtime deps for the site itself):
   - `pnpm add -Dw gray-matter marked highlight.js` (front-matter parsing, markdown conversion, syntax highlighting)

2. Create `site/template.html` - HTML template that:
   - Shares the CRT theme from index.html: same CSS variables (--bg, --surface, --border, --text, --green, etc.), JetBrains Mono font, scanline effect
   - Contains nav with logo "WAIaaS" linking to `/`, nav-links: GitHub, npm, Docker, Blog, Docs (Blog/Docs are new additions for Phase 440, add them now with hrefs `/blog/` and `/docs/`)
   - Has `{{TITLE}}`, `{{DESCRIPTION}}`, `{{CANONICAL_URL}}`, `{{CONTENT}}`, `{{DATE}}`, `{{OG_TITLE}}`, `{{OG_DESCRIPTION}}` placeholders
   - Includes `<link rel="stylesheet" href="/article.css">` for article styles
   - Includes highlight.js CSS inline (monokai-sublime or similar dark theme, to match CRT aesthetic) via a `<link>` to a CDN hljs theme OR inline the styles. Use build-time highlighting so NO client JS is needed for hljs.
   - Footer matching index.html footer style
   - Meta tags: charset, viewport, canonical, og:title, og:description, og:type=article, twitter:card
   - Uses relative paths for favicon: `/favicon.svg`, `/favicon-32.png`, `/favicon-16.png`, `/apple-touch-icon.png`

3. Create `site/build.mjs` (ESM, Node.js 22 native) that:
   - Globs `docs/**/*.md` recursively
   - Parses each file with gray-matter to extract front-matter (title, description, date, section, category, slug)
   - **Validates required fields**: title, description, date. If ANY file is missing ANY required field, print the filename and missing fields, then `process.exit(1)`.
   - Converts markdown body to HTML using `marked` with:
     - highlight.js integration for code blocks (build-time, configure `marked` with a custom renderer or `marked-highlight` package)
     - GFM tables support
     - Proper link handling
   - Determines output path using clean URLs:
     - `docs/why-waiaas/001-ai-agent-wallet-security-crisis.md` -> `site/blog/ai-agent-wallet-security-crisis/index.html` (use slug from front-matter if present, else derive from filename minus numeric prefix)
     - `docs/architecture.md` -> `site/docs/architecture/index.html`
     - Section is determined by front-matter `section: blog|docs` field
   - Reads `site/template.html`, replaces placeholders with front-matter values + rendered HTML content
   - Writes output files, creating directories as needed
   - Prints summary: "Built N pages (M blog, K docs)" on success
   - Does NOT process `docs/admin-manual/` (internal admin docs, not public)
   - Base URL: `https://waiaas.ai` (for canonical URLs)

4. Add script to root `package.json`: `"site:build": "node site/build.mjs"`
   - If a `scripts` section doesn't exist at root level, check if there's a workspace-level package.json. Add it where appropriate.

Note: Use `marked-highlight` package for highlight.js integration with marked (not deprecated `marked` highlight option). Install: `pnpm add -Dw marked-highlight`
  </action>

    - A markdown file with missing front-matter causes build to exit with error
  </done>

<task type="auto">
  <name>Task 2: Article CSS</name>
  <files>site/article.css</files>
  <action>
Create `site/article.css` with CRT-themed article styles for long-form content reading:

1. **Article container** (`.article`):
   - max-width: 720px, centered, padding: 48px 24px top/bottom
   - Uses the same CSS variables from the main site (--bg, --surface, --border, --text, --text-bright, --text-dim, --green, --cyan, --yellow, etc.)

2. **Typography** for readability:
   - h1: 1.6rem, color: var(--text-bright), margin-bottom: 8px, font-weight: 700
   - h2: 1.2rem, color: var(--green), margin-top: 40px, margin-bottom: 16px, `::before { content: "## "; color: var(--text-dim); }`
   - h3: 1rem, color: var(--cyan), margin-top: 32px, margin-bottom: 12px, `::before { content: "### "; color: var(--text-dim); }`
   - Paragraphs: line-height: 1.8 (generous for long reads), margin-bottom: 20px, color: var(--text)
   - WCAG AA contrast: var(--text) #b0b0b0 on var(--bg) #0c0c0c = 9.5:1 ratio (passes AAA)

3. **Article meta** (`.article-meta`):
   - Date display: font-size: 0.8rem, color: var(--text-dim), margin-bottom: 32px
   - `::before { content: "// published "; }`

4. **Code blocks** (`pre code`, inline `code`):
   - Pre: background: var(--surface), border: 1px solid var(--border), padding: 16px 20px, overflow-x: auto, margin: 20px 0, font-size: 0.85rem, line-height: 1.6
   - Inline code: background: var(--surface), padding: 2px 6px, color: var(--cyan), font-size: 0.9em
   - Highlight.js overrides: `.hljs { background: var(--surface); }` to match CRT theme

5. **Lists** (ul, ol):
   - margin: 16px 0, padding-left: 24px
   - li: margin-bottom: 8px, line-height: 1.7
   - Bullet color: var(--green) via `::marker { color: var(--green); }`

6. **Tables**:
   - width: 100%, border-collapse: collapse, margin: 20px 0, font-size: 0.85rem
   - th: padding: 8px 12px, border-bottom: 1px solid var(--border), color: var(--cyan), text-align: left, font-weight: 500
   - td: padding: 6px 12px, border-bottom: 1px solid var(--border)

7. **Blockquotes**:
   - border-left: 2px solid var(--green-dim), padding-left: 16px, color: var(--text-dim), margin: 20px 0, font-style: italic

8. **Links in article**: color: var(--cyan), underline on hover, visited: var(--magenta)

9. **Horizontal rules** (hr):
   - border: none, border-top: 1px solid var(--border), margin: 40px 0

10. **Images**:
    - max-width: 100%, border: 1px solid var(--border)

11. **Responsive** (@media max-width: 768px):
    - h1: 1.3rem
    - Article padding: 32px 16px
    - Pre: font-size: 0.8rem, padding: 12px
    - Table: display: block, overflow-x: auto

The CSS must NOT redefine the :root variables (those come from template.html). It only uses them.
  </action>
