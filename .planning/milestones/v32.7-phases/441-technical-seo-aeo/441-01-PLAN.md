  <name>Task 1: sitemap.xml 자동 생성 + JSON-LD 구조화 데이터 주입</name>
  <files>site/build.mjs, site/template.html</files>
  <action>
1. **template.html에 JSON-LD 플레이스홀더 추가:**
   - `</head>` 바로 위에 `{{JSON_LD}}` 플레이스홀더 추가
   - applyTemplate()에서 JSON-LD를 주입할 위치

2. **build.mjs에 JSON-LD 생성 함수 추가:**
   - `generateJsonLd(frontmatter, canonicalUrl, section)` 함수 생성
   - section이 'blog'이면 `@type: "Article"`, 'docs'이면 `@type: "TechArticle"`
   - 모든 article 페이지에 BreadcrumbList 추가: Home > Section > Title
   - JSON-LD 필드: @context, @type, headline, description, datePublished, url, author(Organization: WAIaaS), publisher(Organization: WAIaaS)
   - BreadcrumbList: `[{Home, /}, {Blog|Docs, /blog/|/docs/}, {Title, canonical}]`
   - 반환값: `<script type="application/ld+json">...</script>` 문자열 (여러 블록 연결)

3. **listing 페이지용 JSON-LD:**
   - `generateListingJsonLd(section, canonicalUrl)` 함수 생성
   - `@type: "CollectionPage"` + BreadcrumbList `[{Home, /}, {Blog|Docs, canonical}]`
   - generateListingPage() 내에서 listing HTML에 JSON-LD 주입

4. **applyTemplate()에 JSON-LD 파라미터 추가:**
   - 기존 시그니처: `applyTemplate(template, frontmatter, htmlContent, canonicalUrl, activeSection)`
   - 새 시그니처: `applyTemplate(template, frontmatter, htmlContent, canonicalUrl, activeSection, jsonLd = '')`
   - `{{JSON_LD}}`를 jsonLd 문자열로 교체
   - article 페이지 빌드 루프에서 generateJsonLd() 호출하여 전달
   - listing 페이지에서 generateListingJsonLd() 호출하여 전달

5. **sitemap.xml 자동 생성:**
   - build() 함수 끝에 (링크 검증 후) `generateSitemap(builtPages)` 호출
   - builtPages: 빌드 과정에서 수집한 `[{url, lastmod, section}]` 배열
   - 홈페이지(/) priority 1.0, blog/docs listing priority 0.8, 개별 article priority 0.6
   - lastmod: front-matter date 또는 오늘 날짜
   - 기존 수동 sitemap.xml을 덮어쓰기 (자동 생성으로 전환)
   - changefreq: homepage=weekly, listing=weekly, article=monthly

6. **기존 수동 sitemap.xml 제거 안내:**
   - 빌드 시 자동 생성되므로 기존 site/sitemap.xml은 빌드 출력물이 됨
   - 빌드 스크립트가 site/sitemap.xml에 직접 작성
