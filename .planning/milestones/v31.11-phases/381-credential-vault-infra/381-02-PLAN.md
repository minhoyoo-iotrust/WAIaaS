Expected REST API endpoints (from CRED-07 requirements):
```typescript
// Per-wallet credential endpoints
GET    /v1/wallets/:walletId/credentials       // sessionAuth — list (metadata only)
POST   /v1/wallets/:walletId/credentials       // masterAuth — create
DELETE /v1/wallets/:walletId/credentials/:ref   // masterAuth — delete
PUT    /v1/wallets/:walletId/credentials/:ref/rotate  // masterAuth — rotate

// Global credential endpoints
POST   /v1/admin/credentials                   // masterAuth — create global
GET    /v1/admin/credentials                   // masterAuth — list global
DELETE /v1/admin/credentials/:ref              // masterAuth — delete global
PUT    /v1/admin/credentials/:ref/rotate       // masterAuth — rotate global
```

Credential metadata shape (from CRED-05):
```typescript
interface CredentialMetadata {
  id: string;           // UUID
  walletId: string | null;
  type: 'api-key' | 'hmac-secret' | 'rsa-private-key' | 'session-token' | 'custom';
  name: string;
  metadata: Record<string, unknown>;
  expiresAt: number | null;
  createdAt: number;
  updatedAt: number;
}
```

다음 섹션을 포함하는 Admin UI Credentials 탭 UX 설계 문서를 작성한다. 설계 문서만 산출하고 구현 코드는 작성하지 않는다.

**1. 개요**
- Admin UI에 Credentials 탭 추가 (기존 메뉴 체계에 통합)
- 두 가지 진입점: 지갑 상세 페이지 내 Credentials 탭 (per-wallet) + Admin > Credentials 페이지 (글로벌)

**2. 지갑별 Credentials 탭 (per-wallet)**
기존 지갑 상세 페이지(`/wallets/:id`)의 탭에 "Credentials" 추가.

목록 뷰:
- 테이블 컬럼: Type (badge), Name, Metadata (요약), Expires At, Created At, Actions
- Type badge 색상: api-key(blue), hmac-secret(purple), rsa-private-key(orange), session-token(green), custom(gray)
- 만료된 credential은 행에 경고 아이콘 + strikethrough
- 빈 상태: "No credentials registered. Add credentials to enable off-chain actions."
- **원문 비노출 원칙**: 테이블에 encrypted_value 컬럼 없음. 복호화된 값은 UI에 절대 표시하지 않음

등록 모달:
- "Add Credential" 버튼 → 모달 오픈
- 필드: Type (select), Name (text), Value (password input, 붙여넣기 허용), Metadata (JSON textarea, optional), Expires At (datetime picker, optional)
- Value 필드: type="password", autocomplete="off", 마스킹 상태 기본. 눈 아이콘으로 토글 가능 (입력 시에만, 저장 후에는 조회 불가)
- 제출 시: POST /v1/wallets/:walletId/credentials → 성공 시 목록 갱신 + 토스트
- masterAuth 필요: 제출 시 master password 확인 다이얼로그

삭제:
- Actions 컬럼에 휴지통 아이콘
- 클릭 시 확인 다이얼로그: "Delete credential '{name}'? This cannot be undone. Actions referencing this credential will fail."
- masterAuth 필요

로테이션:
- Actions 컬럼에 회전 아이콘
- 클릭 시 모달: "Rotate credential '{name}'" + 새 Value 입력 (password)
- masterAuth 필요
- 성공 시 updated_at 갱신 확인 토스트

**3. 글로벌 Credentials 페이지 (Admin > Credentials)**
별도 Admin 메뉴 항목으로 추가 (/admin/credentials).

- per-wallet 탭과 동일한 테이블 구조 + 등록/삭제/로테이션 UX
- 차이점: walletId 컬럼 없음 (모두 글로벌), "Used by" 컬럼 추가 (참조 중인 지갑 수 표시)
- masterAuth 전용 페이지

**4. 보안 원칙**
- 복호화된 credential 값은 REST API 응답에 포함하지 않음 → UI에 표시 불가능
- 등록 시 입력한 값은 서버로 전송 후 UI에서 즉시 폐기 (signal 초기화)
- CSP 정책 유지: 외부 리소스 로드 없음
- masterAuth 확인은 기존 패턴 사용 (master password 입력 다이얼로그)
- Value 필드에 clipboard readText 사용 시 navigator.clipboard API (HTTPS/localhost만)

**5. 컴포넌트 구조 (Preact)**
```
CredentialsTab (per-wallet)
├── CredentialList (테이블)
│   ├── CredentialRow (행)
│   │   ├── TypeBadge (타입별 색상)
│   │   └── ActionButtons (삭제, 로테이션)
│   └── EmptyState
├── AddCredentialModal (등록)
│   ├── CredentialTypeSelect
│   ├── SecureValueInput (password + toggle)
│   └── MetadataEditor (JSON textarea)
├── RotateCredentialModal (로테이션)
│   └── SecureValueInput
└── DeleteConfirmDialog

AdminCredentials (글로벌)
├── CredentialList (동일 컴포넌트 재사용, walletId=null 모드)
├── AddCredentialModal
├── RotateCredentialModal
└── DeleteConfirmDialog
```

**7. 기존 Admin UI 패턴과의 일관성**
- 테이블 스타일: 기존 Policy/Notifications/Token Registry 탭과 동일한 테이블 컴포넌트 재사용
- 모달 스타일: 기존 Add Policy/Add Token 모달 패턴 준수
- 토스트: sonner 사용 (기존 패턴)
- 에러 처리: API 실패 시 토스트 에러 + 폼 상태 유지
- 반응형: 모바일에서 테이블 → 카드 전환 (기존 패턴)

**8. MCP 도구 설계**
```
credential-list    — 지갑의 credential 목록 조회 (메타데이터만)
credential-create  — credential 등록 (masterAuth)
credential-delete  — credential 삭제 (masterAuth)
credential-rotate  — credential 로테이션 (masterAuth)
```
