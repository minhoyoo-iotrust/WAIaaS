```
# External Action Framework 설계 (doc-81)
> v31.11 설계 마일스톤 통합 문서

## 0. 개요
- 목표: ActionProvider 프레임워크를 on-chain 전용에서 off-chain 액션 포괄로 확장
- 설계 원칙 5가지 (기존 무변경, Zod SSoT, kind vs type 분리, credential 간접 참조, 정책 일원화)
- 전체 아키텍처 다이어그램 (ActionProvider -> resolve() -> normalize -> kind별 파이프라인 라우팅)

## D1. ResolvedAction 타입 시스템
- 소스: 380-01 resolved-action-types.md
- 내용: kind 3종 union (contractCall/signedData/signedHttp), Zod 스키마 초안 (SignedDataActionSchema, SignedHttpActionSchema, ContractCallRequest kind 확장), normalizeResolvedAction() 정규화 전략, IActionProvider.resolve() 반환 타입 확장, ApiDirectResult 분리 원칙, 기존 13개 provider 하위 호환 분석

## D2. ISignerCapability + SigningSchemeEnum
- 소스: 380-02 signer-capability-interface.md + 382-01 existing-adapters-signbytes.md + 382-02 hmac-rsapss-registry.md
- 내용: SigningSchemeEnum 7종, ISignerCapability 인터페이스 + SigningParams discriminated union + SigningResult + SigningError, 기존 4종 어댑터(Eip712/Personal/Erc8128/Transaction) 상세, signBytes 2종(ECDSA/Ed25519), HMAC/RSA-PSS 신규 capability, SignerCapabilityRegistry 상세 + daemon 부팅 등록 + connect-info 확장

## D3. CredentialVault 인프라
- 소스: 381-01 credential-vault-design.md + 381-02 credential-vault-admin-ui.md
- 내용: ICredentialVault CRUD 인터페이스, 5종 credential 타입, credentialRef 간접 참조, 스코프 모델(per-wallet + 글로벌 fallback), wallet_credentials DB 스키마(v55), AES-256-GCM + HKDF 암호화, re-encrypt/backup 통합, credential lifecycle, 인증 모델(sessionAuth/masterAuth), REST API 8 엔드포인트, Admin UI Credentials 탭 UX, MCP 4도구, SDK 4메서드

## D4. 파이프라인 라우팅
- 소스: 383-01 pipeline-routing-design.md + 383-02 external-action-interfaces.md
- 내용: 3-way 라우팅(contractCall/signedData/signedHttp), signedData 5-stage 파이프라인(credential 해소 -> SigningParams 조립 -> canSign() -> sign() -> 결과 처리), signedHttp 6-stage 파이프라인, mixed-kind 배열 순차 실행, off-chain DB 기록(v56: action_kind/venue/operation/external_id), 정책 평가 시점 보장(resolve 후 서명 전), REST/MCP/SDK 인터페이스 확장(ActionExecutionResult kind-union, action-list-offchain, connect-info externalActions), 인증 모델

## D5. 정책 확장
- 소스: 384-01 policy-extension-design.md
- 내용: TransactionParam 확장(venue/actionCategory/notionalUsd/leverage/expiry/hasWithdrawCapability), VENUE_WHITELIST 정책(default-deny + Admin Settings 비활성화 가능), ACTION_CATEGORY_LIMIT 정책(카테고리별 USD 누적 한도), ActionDefinition riskLevel 4등급 자동 매핑

## D6. 비동기 추적 확장
- 소스: 384-02 async-tracking-extension-design.md
- 내용: AsyncTrackingResult.state 9종(기존 4 + PARTIALLY_FILLED/FILLED/CANCELED/SETTLED/EXPIRED), bridge_status/bridge_metadata 재사용 결정, AsyncPollingService 쿼리 확장(PARTIALLY_FILLED 폴링), tracker 메타데이터 확장(trackerName/venue/fillPercentage), DB 마이그레이션 v57, 알림 이벤트 6종

## 7. DB 마이그레이션 요약
- v55: wallet_credentials 테이블 (D3)
- v56: transactions 테이블 action_kind/venue/operation/external_id + 인덱스 (D4)
- v57: bridge_status 복합 인덱스 (D6)

## 8. 전체 설계 결정 통합 테이블
- Phase 380~384에서 축적된 모든 설계 결정을 하나의 테이블로 통합 (번호 재매김)
- 각 결정에 출처 Phase 표기

## 9. 구현 우선순위 권장
- Wave 1: 타입 시스템 + Signer (D1 + D2) — 의존성 없음
- Wave 2: CredentialVault (D3) — DB v55
- Wave 3: 파이프라인 라우팅 (D4) — D1+D2+D3 필요, DB v56
- Wave 4: 정책 + 추적 (D5 + D6) — D4 필요, DB v57

## 10. Pitfall 방지 통합 체크리스트
- Phase 380~384의 모든 pitfall 체크리스트를 카테고리별로 통합 정리
- 카테고리: 타입 시스템, 서명, Credential, 파이프라인, 정책, 추적

## 부록 A: 기존 13개 ActionProvider 하위 호환 매트릭스
## 부록 B: Zod 스키마 전체 목록 (구현 시 참조)
## 부록 C: REST API 엔드포인트 전체 목록
```
