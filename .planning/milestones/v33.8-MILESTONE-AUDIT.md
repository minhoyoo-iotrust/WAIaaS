### Design Decisions Verified

1. **CONTRACT_CALL type reuse** -- XRPL DEX uses the existing discriminatedUnion type rather than adding a 10th type. Correct architectural choice matching the calldata JSON pattern.
2. **ApiDirectResult pattern** -- get_orderbook and get_offers bypass the pipeline (Hyperliquid precedent). Correctly implemented with `__apiDirect: true`.
3. **TakerGets as spending amount** -- Semantically correct for policy evaluation (what the account gives away). Consistent between tx-parser (Phase 1) and resolveEffectiveAmountUsd (Phase 3).
4. **IOU notListed fallback** -- IOU TakerGets returns `notListed` instead of `$0` in USD resolution. This is the safe policy path: unknown price triggers escalation rather than silently allowing.
5. **2-step trust line auto-setup** -- When trust line is missing, returns `[TrustSet, OfferCreate]` array. The pipeline processes both sequentially.
6. **xrpl_dex_enabled defaults to false** -- Opt-in provider, consistent with the default-deny policy.

### Known Gaps (Acknowledged, Not Blocking)

1. **IOU USD pricing** -- IPriceOracle does not yet support XRPL IOU tokens. IOU TakerGets returns `notListed`. Documented as out-of-scope for this milestone.
2. **AffectedNodes metadata parsing** -- Actual fill amounts from partial fills are not extracted from on-chain metadata. Listed as v2 requirement (ADV-01).
