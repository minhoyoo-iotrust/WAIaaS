## Critical Findings

### CRITICAL-01: `cors-resource-management.test.ts` imports deleted `NtfyChannel`

- **File:** `packages/daemon/src/__tests__/cors-resource-management.test.ts:15`
- **Issue:** `import { NtfyChannel } from '../notifications/channels/ntfy.js'` -- this file was deleted in Phase 450 (SIGN-05). Test will fail to compile.
- **Impact:** TypeScript compilation of test suite will fail for this test file.
- **Affected requirements:** SIGN-05

### CRITICAL-02: Multiple test files assert `sdk_ntfy` against CHECK constraint that only allows `sdk_push`

- **Files:** `wallet-owner-preset.test.ts:165,171,231,240`, `wallets-api.test.ts:135,141,220,224,278`, `preset-auto-setup.test.ts:133,137,343,348`
- **Issue:** Tests insert `sdk_ntfy` into `owner_approval_method` column and assert the API returns `sdk_ntfy`. But schema-ddl CHECK constraint now only allows `sdk_push`. These tests will fail with SQLite CHECK constraint violations on pre-migration databases, or they test pre-v60 behavior that no longer applies.
- **Impact:** If tests run against v60 schema, INSERT with `sdk_ntfy` will fail. If tests seed their own DB without v60 migration, they use old DDL with `sdk_ntfy` in CHECK and may pass but test obsolete behavior.
- **Affected requirements:** CORE-02, DB-01

### ADVISORY-01: Admin UI notifications page still renders ntfy settings

- **File:** `packages/admin/src/pages/notifications.tsx:73,298-313`
- **Issue:** The notifications settings page renders an "ntfy" FieldGroup with `ntfy_server` input field. The daemon no longer has these config fields (CONF-01), so saving these settings would either fail or be ignored.
- **Impact:** Cosmetic/UX issue -- Admin users see obsolete settings fields.
- **Affected requirements:** CONF-01 (partial gap -- daemon config cleaned, but Admin UI notifications page not updated)

### ADVISORY-02: Admin settings search index references stale ntfy settings

- **File:** `packages/admin/src/utils/settings-search-index.ts:81-82`
- **Issue:** Search entries for `notifications.ntfy_server` and `notifications.ntfy_topic` will appear in settings search results but point to non-existent settings.
- **Impact:** Minor UX issue -- search returns stale results.
