## v0.4 Requirements

### 테스트 레벨 & 모듈 매트릭스

- [x] **TLVL-01**: 6개 테스트 레벨(Unit/Integration/E2E/Chain Integration/Security/Platform)의 범위, 실행 환경, 실행 빈도가 정의됨
- [x] **TLVL-02**: 7개 모노레포 패키지 + Python SDK + Desktop App에 대해 어떤 테스트 레벨을 적용할지 매트릭스로 정의됨
- [x] **TLVL-03**: 패키지별 커버리지 목표가 정의됨 (core/daemon 90%+, adapter-solana 80%+, cli 70%+ 등)

### Mock 경계 & 테스트 인터페이스

- [x] **MOCK-01**: 5개 외부 의존성(블록체인 RPC, 알림 채널, 파일시스템, 시간, Owner 서명)에 대한 Mock 방식이 Unit/Integration/E2E 레벨별로 정의됨
- [x] **MOCK-02**: 기존 v0.2 인터페이스 4개(IChainAdapter, INotificationChannel, ILocalKeyStore, IPolicyEngine)의 Mock 가능성이 검증됨
- [x] **MOCK-03**: 신규 테스트 인터페이스 IClock(시간 추상화)과 ISigner(서명 추상화)의 인터페이스 스펙이 정의됨
- [x] **MOCK-04**: Mock과 실제 구현의 동작 일치를 보장하는 Contract Test 전략이 정의됨

### 보안 테스트 시나리오

- [x] **SEC-01**: Layer 1 세션 인증 공격 시나리오 9개 이상이 정의됨 — 실제 20건 정의 (sessionAuth 12 + ownerAuth 8)
- [x] **SEC-02**: Layer 2 정책 우회 공격 시나리오 6개 이상이 정의됨 — 실제 9건 (TOCTOU, 금액 분할, 시간 조작 포함)
- [x] **SEC-03**: Layer 3 Kill Switch 및 복구 시나리오 6개 이상이 정의됨 — 실제 8건 (캐스케이드, AutoStop, 복구 포함)
- [x] **SEC-04**: 키스토어 보안 시나리오 4개 이상이 정의됨 — 실제 10건 (키스토어 6 + 외부 위협 4)
- [x] **SEC-05**: 경계값 테스트 케이스가 정의됨 — 실제 19건 + E2E 체인 5건

### 블록체인 테스트 전략

- [x] **CHAIN-01**: Solana 테스트 3단계 환경(Mock RPC / Local Validator / Devnet)별 실행 범위와 시나리오가 정의됨
- [x] **CHAIN-02**: Mock RPC 클라이언트의 시나리오(성공/실패/지연/Blockhash 만료)가 명세됨 — 13개 시나리오
- [x] **CHAIN-03**: Local Validator(solana-test-validator) 기반 E2E 테스트 흐름이 정의됨 — 5개 흐름
- [x] **CHAIN-04**: EVM 테스트 범위가 정의됨 — EvmAdapterStub 5개 테스트 항목

### Enum & 설정 일관성 검증

- [x] **ENUM-01**: 9개 Enum SSoT 동기화 검증 방법이 정의됨 — as const 파생 체인 + 빌드타임/Unit/Integration 검증
- [x] **ENUM-02**: config.toml 검증 전략이 정의됨 — 12개 테스트 케이스
- [x] **ENUM-03**: NOTE-01~11 테스트 매핑 완료 — 4건 필요(22 케이스), 7건 불필요

### CI/CD 파이프라인

- [x] **CICD-01**: 4단계 파이프라인 구조가 정의됨 (매 커밋/매 PR/nightly/릴리스)
- [x] **CICD-02**: GitHub Actions 워크플로우 구조가 설계됨 — 4 YAML + 1 composite action
- [x] **CICD-03**: 커버리지 게이트 및 리포트 자동 생성 방식이 정의됨 — Soft/Hard 전환 + 6단계 우선순위

### 배포 타겟별 테스트

- [x] **PLAT-01**: CLI Daemon 테스트 범위가 정의됨 — 32건 시나리오
- [x] **PLAT-02**: Docker 테스트 범위가 정의됨 — 18건 시나리오
- [x] **PLAT-03**: Desktop App(Tauri) 테스트 범위가 정의됨 — 자동 6건 + 수동 QA 28건
- [x] **PLAT-04**: Telegram Bot 테스트 범위가 정의됨 — 34건 시나리오
