### Critical Integration Issue

**Phase 403 migrateAmount -> Pre-existing Provider Tests:**
The migration from `parseTokenAmount()` to `migrateAmount()` changed zero-amount handling. `parseTokenAmount()` threw an error for zero amounts; `migrateAmount()` passes `BigInt(0)` through. This breaks 4 pre-existing tests:

| Test File | Line | Expected Behavior |
|-----------|------|-------------------|
| `packages/actions/src/__tests__/aave-v3-provider.test.ts` | 334 | `rejects.toThrow('Amount must be greater than 0')` |
| `packages/actions/src/__tests__/kamino-provider.test.ts` | 345 | `rejects.toThrow('Amount must be greater than 0')` |
| `packages/actions/src/__tests__/lido-staking.test.ts` | 84 | `rejects.toThrow('Amount must be greater than 0')` |
| `packages/actions/src/__tests__/jito-staking.test.ts` | 107 | `rejects.toThrow('Amount must be greater than 0')` |

**Root cause:** `migrateAmount()` deliberately passes zero through per design decision ("Zero value (0n) passed through without validation -- provider-level validation handles it"), but the provider-level validation for zero amounts was in `parseTokenAmount()`, which is no longer called.

**Fix required:** Either add zero-amount guard in `migrateAmount()` or add explicit zero-amount validation in each provider's resolve method after migration.
