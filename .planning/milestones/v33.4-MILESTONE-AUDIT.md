## 4. E2E Flow Verification

### Flow 1: Exclusive Signing Toggle (Backend)

```
Admin UI radio select → PUT /v1/admin/wallet-apps/{id} {signing_enabled: true}
  → wallet-apps.ts route handler → WalletAppService.update(signingEnabled=true)
    → SQLite transaction: disable peers WHERE wallet_type=? AND signing_enabled=1 AND id!=?
    → UPDATE target app signing_enabled=1
    → Partial unique index enforces at-most-one constraint
  → Response → Admin UI refetches → Radio state updated
```
**Status: COMPLETE** -- verified by TST-01 tests (7 cases) + Admin UI T-ADM-03/T-ADM-05 tests

### Flow 2: Sign Request Routing via signing_enabled

```
Transaction pipeline → PENDING_APPROVAL state
  → ApprovalChannelRouter reads wallet_type from wallets table (line 79-80)
  → Enriches params.walletName = row.wallet_type (line 89)
  → Checks signing_enabled=1 exists for wallet_type (line 95-96)
  → SignRequestBuilder.buildRequest() queries:
      SELECT ... FROM wallet_apps WHERE wallet_type=? AND signing_enabled=1 (line 107)
  → Uses push_relay_url from signing_enabled app (line 175)
  → Sends to Push Relay → Wallet app receives sign request
```
**Status: COMPLETE** -- verified by TST-03 tests (3 integration scenarios) + preferred_wallet ignored test

### Flow 3: Admin UI wallet_type Grouping

```
Page load → GET /v1/admin/wallet-apps
  → Response includes wallet_type per app
  → groupAppsByWalletType() creates Map<string, WalletAppApi[]>
  → Each group rendered with: header, radio group ("None" + per-app options), app cards
  → Radio selection → handleSigningRadioChange() → PUT API → refetch → re-render
```
**Status: COMPLETE** -- verified by T-ADM-01 through T-ADM-06 tests (6 cases)

### Flow 4: Preset Auto-Setup

```
Owner registration with wallet preset
  → PresetAutoSetupService.apply()
    → Step 4: walletAppService.ensureRegistered() + update(signingEnabled=true)
    → Exclusive toggle handles disabling other apps of same wallet_type
    → No preferred_wallet setting written (v33.4 removed)
```
**Status: COMPLETE** -- verified by T-AUTO-01 and T-APP-09 tests
