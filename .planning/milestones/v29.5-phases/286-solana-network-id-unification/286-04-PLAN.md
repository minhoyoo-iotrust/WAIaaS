# Plan 286-04: 테스트 업데이트 + 전 패키지 테스트 통과

## 목적

Plans 286-01~03에서 변경된 SSoT 상수, 인프라, 인터페이스에 맞춰 전 패키지의 테스트 파일에서 Solana 네트워크 참조를 `solana-mainnet`/`solana-devnet`/`solana-testnet`으로 업데이트하고, `pnpm turbo run test:unit`으로 전체 통과를 확인한다.

## 변환 규칙

테스트 파일에서 Solana 네트워크 참조를 다음과 같이 일괄 변환한다. **중요**: Solana 체인 컨텍스트에서만 변환하며, `environment: 'mainnet'`은 변환하지 않는다 (EnvironmentType은 별개 개념).

| 기존값 | 새 값 | 컨텍스트 |
|--------|--------|----------|
| `network: 'mainnet'` (Solana) | `network: 'solana-mainnet'` | wallet/tx/policy 등에서 chain='solana'인 경우 |
| `network: 'devnet'` (Solana) | `network: 'solana-devnet'` | chain='solana'인 경우 |
| `network: 'testnet'` (Solana) | `network: 'solana-testnet'` | chain='solana'인 경우 |
| `'mainnet'` in array/set (Solana 네트워크 목록) | `'solana-mainnet'` | SOLANA_NETWORK_TYPES 참조 |

**변환하지 않는 경우:**
- `environment: 'mainnet'` -- EnvironmentType은 `'mainnet'`/`'testnet'`으로 유지
- `mode: 'mainnet'` -- CLI 모드는 환경(environment) 개념
- EVM 컨텍스트의 `'mainnet'` -- EVM에서의 `mainnet` 참조는 이미 `ethereum-mainnet` 등이므로 해당 없음
- `EVM_CONFIRMATION_THRESHOLDS['mainnet']` → `EVM_CONFIRMATION_THRESHOLDS['ethereum-mainnet']` (Plan 286-02에서 소스 변경됨)
- `createQuickstartFetchMock('mainnet')` -- 이것은 environment 파라미터이므로 유지
