# Plan 286-03: 인터페이스 레이어 업데이트

## 목적

CLI/API 레거시 입력 자동 변환, Admin UI 네트워크 표시, MCP/SDK 네트워크 타입, skills 파일 예시를 `solana-mainnet` 형식으로 업데이트한다.

## 태스크

<task id="286-03-T1">
**레거시 입력 자동 변환 + deprecation 경고**

1. `packages/core/src/enums/chain.ts`에 normalizer 함수 추가:

   ```typescript
   /**
    * Legacy Solana network name mapping.
    * Auto-converts bare 'mainnet'/'devnet'/'testnet' to 'solana-mainnet'/'solana-devnet'/'solana-testnet'.
    * Emits deprecation warning on first use.
    */
   const LEGACY_SOLANA_NETWORK_MAP: Record<string, NetworkType> = {
     'mainnet': 'solana-mainnet',
     'devnet': 'solana-devnet',
     'testnet': 'solana-testnet',
   };

   let _legacyWarned = false;

   /**
    * Normalize a network input string, converting legacy Solana names to new format.
    * Returns the canonical NetworkType if valid, or the input unchanged if not a legacy name.
    * Emits a deprecation warning to stderr on first legacy conversion.
    */
   export function normalizeNetworkInput(network: string): string {
     const mapped = LEGACY_SOLANA_NETWORK_MAP[network];
     if (mapped) {
       if (!_legacyWarned) {
         console.warn(
           `[WAIaaS DEPRECATION] Network name '${network}' is deprecated. Use '${mapped}' instead. ` +
           `Legacy names will be removed in a future release.`
         );
         _legacyWarned = true;
       }
       return mapped;
     }
     return network;
   }
   ```

   `_legacyWarned` 플래그로 경고 스팸 방지 (세션당 1회). `normalizeNetworkInput`은 `@waiaas/core`에서 export.

2. `packages/core/src/index.ts`에서 `normalizeNetworkInput` export 추가.

3. `packages/daemon/src/pipeline/network-resolver.ts`의 `resolveNetwork()` 함수 수정:
   - 입력받은 `requestNetwork`에 `normalizeNetworkInput` 적용:
   ```typescript
   import { normalizeNetworkInput } from '@waiaas/core';
   // ...
   const normalizedRequest = requestNetwork ? normalizeNetworkInput(requestNetwork) as NetworkType : undefined;
   const resolved: NetworkType | null = normalizedRequest ?? getSingleNetwork(chain, environment);
   ```

4. REST API 라우트에서의 `network` 파라미터 처리 -- Zod `NetworkTypeEnum` validation이 `solana-mainnet` 형식만 허용하므로, REST API 쿼리 파라미터 파싱 전에 normalizer를 적용해야 함. 이를 위해 API 라우트의 network 입력 처리 부분에서 `normalizeNetworkInput` 적용:
   - 이 부분은 Zod 스키마 `.transform()` 또는 `.preprocess()`로 처리하는 것이 가장 깔끔함
   - `packages/core/src/schemas/transaction.schema.ts` 등에서 네트워크 필드에 `.transform(normalizeNetworkInput)` 적용 또는
   - REST API 라우트 핸들러에서 직접 normalize 호출

   **판단**: REST API 라우트에서 query param으로 받는 `network` 필드에 `normalizeNetworkInput` 적용. 기존 `NetworkTypeEnum.parse()`가 실패하면 400 에러가 나므로, parse 전에 normalize 필요. Zod의 `.preprocess()` 사용:
   ```typescript
   // packages/core/src/enums/chain.ts에 추가
   export const NetworkTypeEnumWithLegacy = z.preprocess(
     (val) => (typeof val === 'string' ? normalizeNetworkInput(val) : val),
     NetworkTypeEnum,
   );
   ```
   이 스키마를 REST API 트랜잭션 요청 스키마에서 사용. 기존 `NetworkTypeEnum`은 strict validation(DB 저장, 내부 로직)으로 유지.

5. `packages/core/src/index.ts`에서 `normalizeNetworkInput`, `NetworkTypeEnumWithLegacy` export 추가:
   ```typescript
   export {
     // ... existing exports ...
     normalizeNetworkInput,
     NetworkTypeEnumWithLegacy,
   } from './enums/index.js';
   ```

6. `packages/core/src/schemas/transaction.schema.ts`에서 `NetworkTypeEnum` → `NetworkTypeEnumWithLegacy` 교체:
   - 레거시 `mainnet` 입력이 Zod validation 단계에서 400 차단되는 것을 방지하기 위해, 트랜잭션 요청 스키마의 `network` 필드를 `NetworkTypeEnumWithLegacy`로 교체:
   ```typescript
   import { NetworkTypeEnumWithLegacy } from '../enums/chain.js';

   // TransferRequestSchema
   network: NetworkTypeEnumWithLegacy.optional(),

   // TokenTransferRequestSchema
   network: NetworkTypeEnumWithLegacy.optional(),

   // ContractCallRequestSchema
   network: NetworkTypeEnumWithLegacy.optional(),

   // ApproveRequestSchema
   network: NetworkTypeEnumWithLegacy.optional(),

   // BatchRequestSchema
   network: NetworkTypeEnumWithLegacy.optional(),
   ```
   - `TransactionSchema`(응답 스키마)의 `network` 필드는 DB에서 읽은 값이므로 `NetworkTypeEnum`을 유지한다.
   - 이렇게 하면 REST API에서 `POST /v1/transactions/send`로 `"network": "mainnet"` 입력 시 Zod preprocess가 `"solana-mainnet"`으로 변환 후 `NetworkTypeEnum` validation을 통과한다.

7. **MCP 네트워크 파라미터 스키마 (NETID-06)**: MCP 도구들(`packages/mcp/src/tools/*.ts`)은 `network` 파라미터에 `z.string()`을 사용하고, 값을 REST API에 그대로 전달한다. MCP 자체에는 별도의 `NetworkTypeEnum` validation이 없으므로 MCP 파일 수정은 불필요하다. 레거시 입력 normalization은 REST API의 `TransferRequestSchema` 등에서 `NetworkTypeEnumWithLegacy`가 처리하므로, MCP → REST API 경로에서도 레거시 `mainnet` 입력이 정상 변환된다.

</task>
