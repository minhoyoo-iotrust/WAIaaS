Push a new migration with `version: 28`, `managesOwnTransaction: true`:

```typescript
MIGRATIONS.push({
  version: 28,
  description: 'Migrate api_keys to settings table and drop api_keys (v29.5 #214)',
  managesOwnTransaction: true,
  up: (sqlite) => {
    sqlite.exec('BEGIN');
    try {
      // 1. Check if api_keys table exists (may not exist on very old DBs that skipped v11)
      const tableExists = sqlite.prepare(
        "SELECT name FROM sqlite_master WHERE type='table' AND name='api_keys'"
      ).get();

      if (tableExists) {
        // 2. Read all api_keys rows
        const rows = sqlite.prepare('SELECT provider_name, encrypted_key, updated_at FROM api_keys').all() as Array<{
          provider_name: string;
          encrypted_key: string;
          updated_at: number;
        }>;

        // 3. For each row, insert into settings with key = 'actions.{provider_name}_api_key'
        // Only insert if the setting doesn't already exist (avoid overwriting manual settings)
        const insertStmt = sqlite.prepare(
          `INSERT OR IGNORE INTO settings (key, value, encrypted, category, updated_at)
           VALUES (?, ?, 1, 'actions', ?)`
        );

        for (const row of rows) {
          const settingKey = `actions.${row.provider_name}_api_key`;
          insertStmt.run(settingKey, row.encrypted_key, row.updated_at);
        }

        // 4. Drop api_keys table
        sqlite.exec('DROP TABLE IF EXISTS api_keys');
      }

      sqlite.exec('COMMIT');
    } catch (err) {
      sqlite.exec('ROLLBACK');
      throw err;
    }
  },
});
```

**Important notes:**
- The `encrypted_key` values in `api_keys` use the SAME encryption format as `settings.value` (both use `encryptSettingValue` from `settings-crypto.ts`), so they can be copied directly without re-encryption.
- Use `INSERT OR IGNORE` to avoid overwriting if user already set the key via `PUT /admin/settings`.
- The `encrypted` column is set to `1` since API keys are credentials.

**1. Add `hasApiKey(providerName)` and `getApiKeyMasked(providerName)` helper methods to SettingsService:**

```typescript
/**
 * Check if an API key exists for a provider in settings.
 * Looks up 'actions.{providerName}_api_key' and returns true if non-empty value exists in DB.
 */
hasApiKey(providerName: string): boolean {
  const key = `actions.${providerName}_api_key`;
  // Only check DB -- don't fall through to default empty string
  const row = this.db.select().from(settings).where(eq(settings.key, key)).get();
  if (!row) return false;
  // Decrypt and check non-empty
  const val = row.encrypted ? decryptSettingValue(row.value, this.currentPassword) : row.value;
  return val.length > 0;
}

/**
 * Get masked API key for a provider (for display in admin UI).
 * Returns null if no key exists.
 */
getApiKeyMasked(providerName: string): string | null {
  const key = `actions.${providerName}_api_key`;
  const row = this.db.select().from(settings).where(eq(settings.key, key)).get();
  if (!row) return null;
  const val = row.encrypted ? decryptSettingValue(row.value, this.currentPassword) : row.value;
  if (val.length === 0) return null;
  return maskApiKey(val);
}
```
