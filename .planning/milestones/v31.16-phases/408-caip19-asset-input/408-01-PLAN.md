       ```typescript

         address: z.string().min(1).optional(),
         decimals: z.number().int().min(0).max(18).optional(),
         symbol: z.string().min(1).max(10).optional(),

       ```

    2. **TokenInfoSchema superRefine 확장**: 기존 cross-validation을 유지하면서 새로운 규칙 추가:
       - assetId가 없으면: address, decimals, symbol 모두 필수 (기존 동작 유지, AST-01)
       - assetId가 있고 address도 있으면: 기존 대소문자 무시 비교 유지 (AST-04)
       - assetId가 있고 레지스트리에 없으면: address는 assetId에서 추출 가능하므로 optional이지만, decimals/symbol은 필수 (AST-03) -- 이 규칙은 Plan 02에서 레지스트리 resolve 후 적용. 여기서는 "assetId만 있으면 address/decimals/symbol optional" 스키마 수준 변경만.
       - assetId의 네트워크 vs 요청 network 불일치 검증은 이 스키마 레벨이 아닌 요청 레벨에서 처리 (TokenInfo에는 network가 없음).

    3. **superRefine 규칙 (token-level)**:
       ```
       if (!assetId && !address) => error: "address is required when assetId is not provided"
       if (!assetId && decimals === undefined) => error: "decimals is required when assetId is not provided"
       if (!assetId && !symbol) => error: "symbol is required when assetId is not provided"
       if (assetId && address) => 기존 cross-validation (대소문자 무시 비교)
       ```

    4. 기존 `TokenTransferRequestSchema`, `ApproveRequestSchema`는 TokenInfoSchema를 참조하므로 자동 적용.
