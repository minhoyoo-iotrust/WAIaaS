  <files>packages/core/src/caip/response-enrichment.ts, packages/core/src/caip/index.ts, packages/core/src/__tests__/response-enrichment.test.ts</files>
  <behavior>
    - enrichBalance({ network, ... }) -> { ...original, chainId: 'eip155:1' }
    - enrichBalance({ network: 'ethereum-mainnet', ... }) -> chainId = 'eip155:1', assetId = 'eip155:1/slip44:60'
    - enrichAsset({ network, mint, isNative: true, ... }) -> assetId = slip44 namespace
    - enrichAsset({ network, mint, isNative: false, ... }) -> assetId = erc20/token namespace (address from mint)
    - enrichNft({ network, contractAddress, tokenId, standard, ... }) -> assetId = erc721/erc1155/metaplex
    - enrichTransaction({ network, ... }) -> { ...original, chainId }
    - enrichIncomingTx({ network, tokenAddress, chain, ... }) -> { ...original, chainId, assetId? } (assetId only when tokenAddress present or native)
    - 모든 함수는 원본 객체의 모든 필드를 spread하여 유지 (additive only)
    - network가 매핑에 없으면 chainId 필드를 생략 (에러 아닌 graceful skip)
  </behavior>
  <action>
    1. `packages/core/src/caip/response-enrichment.ts` 생성:
       - `enrichBalance<T>(obj: T & { network: string }): T & { chainId?: string; assetId?: string }` -- networkToCaip2로 chainId 생성, nativeAssetId로 네이티브 assetId 생성. try-catch로 unknown network graceful skip.
       - `enrichAsset<T>(obj: T & { network: string; mint: string; isNative: boolean }): T & { chainId?: string; assetId?: string }` -- chainId 생성 + isNative이면 nativeAssetId, 아니면 tokenAssetId(network, mint). Solana의 경우 mint(base58 원본 케이스 유지), EVM의 경우 자동 lowercase.
       - `enrichNft<T>(obj: T & { network: string; contractAddress: string; tokenId: string; standard: string }): T & { chainId?: string; assetId?: string }` -- nftAssetId 호출. standard가 erc721/erc1155/metaplex가 아니면 assetId 생략.
       - `enrichTransaction<T>(obj: T & { network?: string | null }): T & { chainId?: string }` -- network가 있으면 chainId 추가.
       - `enrichIncomingTx<T>(obj: T & { network: string; chain: string; tokenAddress?: string | null }): T & { chainId?: string; assetId?: string }` -- chainId 항상. tokenAddress가 있으면 tokenAssetId, null이면 nativeAssetId.
       - 모든 함수: try-catch로 매핑 없는 network 처리 (graceful, 원본 그대로 반환)
    2. `packages/core/src/caip/index.ts`에 re-export 추가.
    3. 테스트: EVM 네트워크(ethereum-mainnet), Solana 네트워크(solana-mainnet), 미지원 네트워크, 각 함수별 정상/엣지 케이스.
  </action>
