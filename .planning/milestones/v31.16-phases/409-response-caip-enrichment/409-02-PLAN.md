  <name>Task 1: 전체 응답 포인트에 enrichment 적용</name>
  <files>packages/daemon/src/api/routes/wallet.ts, packages/daemon/src/api/routes/transactions.ts, packages/daemon/src/api/routes/incoming.ts, packages/daemon/src/api/routes/nfts.ts, packages/daemon/src/api/routes/defi-positions.ts, packages/daemon/src/api/routes/staking.ts, packages/daemon/src/api/routes/tokens.ts</files>
  <action>
    모든 라우트에서 `@waiaas/core`로부터 enrichment 함수를 import하고, `c.json()` 반환 직전에 enrichment를 적용한다. 패턴: `c.json(enrichXxx({ ...existingResponse }), 200)`. 기존 필드는 절대 제거하지 않는다.

    **wallet.ts (balance + assets):**
    - `enrichBalance`, `enrichAsset` import from `@waiaas/core`.
    - GET /wallet/balance: `c.json(enrichBalance({ walletId, chain, network: targetNetwork, address, balance, decimals, symbol, balanceFormatted, displayBalance, displayCurrency }), 200)`. network=all 경우도 각 balance 결과에 enrichBalance 적용.
    - GET /wallet/assets: 각 asset 객체에 `enrichAsset({ network, mint, isNative, ...rest })` 적용. 최상위에도 `chainId: networkToCaip2(targetNetwork)` 추가 (try-catch).

    **transactions.ts:**
    - `enrichTransaction` import.
    - GET /transactions (list): items 매핑 시 각 tx에 enrichTransaction 적용.
    - GET /transactions/:id (detail): 단일 tx에 enrichTransaction 적용.
    - GET /transactions/pending: items 매핑 시 enrichTransaction 적용.

    **incoming.ts:**
    - `enrichIncomingTx` import.
    - GET /wallet/incoming: data 배열의 각 항목에 enrichIncomingTx 적용.

    **nfts.ts:**
    - `networkToCaip2` import from `@waiaas/core`. NFT 응답은 이미 assetId를 포함하므로 chainId만 추가.
    - GET /wallet/nfts: 각 NFT 아이템에 `chainId` 필드 추가 (try-catch networkToCaip2). list와 grouped 모두.
    - GET /wallet/nfts/:contractAddress/:tokenId: 단일 NFT 메타데이터에 chainId 추가.

    **defi-positions.ts:**
    - `networkToCaip2` import. positions 배열의 각 position에 `chainId` 추가 (position.network가 있으면).

    **staking.ts:**
    - `networkToCaip2` import. staking positions 배열의 각 position에 `chainId` 추가. chain이 'ethereum'이면 mainnet 네트워크 추론, 'solana'이면 solana-mainnet 추론.

    **tokens.ts:**
    - GET /tokens 목록에서 각 토큰에 `chainId` 추가 (query의 network 파라미터로 networkToCaip2).

    **중요:** 모든 enrichment는 try-catch로 감싸서 unknown network에서 에러 없이 기존 응답을 그대로 반환한다. `as never` 캐스트가 필요한 곳은 기존 패턴 따라 적용.
