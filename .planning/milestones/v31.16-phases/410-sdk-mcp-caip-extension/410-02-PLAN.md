1. `packages/mcp/src/tools/resolve-asset.ts` 신규 생성:
   ```typescript
   import { z } from 'zod';
   import type { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
   import { type ApiClient, toToolResult } from '../api-client.js';

   export function registerResolveAsset(server: McpServer, apiClient: ApiClient): void {
     server.tool(
       'resolve_asset',
       'Resolve a CAIP-19 asset identifier to token metadata. Returns address, decimals, symbol, name, network, chainId, isNative, isRegistered.',
       {
         asset_id: z.string().describe('CAIP-19 asset identifier (e.g., "eip155:1/erc20:0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48", "eip155:1/slip44:60")'),
       },
       async (args) => { ... }
     );
   }
   ```

2. 구현 로직:
   - CAIP-19 문자열을 직접 파싱 (MCP 패키지는 @waiaas/core에 의존하지 않으므로 간단한 로컬 파서 사용):
     - `chainId/assetNamespace:assetReference` 형식으로 split
     - `slip44` namespace -> isNative=true, address=null
     - `erc20`/`token` namespace -> isNative=false, address=assetReference
   - 파싱 실패 시 에러 결과 반환 (toToolResult 사용하지 않고 직접 content 반환)
   - 파싱 성공 시 `GET /v1/tokens?network={network}` 호출하여 레지스트리에서 해당 address 검색
     - network는 CAIP-2 chainId를 그대로 전달 (daemon이 CAIP-2 수용 -- Phase 407에서 구현)
   - 레지스트리에서 찾으면: `{ address, decimals, symbol, name, network, chainId, isNative, isRegistered: true }`
   - 못 찾으면: `{ address (파싱 결과), decimals: null, symbol: null, name: null, network, chainId, isNative, isRegistered: false }`
   - 응답은 JSON text content로 반환 (다른 MCP 도구 패턴 따름)

3. `server.ts`에 등록:
   - `import { registerResolveAsset } from './tools/resolve-asset.js';` 추가
   - `registerResolveAsset(server, apiClient);` 호출 추가 (walletContext 불필요 -- 글로벌 도구)

   - Mock ApiClient로 GET /v1/tokens 응답 설정
   - 등록된 토큰: USDC at eip155:1 -> full metadata
   - 미등록 토큰: eip155:1/erc20:0xabc... -> isRegistered=false
   - 네이티브: eip155:1/slip44:60 -> isNative=true
   - 잘못된 형식: "invalid" -> 에러
