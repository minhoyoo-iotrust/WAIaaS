**IMPORTANT -- from address issue:** The safeTransferFrom for both ERC-721 and ERC-1155 requires an explicit `from` address. In the Smart Account context, this is the smart account address itself. The current `buildUserOpCalls` does NOT receive walletPublicKey. Two approaches:
  - (a) Extend `buildUserOpCalls` to accept an optional `walletAddress` parameter (breaking change risk -- check all callers)
  - (b) The smart account is the sender (`msg.sender`) and also the `from` -- so smart account address = sender. The bundler will execute the UserOp from the smart account, which means `from` should be the smart account address. Since `stage5ExecuteSmartAccount` has `ctx.wallet.publicKey`, pass it through.

**Choose approach (a):** Extend `buildUserOpCalls(request, walletAddress?)` with optional second param. Update `stage5ExecuteSmartAccount` to pass `ctx.wallet.publicKey`. This is backward compatible since the param is optional and only used for NFT_TRANSFER. Update the args to use `walletAddress` instead of zero address.
