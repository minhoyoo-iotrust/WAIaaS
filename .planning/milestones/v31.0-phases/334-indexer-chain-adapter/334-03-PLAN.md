    1. Import the new ABI files and NftTransferParams/NftApproveParams types:
       ```typescript
       import { ERC721_ABI } from './abi/erc721.js';
       import { ERC1155_ABI } from './abi/erc1155.js';
       import { ERC165_ABI, ERC_INTERFACE_IDS } from './abi/erc165.js';
       import type { NftTransferParams, NftApproveParams } from '@waiaas/core';
       ```

    2. Add private helper `detectNftStandard(contractAddress: string): Promise<'ERC-721' | 'ERC-1155'>`:
       - Call `supportsInterface(0x80ac58cd)` (ERC-721 interface ID) via viem `readContract`.
       - If true -> 'ERC-721'.
       - Call `supportsInterface(0xd9b67a26)` (ERC-1155 interface ID).
       - If true -> 'ERC-1155'.
       - If neither -> throw `WAIaaSError('UNSUPPORTED_NFT_STANDARD')`.
       - Wrap in try/catch: if readContract fails (contract doesn't support ERC-165), throw UNSUPPORTED_NFT_STANDARD.

    3. `buildNftTransferTx(request: NftTransferParams): Promise<UnsignedTransaction>`:
       - Based on `request.token.standard`:
         - `'ERC-721'`: encodeFunctionData with ERC721_ABI `safeTransferFrom(from, to, tokenId)`.
         - `'ERC-1155'`: encodeFunctionData with ERC1155_ABI `safeTransferFrom(from, to, id, amount, '0x')`.
       - Gas estimation with safety margin: `(estimatedGas * 120n) / 100n`.
       - Get nonce, EIP-1559 fee data (same pattern as buildApprove).
       - Build and serialize EIP-1559 transaction, value=0n.
       - Return UnsignedTransaction with metadata including `nftStandard`, `tokenId`, `contractAddress`.

    4. `transferNft(request: NftTransferParams, privateKey: Uint8Array): Promise<SubmitResult>`:
       - Call `buildNftTransferTx(request)` -> `signTransaction(tx, privateKey)` -> `submitTransaction(signedTx)`.
       - Return SubmitResult.

    5. `approveNft(request: NftApproveParams): Promise<UnsignedTransaction>`:
       - `'single'` + `'ERC-721'`: encode `approve(spender, tokenId)` using ERC721_ABI.
       - `'all'` + `'ERC-721'`: encode `setApprovalForAll(spender, true)` -- note: ERC-721 also has setApprovalForAll, add it to erc721.ts ABI.
       - `'single'` + `'ERC-1155'`: NOT SUPPORTED (ERC-1155 only has setApprovalForAll) -> throw WAIaaSError.
       - `'all'` + `'ERC-1155'`: encode `setApprovalForAll(operator, true)` using ERC1155_ABI.
       - Same gas estimation pattern.

    1. Import NftTransferParams/NftApproveParams. Metaplex NFTs are SPL tokens (amount=1, decimals=0).

    2. `buildNftTransferTx(request: NftTransferParams): Promise<UnsignedTransaction>`:
       - Metaplex NFT transfer is identical to SPL token transfer with decimals=0, amount=1 (or request.amount).
       - Reuse existing SPL transfer logic from buildTokenTransfer:
         - Find source ATA (owner's associated token account for mint).
         - Find/create destination ATA.
         - Create `getTransferCheckedInstruction` with amount and decimals=0.
       - Build message with system instructions (ATA creation if needed) + transfer instruction.
       - Compile and return UnsignedTransaction.

    3. `transferNft(request: NftTransferParams, privateKey: Uint8Array): Promise<SubmitResult>`:
       - Call `buildNftTransferTx(request)` -> `signTransaction(tx, privateKey)` -> `submitTransaction(signedTx)`.

    4. `approveNft(request: NftApproveParams): Promise<UnsignedTransaction>`:
       - Use `getApproveCheckedInstruction` from @solana-program/token (already imported in adapter).
       - Delegates the token to the spender with amount=1, decimals=0.
       - `approvalType: 'all'` is not applicable to Solana (no collection-wide approval) -> throw WAIaaSError('UNSUPPORTED_NFT_STANDARD', { message: 'Solana does not support collection-wide NFT approval' }).
