    **Part A: HeliusNftIndexer** -- Create `packages/daemon/src/infrastructure/nft/helius-nft-indexer.ts`:

    1. `HeliusNftIndexer` class implementing `INftIndexer`:
       - Constructor: `{ apiKey: string, baseUrl?: string }`. Default baseUrl: `https://mainnet.helius-rpc.com`.
       - `provider = 'helius'`, `supportedChains = ['solana']`.
       - Helius DAS API uses JSON-RPC: POST to `{baseUrl}/?api-key={apiKey}`.

    2. `listNfts(options)`:
       - JSON-RPC method: `getAssetsByOwner` with params `{ ownerAddress, limit, page, sortBy: { sortBy: 'created', sortDirection: 'desc' } }`.
       - Map response `items[]` to `NftItem[]`: `id` -> mintAddress (also contractAddress), `content.metadata.name` -> name, `content.links.image` -> image, `content.metadata.description` -> description, standard = 'METAPLEX'.
       - Generate `assetId` using `nftAssetId(network, mintAddress, undefined, 'METAPLEX')`.
       - `tokenId` for Metaplex is the mint address itself.
       - Pagination: Helius uses `page` (1-based), convert from/to cursor by encoding page number.

    3. `getNftMetadata(network, contractAddress, tokenId)`:
       - JSON-RPC method: `getAsset` with params `{ id: contractAddress }` (for Metaplex, contractAddress IS the mint).
       - Map `content.metadata.attributes` to NftMetadata attributes array.

    4. `getNftsByCollection(network, collectionAddress, pageKey?)`:
       - JSON-RPC method: `getAssetsByGroup` with params `{ groupKey: 'collection', groupValue: collectionAddress, limit, page }`.

    5. Error handling: non-200 HTTP -> `INDEXER_API_ERROR`, JSON-RPC error -> `INDEXER_API_ERROR`.

    **Part B: Setting keys** -- Add to `setting-keys.ts` (in actions category, after existing DeFi provider keys):
    ```typescript
    // --- NFT Indexer ---
    { key: 'actions.alchemy_nft_api_key', category: 'actions', configPath: 'actions.alchemy_nft_api_key', defaultValue: '', isCredential: true },
    { key: 'actions.helius_das_api_key', category: 'actions', configPath: 'actions.helius_das_api_key', defaultValue: '', isCredential: true },
    { key: 'actions.nft_indexer_cache_ttl_sec', category: 'actions', configPath: 'actions.nft_indexer_cache_ttl_sec', defaultValue: '300', isCredential: false },
    ```
    Also add to `loader.ts` DaemonConfig actions section with corresponding fields and Zod defaults.

    **Part C: NftIndexerClient** -- Create `packages/daemon/src/infrastructure/nft/nft-indexer-client.ts`:

    1. `NftIndexerClient` class wrapping INftIndexer with cross-cutting concerns:
       - Constructor: `{ settingsService: SettingsService }`.
       - `getIndexer(chain: ChainType): INftIndexer` -- resolves the correct indexer:
         - `solana` -> check `actions.helius_das_api_key` from settings -> if empty, throw `INDEXER_NOT_CONFIGURED` -> else create/cache `HeliusNftIndexer({ apiKey })`.
         - `ethereum` -> check `actions.alchemy_nft_api_key` -> if empty, throw `INDEXER_NOT_CONFIGURED` -> else create/cache `AlchemyNftIndexer({ apiKey })`.

    2. **Retry wrapper** (`withRetry<T>(fn: () => Promise<T>): Promise<T>`):
       - Max 3 retries. Retry on: HTTP 429 (rate limit), 500/502/503 (server error).
       - Exponential backoff: base 1000ms, multiplier 2x (1s, 2s, 4s).
       - Respect `Retry-After` header: if INDEXER_API_ERROR has `retryAfter` in details, wait that many seconds instead of backoff.
       - Non-retryable errors (4xx except 429) throw immediately.

    3. **In-memory cache** (Map-based):
       - Key: `${method}:${JSON.stringify(args)}`.
       - TTL from settings: `actions.nft_indexer_cache_ttl_sec` (default 300s = 5min).
       - `listNfts` and `getNftsByCollection` are cached. `getNftMetadata` is NOT cached here (DB cache in Phase 335).
       - `clearCache()` method for manual invalidation.

    4. Convenience methods that combine getIndexer + retry + cache:
       - `listNfts(chain, options)` -> `withRetry(() => cache.getOrSet(key, () => getIndexer(chain).listNfts(options)))`
       - `getNftMetadata(chain, network, contractAddress, tokenId)` -> `withRetry(() => getIndexer(chain).getNftMetadata(...))`
       - `getNftsByCollection(chain, network, collectionAddress, pageKey?)` -> `withRetry(() => cache.getOrSet(...))`

    Update `packages/daemon/src/infrastructure/nft/index.ts` to export all classes.

    **Tests**: Mock `fetch` for Helius tests (JSON-RPC format). Mock SettingsService for NftIndexerClient tests. Use `vi.useFakeTimers()` for cache TTL tests. Verify retry behavior with sequential fetch mock returns (429 -> 429 -> 200).
