CONTRIBUTING.md를 영문으로 작성한다. 구조:

1. **Contributing to WAIaaS** (서문 -- 기여 환영, CoC 미적용 간소 프로젝트임을 명시)
2. **Development Setup**:
   - Prerequisites: Node.js 22 LTS, pnpm 9+
   - Clone + install: `git clone && pnpm install && pnpm build`
   - Dev mode: `pnpm dev` (daemon watch + admin HMR)
   - Data directory: `waiaas init` (dev mode: `waiaas init --dev`)
3. **Project Structure** (모노레포 9 패키지 간단 설명 + python-sdk)
4. **Code Style**:
   - TypeScript strict, ESLint + Prettier
   - `pnpm lint && pnpm format:check && pnpm typecheck`
   - Zod SSoT: Zod 스키마가 단일 진실 원천 -> TS -> OpenAPI -> Drizzle
   - 코드 주석/변수명은 영어, 기획/설계 문서는 한글
5. **Testing**:
   - `pnpm test` (전체), `pnpm --filter @waiaas/daemon test` (패키지별)
   - Vitest 3.x, 테스트 파일 `*.test.ts` 규칙
   - Coverage: `pnpm --filter @waiaas/daemon exec vitest run --coverage`
6. **Pull Request Process**:
   - Fork -> feature branch -> PR to main
   - Conventional Commits: `feat:`, `fix:`, `docs:`, `test:`, `chore:`, `ci:`
   - PR 체크리스트: 테스트 통과, lint 통과, 타입 체크 통과
   - release-please가 자동으로 버전 범프 + CHANGELOG 관리
7. **Database Migrations** (v1.4부터 필수):
   - 스키마 변경 시 ALTER TABLE 증분 마이그레이션 제공
   - DB 삭제 후 재생성 금지
   - schema_version 테이블로 버전 관리
8. **Interface Sync Rule**:
   - REST API, SDK, MCP 변경 시 `skills/` 파일 동기화 필수
