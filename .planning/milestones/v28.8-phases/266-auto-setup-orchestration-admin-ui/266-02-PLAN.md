**Step A: 프리셋 목록 상수 정의** (wallets.tsx 상단)

Admin SPA는 @waiaas/core를 직접 import할 수 없으므로, 프리셋 목록을 정적으로 정의한다:

```ts
/** Builtin wallet presets — mirrored from @waiaas/core BUILTIN_PRESETS.
 *  When core adds a new preset, this list must be updated manually. */
const WALLET_PRESETS = [
  { value: 'dcent', label: "D'CENT Wallet", description: 'Hardware wallet with WalletConnect signing' },
] as const;
```

**Step B: Owner 등록 폼 UI 변경**

현재 `startEditOwner()` → `handleSaveOwner()` 흐름에서:

1. 새 signal 추가:
   ```ts
   const walletTypeSelect = useSignal<string>(''); // '' = Custom (no preset)
   ```

2. `startEditOwner()` 안에서 `walletTypeSelect.value = ''` 초기화. 만약 `wallet.value?.walletType`이 있으면 해당 값으로 초기화(재편집 시 기존 선택 유지).

3. Owner 등록 폼(ownerEditing.value가 true일 때 렌더링되는 인라인 폼)에 지갑 종류 select를 추가한다. editOwnerAddress input 위에 배치:

   ```tsx
   <div style={{ marginBottom: 'var(--space-3)' }}>
     <label style={{ display: 'block', marginBottom: 'var(--space-1)', fontSize: '0.85rem', fontWeight: 500 }}>
       Wallet Type
     </label>
     <select
       value={walletTypeSelect.value}
       onChange={(e) => { walletTypeSelect.value = (e.target as HTMLSelectElement).value; }}
       style={{
         width: '100%',
         padding: 'var(--space-2) var(--space-3)',
         border: '1px solid var(--color-border)',
         borderRadius: 'var(--radius-md)',
         background: 'var(--color-bg)',
         color: 'var(--color-text)',
         fontSize: '0.9rem',
       }}
     >
       <option value="">Custom (manual setup)</option>
       {WALLET_PRESETS.map((p) => (
         <option key={p.value} value={p.value}>
           {p.label}
         </option>
       ))}
     </select>
     {walletTypeSelect.value && (
       <p style={{ marginTop: 'var(--space-1)', fontSize: '0.8rem', color: 'var(--color-text-secondary)' }}>
         {WALLET_PRESETS.find(p => p.value === walletTypeSelect.value)?.description}
       </p>
     )}
   </div>
   ```

4. Owner 편집 모드(ownerState가 'NONE'이 아니고 이미 address가 설정된 경우)에서는 드롭다운을 표시하지 않는다. 드롭다운은 첫 Owner 등록 시에만 표시한다 (ownerState === 'NONE'인 경우). 이미 Owner가 설정된 상태에서 address 수정 시에는 기존 방식 유지.

**Step C: handleSaveOwner에 wallet_type 전달**

`handleSaveOwner()` 함수 수정:

```ts
const body: Record<string, unknown> = {
  owner_address: editOwnerAddress.value.trim(),
};
if (walletTypeSelect.value) {
  body.wallet_type = walletTypeSelect.value;
}
await apiPut(API.WALLET_OWNER(id), body);
```

성공 시 toast 메시지를 프리셋 여부에 따라 분기:
- 프리셋 선택: `'Owner set with ${preset.label} auto-setup'`
- Custom: `'Owner address updated'` (기존 동작)

**Step D: WalletDetail 인터페이스에 walletType 추가**

```ts
interface WalletDetail extends Wallet {
```
