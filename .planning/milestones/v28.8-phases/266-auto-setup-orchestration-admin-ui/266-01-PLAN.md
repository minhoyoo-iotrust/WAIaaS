**Step B: PresetAutoSetupService 구현** (packages/daemon/src/services/signing-sdk/preset-auto-setup.ts)

새 파일을 생성하고 다음 클래스를 작성한다:

```ts
export class PresetAutoSetupService {
  constructor(
    private readonly settingsService: SettingsService,
    private readonly walletLinkRegistry: WalletLinkRegistry,
  ) {}

  /**
   * Apply preset auto-setup: 4 steps with Settings snapshot rollback.
   * Steps: (1) enable signing SDK, (2) register WalletLinkConfig, (3) set preferred_wallet, (4) set preferred_channel
   * approval_method is handled by the caller (wallets.ts handler saves to DB).
   *
   * @returns { applied: string[] } — list of steps that were applied
   * @throws — re-throws after rolling back Settings snapshot
   */
  apply(preset: WalletPreset): { applied: string[] } { ... }
}
```

구현 세부사항:

1. **Settings 스냅샷 캡처**: `apply()` 진입 시 3개 키의 현재 값을 snapshot으로 저장:
   - `signing_sdk.enabled`
   - `signing_sdk.preferred_wallet`
   - `signing_sdk.preferred_channel`

2. **Step 1: signing SDK 활성화**: `settingsService.set('signing_sdk.enabled', 'true')`. 이미 'true'면 skip하고 applied에 포함하지 않는다.

3. **Step 2: WalletLinkRegistry 등록**: `walletLinkRegistry.registerWallet(preset.walletLinkConfig)` 호출. 이미 등록된 경우(`SIGN_REQUEST_ALREADY_PROCESSED` 에러) catch하여 skip — 멱등성 보장.

4. **Step 3: preferred_wallet 설정**: `settingsService.set('signing_sdk.preferred_wallet', preset.preferredWallet)`. 이미 같은 값이면 skip.

5. **Step 4: preferred_channel 설정**: preset.approvalMethod에 따라 채널을 결정한다:
   - 'walletconnect' → preferred_channel은 건드리지 않음 (WalletConnect는 signing SDK 채널이 아님)
   - 'sdk_ntfy' → `settingsService.set('signing_sdk.preferred_channel', 'ntfy')`
   - 'sdk_telegram' → `settingsService.set('signing_sdk.preferred_channel', 'telegram')`
   - 기타 → skip

6. **롤백**: try/catch로 전체를 감싸고, catch 블록에서 snapshot 3개 키를 순서대로 복원한 뒤 에러를 re-throw한다. snapshot 복원은 각각 try/catch로 감싸서 복원 실패가 다른 키 복원을 막지 않도록 한다.

7. 반환값은 `{ applied: string[] }` 형태로, 실제 적용된 단계 이름 목록을 반환한다 (예: `['signing_sdk_enabled', 'wallet_registered', 'preferred_wallet_set']`).

2. PUT /wallets/:id/owner 핸들러의 `if (walletType)` 분기 안에서:
   - 기존 `preset.approvalMethod` DB 저장 로직은 유지
   - `deps.settingsService && deps.walletLinkRegistry` 일 때 `PresetAutoSetupService`를 인스턴스화하고 `apply(preset)` 호출
   - `apply()` 호출은 SQLite 트랜잭션(`deps.sqlite.transaction(...)`) 안에서 실행하여 DB 변경(wallet_type, approval_method)과 Settings 변경이 원자적으로 수행되도록 한다

   - 성공 시 `apply()` 반환값의 `applied` 목록을 로깅한다

3. 기존 approval_method DB 저장 로직(현재 `deps.sqlite.prepare('UPDATE wallets SET owner_approval_method = ?, wallet_type = ? ...')`)을 트랜잭션 안으로 이동한다.
