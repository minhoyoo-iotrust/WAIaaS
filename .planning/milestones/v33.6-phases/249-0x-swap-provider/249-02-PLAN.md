```ts
import { z } from 'zod';
import { ChainError } from '@waiaas/core';

```

SwapInputSchema:
```ts
const SwapInputSchema = z.object({
  sellToken: z.string().min(1, 'sellToken is required'),
  buyToken: z.string().min(1, 'buyToken is required'),
  sellAmount: z.string().min(1, 'sellAmount is required (in smallest units)'),
  slippageBps: z.number().int().optional(),
});
```

**Constructor:**
- Merge config with ZEROX_SWAP_DEFAULTS
- DO NOT create apiClient in constructor -- it needs chainId which comes from context at resolve time
- Set metadata: `name: 'zerox_swap'`, `chains: ['ethereum']`, `mcpExpose: true`, `requiresApiKey: true`, `requiredApis: ['0x']`, `version: '1.0.0'`
- Single action: `name: 'swap'`, `chain: 'ethereum'`, `riskLevel: 'medium'`, `defaultTier: 'DELAY'`

**resolve() method** -- Key difference from Jupiter: returns `ContractCallRequest[]` (not single):

1. Validate actionName === 'swap', throw ChainError('INVALID_INSTRUCTION', 'ethereum') if not
2. Parse input with SwapInputSchema
3. SAFE-05: Block same-token swap (`sellToken === buyToken`)
4. Resolve chainId from `context.chain` + action chain. Use `CHAIN_ID_MAP` with the network from context if available, fallback to 1 (ethereum-mainnet). The ActionContext does not have network field currently, so resolve chainId by checking if context has any network hint, otherwise default to chainId 1.

   **Important:** Since ActionContext only has `chain` and `walletAddress` and `walletId`, and doesn't carry network, the provider should accept an optional `chainId` in the swap input schema OR use a default. Add `chainId: z.number().int().optional()` to SwapInputSchema. If not provided, default to 1 (Ethereum mainnet).

5. ZXSW-06: Clamp slippage -- use `clampSlippageBps()` with defaults from config. Default 100bps (1%), max 500bps (5%).
6. Create ZeroExApiClient with config + resolved chainId
7. Get quote via `apiClient.getQuote({ sellToken, buyToken, sellAmount, taker: context.walletAddress, slippageBps })`
8. ZXSW-07: Check `quote.liquidityAvailable === false` -> throw `ChainError('INVALID_INSTRUCTION', 'ethereum', { message: 'No liquidity available for this swap pair' })`
9. ZXSW-09: Validate AllowanceHolder -- `quote.transaction.to` must match `getAllowanceHolderAddress(chainId)`. Throw if mismatch.

10. **Build result array** (ZXSW-04/ZXSW-05):

    Detect native ETH sell: `sellToken.toLowerCase() === '0xeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeee'` (the 0x native token placeholder).

    **If ERC-20 sell (not native)** -- Return 2-element array:
    ```ts
    const approveRequest: ContractCallRequest = {
      type: 'CONTRACT_CALL',
      to: input.sellToken,  // ERC-20 token contract
      calldata: encodeApproveCalldata(getAllowanceHolderAddress(chainId), input.sellAmount),
      value: '0',
    };
    const swapRequest: ContractCallRequest = {
      type: 'CONTRACT_CALL',
      to: quote.transaction.to,
      calldata: quote.transaction.data,
      value: quote.transaction.value,
    };
    return [approveRequest, swapRequest];
    ```

    For `encodeApproveCalldata`: encode ERC-20 approve(address spender, uint256 amount). The function selector is `0x095ea7b3`. Encode manually:
    ```ts
    function encodeApproveCalldata(spender: string, amount: string): string {
      const selector = '0x095ea7b3';
      const paddedSpender = spender.slice(2).toLowerCase().padStart(64, '0');
      const paddedAmount = BigInt(amount).toString(16).padStart(64, '0');
      return `${selector}${paddedSpender}${paddedAmount}`;
    }
    ```

    **If native ETH sell** -- Return 1-element array:
    ```ts
    const swapRequest: ContractCallRequest = {
      type: 'CONTRACT_CALL',
      to: quote.transaction.to,
      calldata: quote.transaction.data,
      value: quote.transaction.value,
    };
    return [swapRequest];
    ```
