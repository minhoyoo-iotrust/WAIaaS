  <action>
    Replace the transaction pipeline stubs in RippleAdapter with real implementations.

    **buildTransaction(request: TransferRequest) (per XRP-01, XRP-03, XRP-04, XRP-10):**
    1. Decode destination address: if isXAddress(request.to), decode to classicAddress + tag
    2. Parse Destination Tag from request.memo:
       - If memo is a numeric string (e.g., "12345"), use as DestinationTag
       - If memo is JSON with destinationTag field, extract it
       - If X-address was decoded with tag, use that tag (X-address tag takes priority if no explicit tag)
    3. Build XRPL Payment transaction object:
       ```
       const payment: Payment = {
         TransactionType: 'Payment',
         Account: request.from,
         Destination: classicAddress,
         Amount: request.amount.toString(), // drops as string
         ...(destinationTag !== undefined && { DestinationTag: destinationTag }),
       }
       ```
    4. Call client.autofill(payment) to populate Sequence, Fee, LastLedgerSequence (per XRP-04)
       - autofill automatically sets LastLedgerSequence = current_ledger + 20 (default)
    5. Apply fee safety margin: overwrite Fee = ((BigInt(autofilled.Fee) * 120n) / 100n).toString() (per XRP-05)
    6. Serialize the autofilled transaction to JSON, encode as Uint8Array (TextEncoder)
    7. Calculate expiry from LastLedgerSequence (approximate: ~4s per ledger)
    8. Return UnsignedTransaction:
       - chain: 'ripple'
       - serialized: encoded transaction JSON bytes
       - estimatedFee: BigInt(autofilled.Fee) (after safety margin)
       - expiresAt: approximate expiry Date
       - metadata: { Sequence, LastLedgerSequence, Fee, DestinationTag, originalTx: autofilled }

    **simulateTransaction(tx: UnsignedTransaction) (per XRP-08):**
    1. Deserialize tx.serialized back to JSON transaction object
    2. Use client.autofill() on the transaction as a dry-run validation:
       - autofill validates account existence, balance sufficiency, sequence correctness
       - If autofill succeeds, the transaction is structurally valid
    3. Return SimulationResult { success: true, logs: ['autofill validation passed'] }
    4. On error, return { success: false, logs: [], error: err.message }

    **signTransaction(tx: UnsignedTransaction, privateKey: Uint8Array) (per XRP-01):**
    1. Deserialize tx.serialized back to JSON transaction object
    2. The privateKey from KeyStore is the 32-byte Ed25519 seed (NOT the full 64-byte sodium key)
    3. Create xrpl Wallet from seed:
       - Convert seed to hex: `Buffer.from(privateKey).toString('hex')`
       - Use `Wallet.fromSeed(seedHex, { algorithm: 'ed25519' })` -- this creates a wallet from the seed
       - IMPORTANT: Verify that Wallet.fromSeed matches the account address in the transaction
    4. Sign: `const { tx_blob } = wallet.sign(txJson)` -- returns hex-encoded signed transaction
    5. Encode tx_blob hex to Uint8Array: `Buffer.from(tx_blob, 'hex')`
    6. Return the signed bytes

    **submitTransaction(signedTx: Uint8Array):**
    1. Convert signedTx bytes back to hex string
    2. Submit: `client.request({ command: 'submit', tx_blob: hexString })`
    3. Parse result:
       - engine_result: 'tesSUCCESS' = submitted successfully
       - engine_result starts with 'tec' = transaction will be in ledger but failed
       - engine_result starts with 'tef'/'tel'/'tem'/'ter' = rejected
    4. Return SubmitResult:
       - txHash: result.tx_json?.hash or compute from tx_blob
       - status: 'submitted' (not yet confirmed)

    **waitForConfirmation(txHash: string, timeoutMs?: number) (per XRP-07):**
    1. Default timeout: 30000ms (covers ~7-8 ledgers)
    2. Poll using `client.request({ command: 'tx', transaction: txHash })`
    3. Check result.validated === true (per XRP-07 "validated ledger" requirement)
    4. If validated:
       - Parse TransactionResult (meta.TransactionResult)
       - tesSUCCESS -> status: 'confirmed'
       - tec* -> status: 'failed'
       - Return with blockNumber = ledger_index, fee = BigInt(Fee in drops)
    5. If not yet validated, wait 2000ms and retry
    6. On timeout, return status: 'submitted' (still pending)

    **parseTransaction(rawTx: string) (sign-only support):**
    Create src/tx-parser.ts with parseRippleTransaction():
    1. Parse rawTx as JSON (XRPL transactions are JSON)
    2. Detect TransactionType:
       - 'Payment' with Amount as string (drops) -> NATIVE_TRANSFER
       - 'Payment' with Amount as object { currency, issuer, value } -> TOKEN_TRANSFER
       - 'TrustSet' -> APPROVE
       - Other -> UNKNOWN
    3. Extract to/amount/token from the parsed transaction
    4. Return ParsedTransaction { operations, rawTx }

    **signExternalTransaction(rawTx: string, privateKey: Uint8Array):**
    1. Parse rawTx as JSON transaction
    2. Create Wallet from seed (same as signTransaction)
    3. Sign: `wallet.sign(txJson)`
    4. Return SignedTransaction { signedTransaction: tx_blob, txHash: hash }

    Update src/index.ts to also export from tx-parser.ts if needed.
