**Section 21.1: 인터페이스 정의**
Define IPerpProvider extending IActionProvider with 3 query methods:
- `getPosition(walletId: string, context: ActionContext): Promise<PerpPositionSummary[]>` — per-market position summaries
- `getMarginInfo(walletId: string, context: ActionContext): Promise<MarginInfo>` — aggregate account-level margin info (NOT per-position, per research cross-margin insight)
- `getMarkets(chain: string, network?: string): Promise<PerpMarketInfo[]>` — available perp markets

Same `resolve()` signature as ILendingProvider: returns `ContractCallRequest | ContractCallRequest[]`.

**Section 21.2: ActionDefinition 5개**
Define 5 action definitions with Zod input schemas:

1. `open_position` — params: market (string), direction (LONG/SHORT), size (string, base asset amount), leverage (number, optional — account-level), orderType (MARKET/LIMIT), limitPrice (string, optional). riskLevel: high.
2. `close_position` — params: market (string), size (string, optional — partial close). riskLevel: medium.
3. `modify_position` — params: market (string), newSize (string, optional), newLimitPrice (string, optional). riskLevel: high.
4. `add_margin` — params: amount (string), asset (CAIP-19). riskLevel: low.
5. `withdraw_margin` — params: amount (string), asset (CAIP-19). riskLevel: medium.

For each, define Zod input schema (z.object) with `.describe()` annotations for AI agent consumption. Use string for amount fields (DEC-LEND-06 precedent). Use CAIP-19 for asset identifiers (DEC-LEND-07/DEC-YIELD-08 precedent).

**Section 21.3: IPositionProvider 동시 구현 패턴**
Document DriftProvider dual interface implementation:
```typescript
class DriftProvider implements IPerpProvider, IPositionProvider {
  // IPerpProvider (5 actions + 3 queries)
  // IPositionProvider (getPositions, getProviderName, getSupportedCategories)
}
```
- `getSupportedCategories()` returns `['PERP']`
- `getPositions()` maps DriftClient perp positions to Phase 268 PositionUpdate format with PerpMetadata

**Section 21.4: 설계 결정**
Record design decisions:
- DEC-PERP-01: IPerpProvider extends IActionProvider (same pattern as ILendingProvider/IYieldProvider)
- DEC-PERP-02: 3 query methods directly on IPerpProvider (no service indirection)
- DEC-PERP-03: open_position riskLevel: high (leveraged trading), add_margin riskLevel: low (adding safety)
- DEC-PERP-04: MarginInfo is account-level aggregate, not per-position (cross-margin model per research)
- DEC-PERP-05: leverage param is optional on open_position — account-level leverage determined by position size vs collateral
- DEC-PERP-06: Perp action identification via ContractCallRequest.metadata (same as DEC-LEND-09)

**Section 22.1: PerpPositionSummary Zod 스키마 (API 응답용)**
Define PerpPositionSummary as API response type (superset of DB PerpMetadataSchema from section 5.3):
```typescript
const PerpPositionSummarySchema = z.object({
  positionId: z.string(),
  provider: z.string(),        // 'drift'
  market: z.string(),          // 'SOL-PERP'
  direction: z.enum(['LONG', 'SHORT']),
  size: z.string(),            // base asset amount (string for JSON)
  entryPrice: z.string(),      // average entry price
  markPrice: z.string(),       // current mark price
  leverage: z.string(),        // current effective leverage
  unrealizedPnl: z.string(),   // unrealized PnL in USD
  realizedPnl: z.string(),     // realized PnL in USD
  liquidationPrice: z.string().nullable(), // null if no liquidation risk
  marginUsed: z.string(),      // margin allocated to this position
  fundingRate: z.string(),     // current funding rate
  status: z.enum(['OPEN', 'CLOSING', 'CLOSED']),
  openedAt: z.string(),        // ISO timestamp
  updatedAt: z.string(),       // ISO timestamp
});
```

Document DB PerpMetadataSchema → API PerpPositionSummary field mapping table (same pattern as section 14.3 / 19.5). Fields like markPrice, fundingRate are computed at query time, not stored in DB.

**Section 22.2: MarginInfo 타입**
Define aggregate account-level margin info:
```typescript
const MarginInfoSchema = z.object({
  totalCollateral: z.string(),       // total deposited collateral value (USD)
  usedMargin: z.string(),            // margin currently used by positions
  availableMargin: z.string(),       // free collateral for new positions
  maintenanceMargin: z.string(),     // minimum required margin before liquidation
  marginRatio: z.string(),           // totalCollateral / usedMargin ratio
  accountLeverage: z.string(),       // total notional / totalCollateral
  unrealizedPnlTotal: z.string(),    // sum of all position unrealized PnL
  liquidatable: z.boolean(),         // true if marginRatio < maintenance threshold
});
```

Note: All values are account-level aggregates (cross-margin model, DEC-PERP-04). Per-position margin is in PerpPositionSummary.marginUsed.

**Section 22.4: PerpPolicyEvaluator 설계**
Design 3 policy types following LendingPolicyEvaluator (section 15) pattern:

1. **PERP_LEVERAGE_LIMIT**: Max account-level leverage limit.
```typescript
const PerpLeverageLimitRulesSchema = z.object({
  maxLeverage: z.number().min(1).max(100).describe('Maximum allowed leverage (e.g., 10 = 10x)'),
  warningLeverage: z.number().min(1).max(100).describe('Warning threshold — forces DELAY tier for human review'),
});
```
- Checks ACCOUNT-LEVEL leverage (not per-position), consistent with cross-margin model
- If projected leverage > maxLeverage → DENY
- If projected leverage > warningLeverage → upgrade to DELAY tier (same as DEC-LEND-11)
- Reads current account leverage from defi_positions cache (same as DEC-LEND-10)

2. **PERP_POSITION_SIZE_LIMIT**: Max position notional value in USD.
```typescript
const PerpPositionSizeLimitRulesSchema = z.object({
  maxPositionSizeUsd: z.string().describe('Maximum position notional value in USD per market'),
  maxTotalExposureUsd: z.string().describe('Maximum total notional exposure across all markets'),
});
```
- Per-market and aggregate limits
- Projected position = current + new order notional

3. **PERP_MARKET_WHITELIST**: Allowed markets, default-deny.
```typescript
const PerpMarketWhitelistRulesSchema = z.object({
  allowedMarkets: z.array(z.string()).describe('Allowed market symbols (e.g., ["SOL-PERP", "BTC-PERP"])'),
});
```
- Default-deny: if PERP_MARKET_WHITELIST not configured → deny all perp actions
- Follows LENDING_ASSET_WHITELIST (DEC-LEND-08) and CONTRACT_WHITELIST patterns

**PolicyEngine integration point**: Step 4h-c (after lending 4h-a, 4h-b). Execution order: 4h-c PERP_LEVERAGE_LIMIT, 4h-d PERP_POSITION_SIZE_LIMIT, 4h-e PERP_MARKET_WHITELIST.
