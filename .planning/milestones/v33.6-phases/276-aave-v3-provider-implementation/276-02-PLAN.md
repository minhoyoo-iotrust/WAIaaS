<objective>
RPC response decoding, health factor simulation, and APY conversion utilities for Aave V3 on-chain read operations.

Purpose: Provide the read-side infrastructure that AaveV3LendingProvider (Plan 276-03) uses to implement ILendingProvider.getPosition(), getHealthFactor(), and getMarkets(). The actual eth_call invocation is delegated to an IRpcCaller injected into the provider.

Create `packages/actions/src/providers/aave-v3/aave-rpc.ts`:

1. Import MAX_UINT256 from './aave-contracts.js'.

2. Define and export `IRpcCaller` interface:
   ```typescript
   export interface IRpcCaller {
     call(params: { to: string; data: string; chainId?: number }): Promise<string>;
   }
   ```
   This is the injection point for real RPC or mock in tests.

3. Define and export `UserAccountData` type:
   ```typescript
   export interface UserAccountData {
     totalCollateralBase: bigint;  // 8 decimals, USD
     totalDebtBase: bigint;        // 8 decimals, USD
     availableBorrowsBase: bigint; // 8 decimals, USD
     currentLiquidationThreshold: bigint; // basis points (e.g. 8250 = 82.5%)
     ltv: bigint;                  // basis points (e.g. 8000 = 80%)
     healthFactor: bigint;         // 18 decimals (1e18 = HF 1.0)
   }
   ```

4. Export `decodeGetUserAccountData(hexResponse: string): UserAccountData`:
   - Strip '0x' prefix if present
   - Slice into 6 x 64-char chunks
   - Convert each to bigint via BigInt('0x' + chunk)
   - Return typed object
   - Validate hex string length >= 384 chars (6 * 64), throw if short

5. Define and export `ReserveData` type (simplified -- only fields we need):
   ```typescript
   export interface ReserveData {
     liquidityRate: bigint;       // ray (1e27), current supply rate
     variableBorrowRate: bigint;  // ray (1e27), current variable borrow rate
     liquidityIndex: bigint;      // ray
     variableBorrowIndex: bigint; // ray
   }
   ```

6. Export `decodeGetReserveData(hexResponse: string): ReserveData`:
   - getReserveData returns multiple fields. We need positions:
     - [3] liquidityRate (index 3 = chars 192..256)
     - [4] variableBorrowRate (index 4 = chars 256..320)
     - [1] liquidityIndex (index 1 = chars 64..128)
     - [2] variableBorrowIndex (index 2 = chars 128..192)
   - NOTE: The actual offset depends on ABI encoding. Aave V3's PoolDataProvider.getReserveData returns a tuple. The research says selector 0x35ea6a75.
   - For safety, parse: strip 0x, slice by 64-char windows. The function returns (DataTypes.ReserveData) which is a packed struct. The key fields for APY are:
     - currentLiquidityRate at offset [3] (4th 32-byte word)
     - currentVariableBorrowRate at offset [4] (5th 32-byte word)
   - Implement with defensive length check.

7. Export `rayToApy(rayRate: bigint): number`:
   - Linear approximation: `Number(rayRate) / 1e27`
   - Returns as decimal (e.g. 0.035 for 3.5% APY)
   - This is acceptable precision for display purposes (research confirms)

8. Export HF threshold constants:
   ```typescript
   export const LIQUIDATION_THRESHOLD_HF = 1_000_000_000_000_000_000n; // 1.0 in 18 decimals
   export const WARNING_THRESHOLD_HF = 1_200_000_000_000_000_000n;     // 1.2 in 18 decimals
   ```

9. Export `simulateHealthFactor(currentData: { totalCollateralBase: bigint; totalDebtBase: bigint; currentLiquidationThreshold: bigint }, action: 'borrow' | 'withdraw', amountBase: bigint): bigint`:
   - For 'borrow': `newTotalDebt = totalDebtBase + amountBase`
   - For 'withdraw': `newCollateral = totalCollateralBase - amountBase`, debt stays same
   - Compute: `collateralThresholdAdjusted = (collateral * currentLiquidationThreshold) / 10000n`
   - If newTotalDebt === 0n: return MAX_UINT256 (infinite HF)
   - Return: `(collateralThresholdAdjusted * 10n**18n) / newTotalDebt`
   - All arithmetic in bigint, NEVER convert to Number for this calculation (Research Flag C2)

10. Export `hfToNumber(hf: bigint): number`:
    - Converts 18-decimal bigint HF to display number
    - `Number(hf) / 1e18`
    - This is ONLY for API display responses, NOT for threshold comparisons

IMPORTANT: Do NOT import viem or any ABI encoding library. All hex parsing is raw string operations.
