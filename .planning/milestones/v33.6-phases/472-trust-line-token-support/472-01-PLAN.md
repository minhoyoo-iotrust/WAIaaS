1. Create `packages/adapters/ripple/src/currency-utils.ts`:
   - `isValidCurrencyCode(code: string): boolean` -- accepts 3-char ISO codes (e.g., "USD", "EUR") and 40-char hex codes (uppercase hex, exactly 40 chars). Reject "XRP" (reserved for native).
   - `normalizeCurrencyCode(code: string): string` -- 3-char codes: uppercase + right-pad with null bytes to match XRPL representation. 40-char hex: uppercase. Pass through as-is for XRPL API (xrpl.js handles encoding).
   - `parseTrustLineToken(tokenAddress: string): { currency: string; issuer: string }` -- splits "{currency}.{issuer}" format. Throws ChainError('INVALID_INSTRUCTION') if format invalid. For 40-char hex currencies the format is just the hex string without issuer dot separator -- but in our CAIP-19 format it's always "{hex}.{issuer}".
   - `IOU_DECIMALS = 15` constant -- XRPL IOU tokens have up to 15 significant digits.
   - `iouToSmallestUnit(value: string, decimals: number): bigint` -- converts decimal string to bigint with given decimals (multiply by 10^decimals).
   - `smallestUnitToIou(amount: bigint, decimals: number): string` -- converts bigint back to decimal string for XRPL Amount.value field.
   - Export from index.ts.

2. Replace `buildApprove` stub in adapter.ts with TrustSet implementation:
   - Parse `request.token.address` via `parseTrustLineToken()` to get {currency, issuer}.
   - `request.spender` is the issuer address (validate it matches parsed issuer or use parsed issuer).
   - Build XRPL TrustSet transaction:
     ```
     {
       TransactionType: 'TrustSet',
       Account: request.from,
       LimitAmount: {
         currency: currency,
         issuer: issuer,
         value: smallestUnitToIou(request.amount, request.token.decimals)
       },
       Flags: 131072  // tfSetNoRipple (0x00020000) per TRUST-02
     }
     ```
   - autofill + fee safety margin (same pattern as buildTransaction).
   - Serialize to JSON bytes, return UnsignedTransaction.
   - Note: TrustSet increases OwnerCount by 1 (adds 2 XRP owner reserve).

3. Replace `buildTokenTransfer` stub with IOU Payment implementation:
   - Parse `request.token.address` via `parseTrustLineToken()` to get {currency, issuer}.
   - Build XRPL Payment transaction:
     ```
     {
       TransactionType: 'Payment',
       Account: request.from,
       Destination: destinationAddress,
       Amount: {
         currency: currency,
         issuer: issuer,
         value: smallestUnitToIou(request.amount, request.token.decimals)
       },
       ...(destinationTag if X-address or memo)
     }
     ```
   - Handle X-address decode and Destination Tag from memo (reuse existing parseDestinationTag pattern).
   - autofill + fee safety margin + serialize (same as buildTransaction).
   - Return UnsignedTransaction.

4. Export new utilities from `packages/adapters/ripple/src/index.ts`.
