1. In `packages/admin/src/api/endpoints.ts`:
   - Add `ACTIONS_PROVIDERS: '/v1/actions/providers'` to the API object.
   - This endpoint already exists in the daemon (actions.ts listProvidersRoute) and returns provider metadata.

2. In `packages/admin/src/pages/actions.tsx`:
   Create a new Actions page component following the existing page patterns (useSignal, apiGet/apiPost/apiPut/apiDelete, FormField, Button, Badge, showToast, getErrorMessage).

   **State:**
   - `providers` signal: array of provider data from GET /v1/actions/providers (name, description, version, chains, requiresApiKey, hasApiKey, actions[])
   - `settings` signal: object from GET /v1/admin/settings (to read actions.* enabled states)
   - `apiKeys` signal: array from GET /v1/admin/api-keys (for masked keys)
   - `dirty` signal: track changed settings for save
   - `saving`, `loading` signals

   **Data fetching (on mount):**
   - Fetch settings via `apiGet(API.ADMIN_SETTINGS)` to read `actions` category (actions.jupiter_swap_enabled, actions.zerox_swap_enabled, etc.)
   - Fetch API keys via `apiGet(API.ADMIN_API_KEYS)` for masked key display
   - Fetch providers via `apiGet(API.ACTIONS_PROVIDERS)` for provider metadata
   - Note: settings fetch is the source of truth for enabled state. The providers endpoint shows what's currently registered.

   **Built-in providers definition (client-side):**
   Define a static array of known built-in providers for display even when they're not registered:
   ```typescript
   const BUILTIN_PROVIDERS = [
     { key: 'jupiter_swap', name: 'Jupiter Swap', description: 'Solana DEX aggregator', chain: 'solana', requiresApiKey: false },
     { key: 'zerox_swap', name: '0x Swap', description: 'EVM DEX aggregator (AllowanceHolder)', chain: 'evm', requiresApiKey: true },
   ];
   ```

   **Layout structure:**
   - Page title: "Actions" with subtitle "Manage DeFi action providers and API keys"
   - For each built-in provider, render a card/section:
     - **Header**: Provider name (bold) + version badge + chain badge(s) (solana=purple, evm=blue)
     - **Description**: Provider description text
     - **Enabled toggle**: Reads from settings `actions.{key}_enabled`. On change: call `apiPut(API.ADMIN_SETTINGS, { key: 'actions.{key}_enabled', value: newValue })` and refetch.
     - **API Key section** (only if requiresApiKey=true):
       - If key exists: show masked key + "Update" and "Delete" buttons
       - If no key: show text input + "Save" button
       - Use `apiPut(API.ADMIN_API_KEY(providerKey), { apiKey: value })` to save
       - Use `apiDelete(API.ADMIN_API_KEY(providerKey))` to delete
     - **Actions list**: Show table/list of registered actions (name, chain, risk level, default tier) -- only when provider is enabled and registered
     - **Status indicator**: "Active" badge (green) when provider is enabled AND registered in daemon, "Inactive" (gray) when disabled, "Requires API Key" (yellow) when enabled but missing required key

   **CSS**: Follow existing Preact inline style patterns used in other pages (no external CSS files). Use CSS variables from global.css (--bg-*, --text-*, --border-*, --accent-*).

3. In `packages/admin/src/components/layout.tsx`:
   - Import the new ActionsPage: `import ActionsPage from '../pages/actions';`
   - Add to PAGE_TITLES: `'/actions': 'Actions'`
   - Add to PAGE_SUBTITLES: `'/actions': 'Manage DeFi action providers and API keys'`
   - Add to NAV_ITEMS array (after '/tokens', before '/policies'):
     `{ path: '/actions', label: 'Actions' }`
   - Add route in PageRouter:
     `if (path === '/actions') return <ActionsPage />;`
