       - Add Ripple-specific notes to wallet creation:
         - chain: 'ripple', environment: 'testnet' or 'mainnet'
         - Networks: xrpl-mainnet, xrpl-testnet, xrpl-devnet
         - XRP uses Ed25519 keys, address format: r-address (e.g., rN7n3473SaZBCG4dFL83w7p1W9cgZw6w3c)
         - Reserve requirement: 10 XRP base reserve + 2 XRP per owned object
       - Add to "Assets" section: getAssets returns native XRP + Trust Line tokens for ripple wallets
       - Note: Smart accounts (ERC-4337) are NOT supported on ripple chain

    3. **transactions.skill.md**:
       - Add "Ripple (XRPL) Transactions" section:
         - **TRANSFER**: Native XRP transfer in drops. Destination Tag via memo field (numeric string or JSON `{"destinationTag":12345}`). X-address auto-decoded.
         - **TOKEN_TRANSFER**: IOU token transfer via Trust Line. Token address format: `{currency}.{issuer}` (e.g., `USD.rIssuerAddress`). 3-char or 40-hex currency codes supported.
         - **APPROVE**: Maps to TrustSet -- sets up a Trust Line. Amount = credit limit. tfSetNoRipple auto-applied.
         - **Unsupported**: CONTRACT_CALL, BATCH, CONTRACT_DEPLOY -- return explicit error (XRPL has no smart contracts)
         - Gas/Fee: Fees in drops (typically 12 drops), 120% safety margin applied automatically
         - Confirmation: validated ledger-based (not block confirmations)
       - Add curl examples for each supported type

    4. **nft.skill.md**:
       - Add "XLS-20 NFT (XRPL)" section:
         - Standard: 'XLS-20' in NFT_TRANSFER requests
         - Transfer model: 2-step (CreateOffer + AcceptOffer). WAIaaS creates sell offer for the sender.
         - The recipient must accept the offer (outside WAIaaS scope, or via a second WAIaaS wallet)
         - `token.address`: Not applicable for XLS-20 (use empty string or NFTokenID)
         - `token.tokenId`: The 64-character hex NFTokenID
         - NFT listing: Use GET /v1/wallet/assets (native RPC, not external indexer)
         - Note: NFT approvals (approveNft) are not supported -- XLS-20 uses offer-based model
       - Add curl example for XLS-20 NFT transfer
