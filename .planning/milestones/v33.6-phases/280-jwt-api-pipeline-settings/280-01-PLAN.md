# Plan 280-01: JWT/Auth Middleware + Telegram Bot Default Wallet Logic Removal

## Goal
Remove the `wlt` claim from JWT payloads and the `defaultWalletId` context variable from auth middleware. Telegram bot session creation must stop using `wlt` and `is_default`. Existing JWTs with `wlt` claim must be silently ignored for backward compatibility.

## Context

Phase 279 established that `resolveWalletId` no longer depends on JWT-cached `defaultWalletId` -- it queries `session_wallets` at resolution time. The `wlt` claim and `defaultWalletId` context are now dead code paths. The `session_wallets.is_default` column was already removed in migration v27.

**Key files from Phase 279:**
- `resolveWalletId` now uses 2-priority (body > query) + single-wallet auto-resolve
- `session_wallets` no longer has `is_default` column
- `JwtPayload.wlt` is the last remaining reference to default wallet in auth layer
