# Plan 280-03: Pipeline/Infrastructure Services + Admin Settings Default Network Removal

## Goal
Remove all `defaultNetwork` references from pipeline context/stages, daemon lifecycle, notification service, adapter pool, and balance monitor. Remove `rpc.evm_default_network` from settings keys, config loader, and hot-reload. Balance monitor must iterate all networks for each wallet instead of using a single default network.

## Context

Phase 279 established:
- `resolveNetwork` now takes 3 params (request, environment, chain) -- no walletDefaultNetwork
- `getSingleNetwork` replaces `getDefaultNetwork` (EVM returns null)
- `wallets.defaultNetwork` column removed from DB and Drizzle schema

Plan 280-01/280-02 (wave 1) removes JWT wlt, auth middleware, API endpoints. This plan (wave 2) depends on those to ensure the full pipeline and infrastructure layer is clean.

**Key insight for BalanceMonitor:** Currently checks only `wallet.default_network ?? getDefaultNetwork(chain, env)` -- a single network. After this change, it must check ALL networks returned by `getNetworksForEnvironment(chain, env)` for each wallet, matching the IncomingTxMonitor pattern (D7 decision).

     ```typescript
     private async checkWallet(wallet: WalletRow): Promise<void> {
       const chain = wallet.chain as ChainType;
       const env = wallet.environment as EnvironmentType;
       const networks = getNetworksForEnvironment(chain, env);

       for (const network of networks) {
         try {
           const rpcUrl = resolveRpcUrl(this.rpcConfig, chain, network);
           if (!rpcUrl) continue; // No RPC URL configured -- skip this network

           const adapter = await this.adapterPool.resolve(chain, network, rpcUrl);
           const balanceInfo = await adapter.getBalance(wallet.public_key);

           const decimalBalance = Number(balanceInfo.balance) / 10 ** balanceInfo.decimals;
           const threshold = this.getThreshold(chain);
           const currency = balanceInfo.symbol;

           // Use walletId:network as the dedup key for per-network tracking
           const dedupKey = `${wallet.id}:${network}`;

           if (decimalBalance <= threshold) {
             if (this.shouldNotify(dedupKey)) {
               this.notifyLowBalance(wallet.id, decimalBalance, currency, threshold);
             }
             this.lastNotified.set(dedupKey, { timestamp: Date.now(), wasLow: true });
           } else {
             this.markRecovered(dedupKey);
           }
         } catch {
           // Per-network error isolation: skip this network
         }
       }
     }
     ```
   - NOTE: The `lastNotified` Map key changes from walletId to `walletId:network` for per-network tracking. The `shouldNotify` and `markRecovered` methods take a string key and already work generically.
