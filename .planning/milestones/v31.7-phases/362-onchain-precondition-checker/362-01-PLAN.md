  <name>Task 1: PreconditionChecker 타입 및 클래스 구현</name>
  <files>packages/e2e-tests/src/helpers/precondition-checker.ts, packages/e2e-tests/src/helpers/index.ts</files>
  <behavior>
    - CheckResult: { name, passed, required, actual, message } 구조로 개별 체크 결과 표현
    - PreconditionReport: { allPassed, checks: CheckResult[], summary: string } 구조로 전체 리포트
    - NetworkFilter: { networks?: string[], protocols?: string[] } 옵션 타입
    - checkDaemon(): 데몬 /v1/health 200 응답 시 passed=true, 접속 실패 시 passed=false
    - checkWallets(): 체인별(ethereum, solana) 지갑 존재 확인, 없으면 passed=false + 어떤 체인 누락인지 메시지
    - checkBalance(walletId, network, minBalance): 잔액 조회, minBalance 미만 시 passed=false + 현재 잔액/필요 잔액 메시지
    - runAll(filter?): 모든 체크를 실행하고 PreconditionReport 반환. filter.networks가 있으면 해당 네트워크만, filter.protocols가 있으면 해당 프로토콜만
    - generateReport(): 텍스트 기반 리포트 생성 (passed/failed 항목 구분, 부족한 항목에 대한 안내 메시지)
  </behavior>
  <action>
    1. `packages/e2e-tests/src/helpers/precondition-checker.ts` 생성:
       - CheckResult 인터페이스: { name: string, passed: boolean, required: string, actual: string, message: string }
       - PreconditionReport 인터페이스: { allPassed: boolean, checks: CheckResult[], summary: string }
       - NetworkFilter 인터페이스: { networks?: string[], protocols?: string[] }
       - OnchainRequirement 타입: 네트워크별 최소 잔액 요구 사항 정의 (예: sepolia ETH 0.01, devnet SOL 0.5)
       - DEFAULT_REQUIREMENTS 상수: 기본 네트워크별 최소 잔액 맵
       - PreconditionChecker 클래스:
         - constructor(baseUrl: string, masterPassword: string)
         - checkDaemon(): Promise<CheckResult> -- GET /v1/health 호출
         - checkWallets(requiredChains: string[]): Promise<CheckResult[]> -- GET /v1/wallets (X-Master-Password), 체인별 지갑 존재 확인
         - checkBalance(walletId: string, minBalance: string, network: string): Promise<CheckResult> -- GET /v1/wallet/balance?walletId=X, 잔액 비교
         - runAll(filter?: NetworkFilter): Promise<PreconditionReport> -- 데몬 -> 지갑 -> 잔액 순서 체크, 앞 단계 실패 시 뒤 단계 스킵
         - generateReport(report: PreconditionReport): string -- 콘솔 출력용 텍스트 리포트 (PASS/FAIL 마커, 부족 항목 안내)
       - filter.networks가 있으면 해당 네트워크의 잔액만 확인
       - filter.protocols가 있으면 해당 프로토콜에 필요한 네트워크만 확인 (예: 'staking' -> holesky, 'swap' -> sepolia)
       - PROTOCOL_NETWORK_MAP 상수: 프로토콜 -> 필요 네트워크 매핑
    2. `packages/e2e-tests/src/helpers/index.ts`에 re-export 추가:
       - export { PreconditionChecker, type PreconditionReport, type CheckResult, type NetworkFilter } from './precondition-checker.js';

    Note: E2EHttpClient를 내부적으로 사용하여 REST API 호출. 잔액 비교는 BigInt로 수행 (lamports/wei 단위).
