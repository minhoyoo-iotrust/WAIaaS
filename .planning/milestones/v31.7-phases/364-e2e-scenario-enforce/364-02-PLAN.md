기존 ci.yml의 stage1 job에 E2E 커버리지 검증 스텝을 추가한다. stage1은 push와 PR 모두에서 실행되므로 모든 코드 변경에 대해 검증이 이루어진다.

**추가 위치:** stage1 job의 마지막 스텝(Unit Tests 다음)에 다음 스텝을 추가:

```yaml
      - name: E2E Coverage Check
        run: pnpm verify:e2e-coverage
```

이 스텝은 별도 빌드 불필요(tsx가 직접 실행), 의존성 설치는 이미 Setup 스텝에서 완료됨.

**stage1에 추가하는 이유:**
- stage1은 PR과 push 모두에서 실행 (가장 넓은 커버리지)
- lint/typecheck와 같은 정적 검증 성격 (빌드 불필요)
- stage2에 넣으면 PR only + stage1 성공 후에만 실행되어 피드백이 느림

**주의사항:**
- 기존 스텝을 수정하지 않음. 새 스텝만 추가
- 기존 affected 기반 lint/typecheck/test와 독립적으로 항상 전체 검증 실행 (coverage map은 전역 상태이므로 affected 필터링 불가)
