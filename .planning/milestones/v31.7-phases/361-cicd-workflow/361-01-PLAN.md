GitHub Actions workflow 파일을 생성한다.

**트리거:**
- `release: published` 이벤트 (release.yml과 동일 패턴, RC/stable 모두 포함)
- `workflow_dispatch` 수동 트리거, inputs로 `version` (string, 선택, 기본값 빈 문자열 — 비어 있으면 release 이벤트의 tag_name 사용)

**permissions:** contents: read

**concurrency:** `e2e-smoke-${{ github.run_id }}` (동시 실행 방지하지 않음 — release 이벤트마다 독립 실행)

**Job: `e2e-smoke`**
runs-on: ubuntu-latest
timeout-minutes: 15

Steps:
1. `actions/checkout@v4`
2. `.github/actions/setup` (기존 composite action — pnpm + node 22 + turbo cache)
3. **Determine version**:
   ```
   if workflow_dispatch and inputs.version != '': VERSION=inputs.version
   elif release event: VERSION=github.event.release.tag_name (strip 'v' prefix)
   else: VERSION='latest'
   ```
   Output: `version` step output
4. **Build e2e-tests package**: `pnpm turbo run build --filter=@waiaas/e2e-tests`
   (e2e-tests가 @waiaas/sdk를 devDependency로 의존하므로 빌드 필요)
5. **Install daemon globally**: `npm install -g @waiaas/daemon@${VERSION}`
   - npx 대신 global install로 데몬 바이너리를 PATH에 배치
   - RC 버전이면 `@waiaas/daemon@3.0.0-rc.1` 형태
6. **Run offchain E2E tests**: `pnpm --filter @waiaas/e2e-tests test:offchain`
   - `env: E2E_DAEMON_INSTALL_MODE: global` (이미 설치된 데몬 사용, lifecycle helper가 npx 대신 global 바이너리 탐색)
   - 실패해도 다음 step 진행 (결과 수집 위해 `continue-on-error: false`이되 step 7은 `if: always()`)
7. **Generate summary** (if: always()):
   ```bash
   if [ -f packages/e2e-tests/e2e-report.md ]; then
     echo "## E2E Smoke Test Report" >> $GITHUB_STEP_SUMMARY
     echo "" >> $GITHUB_STEP_SUMMARY
     echo "**Version:** ${VERSION}" >> $GITHUB_STEP_SUMMARY
     echo "" >> $GITHUB_STEP_SUMMARY
     cat packages/e2e-tests/e2e-report.md >> $GITHUB_STEP_SUMMARY
   else
     echo "## E2E Smoke Test Report" >> $GITHUB_STEP_SUMMARY
     echo "" >> $GITHUB_STEP_SUMMARY
     echo "**Version:** ${VERSION}" >> $GITHUB_STEP_SUMMARY
     echo "" >> $GITHUB_STEP_SUMMARY
     echo "No report file generated. Check test output above." >> $GITHUB_STEP_SUMMARY
   fi
   ```

실제로는 vitest의 reporter 기능보다, globalTeardown에서 reporter 결과를 `e2e-report.md`에 쓰는 방식이 더 적합하다. 이를 위해 `packages/e2e-tests/src/helpers/report-writer.ts` 유틸리티를 생성:
- `CI` 환경변수가 true일 때 `e2e-report.md` 파일을 packages/e2e-tests/ 디렉토리에 자동 생성
- vitest의 globalTeardown 또는 afterAll hook에서 호출
- E2EReporter 인스턴스의 markdownSummary()를 파일로 출력

CI 환경에서 E2E 테스트 결과를 마크다운 파일로 출력하는 유틸리티를 생성한다.

**report-writer.ts:**
```typescript
import { writeFileSync } from 'node:fs';
import { join } from 'node:path';
import type { E2EReporter } from '../reporter.js';

/**
 * Write E2E report markdown file for GitHub Actions summary.
 * Only writes in CI environment (CI=true).
 */
export function writeReportFile(reporter: E2EReporter, outputDir?: string): void {
  const dir = outputDir ?? join(import.meta.dirname, '../..');
  const content = reporter.markdownSummary();
  const header = `Total: ${reporter.total} | Passed: ${reporter.passed} | Failed: ${reporter.failed} | Skipped: ${reporter.skipped}\n\n`;
  writeFileSync(join(dir, 'e2e-report.md'), header + content, 'utf-8');
}
```

**index.ts에 export 추가:**
`export { writeReportFile } from './helpers/report-writer.js';`

**기존 테스트 파일에서 활용:** 각 E2E 시나리오 테스트 파일의 afterAll에서 CI 환경일 때 writeReportFile을 호출하도록 가이드. 그러나 여러 테스트 파일이 각각 reporter를 가지므로, 전체를 통합하는 것은 vitest의 global reporter 방식이 더 적합.

대안으로, `packages/e2e-tests/src/helpers/ci-reporter.ts` 커스텀 vitest reporter를 구현:
- vitest의 Reporter 인터페이스를 구현
- `onFinished` hook에서 전체 테스트 결과를 파싱하여 e2e-report.md 생성
- vitest.config.ts의 reporters 배열에 추가: `reporters: ['default', './src/helpers/ci-reporter.ts']`

이 방식이 개별 시나리오 파일을 수정하지 않아도 되므로 더 깔끔함. ci-reporter.ts를 구현하고 vitest.config.ts에 등록한다.

**ci-reporter.ts 구현:**
- vitest의 `Reporter` 인터페이스에서 `onFinished(files)` 구현
- files의 각 test result를 순회하여 passed/failed/skipped 카운트 + 마크다운 테이블 생성
- `process.env.CI === 'true'`일 때만 파일 생성 (로컬에서는 생성하지 않음)
- 출력 경로: `packages/e2e-tests/e2e-report.md`

**vitest.config.ts 수정:**
- reporters 배열에 커스텀 reporter 경로 추가
