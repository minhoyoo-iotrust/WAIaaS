**1. nightly.yml의 full-suite job 끝에 배지 업데이트 step 추가:**

permissions 수정 — nightly.yml 최상단 `permissions`에 `contents: read` 외 변경 없음 (Gist 접근은 PAT로 처리).

full-suite job의 마지막 step들 추가:

```yaml
      - name: Collect test counts
        if: success()
        id: test-count
        run: |
          # vitest --reporter=json 출력에서 테스트 수 합산
          # turbo 출력에서 "Tests  N passed" 패턴 추출
          TOTAL=$(pnpm turbo run test:unit 2>&1 | grep -oP 'Tests\s+\K[0-9]+(?=\s+passed)' | awk '{s+=$1} END {print s}')
          if [ -z "$TOTAL" ] || [ "$TOTAL" = "0" ]; then
            echo "Warning: Could not parse test count, using fallback"
            TOTAL="unknown"
          fi
          echo "total=$TOTAL" >> "$GITHUB_OUTPUT"
          echo "Test count: $TOTAL"

      - name: Update test badge
        if: success() && steps.test-count.outputs.total != 'unknown'
        uses: schneegans/dynamic-badges-action@v1.7.0
        with:
          auth: ${{ secrets.GIST_SECRET }}
          gistID: ${{ vars.TEST_BADGE_GIST_ID }}
          filename: waiaas-test-badge.json
          label: Tests
          message: "${{ steps.test-count.outputs.total }} passing"
          color: brightgreen
```

**주의:**
- `secrets.GIST_SECRET`은 Gist 쓰기 권한 PAT — 사용자가 사전에 Repository Secret으로 등록 필요
- `vars.TEST_BADGE_GIST_ID`는 Repository Variable — Gist ID를 secrets가 아닌 variables로 저장 (민감하지 않은 값)
- `if: success()` — 테스트가 실패하면 배지 업데이트 하지 않음 (이전 값 유지)
- `test:unit`이 이미 full-suite에서 실행되었으므로 turbo cache hit으로 빠르게 재실행됨
