Admin Settings API (daemon):
- PUT /v1/admin/settings with X-Master-Password header
  Body: JSON with key-value settings pairs
  DeFi-related settings keys (examples):
  - "defi.default_slippage_bps": "50" (0.5%)
  - "defi.max_slippage_bps": "500" (5%)
  - "defi.gas_limit_multiplier": "1.2"
  - "swap.0x_api_key": "test-key"
  - "swap.jupiter_api_url": "https://quote-api.jup.ag/v6"
  - "staking.lido_max_stake": "100000000000000000000"
  - "bridge.lifi_api_key": "test-key"
  - "lending.aave_max_ltv_percent": "80"
- GET /v1/admin/settings with X-Master-Password -> 200 { settings: { ... } }

Push Relay device API:
- POST /devices { walletName, pushToken, platform } -> 201 { status: 'registered', subscription_token }
- GET /devices/:token/subscription-token -> 200 { subscription_token }
- DELETE /devices/:token -> 204
- GET /health -> 200 { status, version, ntfy, push, devices }

PushRelayInstance has baseUrl property for accessing Push Relay API.
Push Relay requires API key auth: X-API-Key header (configured in push-relay config.toml).
PushRelayManager creates config with api_key = 'e2e-test-key'.
