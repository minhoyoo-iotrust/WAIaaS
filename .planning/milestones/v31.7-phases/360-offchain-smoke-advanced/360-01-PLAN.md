Key API endpoints:
- POST /v1/wallets { name, chain, environment, accountType: 'smart' } -> 201 { id, publicKey, accountType, factoryAddress, entryPoint }
- POST /v1/wallets/:id/userop/build { request: { type, to, amount }, network } -> 200 { sender, nonce, callData, buildId }
- POST /v1/wallets/:id/userop/sign { buildId, userOperation: {...} } -> 200 { signedUserOperation, txId }
- GET /v1/nonce -> 200 { nonce } (SIWE/SIWS challenge nonce)
- PUT /v1/wallets/:id/owner { address, method } -> 200 (register owner for SIWE/SIWS)
- POST /v1/wallets/:id/owner/verify { message, signature } -> 200 (verify SIWE/SIWS signature)

Notes:
- Smart account creation requires smart_account.enabled setting to be 'true'
- UserOp Build only works for accountType=smart + chain=ethereum
- UserOp Build requires RPC URL configured (rpc.evm_*) -- test uses daemon with default config
- smart_account.enabled must be set via PUT /v1/admin/settings before creating smart accounts
- Owner Auth: PUT /v1/wallets/:id/owner sets approval_method, POST /v1/wallets/:id/owner/verify verifies SIWE message
