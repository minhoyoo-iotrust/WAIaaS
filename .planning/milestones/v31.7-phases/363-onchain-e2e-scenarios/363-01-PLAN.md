   - `sol-transfer`: track 'onchain', category 'transfer', networks ['devnet']
   - `erc20-transfer`: track 'onchain', category 'transfer', networks ['sepolia'], protocols ['transfer']
   - `spl-transfer`: track 'onchain', category 'transfer', networks ['devnet'], protocols ['transfer']

2. `__tests__/onchain-transfer.e2e.test.ts` — 실제 testnet E2E:
   - beforeAll: DaemonManager는 사용하지 않는다. 온체인 테스트는 이미 실행 중인 데몬에 연결한다 (run-onchain.ts가 사전 조건 확인 후 vitest를 실행하므로). `WAIAAS_E2E_DAEMON_URL` (기본 http://127.0.0.1:3000)과 `WAIAAS_E2E_MASTER_PASSWORD` 환경변수로 연결. setupDaemonSession은 사용하지 않는다 — 이미 존재하는 funded 지갑의 세션을 생성한다.
   - 헬퍼: `getWalletByChain(chain: string)` — GET /v1/wallets로 첫 번째 해당 체인 지갑 찾기. `createSessionForWallet(walletId: string)` — POST /v1/sessions로 세션 생성.
   - 각 describe 블록에서 `it.skipIf(shouldSkipNetwork('sepolia'))` / `it.skipIf(shouldSkipNetwork('devnet'))` 패턴으로 skip 처리.

   a) `eth-transfer` describe:
      - 자기 지갑 주소로 소량(1 wei) ETH 전송: POST /v1/transactions/send `{ type: 'TRANSFER', to: walletAddress, amount: '1', network: 'sepolia' }`
      - 201 응답 확인, body.id 존재 확인
      - GET /v1/transactions/{id}로 상태 폴링 (최대 60초, 1초 간격). status가 'CONFIRMED' 또는 'COMPLETED'가 될 때까지.
      - txId가 0x로 시작하는지 확인.

   b) `sol-transfer` describe:
      - 자기 지갑 주소로 소량(1 lamport) SOL 전송: POST /v1/transactions/send `{ type: 'TRANSFER', to: walletAddress, amount: '1' }`
      - 201 응답 확인, 상태 폴링, txId(base58 형식) 존재 확인.

   c) `erc20-transfer` describe:
      - Skip 조건 추가: ERC-20 테스트용 토큰이 없을 수 있으므로, 전송 실패 시(토큰 잔액 0) skip 처리.
      - 알려진 Sepolia 테스트 ERC-20 토큰 사용 (예: USDC mock). 토큰 정보가 없으면 skip.
      - POST /v1/transactions/send `{ type: 'TOKEN_TRANSFER', to: walletAddress, amount: '1', token: { address, decimals, symbol }, network: 'sepolia' }`
      - 201 또는 4xx 시 skip (토큰 잔액 부족이면 graceful skip).

   d) `spl-transfer` describe:
      - 동일 패턴. SPL 테스트 토큰이 없으면 skip.
      - POST /v1/transactions/send `{ type: 'TOKEN_TRANSFER', to: walletAddress, amount: '1', token: { address, decimals, symbol } }`

   - afterAll: 생성한 세션 삭제.

   중요: 테스트는 실제 testnet과 상호작용하므로 금액을 최소화하고, 자기 자신에게 전송하여 잔액을 유지한다. 토큰 전송은 토큰 잔액이 없으면 graceful skip 한다 (expect 실패가 아닌 test.skip 호출).
