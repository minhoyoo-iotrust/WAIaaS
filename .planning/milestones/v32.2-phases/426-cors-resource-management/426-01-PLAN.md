**1. CORS middleware in server.ts (CORS-01, CORS-02)**

Import `cors` from `hono/cors`. Register CORS middleware in `createApp()` BEFORE other middleware (first middleware after app creation, before requestId). Use a dynamic origin function that reads `cors_origins` from `deps.settingsService` for hot-reload support, falling back to `deps.config?.daemon?.cors_origins` for initial config:

```typescript
import { cors } from 'hono/cors';

// In createApp(), as the first middleware:
app.use('*', cors({
  origin: (origin) => {
    const origins = deps.settingsService?.get('cors_origins') as string[] | undefined
      ?? deps.config?.daemon?.cors_origins
      ?? ['http://localhost:3100', 'http://127.0.0.1:3100'];
    return origins.includes(origin) ? origin : null;
  },
  allowMethods: ['GET', 'POST', 'PUT', 'DELETE', 'PATCH', 'OPTIONS'],
  allowHeaders: ['Content-Type', 'Authorization', 'X-Master-Password', 'X-Request-Id'],
}));
```

Place this BEFORE the `requestId` middleware so CORS preflight responses don't need to go through the full middleware chain. The `origin` callback returns the matched origin or `null` (which means hono/cors won't set any CORS headers for disallowed origins).

Update the doc comment at the top of server.ts to add CORS to the middleware registration order list (position 1, shifting others down).
