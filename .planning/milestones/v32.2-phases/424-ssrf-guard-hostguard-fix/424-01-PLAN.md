   - `validateUrlSafety()`에서 HTTPS 전용 강제를 제거하거나, 옵션 파라미터 `options?: { allowHttp?: boolean }` 추가하여 RPC test에서 HTTP URL도 검증 가능하게 함 (RPC URL은 http://도 사용). 기존 x402 호출은 allowHttp 미지정이므로 HTTPS 강제 유지. 포트 제한(443 only)도 allowHttp일 때는 건너뛰도록 처리

2. **Admin RPC Test SSRF 방어 (SSRF-01):**
   - `admin-settings.ts`의 `testRpcRoute` 핸들러에서 `fetch(url, ...)` 호출 전에 `validateUrlSafety(url, { allowHttp: true })` 호출 추가
   - import 추가: `import { validateUrlSafety } from '../../infrastructure/security/ssrf-guard.js';`
   - validateUrlSafety가 throw하면 catch에서 WAIaaSError로 변환하여 403 또는 기존 에러 형식으로 반환. try-catch 패턴은 이미 해당 핸들러에 존재하므로, catch 블록에서 SSRF 에러를 re-throw하거나 `{ success: false, error: 'SSRF blocked: ...' }` 형태로 응답

3. **hostGuard startsWith 제거 (SSRF-03):**
   - `host-guard.ts`의 `isLocalhost` 판정 로직에서 `hostname.startsWith(pattern)` 부분을 제거
   - 변경 전: `hostname === pattern || hostname.startsWith(pattern)`
   - 변경 후: `hostname === pattern` (정확 매칭만)
   - 이렇게 하면 `localhost.evil.com`은 `localhost`와 `===` 비교에서 false이므로 차단됨

4. **기존 테스트 import 경로 업데이트:**
