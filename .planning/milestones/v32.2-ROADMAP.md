# Roadmap: WAIaaS v32.2 보안 패치

## Overview

SSRF 방어, Rate Limiting, CORS 미들웨어, 리소스 관리 개선을 통해 데몬의 보안과 안정성을 강화한다. 3개 페이즈로 구성되며 각각 독립적으로 검증 가능한 보안 계층을 추가한다.

## Phases

- [ ] **Phase 424: SSRF Guard + hostGuard Fix** - SSRF 가드를 범용화하고 hostGuard prefix 우회를 차단한다
- [ ] **Phase 425: Rate Limit Middleware** - IP/세션/트랜잭션 3계층 인메모리 Rate Limiter를 구현한다
- [ ] **Phase 426: CORS + Resource Management** - CORS 미들웨어 등록, 알림 채널 타임아웃, EventBus 정리를 구현한다

## Phase Details

### Phase 424: SSRF Guard + hostGuard Fix
**Goal**: 외부 요청을 수행하는 모든 엔드포인트에서 로컬/내부 IP 접근과 hostname prefix 우회가 차단된다
**Depends on**: Nothing (first phase)
**Requirements**: SSRF-01, SSRF-02, SSRF-03, SSRF-04
**Success Criteria** (what must be TRUE):
  1. Admin RPC Test 엔드포인트에 169.254.x.x, 10.x.x.x, 172.16.x.x, 192.168.x.x 주소를 입력하면 요청이 거부된다
  2. ssrf-guard.ts가 infrastructure/security/ 경로에 위치하여 모든 외부 fetch에서 import 가능하다
  3. hostGuard에 `localhost.evil.com` 같은 prefix 우회 도메인을 전달하면 차단된다 (정확 매칭만 허용)
  4. SSRF 차단 및 hostGuard 우회 방지를 검증하는 테스트가 통과한다

### Phase 425: Rate Limit Middleware
**Goal**: API 요청에 IP/세션/트랜잭션 3계층 Rate Limit이 적용되어 남용을 방지한다
**Depends on**: Phase 424
**Requirements**: RATE-01, RATE-02, RATE-03, RATE-04, RATE-05
**Success Criteria** (what must be TRUE):
  1. 동일 IP에서 설정된 RPM을 초과하면 HTTP 429 + Retry-After 헤더가 반환된다
  2. 동일 세션에서 설정된 RPM을 초과하면 HTTP 429가 반환된다
  3. 트랜잭션 엔드포인트에서 설정된 RPM을 초과하면 HTTP 429가 반환된다
  4. Admin Settings에서 rate_limit_* 값을 변경하면 재시작 없이 즉시 반영된다
  5. Rate limit 한도 내 요청은 정상 통과하며, 한도 초과/설정 변경 테스트가 통과한다

### Phase 426: CORS + Resource Management
**Goal**: CORS preflight가 올바르게 처리되고, 알림 채널에 타임아웃이 적용되며, AutoStop 재시작 시 리스너 누수가 없다
**Depends on**: Phase 424
**Requirements**: CORS-01, CORS-02, RSRC-01, RSRC-02, RSRC-03
**Success Criteria** (what must be TRUE):
  1. cors_origins 설정에 포함된 origin에서 OPTIONS 요청 시 200 + CORS 헤더가 반환된다
  2. cors_origins에 없는 origin에서 요청 시 CORS 헤더가 포함되지 않는다
  3. Ntfy/Discord/Slack 알림 채널의 fetch가 10초 내 응답 없으면 타임아웃으로 실패한다
  4. AutoStop start/stop 사이클 반복 시 EventBus 리스너가 누적되지 않는다
