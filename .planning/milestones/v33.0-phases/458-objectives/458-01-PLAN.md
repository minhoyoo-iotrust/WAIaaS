**A. 동적 포트 할당 프로토콜 설계 (STR-01)**

섹션 4.2와 4.3 사이에 새 서브섹션 "4.2.1 데몬 포트 동적 할당 프로토콜"을 추가한다.

내용:
1. **문제**: 데몬이 고정 포트(config.toml [daemon].port)를 사용하면 다중 인스턴스 충돌, 다른 서비스와 포트 충돌 가능
2. **프로토콜**:
   - Desktop 모드에서 데몬은 config.toml port=0 (또는 CLI --port=0)으로 시작
   - 데몬이 Node.js net.createServer()에서 TCP bind(0)으로 OS가 빈 포트 할당
   - 할당된 포트를 두 가지 채널로 Sidecar Manager에 전달:
     a. **stdout**: 데몬 시작 시 `WAIAAS_PORT={port}` 형식의 한 줄을 stdout에 출력 (Sidecar Manager가 stdout 파이프를 파싱)
     b. **tempfile fallback**: `{data_dir}/daemon.port` 파일에 포트 번호 기록 (stdout 파싱 실패 시 폴백)
   - Sidecar Manager가 포트를 수신하면 WebView URL을 `http://127.0.0.1:{port}/admin`으로 설정
3. **시퀀스 다이어그램** (mermaid):
   - SidecarManager -> Daemon: spawn with --port=0
   - Daemon -> OS: TCP bind(0)
   - OS -> Daemon: assigned port N
   - Daemon -> stdout: "WAIAAS_PORT=N"
   - Daemon -> file: write {data_dir}/daemon.port
   - SidecarManager: parse stdout for WAIAAS_PORT=
   - SidecarManager -> WebView: navigate to http://127.0.0.1:N/admin
4. **SidecarManager.start() 수정**: 섹션 4.3의 start() 주석에 포트 파싱 단계 추가 (step 2.5: parse WAIAAS_PORT from stdout event)
5. **DaemonStatus.port 필드**: 이미 존재하는 port: u16 필드가 동적으로 설정됨을 명시
