**3.5 Desktop 환경 감지 (IPC-01):**
- `isDesktop()` 함수의 정확한 구현: `window.__TAURI_INTERNALS__` 체크 (NOT `__TAURI__` -- Tauri 2.x에서는 `__TAURI_INTERNALS__`가 내부 전역 객체)
- 파일 위치: `packages/admin/src/utils/platform.ts`
- SSR/prerender 안전 패턴: `typeof window !== 'undefined'` 가드
- 반환값 캐싱 전략 (매 호출마다 window 체크 vs 모듈 로드 시 1회 체크)
- `isDesktop()` 사용 규칙: Desktop 전용 코드의 모든 진입점에서 가드로 사용, static import 금지

**3.6 IPC 브릿지 상세 명세 (IPC-02):**
기존 섹션 3.2의 간략한 테이블을 확장하여 6개 명령 각각에 대해:
- Rust #[tauri::command] 시그니처 (함수명, 인자 타입, 반환 타입)
- TypeScript invoke 래퍼 시그니처 (`packages/admin/src/utils/tauri-bridge.ts`)
- 에러 케이스와 에러 타입 (Rust tauri::Error -> JS catch)
- 타임아웃/재시도 정책

6개 명령 상세:
1. `start_daemon` -- args: StartDaemonArgs { port?: u16, config_path?: String }, returns: DaemonStatus { running: bool, pid: u32, port: u16, uptime_secs: u64 }
2. `stop_daemon` -- args: StopDaemonArgs { force?: bool }, returns: () (graceful 5s -> SIGKILL)
3. `restart_daemon` -- args: none, returns: DaemonStatus
4. `get_daemon_status` -- args: none, returns: DaemonStatus (process alive + /health 조합)
5. `get_daemon_logs` -- args: GetLogsArgs { lines?: u32, since?: String }, returns: Vec<String>
6. `send_notification` -- args: NotificationArgs { title: String, body: String, icon?: String }, returns: ()

각 명령의 DaemonStatus 타입 정의 (Rust struct + TypeScript interface 동시 명세)

**3.7 Tauri Capability 설정 (IPC-03):**
- `tauri.conf.json`의 `capabilities` 섹션 구조
- `CapabilityBuilder.remote()` 사용 시 URL 패턴: `http://127.0.0.1:*/*` (localhost에서 로드되는 Admin Web UI에 IPC 권한 부여)
- 허용 커맨드 목록과 최소 권한 원칙
- `capabilities/default.json` 예시 (IPC 커맨드 6개 + shell:allow-execute-in-sidecar + notification:default + updater:default)

**3.8 CSP 예외 전략 (IPC-04):**
- Admin Web UI 기존 CSP: `default-src 'none'; script-src 'self'; style-src 'self' 'unsafe-inline'; connect-src 'self'; img-src 'self' data:; font-src 'self'`
- Tauri WebView에서의 CSP 충돌점:
  - `connect-src`: IPC 통신을 위해 `ipc:` 스키마 허용 필요 (Tauri 내부 IPC)
  - `connect-src`: WalletConnect relay 연결을 위해 `wss://relay.walletconnect.com` 허용 (Desktop 전용)
  - `script-src`: @reown/appkit이 inline script를 사용할 가능성 -> `'unsafe-eval'` 회피 전략
- Tauri `tauri.conf.json`의 `security.csp` 설정 vs Admin Web UI HTML meta tag CSP 우선순위
- 전략: Tauri의 `security.csp`에서 Desktop 전용 CSP를 오버라이드하되, Admin Web UI의 HTML meta CSP는 브라우저용으로 유지
- 플랫폼별 차이:
  - macOS/Linux: `tauri://localhost` origin의 CSP
  - Windows: `http://tauri.localhost` origin의 CSP
- CSP 설정 예시 (`tauri.conf.json` security 블록)
