## 7. Findings

### FINDING-01: [MEDIUM] Skill File Missing (CLAUDE.md Violation)

**Rule:** CLAUDE.md states "Create a new skill file when a new domain is added."
**Issue:** A new REST API endpoint (`POST /v1/rpc-evm/:walletId/:chainId`), MCP tool (`get_rpc_proxy_url`), and SDK method (`getRpcProxyUrl()`) were added, but no `skills/rpc-proxy.skill.md` was created. No existing skill file was updated with RPC proxy information.
**Impact:** AI agents cannot discover RPC proxy capabilities via skill files.
**Required action:** Create `skills/rpc-proxy.skill.md` with endpoint docs, MCP tool reference, SDK method reference, and the mandatory security notice.

### FINDING-02: [LOW] CLAUDE.md Not Updated (9-type)

**Rule:** Objective doc D3 identifies that CLAUDE.md `discriminatedUnion 8-type` must be updated to `9-type` with CONTRACT_DEPLOY.
**Issue:** `CLAUDE.md` line 17 still reads `discriminatedUnion 8-type` listing 8 types without CONTRACT_DEPLOY.
**Impact:** Future implementations may reference stale 8-type count and miss CONTRACT_DEPLOY in switch/case coverage.
**Required action:** Update CLAUDE.md line 17 to `discriminatedUnion 9-type` and add CONTRACT_DEPLOY to the type list.

### FINDING-03: [LOW] Objective File Status Not Updated

**Rule:** CLAUDE.md states "When a milestone is shipped, update the corresponding objective status: PLANNED -> SHIPPED."
**Issue:** `internal/objectives/m31-14-rpc-proxy.md` still shows `Status: PLANNED`. Should be `SHIPPED` with `Completed: 2026-03-13`.
**Impact:** Status tracking inconsistency.
**Required action:** Update status to SHIPPED and add Completed date.

### FINDING-04: [INFO] No VERIFICATION.md Files for Any Phase

**Observation:** Phases 398-401 have no `*-VERIFICATION.md` files. All 11 SUMMARY files exist and contain self-check notes, but formal phase-level verification documents were not generated.
**Impact:** Minimal -- summaries contain verification details inline. However, prior milestones consistently included VERIFICATION.md per phase.
