**Script 1: `scripts/verify-agent-uat-provider-map.ts`**

`scripts/verify-e2e-coverage.ts` 패턴을 참고하여 작성한다. `#!/usr/bin/env tsx` shebang 사용.

로직:
1. `packages/actions/src/providers/` 하위 디렉토리 목록을 읽는다
2. 각 provider 디렉토리에 대해 `agent-uat/defi/` 하위에 대응하는 시나리오 `.md` 파일이 있는지 확인한다
3. Provider -> scenario 매핑은 스크립트 상단에 `PROVIDER_SCENARIO_MAP` 상수로 정의한다:
   ```typescript
   const PROVIDER_SCENARIO_MAP: Record<string, string> = {
     'aave-v3': 'defi/aave-lending.md',
     'across': 'defi/across-bridge.md',
     'dcent-swap': 'defi/dcent-swap.md',
     'drift': 'defi/drift-perp.md',
     'erc8004': 'advanced/smart-account.md',  // ERC-8004 is Smart Account related
     'hyperliquid': 'defi/hyperliquid-mainnet.md',
     'jito-staking': 'defi/jito-staking.md',
     'jupiter-swap': 'defi/jupiter-swap.md',
     'kamino': 'defi/kamino-lending.md',
     'lido-staking': 'defi/lido-staking.md',
     'lifi': 'defi/lifi-bridge.md',
     'pendle': 'defi/pendle-yield.md',
     'zerox-swap': 'defi/0x-swap.md',
   };
   ```
4. 매핑에 없는 새 provider가 추가되면 에러 출력 + fix hint (매핑 추가 안내)
5. 매핑된 시나리오 파일이 실제로 존재하지 않으면 에러 출력
6. 성공 시 `Provider-Scenario mapping: ALL PASSED (N providers)` 출력, exit 0
7. 실패 시 에러 목록 출력, exit 1

**Script 2: `scripts/verify-agent-uat-format.ts`**

로직:
1. `agent-uat/` 하위 모든 `.md` 파일을 재귀 탐색한다 (`_template.md`, `_index.md`, `README.md` 제외)
2. 각 파일에 대해:
   a. YAML frontmatter 존재 확인 (`---` ... `---` 블록)
   b. frontmatter 필수 필드 확인: `id`, `title`, `category`, `network`, `requires_funds`, `estimated_cost_usd`, `risk_level`
   c. 필수 섹션 4개 존재 확인 (CI-02 요구사항): `## Metadata`, `## Prerequisites`, `## Scenario Steps`, `## Verification`
   d. 추가 권장 섹션 2개 경고: `## Estimated Cost`, `## Troubleshooting` (경고만, 실패 아님)
   e. `## Scenario Steps` 아래에 최소 1개 `### Step` 이 있는지 확인 (빈 시나리오 방지)
3. 에러/경고 구분 출력. 에러 있으면 exit 1, 경고만이면 exit 0

두 스크립트 모두 `import.meta.dirname` 기반으로 ROOT 경로를 resolve한다 (`path.resolve(import.meta.dirname, '..')`).

   b. `_index.md`에 있지만 파일이 없는 시나리오 -> 에러 ("팬텀 항목")

**Script 4: `scripts/verify-admin-route-consistency.ts`**

로직:
1. `packages/admin/src/components/layout.tsx` 파일을 텍스트로 읽는다
2. 3가지 소스에서 라우트를 추출한다:
   a. `NAV_ITEMS` 배열에서 path 추출 (정규식: `/path:\s*'([^']+)'/g`)
   b. `PAGE_TITLES` 객체에서 키 추출 (정규식: `/'([^']+)':\s*'/g` within PAGE_TITLES block)
   c. `PageRouter` 함수에서 라우트 추출 (정규식: `/path\s*===\s*'([^']+)'/g` + `/path\.startsWith\('([^']+)'/g`)
3. 레거시 리다이렉트 라우트는 제외한다: `/incoming`, `/actions`, `/telegram-users`, `/settings`, `/walletconnect`, `/erc8004` (layout.tsx에서 `window.location.hash =` 으로 리다이렉트하는 것들)
   - `LEGACY_REDIRECTS` 상수로 정의
4. 일관성 검증:
   a. `NAV_ITEMS`에 있지만 `PageRouter`에서 라우트 처리가 없는 경우 -> 에러
   b. `NAV_ITEMS`에 있지만 `PAGE_TITLES`에 없는 경우 -> 에러
   c. `PAGE_TITLES`에 있지만 `NAV_ITEMS`에 없는 경우 -> 경고 (숨겨진 페이지일 수 있음)
5. 성공: `Admin route consistency: ALL PASSED (N routes)`, exit 0
6. 실패: 에러 목록, exit 1

두 스크립트 모두 동일한 shebang, ROOT 패턴을 사용한다.
