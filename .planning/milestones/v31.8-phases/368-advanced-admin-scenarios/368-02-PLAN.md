**page-access.md** (id: admin-01):
- frontmatter: category=admin, network=[all], requires_funds=false, estimated_cost_usd=0, risk_level=none, tags=[admin, ui, page-access, http]
- Prerequisites: 데몬 실행, Admin UI 접근 가능 (http://localhost:3100/admin), 마스터 패스워드 보유
- Scenario Steps:
  1. Admin UI 루트 접근 (GET /admin -> 200 또는 로그인 리다이렉트)
  2. 마스터 패스워드로 인증 (POST /v1/admin/auth)
  3. 전체 메뉴 페이지 순회 접근 -- 각 URL에 대해 HTTP 200 확인:
     - /admin/dashboard
     - /admin/wallets
     - /admin/policies
     - /admin/settings
     - /admin/notifications
     - /admin/audit-logs
     - /admin/backup
     - /admin/tokens
     - /admin/stats
     - /admin/nft (NFT 탭)
     - /admin/defi (DeFi 탭)
  4. 각 페이지에서 기본 렌더링 확인 (에러 페이지가 아닌지 확인)
- NOTE: Admin UI는 Preact SPA이므로, 실제로는 API 엔드포인트의 200 응답을 확인한다. UI 렌더링은 사용자가 브라우저에서 확인한다.
- Verification: 모든 메뉴 URL이 HTTP 200 반환, 에러 페이지 없음
- Estimated Cost: $0
- Troubleshooting: 데몬 미실행, CSP 오류, 인증 실패, 라우트 미등록

**authentication.md** (id: admin-02):
- frontmatter: category=admin, network=[all], requires_funds=false, estimated_cost_usd=0, risk_level=none, tags=[admin, auth, master-password, session]
- Prerequisites: 데몬 실행, 마스터 패스워드 보유
- Scenario Steps:
  1. 마스터 패스워드 없이 Admin API 접근 시도 (GET /v1/admin/stats -> 401)
  2. 잘못된 패스워드로 인증 시도 (POST /v1/admin/auth, wrong password -> 401)
  3. 올바른 마스터 패스워드로 인증 (POST /v1/admin/auth -> 200, 토큰 반환)
  4. 인증 토큰으로 Admin API 접근 (GET /v1/admin/stats -> 200)
  5. X-Master-Password 헤더로 직접 접근 확인 (GET /v1/admin/settings -> 200)
- NOTE: 에이전트는 마스터 패스워드를 직접 요청하지 않는다. 사용자에게 입력을 안내한다.
- Verification: 미인증 시 401, 인증 성공 시 200, 토큰 기반 접근 동작
- Estimated Cost: $0
- Troubleshooting: Argon2id 해싱 오류, 토큰 만료, 패스워드 미설정
