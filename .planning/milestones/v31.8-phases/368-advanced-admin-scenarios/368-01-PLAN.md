**smart-account.md** (id: advanced-01):
- frontmatter: category=advanced, network=[ethereum-sepolia], requires_funds=true, estimated_cost_usd=0.02, risk_level=low, tags=[smart-account, userop, erc-4337, sepolia]
- Prerequisites: 데몬 실행, 세션 토큰, Smart Account 지갑(accountType=smart) 보유, Sepolia ETH (가스비+paymaster 또는 직접), AA Provider 설정 완료(Lite 또는 Full 모드)
- Scenario Steps:
  1. Smart Account 지갑 목록 조회 (GET /v1/wallets, accountType=smart 필터)
  2. connect-info에서 userop capability 확인 (GET /v1/connect-info)
  3. UserOp Build (POST /v1/userop/build, ETH self-transfer, network=ethereum-sepolia)
  4. UserOp Sign (POST /v1/userop/sign, build 결과의 userOpHash 사용)
  5. UserOp 상태 확인 (GET /v1/userop/builds/<BUILD_ID> -- TTL 10min)
  6. 트랜잭션 결과 확인 (잔액 변화 확인)
- Verification: build 성공, sign 성공, userOpHash 반환, status 확인
- Estimated Cost: Sepolia gas ~21,000, ~$0.02
- Troubleshooting: AA Provider 미설정, Lite/Full 모드 차이, build TTL 만료, EntryPoint 버전 불일치

**walletconnect-approval.md** (id: advanced-02):
- frontmatter: category=advanced, network=[ethereum-mainnet], requires_funds=false, estimated_cost_usd=0, risk_level=none, tags=[walletconnect, owner, approval, signing]
- Prerequisites: 데몬 실행, 세션 토큰, Owner 등록 완료(approval_method=walletconnect), WalletConnect 클라이언트(MetaMask 등)
- Scenario Steps:
  1. Owner 상태 확인 (GET /v1/owner/status, state=LOCKED 확인)
  2. 승인 필요 트랜잭션 설정 확인 (정책에 owner_approval 조건 있는지 확인)
  3. 트랜잭션 요청 생성 (dry-run으로 승인 대기 상태 트리거)
  4. WalletConnect 세션에서 승인 요청 수신 확인 (사용자가 MetaMask/지갑앱에서 확인)
  5. Owner가 승인 서명 (사용자 수동 -- WalletConnect 지갑에서 서명)
  6. 승인 완료 후 트랜잭션 상태 확인
- NOTE: 이 시나리오는 WalletConnect 세션이 이미 설정된 환경에서 실행한다. WalletConnect pairing은 사용자가 수동으로 수행해야 한다.
- Verification: Owner 상태 확인, 승인 요청 전송, WalletConnect 서명 완료, 트랜잭션 진행
- Estimated Cost: $0 (서명만, 실행하지 않음)
- Troubleshooting: WalletConnect 세션 만료, Owner 미등록, approval_method 불일치

  6. 결제 후 서비스 재요청 (200 OK 확인)
- NOTE: x402 지원 서비스가 없는 경우 Step 2에서 SKIP하고 dry-run까지만 진행한다.
- Verification: 402 응답 파싱, 결제 트랜잭션 성공, 서비스 접근 성공
- Estimated Cost: USDC approve + transfer ~$1.00
- Troubleshooting: x402 서비스 미지원, USDC 잔액 부족, approve 실패

**incoming-tx-mainnet.md** (id: advanced-04):
- frontmatter: category=advanced, network=[ethereum-mainnet, solana-mainnet], requires_funds=true, estimated_cost_usd=0.50, risk_level=medium, tags=[incoming, monitor, mainnet, detection]
- testnet-08 (testnet/incoming-tx.md)과 동일한 구조이나, 메인넷에서 실행한다.
- Prerequisites: 데몬 실행, 세션 토큰, IncomingTxMonitor 활성화, ETH 보유(Ethereum Mainnet, 최소 0.002 ETH), SOL 보유(Solana Mainnet, 최소 0.01 SOL), 알림 채널 설정(선택)
- Scenario Steps:
  1. IncomingTxMonitor 상태 확인 (GET /v1/admin/stats, masterAuth)
  2. Mainnet 지갑 주소 확인 (GET /v1/wallets)
  3. Ethereum Mainnet 자기 전송 실행 (POST /v1/transactions, type=TRANSFER, to=own address, value=0.001 ETH)
  4. 수신 트랜잭션 감지 확인 (~15초 후, GET /v1/wallets/<ID>/transactions?direction=incoming)
  5. Solana Mainnet 자기 전송 실행 (POST /v1/transactions, type=TRANSFER, to=own address, value=0.001 SOL)
  6. Solana 수신 트랜잭션 감지 확인 (~5초 후)
  7. 감사 로그에서 incoming_tx 이벤트 확인 (GET /v1/admin/audit-logs?event=incoming_tx)
- Verification: 양 네트워크 자기 전송 성공, 수신 TX 감지, 감사 로그 기록
- Estimated Cost: ETH transfer ~$0.50, SOL transfer ~$0.001
- Troubleshooting: 모니터 비활성, 블록 확인 지연, 구독 미설정 (#164 known issue)

**balance-monitoring.md** (id: advanced-05):
- frontmatter: category=advanced, network=[ethereum-mainnet, solana-mainnet], requires_funds=true, estimated_cost_usd=0.50, risk_level=medium, tags=[balance, monitoring, alert, notification]
- Prerequisites: 데몬 실행, 세션 토큰, 잔액 모니터링 활성화(Admin Settings), 알림 채널 설정(ntfy 등), ETH/SOL 보유
- Scenario Steps:
  1. 잔액 모니터링 설정 확인 (GET /v1/admin/settings, masterAuth, balance_monitor 관련 설정)
  2. 현재 잔액 기록 (GET /v1/wallets/<ID>/balance)
  3. 잔액 변동 임계값 설정 확인 (Admin Settings에서 threshold 값 확인)
  4. 자기 전송으로 잔액 변동 발생 (POST /v1/transactions, type=TRANSFER, 소액 전송)
  5. 잔액 변동 알림 수신 확인 (ntfy 채널 또는 Admin UI 알림 로그)
  6. Stats API에서 잔액 모니터링 이벤트 확인 (GET /v1/admin/stats)
- Verification: 잔액 변동 감지, 알림 발송 확인, Stats에 이벤트 기록
- Estimated Cost: self-transfer gas ~$0.50
- Troubleshooting: 모니터링 비활성, 알림 채널 미설정, 임계값 미도달
