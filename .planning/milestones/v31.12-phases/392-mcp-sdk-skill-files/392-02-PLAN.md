ResolvedAction 3-kind: contractCall (기존 on-chain), signedData (off-chain 서명), signedHttp (HTTP 서명)

SigningScheme 7종: eip712, personalSign, erc8128, hmac-sha256, rsa-pss-sha256, ecdsa-secp256k1, ed25519

CredentialVault: per-wallet/글로벌 AES-256-GCM 암호화 저장

REST APIs:
- POST /v1/actions/:provider/:action (기존 -- off-chain 자동 분기)
- GET /v1/wallets/:id/actions (off-chain action 목록)
- GET /v1/wallets/:id/actions/:actionId (off-chain action 상세)
- GET/POST/DELETE/PUT /v1/wallets/:id/credentials (per-wallet credential CRUD)
- GET/POST/DELETE/PUT /v1/admin/credentials (글로벌 credential CRUD)

Policy types:
- VENUE_WHITELIST: venue 화이트리스트, default-deny, venue_whitelist_enabled 토글
- ACTION_CATEGORY_LIMIT: 카테고리별 USD 한도 (daily/monthly/per_action), tier_on_exceed

내용 구성 (150줄 이상):

1. **개요**: External Action 프레임워크란 무엇인가 -- AI 에이전트가 on-chain 트랜잭션 외에 off-chain 서명(CLOB DEX, CEX API, HTTP 서명 등)을 수행하는 프레임워크. ResolvedAction 3-kind (contractCall/signedData/signedHttp) 설명.

2. **Security Notice**: `> AI agents must NEVER request the master password. Use only your session token.`

3. **Permissions**:
   - Agent (sessionAuth): action 실행, off-chain action 이력 조회, credential 목록 조회
   - Admin (masterAuth): credential CRUD, venue whitelist 설정, category limit 정책 설정

4. **Pipeline Routing**: kind 필드에 따른 자동 분기 설명. contractCall -> 기존 6-stage, signedData -> credential+policy+sign+track, signedHttp -> sign+execute+track.

5. **Signing Schemes**: 7종 설명 테이블 (eip712, personalSign, erc8128, hmac-sha256, rsa-pss-sha256, ecdsa-secp256k1, ed25519). 각 scheme의 용도와 credential 필요 여부.

6. **Credential Management**: per-wallet vs 글로벌 credential 개념. credentialRef 참조 모델. curl 예시:
   - GET /v1/wallets/:id/credentials (목록 조회)
   - POST /v1/wallets/:id/credentials (등록 -- masterAuth)
   - DELETE /v1/wallets/:id/credentials/:ref (삭제)
   - PUT /v1/wallets/:id/credentials/:ref/rotate (로테이션)

7. **Off-chain Action 실행**: curl 예시 -- POST /v1/actions/:provider/:action (동일한 endpoint, off-chain provider가 등록되어 있으면 자동 분기)

8. **Off-chain Action 조회**: curl 예시:
   - GET /v1/wallets/:id/actions (목록 -- venue/status 필터)
   - GET /v1/wallets/:id/actions/:actionId (상세)

9. **Venue/Category 정책**: VENUE_WHITELIST 토글과 ACTION_CATEGORY_LIMIT 정책 참조 (policies.skill.md로 링크)

10. **MCP 도구**: list_offchain_actions, list_credentials 도구 설명

11. **SDK 메서드**: listOffchainActions(), getActionResult(), listCredentials(), createCredential(), deleteCredential(), rotateCredential() 코드 예시

12. **Error Codes**: CREDENTIAL_NOT_FOUND, CREDENTIAL_EXPIRED, SIGNING_SCHEME_UNSUPPORTED, CAPABILITY_NOT_FOUND, VENUE_NOT_ALLOWED, EXTERNAL_ACTION_FAILED
