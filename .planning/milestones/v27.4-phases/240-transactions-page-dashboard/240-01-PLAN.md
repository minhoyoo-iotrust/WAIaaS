**1. Add endpoint constant** in `packages/admin/src/api/endpoints.ts`:
Add `ADMIN_TRANSACTIONS: '/v1/admin/transactions'` to the API object.

**2. Create `packages/admin/src/pages/transactions.tsx`:**

Define a `TransactionItem` interface matching the API response shape:
```
id, walletId, walletName (nullable), type, status, tier (nullable), toAddress (nullable), amount (nullable), amountUsd (nullable), network (nullable), txHash (nullable), chain, createdAt (nullable)
```

Define a `TransactionsResponse` interface: `{ items: TransactionItem[], total: number, offset: number, limit: number }`.

The page component `TransactionsPage` (default export) should:

a) **State management** using `@preact/signals` (`useSignal`):
   - `items` (TransactionItem[]), `total` (number), `loading` (boolean), `error` (string|null)
   - `filters` (Record<string, string>) for wallet_id, type, status, network, since, until
   - `search` (string) for txHash/address search
   - `page` (number, 0-indexed) and a constant `PAGE_SIZE = 20`
   - `expandedId` (string|null) tracking which row is expanded
   - `displayCurrency` (string) and `displayRate` (number|null)

b) **Data fetching**: Create a `fetchTransactions()` function that:
   - Builds query params from `filters`, `search`, `page * PAGE_SIZE` as offset, `PAGE_SIZE` as limit
   - Calls `apiGet<TransactionsResponse>(API.ADMIN_TRANSACTIONS + '?' + params.toString())`
   - Updates items, total, loading, error signals
   - Called on mount via `useEffect`, and re-called when filters, search, or page change

c) **Display currency**: On mount, call `fetchDisplayCurrency()` to get displayCurrency + displayRate (same pattern as dashboard.tsx).

d) **FilterBar integration**: Use the `FilterBar` component with these fields:
   - wallet_id: type 'select' (initially empty options, or just text-like — actually, since we don't have a wallet list in the FilterBar select, use the SearchInput pattern for wallet instead. Better approach: make wallet_id a select field that gets populated. But since FilterBar only supports 'select' and 'date' types, and we don't have the wallet list loaded... Use a select dropdown. On mount, fetch wallet list from `API.WALLETS` to populate options.)
   - Actually, simpler: Fetch wallets on mount to populate wallet filter options. Use `apiGet` for `/v1/wallets`.
   - type: select with options TRANSFER, TOKEN_TRANSFER, CONTRACT_CALL, APPROVE, BATCH
   - status: select with options PENDING, APPROVED, SUBMITTED, CONFIRMED, FAILED
   - network: select with common networks (mainnet, devnet, testnet, ethereum-mainnet, ethereum-sepolia, polygon-mainnet, polygon-amoy, arbitrum-mainnet, arbitrum-sepolia, optimism-mainnet, optimism-sepolia, base-mainnet, base-sepolia)
   - since: date field
   - until: date field

   When filters change, reset page to 0 and re-fetch.

e) **SearchInput integration**: Use the `SearchInput` component with placeholder "Search by txHash or recipient address...". When search changes, reset page to 0 and re-fetch.

f) **Table rendering**: Use the `Table` component with these columns:
   - Time: render `formatDate(tx.createdAt)` or "\u2014"
   - Wallet: render `tx.walletName ?? tx.walletId.slice(0, 8)`
   - Type: render `<Badge variant="info">{tx.type}</Badge>`
   - To: render `formatAddress(tx.toAddress)` or "\u2014"
   - Amount: render amount + USD display using `formatWithDisplay` (same as dashboard pattern)
   - Network: render `tx.network ?? tx.chain`
   - Status: render `<Badge variant={statusVariant}>{tx.status}</Badge>` where statusVariant maps CONFIRMED->success, FAILED->danger, else->warning
   - Tx Hash: render `<ExplorerLink network={tx.network ?? ''} txHash={tx.txHash} />`

   Set `onRowClick` to toggle `expandedId`.

g) **Row expansion**: After each table row, if `expandedId === tx.id`, render an expanded detail section showing ALL fields:
   - Since the Table component doesn't natively support expansion, implement this by NOT using the Table component for the main view. Instead, build a custom table that renders:
     - Standard `<thead>` with the 8 column headers
     - For each item, a `<tr>` row + conditionally a `<tr class="row-expand">` below it with a single `<td colSpan={8}>` containing a detail grid
   - The detail grid shows: ID, Wallet ID, Wallet Name, Type, Status, Tier, To Address (full), Amount, Amount USD, Network, Chain, Tx Hash (full), Created At
   - Apply `cursor: pointer` to clickable rows and a `.row-expanded` class to the active row

h) **Pagination controls**: Below the table, render pagination:
   - Show "Showing {offset+1}-{Math.min(offset+PAGE_SIZE, total)} of {total}"
   - Previous/Next buttons (disabled at boundaries)
   - `page` signal updates trigger re-fetch

i) **Layout**: Wrap everything in `<div class="page">`:
   - Error banner (if error) with Retry button
   - FilterBar
   - SearchInput
   - Custom table (with expansion support)
   - Pagination controls

**3. Register route in `packages/admin/src/components/layout.tsx`:**

   - Add `if (path === '/transactions') return <TransactionsPage />;` in PageRouter (before the sessions check)
   - Handle path starting with '/transactions' for active state (like wallets does)
