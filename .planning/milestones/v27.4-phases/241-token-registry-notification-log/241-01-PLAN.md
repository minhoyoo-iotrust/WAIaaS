  <action>
Create `packages/admin/src/pages/tokens.tsx` with the following structure:

**State:**
- `network` signal: currently selected EVM network (default: 'ethereum-mainnet')
- `tokens` signal: array of token objects from API
- `loading` signal: boolean
- `showAddForm` signal: boolean toggle for add form visibility
- Add form fields: `address`, `symbol`, `name`, `decimals` (all string signals)
- `adding` signal: boolean for form submission loading
- `deleting` signal: string | null (address being deleted)

**Network Filter:**
- A `<select>` dropdown with all 10 EVM networks from the inlined EVM_NETWORKS constant (same pattern as transactions.tsx which inlines network list since admin SPA cannot import @waiaas/core):
  `ethereum-mainnet`, `ethereum-sepolia`, `polygon-mainnet`, `polygon-amoy`, `arbitrum-mainnet`, `arbitrum-sepolia`, `optimism-mainnet`, `optimism-sepolia`, `base-mainnet`, `base-sepolia`
- On change: update `network` signal and re-fetch tokens

**Token Table:**
- Use a standard `<table>` (not the Table component, to match transactions.tsx pattern for custom rendering)
- Columns: Symbol, Name, Address, Decimals, Source, Actions
- Address column: show truncated address (first 6 + last 4 chars) with full address as title attribute
- Source column: `<Badge variant={source === 'builtin' ? 'neutral' : 'info'}>` showing "Built-in" or "Custom"
- Actions column: For custom tokens, show a Delete button. For builtin tokens, show a dash or nothing.
- Show loading skeleton when fetching, empty state when no tokens

**Add Token Form:**
- Toggle visibility with an "Add Token" button above the table
- When visible: 4 fields (Contract Address input, Symbol input, Name input, Decimals number input) + Submit/Cancel buttons
- On submit: `apiPost(API.TOKENS, { network: network.value, address, symbol, name, decimals: Number(decimals) })`
- On success: close form, re-fetch token list, show success toast
- On error: show error toast
- Decimals field should default to 18

**Delete:**
- Clicking Delete on a custom token: call `apiDelete(API.TOKENS, { network: network.value, address: token.address })`
- Note: The DELETE /v1/tokens endpoint expects a JSON body (not URL param). Use the apiDelete helper or construct fetch with body.
- Check if `apiDelete` exists in client.ts. If not, add it following the same pattern as apiPost but with method 'DELETE'.
- On success: re-fetch token list, show success toast
- On error: show error toast

**Fetch function:**
- `fetchTokens()`: `apiGet(API.TOKENS + '?network=' + encodeURIComponent(network.value))`
- Call on mount and whenever network changes

**Register in layout.tsx:**
- Import TokensPage from '../pages/tokens'
- Add to PAGE_TITLES: `'/tokens': 'Token Registry'`
- Add to PAGE_SUBTITLES: `'/tokens': 'Manage EVM token registry per network'`
- Add to NAV_ITEMS between Sessions and Policies: `{ path: '/tokens', label: 'Tokens' }`
- Add route in PageRouter: `if (path === '/tokens') return <TokensPage />;`

**Verify API.TOKENS exists in endpoints.ts** (it already does: `TOKENS: '/v1/tokens'`). No change needed there.
