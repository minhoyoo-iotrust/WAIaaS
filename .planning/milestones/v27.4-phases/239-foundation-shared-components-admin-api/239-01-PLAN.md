**ExplorerLink component** (`explorer-link.tsx`):
- Import `getExplorerTxUrl` from `@waiaas/core` (already exists in `packages/core/src/utils/explorer-url.ts` with 13-network EXPLORER_MAP).
- Props: `{ network: string; txHash: string | null | undefined; label?: string }`.
- If `txHash` is null/undefined/empty, render nothing (return `null`).
- Call `getExplorerTxUrl(network, txHash)`. If result is null (unknown network), render txHash as truncated plain text (first 8 + "..." + last 6 chars).
- If result is a URL, render `<a href={url} target="_blank" rel="noopener noreferrer" class="explorer-link">{label ?? truncatedHash}</a>`.
- Truncation helper: `txHash.slice(0, 8) + '...' + txHash.slice(-6)`.

**FilterBar component** (`filter-bar.tsx`):
- Generic reusable filter bar. Props:
  ```ts
  interface FilterField {
    key: string;
    label: string;
    type: 'select' | 'date';
    options?: Array<{ value: string; label: string }>;  // for 'select'
    placeholder?: string;
  }

  interface FilterBarProps {
    fields: FilterField[];
    values: Record<string, string>;
    onChange: (values: Record<string, string>) => void;
    syncUrl?: boolean;  // default true
  }
  ```
- On mount, if `syncUrl` is true (default), read initial values from `window.location.hash` query params (hash-based routing: parse after `?` in the hash fragment). Populate `values` accordingly and call `onChange` with the merged values.
- When a filter value changes, update the local state, call `onChange`, and if `syncUrl` is true, update the URL hash query params via `window.location.replace(...)` (do NOT push new history entries -- use replaceState to update only the hash).
- Render each field: `select` type renders a `<select>` with an empty "All" option; `date` type renders `<input type="date">`.
- Include a "Clear" button that resets all filters to empty strings and updates URL.
- CSS class: `filter-bar` wrapper, `filter-field` for each field group.

**SearchInput component** (`search-input.tsx`):
- Props: `{ value: string; onSearch: (query: string) => void; placeholder?: string; debounceMs?: number }`.
- `debounceMs` defaults to 300ms.
- Use `useRef` for timeout ID. On each keystroke, clear previous timeout and set a new one. When timeout fires, call `onSearch(currentValue)`.
- Render `<div class="search-input"><input type="text" .../>{value && <button class="search-clear" onClick={clear}>x</button>}</div>`.
- Clear button sets input to empty and calls `onSearch('')` immediately (no debounce).
- The component is controlled: parent manages `value` state.

Follow existing Preact component patterns (functional components, Preact signals not required here since these are controlled components). Use `import type { ... } from 'preact'` style imports.
