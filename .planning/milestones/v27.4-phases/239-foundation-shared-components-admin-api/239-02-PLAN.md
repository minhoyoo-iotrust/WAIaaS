**1. GET /v1/admin/transactions** (cross-wallet transaction list)

Add a Zod query schema:
```ts
const adminTransactionsQuerySchema = z.object({
  offset: z.coerce.number().int().min(0).default(0).optional(),
  limit: z.coerce.number().int().min(1).max(100).default(20).optional(),
  wallet_id: z.string().uuid().optional(),
  type: z.string().optional(),       // TRANSFER | TOKEN_TRANSFER | CONTRACT_CALL | APPROVE | BATCH
  status: z.string().optional(),     // PENDING | QUEUED | DELAYED | PENDING_APPROVAL | EXECUTING | CONFIRMED | FAILED | CANCELLED
  network: z.string().optional(),
  since: z.coerce.number().optional(),  // epoch seconds lower bound on createdAt
  until: z.coerce.number().optional(),  // epoch seconds upper bound on createdAt
  search: z.string().optional(),         // search txHash or toAddress
});
```

Route definition (createRoute):
- method: 'get', path: '/admin/transactions', tags: ['Admin']
- summary: 'List cross-wallet transactions with filters and pagination'
- Response schema: `{ items: Array<{id, walletId, walletName, type, status, tier, toAddress, amount, amountUsd, network, txHash, chain, createdAt}>, total: number, offset: number, limit: number }`

Handler implementation:
- Build WHERE conditions from query params:
  - `wallet_id` -> `eq(transactions.walletId, wallet_id)`
  - `type` -> `eq(transactions.type, type)`
  - `status` -> `eq(transactions.status, status)`
  - `network` -> `eq(transactions.network, network)`
  - `since` -> `sql\`${transactions.createdAt} >= ${since}\`` (epoch seconds integer comparison)
  - `until` -> `sql\`${transactions.createdAt} <= ${until}\`` (epoch seconds integer comparison)
  - `search` -> `sql\`(${transactions.txHash} LIKE ${'%' + search + '%'} OR ${transactions.toAddress} LIKE ${'%' + search + '%'})\``
- Use `leftJoin(wallets, eq(transactions.walletId, wallets.id))` to get walletName.
- Count total with same WHERE (without offset/limit).
- Query with ORDER BY `desc(transactions.createdAt)`, offset, limit.
- Map rows: convert `createdAt` (Date | number) to epoch seconds. Include `walletName` from the join.

**2. GET /v1/admin/incoming** (cross-wallet incoming transaction list)

Add a Zod query schema:
```ts
const adminIncomingQuerySchema = z.object({
  offset: z.coerce.number().int().min(0).default(0).optional(),
  limit: z.coerce.number().int().min(1).max(100).default(20).optional(),
  wallet_id: z.string().uuid().optional(),
  chain: z.string().optional(),
  status: z.string().optional(),
  suspicious: z.enum(['true', 'false']).optional(),
});
```

Route definition (createRoute):
- method: 'get', path: '/admin/incoming', tags: ['Admin']
- summary: 'List cross-wallet incoming transactions with filters and pagination'
- Response schema: `{ items: Array<{id, txHash, walletId, walletName, fromAddress, amount, tokenAddress, chain, network, status, blockNumber, detectedAt, confirmedAt, suspicious}>, total: number, offset: number, limit: number }`

Handler implementation:
- Build WHERE conditions from query params:
  - `wallet_id` -> `eq(incomingTransactions.walletId, wallet_id)`
  - `chain` -> `eq(incomingTransactions.chain, chain)`
  - `status` -> `eq(incomingTransactions.status, status)`. Unlike the per-wallet API, do NOT default to CONFIRMED (admin sees all statuses).
  - `suspicious` -> `eq(incomingTransactions.isSuspicious, suspicious === 'true')`
- Use `leftJoin(wallets, eq(incomingTransactions.walletId, wallets.id))` to get walletName.
- Count total with same WHERE (without offset/limit).
- Query with ORDER BY `desc(incomingTransactions.detectedAt)`, offset, limit.
- Map rows: convert `detectedAt`/`confirmedAt` (Date) to epoch seconds. Map `isSuspicious` -> `suspicious`.

**Also import `incomingTransactions` from schema** (if not already imported). The existing admin.ts imports `wallets, sessions, sessionWallets, notificationLogs, policies, transactions` -- add `incomingTransactions` to this import.

**Update the JSDoc comment** at the top of admin.ts to document the two new endpoints.
