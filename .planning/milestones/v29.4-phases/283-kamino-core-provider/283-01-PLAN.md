Create `packages/actions/src/providers/kamino/kamino-sdk-wrapper.ts`:

This file abstracts @kamino-finance/klend-sdk behind a testable interface. The key insight is that Kamino K-Lend SDK builds TransactionInstruction objects which we convert to ContractCallRequest format (programId + base64 instructionData + accounts array) -- the same pattern as jito-stake-pool.ts.

1. Define the SDK wrapper interface:
   ```typescript
   /** Kamino instruction result (converted from TransactionInstruction). */
   export interface KaminoInstruction {
     programId: string;
     instructionData: string;  // base64-encoded
     accounts: Array<{ pubkey: string; isSigner: boolean; isWritable: boolean }>;
   }

   /** Abstraction over @kamino-finance/klend-sdk for testability. */
   export interface IKaminoSdkWrapper {
     buildSupplyInstruction(params: {
       market: string;
       asset: string;
       amount: bigint;
       walletAddress: string;
     }): Promise<KaminoInstruction[]>;

     buildBorrowInstruction(params: {
       market: string;
       asset: string;
       amount: bigint;
       walletAddress: string;
     }): Promise<KaminoInstruction[]>;

     buildRepayInstruction(params: {
       market: string;
       asset: string;
       amount: bigint | 'max';
       walletAddress: string;
     }): Promise<KaminoInstruction[]>;

     buildWithdrawInstruction(params: {
       market: string;
       asset: string;
       amount: bigint | 'max';
       walletAddress: string;
     }): Promise<KaminoInstruction[]>;

     /** Get obligation (position) data for a wallet. */
     getObligation(params: {
       market: string;
       walletAddress: string;
     }): Promise<KaminoObligation | null>;

     /** Get reserve data for market data queries. */
     getReserves(market: string): Promise<KaminoReserve[]>;
   }
   ```

2. Define data types:
   ```typescript
   export interface KaminoObligation {
     deposits: Array<{
       mintAddress: string;
       amount: bigint;
       marketValueUsd: number;
     }>;
     borrows: Array<{
       mintAddress: string;
       amount: bigint;
       marketValueUsd: number;
     }>;
     /** Pre-calculated by SDK from obligation account data. */
     loanToValue: number;
   }

   export interface KaminoReserve {
     mintAddress: string;
     symbol: string;
     supplyApy: number;
     borrowApy: number;
     ltvPct: number;
     availableLiquidity: string;
   }
   ```

3. Create `MockKaminoSdkWrapper` class implementing IKaminoSdkWrapper that returns deterministic mock data. This is critical for unit testing -- it builds fake but structurally valid KaminoInstruction arrays. Each instruction should have:
   - `programId`: KAMINO_PROGRAM_ID from config.ts
   - `instructionData`: a base64-encoded buffer with a recognizable prefix byte (supply=0, borrow=1, repay=2, withdraw=3) + amount as LE u64
   - `accounts`: a minimal but correct-structure array with the wallet as signer

   The mock uses simple instruction encoding, NOT real Kamino CPI layout:
   ```typescript
   function encodeMockInstructionData(actionIndex: number, amount: bigint): string {
     const buffer = new Uint8Array(9);
     buffer[0] = actionIndex;
     const view = new DataView(buffer.buffer);
     view.setBigUint64(1, amount, true);
     return Buffer.from(buffer).toString('base64');
   }
   ```

   For 'max' amounts in repay/withdraw, use `BigInt('0xFFFFFFFFFFFFFFFF')` (u64 max).

   Mock getObligation returns a sample obligation with 1 deposit + 1 borrow.
   Mock getReserves returns 3 sample reserves.

4. Create `KaminoSdkWrapper` class (real implementation) that wraps @kamino-finance/klend-sdk. For now, this class lazily imports `@kamino-finance/klend-sdk` and `@solana/web3.js` (both are optional peer dependencies). The real implementation:

   - `kamino_supply`: chain 'solana', inputSchema KaminoSupplyInputSchema, riskLevel 'medium', defaultTier 'DELAY'
   - `kamino_borrow`: chain 'solana', inputSchema KaminoBorrowInputSchema, riskLevel 'high', defaultTier 'APPROVAL'
   - `kamino_withdraw`: chain 'solana', inputSchema KaminoWithdrawInputSchema, riskLevel 'high', defaultTier 'APPROVAL'
   - `kamino_repay`: chain 'solana', inputSchema KaminoRepayInputSchema, riskLevel 'medium', defaultTier 'DELAY'

4. **resolve()**: Switch on actionName, delegate to private resolveSupply/resolveBorrow/resolveRepay/resolveWithdraw.

5. **resolveSupply**: Parse input via KaminoSupplyInputSchema, resolve market via resolveMarketAddress, parse amount with parseTokenAmount (SPL tokens use variable decimals -- default 6 for USDC-like, but accept the human-readable string and let the SDK handle decimals). Call sdkWrapper.buildSupplyInstruction(), convert KaminoInstruction[] to ContractCallRequest[] (each instruction becomes a ContractCallRequest with type='CONTRACT_CALL', to=market address, programId, instructionData, accounts, network='solana-mainnet').

6. **resolveBorrow**: Same pattern as supply. Returns ContractCallRequest or ContractCallRequest[].

7. **resolveRepay**: Supports 'max'. If amount === 'max', pass 'max' to sdkWrapper.buildRepayInstruction. Otherwise parse amount as bigint.

8. **resolveWithdraw**: Supports 'max'. Same pattern as repay.

9. **Amount parsing helper** `parseTokenAmount(amount: string, decimals: number = 6): bigint` -- Same logic as AaveV3's parseTokenAmount but defaults to 6 decimals (most Solana SPL tokens like USDC). Throws ChainError('INVALID_INSTRUCTION', 'solana') for zero/negative.

10. **ILendingProvider methods** (stub implementations for now -- real data in plan 283-03):
    - `getPosition()`: Returns `[]` if no sdkWrapper or obligation is null
    - `getHealthFactor()`: Returns safe defaults `{ factor: Infinity, totalCollateralUsd: 0, totalDebtUsd: 0, currentLtv: 0, status: 'safe' }`
    - `getMarkets()`: Returns `[]`

11. **IPositionProvider methods**:
    - `getPositions()`: Returns `[]`
    - `getProviderName()`: Returns `'kamino'`
    - `getSupportedCategories()`: Returns `['LENDING']`
