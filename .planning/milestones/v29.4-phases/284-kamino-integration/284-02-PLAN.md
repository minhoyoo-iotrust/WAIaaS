After Step 4f completes (around line 1141, after the `catch` block), add a new step to register IPositionProvider implementations with PositionTracker:

```typescript
    // ------------------------------------------------------------------
    // Step 4f-5: Register IPositionProvider implementations with PositionTracker
    // ------------------------------------------------------------------
    if (this.positionTracker && this.actionProviderRegistry) {
      try {
        const { IPositionProvider: _ignore, ..._ } = {} as any; // type-only reference
        // Check each registered provider for IPositionProvider interface
        for (const meta of this.actionProviderRegistry.listProviders()) {
          const provider = this.actionProviderRegistry.getProvider(meta.name);
          if (provider && 'getPositions' in provider && 'getSupportedCategories' in provider && 'getProviderName' in provider) {
            this.positionTracker.registerProvider(provider as unknown as import('@waiaas/core').IPositionProvider);
            console.debug(`Step 4f-5: Registered ${meta.name} with PositionTracker`);
          }
        }
        console.debug(`Step 4f-5: PositionTracker has ${this.positionTracker.providerCount} providers`);
      } catch (err) {
        console.warn('Step 4f-5 (fail-soft): PositionTracker provider registration warning:', err);
      }
    }
```

**Important design notes:**
- Uses duck-typing (`'getPositions' in provider`) to detect IPositionProvider interface, since TypeScript interfaces are erased at runtime. This is the same pattern used elsewhere in the codebase for interface detection.
- This naturally registers both Aave V3 and Kamino (and any future lending providers) because they both implement IPositionProvider.
- Ordering: PositionTracker is already started (Step 4c-10.5) before providers are registered (Step 4f). The initial sync fires on start but with 0 providers (no-op). The first real sync happens at the next interval tick (5 min for LENDING) after providers are registered here.
- For immediate first sync after registration, add after the loop:
```typescript
        // Trigger immediate LENDING sync now that providers are registered
        if (this.positionTracker.providerCount > 0) {
          void this.positionTracker.syncCategory('LENDING');
        }
```

In the `start()` method, update the LENDING interval override section (lines 94-101) to:

```typescript
      if (category === 'LENDING' && this.settingsService) {
        try {
          // Read both Aave V3 and Kamino sync intervals; use the minimum (most frequent)
          const aaveVal = this.settingsService.get('actions.aave_v3_position_sync_interval_sec');
          const aaveParsed = Number(aaveVal);
          const aaveMs = (!Number.isNaN(aaveParsed) && aaveParsed > 0) ? aaveParsed * 1000 : intervalMs;

          // Kamino uses the same default (300s) but can be configured independently
          // For now, LENDING interval = min(aave, default) since Kamino doesn't have a separate interval key
          // If we add a kamino-specific interval key later, it would be compared here
          intervalMs = aaveMs;
        } catch { /* fallback to default */ }
      }
```

Note: This is a minimal change. The PositionTracker is already provider-agnostic for the actual sync -- it calls `provider.getPositions()` on all registered providers for the LENDING category.
