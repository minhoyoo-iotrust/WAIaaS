## E2E Flow Verification (Design Consistency)

For a design-only milestone, E2E flows are verified as design coherence rather than runtime behavior.

### Flow 1: WebView Loads Admin Web UI

```
Tauri App Start -> Sidecar Manager spawns daemon -> TCP bind(0) assigns port
-> stdout WAIAAS_PORT={port} -> Manager reads port -> WebView loads http://127.0.0.1:{port}/admin
-> Admin Web UI renders (Preact 10.x) -> apiCall() uses same-origin relative paths
```

**Status:** COMPLETE. All steps are designed and documented across sections 2.1, 3.3, 4.2.1, 6.1.

### Flow 2: Desktop-Only Feature Activation

```
WebView loaded -> isDesktop() checks __TAURI_INTERNALS__ -> returns true
-> Router adds /setup-wizard route -> desktopComponent() lazy loads Setup Wizard
-> Sidecar status card rendered on dashboard -> WalletConnect QR available in wallet detail
```

**Status:** COMPLETE. Documented across sections 3.5, 3.9, 7.2-7.4.

### Flow 3: Browser Bundle Isolation

```
pnpm build (mode=browser) -> __DESKTOP__=false -> isDesktop() guard blocks dynamic imports
-> Vite tree-shakes Desktop chunks -> rollupOptions.external excludes @tauri-apps/*
-> CI verification script scans bundle for forbidden strings -> PASS
```

**Status:** COMPLETE. 4-layer strategy documented in sections 6.4-6.7.
