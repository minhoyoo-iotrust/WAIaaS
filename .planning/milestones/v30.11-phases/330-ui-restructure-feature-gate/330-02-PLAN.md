**erc8004.tsx changes (TOGL-02 through TOGL-05, GATE-01):**

1. **GATE-01: Unify settings parsing.** Currently the page reads `settings['actions.erc8004_agent_enabled']?.value`. Change to use the same pattern as actions.tsx: load settings via `apiGet<SettingsData>(API.ADMIN_SETTINGS)` which returns `Record<string, Record<string, string>>`, then check `settings['actions']?.['erc8004_agent_enabled'] === 'true'`. Import `SettingsData` from `../utils/settings-helpers` (already imported but used differently).

2. **TOGL-02: Add toggle at page top.** Replace the current `EmptyState` disabled-state block with a toggle section that is ALWAYS rendered at the top of the page (above the TabNav). Use `FormField` component with `type="checkbox"`:
   ```tsx
   <div class="settings-category" style={{ marginBottom: 'var(--space-4)' }}>
     <div class="settings-category-header">
       <h3 style={{ margin: 0 }}>ERC-8004 Agent Identity</h3>
     </div>
     <div class="settings-category-body">
       <div style={{ display: 'flex', alignItems: 'center', gap: 'var(--space-3)' }}>
         <FormField
           label="Enabled"
           name="actions.erc8004_agent_enabled"
           type="checkbox"
           value={featureEnabled.value}
           onChange={(v) => handleToggleEnabled(!!v)}
           disabled={toggleSaving.value}
         />
         {toggleSaving.value && <span style={{ fontSize: 'var(--font-size-sm)', color: 'var(--text-secondary)' }}>Saving...</span>}
       </div>
     </div>
   </div>
   ```

3. **TOGL-03: Add toggle handler.** Add `handleToggleEnabled` function that calls `apiPut(API.ADMIN_SETTINGS, { settings: [{ key: 'actions.erc8004_agent_enabled', value: String(newEnabled) }] })` and updates `featureEnabled` signal + refreshes data. Import `apiPut` (add to existing import). Add `toggleSaving = useSignal(false)` state.

4. **TOGL-04: Always show toggle + disabled message.** The toggle is always visible. When disabled, show a message below the toggle:
   ```tsx
   {!featureEnabled.value && (
     <div class="empty-state" style={{ marginTop: 'var(--space-3)' }}>
       <p>Agent Identity features are disabled. Enable the toggle above to manage agent registrations, reputation, and wallet linking.</p>
     </div>
   )}
   ```

5. **TOGL-05: Read-only mode when disabled.** When `featureEnabled.value` is false:
   - Still show the Identity tab content but make the Registered Agents table read-only: hide the "Register Agent" button, hide the "Link Wallet" / "Unlink" action buttons in the table
   - Hide the Registration File and Reputation tabs (remove them from the rendered TabNav or filter TABS)
   - If there are registered agents (from a previously-enabled state), show them in the table with Tier cells but no action buttons

   When `featureEnabled.value` is true:
   - Show all 3 tabs and full functionality (current behavior)

   Implementation approach: Filter `TABS` based on `featureEnabled.value` -- only show 'identity' tab when disabled. In the identity tab rendering, conditionally hide action buttons and the Register Agent button based on `featureEnabled.value`.

6. **Loading flow adjustment:** Currently `loadData()` returns early when feature is disabled. Change: always load wallets and agent entries even when disabled (to show read-only table), but skip reputation data loading when disabled.
