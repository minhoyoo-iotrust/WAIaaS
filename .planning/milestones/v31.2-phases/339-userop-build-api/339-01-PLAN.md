3. **POST `/wallets/:walletId/userop/build`** route:
   - Auth: sessionAuth + masterAuth (same as transactions.ts pattern)
   - Validate request body with `UserOpBuildRequestSchema`
   - Load wallet from DB, verify:
     - Wallet exists (404 if not)
     - accountType === 'smart' (400 ACTION_VALIDATION_FAILED if not)
     - chain === 'evm' (400 ACTION_VALIDATION_FAILED if Solana)
   - Resolve RPC URL via `rpcUrlResolver(network)`
   - Create publicClient with `createPublicClient({ chain, transport: http(rpcUrl) })`
   - Decrypt signer key, create LocalAccount via `privateKeyToAccount`
   - Create SmartAccount via `SmartAccountService.createSmartAccount({ owner, client })`
   - **Nonce**: Read from EntryPoint v0.7 contract via `publicClient.readContract({ address: entryPoint07Address, abi: entryPoint07Abi, functionName: 'getNonce', args: [sender, 0n] })` -- returns bigint, convert to hex string with `toHex(nonce)`
   - **callData**: Build calls via `buildUserOpCalls(request)`, then `account.encodeCalls(calls)`
   - **Factory detection (BUILD-07, BUILD-08)**: If `wallet.deployed === false`, call `account.getFactoryArgs()` which returns `{ factory, factoryData }` or undefined. Also check on-chain deployment status via `publicClient.getCode({ address: sender })` -- if bytecode exists, update `wallet.deployed = true` in DB. If getFactoryArgs returns undefined (already deployed), set factory/factoryData to null.
   - **Key release**: Always release key in finally block via `keyStore.releaseKey(walletId)`
   - **Build data persistence (BUILD-09)**: Generate UUIDv7 buildId, insert into userop_builds table with walletId, sender (hex), nonce (hex), callData (hex), entryPoint, createdAt=now, expiresAt=now+600 (10 min TTL), used=0
   - **Response (BUILD-03, BUILD-11)**: Return `{ sender, nonce, callData, factory, factoryData, entryPoint, buildId }` -- NO gas/paymaster fields

4. **Wire into server.ts**: Import `userOpRoutes` in server.ts, register with `app.route('/v1', userOpRoutes({ ... }))` after nftRoutes block. Add export to `routes/index.ts`.

5. **OpenAPI spec**: Use `createRoute()` with proper request/response schemas for OpenAPI generation. Use `UserOpBuildRequestSchema` for body and `UserOpBuildResponseSchema` for 200 response.

Important implementation notes:
- Use `viem` `toHex()` for bigint-to-hex nonce conversion (not manual conversion)
- `SmartAccount.encodeCalls()` is async -- await it
- `SmartAccount.getFactoryArgs()` returns `{ factory: Address, factoryData: Hex } | undefined` -- check for undefined
- `publicClient.getCode()` returns bytecode hex or `undefined`/`'0x'` if not deployed
- UUID v7 via `crypto.randomUUID()` (Node.js 22 built-in) for buildId
- Error responses use WAIaaSError with error codes from error-codes.ts
- Use EVM_CHAIN_MAP for viem Chain resolution (same pattern as stage5ExecuteSmartAccount)
