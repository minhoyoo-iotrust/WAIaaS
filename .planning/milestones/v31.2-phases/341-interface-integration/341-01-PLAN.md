wallets.tsx에서 다음 4가지를 변경한다:

1. **AA_PROVIDER_OPTIONS에 None 추가** (ADMIN-01):
   배열 맨 앞에 `{ value: 'none', label: 'None (Lite mode)' }` 추가.

2. **Create form에서 None 선택 시 필드 숨김** (ADMIN-01):
   `formProvider.value === 'none'`일 때 API Key/URL 필드를 숨긴다.
   handleCreate에서 `formProvider.value === 'none'`이면 aaProvider를 설정하지 않는다 (accountType='smart'만 전송).
   기존 로직:
   ```
   if (formAccountType.value === 'smart') {
     createBody.accountType = 'smart';
     if (formProvider.value === 'custom') { ... } else { ... }
   }
   ```
   변경:
   ```
   if (formAccountType.value === 'smart') {
     createBody.accountType = 'smart';
     if (formProvider.value !== 'none') {
       if (formProvider.value === 'custom') { ... } else { ... }
     }
     // 'none' -> aaProvider 미설정 = Lite mode
   }
   ```
   formProvider 초기값을 'none'으로 변경 (Lite mode가 기본).
   handleChainChange에서도 formProvider.value = 'none'으로 리셋.

3. **지갑 상세 Overview에 Mode 행 추가** (ADMIN-02):
   accountType === 'smart' 블록 안에, Deployed 행 다음에 Mode 행 추가:
   ```tsx
   <DetailRow label="Mode">
     <Badge variant={wallet.value.provider ? 'success' : 'warning'}>
       {wallet.value.provider ? 'Full' : 'Lite'}
     </Badge>
     {!wallet.value.provider && (
       <span style={{ marginLeft: '0.5rem', fontSize: '0.85rem', color: 'var(--color-text-secondary)' }}>
         Use UserOp API for gas-sponsored transactions
       </span>
     )}
   </DetailRow>
   ```

4. **지갑 목록 컬럼에 Type 추가** (ADMIN-03):
   listColumns에 'environment' 다음, 'publicKey' 전에 새 컬럼 추가:
   ```tsx
   {
     key: 'accountType',
     header: 'Type',
     render: (a) => {
       if (a.accountType !== 'smart') return <Badge variant="default">EOA</Badge>;
       return (
         <Badge variant={a.provider ? 'info' : 'warning'}>
           {a.provider ? 'Smart (Full)' : 'Smart (Lite)'}
         </Badge>
       );
     },
   },
   ```

5. **ADMIN-04 (No change)**: Provider 편집 UI는 기존 동작 그대로 유지. 변경하지 않는다.

주의: Badge 컴포넌트의 variant 값은 기존 패턴을 따른다 (success, warning, info, default, danger, neutral).
