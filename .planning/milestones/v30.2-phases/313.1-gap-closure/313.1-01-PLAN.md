**RPC 카운터 (rpc.calls, rpc.errors, rpc.latency) 에 대해:**
RPC 호출은 IChainAdapter 메서드를 통해 간접 실행되어, 각 RPC 호출을 개별 추적하려면 어댑터 래핑이 필요하다. 현재 스코프에서는 tx.submitted, tx.failed, autostop.triggered 3개 카운터를 먼저 와이어링한다. RPC 카운터는 pipeline stage5의 adapter 호출 전후에 측정하는 것으로 충분한 proxy가 된다:
- stage5b (simulateTransaction) 전후로 rpc.calls increment + latency 측정
- stage5d (submitTransaction) 전후로 rpc.calls increment + latency 측정
- catch 블록에서 ChainError 발생 시 rpc.errors increment

구현:
```typescript
// stage5b simulate
const simStart = Date.now();
ctx.metricsCounter?.increment('rpc.calls', { network: ctx.resolvedNetwork });
const simResult = await ctx.adapter.simulateTransaction(ctx.unsignedTx);
ctx.metricsCounter?.recordLatency('rpc.latency', Date.now() - simStart, { network: ctx.resolvedNetwork });
if (!simResult.success) {
  ctx.metricsCounter?.increment('rpc.errors', { network: ctx.resolvedNetwork });
}

// stage5d submit
const submitStart = Date.now();
ctx.metricsCounter?.increment('rpc.calls', { network: ctx.resolvedNetwork });
ctx.submitResult = await ctx.adapter.submitTransaction(ctx.signedTx);
ctx.metricsCounter?.recordLatency('rpc.latency', Date.now() - submitStart, { network: ctx.resolvedNetwork });
```

catch ChainError 블록 각각에:
```typescript
ctx.metricsCounter?.increment('rpc.errors', { network: ctx.resolvedNetwork });
```

- 기존 테스트에 영향 없음 (metricsCounter는 undefined로 남음)
- IMetricsCounter import는 @waiaas/core에서 type import로 처리
