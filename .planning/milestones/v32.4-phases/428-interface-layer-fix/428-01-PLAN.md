IChainSubscriber 인터페이스에 optional 메서드 3개를 추가한다. 모든 subscriber가 모든 메서드를 구현하는 것은 아니므로 optional로 선언한다:

1. `pollAll?(): Promise<void>` -- EVM/Solana 모두 구현
2. `checkFinalized?(txHash: string): Promise<boolean>` -- Solana만 구현
3. `getBlockNumber?(): Promise<bigint>` -- EVM만 구현

인터페이스 JSDoc 업데이트:
- `v27.1 scope: 6 methods` 주석을 `v32.4: 9 methods (6 base + 3 optional polling/confirmation)` 으로 변경
- 각 optional 메서드에 JSDoc 추가 (사용처, chain-specific 여부 명시)

구현체 확인:
- EvmIncomingSubscriber: pollAll()과 getBlockNumber()가 이미 존재하므로 변경 불필요. 다만 `implements IChainSubscriber`가 명시되어 있는지 확인하고, 없으면 추가하지 않아도 됨 (structural typing으로 충분)
- SolanaIncomingSubscriber: pollAll()과 checkFinalized()가 이미 존재하므로 변경 불필요

주의: 기존 6개 메서드(subscribe, unsubscribe, subscribedWallets, connect, waitForDisconnect, destroy)는 절대 변경하지 말 것.
