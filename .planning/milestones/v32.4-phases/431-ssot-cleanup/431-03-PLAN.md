1. **Settings configPath 감사 (CLN-02):**
   SETTING_DEFINITIONS의 configPath는 config.toml의 TOML 키 경로를 나타낸다. SettingsService가 config.toml에서 초기값을 읽을 때 이 경로를 사용한다.

   검증 대상 5개 카테고리의 configPath가 실제 config.toml 구조와 일치하는지 확인:
   - `oracle` 카테고리: configPath가 `oracle.coingecko_api_key`, `oracle.cross_validation_threshold` -- config.toml의 `[oracle]` 섹션과 일치하는지 확인. 실제 config loader에 oracle 섹션이 없으면 configPath가 phantom이다.
   - `signing_sdk` 카테고리: configPath가 `signing_sdk.*` -- DaemonConfigSchema에 signing_sdk 섹션이 없으므로 config.toml에서 읽을 수 없는 phantom configPath다. 이 경우 configPath를 빈 문자열 또는 실제 DB 키와 동일하게 맞춘다.
   - `gas_condition` 카테고리: configPath가 `gas_condition.*` -- DaemonConfigSchema에 gas_condition이 없으므로 phantom이다.
   - `rpc_pool` 카테고리: configPath가 `rpc_pool.*` -- DaemonConfigSchema에 rpc_pool이 없으므로 phantom이다.
   - `position_tracker` 카테고리: configPath가 `position_tracker.*` -- phantom이다.

   이들은 Admin Settings DB에서만 관리되고 config.toml fallback이 없다. configPath가 사용되지 않는 경우, 혼동을 줄이기 위해 실제 상태를 반영하도록 주석을 추가한다: `// configPath: DB-only, no config.toml fallback`

   NOTE: configPath 값 자체를 변경하면 SettingsService의 config.toml 로딩 로직에 영향을 줄 수 있으므로, 값은 유지하되 주석으로 "이 카테고리는 config.toml에 대응 섹션이 없으며 Admin Settings DB에서만 관리됨"을 명시한다.

1. **packages/daemon/src/__tests__/settings-configpath-audit.test.ts:**
   ```typescript
   import { describe, it, expect } from 'vitest';
   import { SETTING_DEFINITIONS } from '../infrastructure/settings/setting-keys.js';

   describe('Settings configPath audit', () => {
     it('모든 설정 키가 유효한 configPath를 가진다', () => {
       for (const def of SETTING_DEFINITIONS) {
         expect(def.configPath, `key=${def.key} has empty configPath`).toBeTruthy();
       }
     });

     it('중복 key가 없다', () => {
       const keys = SETTING_DEFINITIONS.map(d => d.key);
       const unique = new Set(keys);
       expect(keys.length).toBe(unique.size);
     });

     it('중복 configPath가 없다', () => {
       const paths = SETTING_DEFINITIONS.map(d => d.configPath);
       const unique = new Set(paths);
       expect(paths.length).toBe(unique.size);
     });

     it('카테고리별 configPath 패턴이 일관적이다', () => {
       // DB-only 카테고리의 configPath는 key와 동일한 패턴이어야 함
       const dbOnlyCategories = ['gas_condition', 'rpc_pool', 'position_tracker'];
       for (const def of SETTING_DEFINITIONS) {
         if (dbOnlyCategories.includes(def.category)) {
           expect(def.configPath).toBeTruthy();
         }
       }
     });
   });
   ```
