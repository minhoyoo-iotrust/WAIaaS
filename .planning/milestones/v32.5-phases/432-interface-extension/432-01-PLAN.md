    - provider.getPositions(ctx)에 PositionQueryContext를 전달한다
    - PositionTracker 생성자가 rpcConfig를 수용한다 (rpcUrls 구성에 필요)
    - 기존 테스트가 새 시그니처에 맞게 업데이트된다
  </behavior>
  <action>
    1. PositionTracker 생성자에 rpcConfig 파라미터 추가:
       ```typescript
       constructor(opts: {
         sqlite: Database;
         settingsService?: SettingsService;
         rpcConfig?: Record<string, string>;
       })
       ```
       this.rpcConfig = opts.rpcConfig ?? {} 로 저장.

    2. syncCategory() 내 wallets 쿼리를 변경:
       기존: `SELECT id FROM wallets WHERE status = 'ACTIVE'`
       변경: `SELECT id, chain, environment FROM wallets WHERE status = 'ACTIVE'`
       결과 타입: `Array<{ id: string; chain: string; environment: string }>`

    3. 각 wallet에 대해 PositionQueryContext 구성:
       ```typescript
       import { getNetworksForEnvironment } from '@waiaas/core';
       import { resolveRpcUrl } from '../../infrastructure/adapter-pool.js';

       const networks = getNetworksForEnvironment(
         wallet.chain as ChainType,
         wallet.environment as EnvironmentType,
       );
       const rpcUrls: Record<string, string> = {};
       for (const net of networks) {
         const url = resolveRpcUrl(this.rpcConfig, wallet.chain, net);
         if (url) rpcUrls[net] = url;
       }
       const ctx: PositionQueryContext = {
         walletId: wallet.id,
         chain: wallet.chain as ChainType,
         networks,
         environment: wallet.environment as EnvironmentType,
         rpcUrls,
       };
       ```

    4. provider.getPositions(wallet.id) -> provider.getPositions(ctx) 변경.
