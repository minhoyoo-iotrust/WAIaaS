  <name>ContractNameRegistry 4-Tier Resolution</name>
  <files>packages/core/src/services/contract-name-registry.ts, packages/core/src/__tests__/contract-name-registry.test.ts</files>
  <behavior>
    ## Core Resolution
    - resolve('0xABCD...', 'ethereum') returns { name: 'Uniswap V3 Router', source: 'well_known' } for known contract
    - resolve('0xabcd...', 'ethereum') returns same result as mixed-case (lowercase normalization)
    - resolve('0x0000000000000000000000000000000000000000', 'ethereum') returns fallback { name: '0x0000...0000', source: 'fallback' }
    - resolve('unknownAddr', 'solana-mainnet') returns fallback with abbreviated address

    ## 4-Tier Priority
    - Tier 1 (Action Provider): If address is registered via registerProvider(), returns { name: displayName, source: 'action_provider' }
    - Tier 2 (Well-known): If address found in WELL_KNOWN_CONTRACTS, returns { name, source: 'well_known' }
    - Tier 3 (Whitelist): If address registered via registerWhitelist(), returns { name, source: 'whitelist' }
    - Tier 4 (Fallback): Returns truncated address, { name: '0xabcd...1234', source: 'fallback' }
    - Priority test: Same address registered in all tiers → action_provider wins

    ## Per-Network Resolution (REG-05)
    - Same address '0xabc' on 'ethereum' → 'Uniswap Router', on 'base' → 'Aerodrome Router'
    - resolve('0xabc', 'ethereum') !== resolve('0xabc', 'base') when both registered differently

    ## EVM Case-Insensitivity (REG-04)
    - resolve('0xABCDEF...', network) === resolve('0xabcdef...', network)
    - Solana addresses are case-sensitive (base58)

    ## Fallback Format (REG-06)
    - EVM: '0x' + first 4 hex chars + '...' + last 4 hex chars (e.g., '0xabcd...1234')
    - Solana: first 4 chars + '...' + last 4 chars

    ## Provider Registration
    - registerProvider(metadata, contractAddresses) adds entries for Tier 1
    - contractAddresses: Array<{ address: string; network: string }>
    - Uses metadata.displayName if set, falls back to snakeCaseToDisplayName(metadata.name)

    ## Whitelist Registration
    - registerWhitelist(contracts, network) adds entries for Tier 3
    - contracts: Array<{ address: string; name?: string }>
    - If name is undefined, skip (don't add to whitelist tier)

    ## Source Field (REG-03)
    - Every result includes source: 'action_provider' | 'well_known' | 'whitelist' | 'fallback'
  </behavior>
