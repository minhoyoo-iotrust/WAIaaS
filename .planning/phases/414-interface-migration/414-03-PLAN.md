**wallets.tsx 마이그레이션 (3417 lines, 16 interfaces, 37 api calls):**

1. `import { apiGet, apiPost, apiPut, apiDelete, ApiError } from '../api/client'` → `import { api, ApiError } from '../api/typed-client'`
2. 16개 interface를 generated type aliases로 교체:
   - Wallet → `components['schemas']['WalletResponse']` 또는 해당 스키마
   - WalletDetail → `components['schemas']['WalletDetailResponse']` 또는 path-level
   - NetworkInfo, McpTokenResult, NetworkBalance, WalletBalance, WalletTransaction → 각각 매칭되는 스키마
   - WcSession, WcPairingResult, WcPairingStatus → WalletConnect 관련 스키마
   - NftItem, NftMetadata → NFT 관련 스키마
   - StakingPosition, StakingPositionsResponse → staking 관련 스키마
   - ApprovalSettingsInfo → 해당 스키마 또는 UI-only 유지
   - UrlEntry → UI-only 가능성 높음 (유지)

3. 37개 API 호출을 typed client 패턴으로 교체. 주요 패턴:
   - `apiGet<WalletDetail>(API.WALLET(id))` → `api.GET('/v1/wallets/{walletId}', { params: { path: { walletId: id } } })`
   - `apiPost<Wallet>(API.WALLETS, body)` → `api.POST('/v1/wallets', { body })`
   - `apiPut<WalletDetail>(API.WALLET(id), { name })` → `api.PUT('/v1/wallets/{walletId}', { params: { path: { walletId: id } }, body: { name } })`
   - `apiDelete(API.WALLET(id))` → `api.DELETE('/v1/wallets/{walletId}', { params: { path: { walletId: id } } })`
   - query string URL 패턴 (`\`${url}?${params}\``) → `params: { query: { ... } }` 사용

4. SettingsData, RpcTestResult 등 settings-helpers 타입은 유지 (Plan 01에서 이미 처리)

**테스트 마이그레이션 (7개 파일):**
- `vi.mock('../api/client')` → `vi.mock('../api/typed-client')`
- mock 구조를 `{ api: { GET: vi.fn(), POST: vi.fn(), PUT: vi.fn(), DELETE: vi.fn() }, ApiError }` 로 변경
- mock return을 `{ data: ... }` 구조로 변경
- mock 객체에 `satisfies` 타입 검증 추가

**주의사항:**
- wallets.tsx는 내부에 여러 sub-component 함수가 있음 (WalletDetailPanel, NftGallery 등) — 각 함수 내부의 API 호출도 빠짐없이 교체
- `apiGet<{ items: Wallet[] }>(API.WALLETS)` 같은 인라인 타입 파라미터는 typed client에서 자동 추론되므로 제거
- credential 관련 API 호출 (`WALLET_CREDENTIALS`, `WALLET_CREDENTIAL_DELETE`, `WALLET_CREDENTIAL_ROTATE`)도 포함
