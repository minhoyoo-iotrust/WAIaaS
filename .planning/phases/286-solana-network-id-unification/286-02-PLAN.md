# Plan 286-02: 인프라 레이어 업데이트

## 목적

adapter-pool의 `rpcConfigKey()`/`configKeyToNetwork()` 양방향 매핑, config.toml 파싱, hot-reload의 네트워크 판별, oracle 피드 매핑, DeFi 프로바이더의 네트워크 참조를 `solana-mainnet` 형식으로 전환한다. config.toml 키는 기존 `solana_mainnet` 유지하되 변환 함수로 양방향 매핑한다.

## 태스크

<task id="286-02-T1">
**adapter-pool rpcConfigKey + configKeyToNetwork 양방향 매핑**

1. `packages/daemon/src/infrastructure/adapter-pool.ts` 수정:

   a. `rpcConfigKey()` 함수 업데이트 -- Solana 네트워크에서 `solana-` prefix strip:
   ```typescript
   export function rpcConfigKey(chain: string, network: string): string {
     if (chain === 'solana') {
       // solana-mainnet -> solana_mainnet, solana-devnet -> solana_devnet
       // Strip 'solana-' prefix to get config.toml key (e.g., solana_mainnet)
       const suffix = network.startsWith('solana-') ? network.slice('solana-'.length) : network;
       return `solana_${suffix}`;
     }
     return `evm_${network.replace(/-/g, '_')}`;
   }
   ```
   주의: legacy `mainnet` 입력도 처리 -- `network.startsWith('solana-')` false이면 `suffix = network` (= 'mainnet') → `solana_mainnet` 생성. 이 경로는 deprecation 기간 동안 하위 호환성 유지.

   b. `configKeyToNetwork()` 함수 업데이트 -- `solana-` prefix 추가:
   ```typescript
   export function configKeyToNetwork(configKey: string): string | null {
     if (configKey.startsWith('solana_ws_')) return null;
     if (configKey.startsWith('solana_')) {
       // solana_mainnet -> solana-mainnet, solana_devnet -> solana-devnet
       return `solana-${configKey.slice('solana_'.length)}`;
     }
     if (configKey.startsWith('evm_')) {
       return configKey.slice('evm_'.length).replace(/_/g, '-');
     }
     return null;
   }
   ```

   c. JSDoc 주석 업데이트:
   - `rpcConfigKey`: `solana + 'solana-mainnet' -> solana_mainnet`
   - `configKeyToNetwork`: `solana_mainnet -> solana-mainnet`

</task>

       // solana-mainnet -> solana_mainnet (strip 'solana-', prefix 'solana_')
