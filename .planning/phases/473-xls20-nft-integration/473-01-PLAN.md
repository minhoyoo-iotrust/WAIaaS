       - `buildNftTransferTx(request: NftTransferParams)`:
         - XLS-20 NFT transfer requires 2-step: seller creates sell offer, buyer accepts
         - For WAIaaS (where we control sender's key), create NFTokenCreateOffer with:
           - `TransactionType: 'NFTokenCreateOffer'`
           - `NFTokenID: request.token.tokenId` (the 64-char hex NFTokenID)
           - `Destination: request.to` (restrict offer to recipient)
           - `Amount: '0'` (free transfer, not a sale)
           - `Flags: 1` (tfSellNFToken -- selling/giving away)
         - autofill, fee safety margin, serialize as UnsignedTransaction
         - Set `metadata.pendingAccept = true` to signal 2-step flow
         - Set `metadata.nftTokenId = request.token.tokenId`
       
       - `transferNft(request: NftTransferParams, privateKey: Uint8Array)`:
         - Build + sign + submit the NFTokenCreateOffer
         - Return SubmitResult with `status: 'submitted'` and metadata indicating offer was created
         - Note: The recipient must call NFTokenAcceptOffer to complete -- this is XRPL's design
         - Per NFT-02: return result with metadata `{ pendingAccept: true, offerIndex: txHash }`
       
       - `approveNft` remains throwing INVALID_INSTRUCTION (XLS-20 doesn't use approval model)
