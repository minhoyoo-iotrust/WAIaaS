Step 1: ACTION_VALIDATION_FAILED의 일반 Zod 에러 오용을 VALIDATION_FAILED로 교체

ACTION_VALIDATION_FAILED는 "action provider가 반환한 validation 에러"에 사용하는 코드이다. 일반 Zod 스키마 검증 실패(request body parsing 등)에는 VALIDATION_FAILED를 사용해야 한다.

파일들을 읽고 각 사용처를 분석:
- sign-message.ts Line ~77: `SignMessageRequestSchema.safeParse(body)` 실패 시 -> 이것은 request body Zod 검증이므로 VALIDATION_FAILED로 교체
- sign-message.ts Line ~109: `signType "typedData" is only supported for EVM wallets` -> 이것은 action 타입 제약이므로 ACTION_VALIDATION_FAILED 유지
- dry-run.ts Line ~124: `TransactionRequestSchema.safeParse(request)` 실패 시 -> request body Zod 검증이므로 VALIDATION_FAILED로 교체
- stages.ts: `(ctx.request as any).type` 관련 -> 읽어서 확인 필요

각 파일에서:
- 일반 Zod safeParse 실패: `WAIaaSError('ACTION_VALIDATION_FAILED', ...)` -> `WAIaaSError('VALIDATION_FAILED', ...)`
- action provider 고유 검증 실패: ACTION_VALIDATION_FAILED 유지

주의: import 추가가 필요할 수 있음 (WAIaaSError는 이미 import되어 있을 것이므로 에러 코드 문자열만 변경)
