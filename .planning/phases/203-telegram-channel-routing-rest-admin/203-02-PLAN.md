**Part A: Extend SetOwnerRequestSchema in openapi-schemas.ts**

Modify `SetOwnerRequestSchema` in `packages/daemon/src/api/routes/openapi-schemas.ts`:

Current:
```ts
export const SetOwnerRequestSchema = z
  .object({
    owner_address: z.string().min(1),
  })
  .openapi('SetOwnerRequest');
```

Change to:
```ts
import { ApprovalMethodSchema } from '@waiaas/core';

export const SetOwnerRequestSchema = z
  .object({
    owner_address: z.string().min(1),
    approval_method: ApprovalMethodSchema.optional(),
  })
  .openapi('SetOwnerRequest');
```

This makes `approval_method` optional. When provided, it must be one of the 5 valid values (`sdk_ntfy`, `sdk_telegram`, `walletconnect`, `telegram_bot`, `rest`). When omitted, the existing behavior is preserved. Invalid values get automatic 400 from Zod validation via OpenAPI validation hook.

**Part B: Update PUT /wallets/:id/owner handler in wallets.ts**

In the `setOwnerRoute` handler within `walletCrudRoutes()`:

1. After parsing body (`const body = c.req.valid('json');`), extract `approval_method`:
   ```ts
   const approvalMethod = body.approval_method ?? null;
   ```

2. In the `ownerLifecycle.setOwner(walletId, normalizedAddress)` call area, add a direct SQL update to set `owner_approval_method` column:
   ```ts
   // Update approval_method if provided
   if (approvalMethod !== null) {
     deps.sqlite.prepare(
       'UPDATE wallets SET owner_approval_method = ? WHERE id = ?'
     ).run(approvalMethod, walletId);
   }
   ```
   Use `deps.sqlite` (raw SQLite) which is already available in `WalletCrudRouteDeps`.

3. In the response JSON, add `approvalMethod` field by reading from the updated wallet:
   ```ts
   const updated = await deps.db.select().from(wallets).where(eq(wallets.id, walletId)).get();
   ```
   Add to the response object: `approvalMethod: updated!.ownerApprovalMethod ?? null`

4. Also update the `WalletOwnerResponseSchema` in openapi-schemas.ts to include the new field:
   ```ts
   approvalMethod: z.string().nullable().optional(),
   ```

5. Also update the `WalletDetailResponseSchema` to include `approvalMethod` for the GET /wallets/:id endpoint so Admin UI can read it:
   ```ts
   approvalMethod: z.string().nullable().optional(),
   ```

6. Update the GET /wallets/:id handler to return `approvalMethod: wallet.ownerApprovalMethod ?? null`.
