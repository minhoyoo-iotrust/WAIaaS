1. `packages/actions/src/__tests__/dcent-policy-integration.test.ts` 생성:

정책 엔진 통합 관점에서 DcentSwapActionProvider의 출력을 검증한다.

- DEX Swap resolve() 결과가 ContractCallRequest[] 형태이고, 각 요소의 `to`, `calldata`, `value` 필드가 올바른지 확인
- ERC-20 sell의 경우 첫 번째 request가 approve (to = token contract, calldata = approve selector), 두 번째가 swap (to = DEX router, calldata = swap data)
- Native sell의 경우 단일 request (to = DEX router, value > 0)
- 이 ContractCallRequest가 기존 파이프라인에 입력되면 CONTRACT_WHITELIST(DEX 라우터), ALLOWED_TOKENS, SPENDING_LIMIT 정책이 평가됨을 검증
- Exchange의 executeExchangeAction() 결과가 TRANSFER 타입인지 확인 → CONTRACT_WHITELIST 미적용 확인 (DS-09)

Mock 전략: DcentSwapApiClient mock (기존 테스트 패턴 재사용)
