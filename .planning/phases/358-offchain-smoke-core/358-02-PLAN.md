WAIaaS REST API endpoints (relevant):
- POST /v1/policies — create policy (masterAuth, body: { walletId, type, params })
  - type examples: DAILY_LIMIT, TRANSACTION_LIMIT, ALLOWED_TOKENS, CONTRACT_WHITELIST, APPROVED_SPENDERS, TOKEN_SPEND_LIMIT
  - params structure varies by type, e.g., DAILY_LIMIT: { maxAmount: string, currency: string }
- GET /v1/policies — list policies (masterAuth, query: ?walletId=xxx)
- PUT /v1/policies/:id — update policy (masterAuth, body: { params })
- DELETE /v1/policies/:id — delete policy (masterAuth)
- POST /v1/transactions/simulate — dry-run evaluation (sessionAuth, body: { to, value, chain?, walletId? })
  - Returns: { allowed: boolean, violations: [...], policies: [...] }

   - POST /v1/policies (masterAuth) — DAILY_LIMIT 정책 생성:
     ```json
     { "walletId": "{walletId}", "type": "DAILY_LIMIT", "params": { "maxAmount": "1000000000000000000", "currency": "native" } }
     ```
     → 201 + id 반환 확인
   - GET /v1/policies?walletId={walletId} (masterAuth) → 1개 이상 존재, 방금 생성한 id 포함 확인
   - PUT /v1/policies/:id (masterAuth) — maxAmount를 2 ETH로 변경:
     ```json
     { "params": { "maxAmount": "2000000000000000000", "currency": "native" } }
     ```
     → 200 확인
   - GET /v1/policies?walletId={walletId} (masterAuth) → 업데이트된 값 확인
   - DELETE /v1/policies/:id (masterAuth) → 200 확인
   - GET /v1/policies?walletId={walletId} (masterAuth) → 삭제된 id 없음 확인

   **dry-run 평가 테스트:**
   - POST /v1/policies (masterAuth)로 TRANSACTION_LIMIT 정책 생성:
     ```json
     { "walletId": "{walletId}", "type": "TRANSACTION_LIMIT", "params": { "maxAmount": "500000000000000000" } }
     ```
   - POST /v1/transactions/simulate (sessionAuth)로 정책 내 금액 시뮬레이션:
     ```json
     { "to": "0x0000000000000000000000000000000000000001", "value": "100000000000000000" }
     ```
     → 200 + allowed: true (또는 violations 빈 배열) 확인
   - POST /v1/transactions/simulate (sessionAuth)로 정책 초과 금액 시뮬레이션:
     ```json
     { "to": "0x0000000000000000000000000000000000000001", "value": "600000000000000000" }
     ```
     → 200 + allowed: false (또는 violations 비어있지 않음) 확인
   - 정리: 생성한 정책 삭제
