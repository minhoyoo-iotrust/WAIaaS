  <action>
Modify the `WalletListContent` function in `wallets.tsx` to add:

**1. SearchInput for name/publicKey filtering:**
- Import `SearchInput` from `../components/search-input`.
- Add a `search` signal (string).
- Place `SearchInput` above the table with placeholder "Search by name or public key...".
- Client-side filter: filter `wallets.value` array by checking if `wallet.name.toLowerCase()` or `wallet.publicKey.toLowerCase()` includes `search.value.toLowerCase()`.

**2. FilterBar for chain/environment/status:**
- Import `FilterBar` and `FilterField` from `../components/filter-bar`.
- Add a `filters` signal with keys: `chain`, `environment`, `status`.
- Define filter fields:
  - `chain` (select): options = `[{value:'solana',label:'Solana'},{value:'ethereum',label:'Ethereum'}]`
  - `environment` (select): options = `[{value:'testnet',label:'Testnet'},{value:'mainnet',label:'Mainnet'}]`
  - `status` (select): options = `[{value:'ACTIVE',label:'ACTIVE'},{value:'SUSPENDED',label:'SUSPENDED'},{value:'TERMINATED',label:'TERMINATED'}]`
- Place `FilterBar` above SearchInput, with `syncUrl={false}` (wallets page uses hash routing for tabs).
- Client-side filter: if a filter value is set, only show wallets matching that filter.

**3. Balance + USD column in the table:**
- Import `fetchDisplayCurrency` and `formatWithDisplay` from `../utils/display-currency`.
- Add signals: `balances` (Record<string, { balance: string; symbol: string; usd: string } | null>), `displayCurrency` (string, default 'USD'), `displayRate` (number | null, default 1).
- On mount, call `fetchDisplayCurrency()` to get currency and rate.
- After wallets are fetched, for each wallet, call `apiGet(API.ADMIN_WALLET_BALANCE(wallet.id))` to get balances, and store the **default network** native balance and symbol. To get the USD value, use the price oracle data if `amountUsd` is available from the balance API (currently not available in balance API, so just display the raw balance).
  - IMPORTANT: The balance API at `GET /v1/admin/wallets/:id/balance` does NOT return USD values. So for the USD column, skip USD conversion on the balance API response. Instead, just show the native balance (e.g., "1.5 SOL").
  - The wallet list may have many wallets; fetch balances lazily: fetch all wallet balances in parallel after the wallet list loads, but cap at a reasonable number (first 50 wallets).
- Add a new column to `walletColumns` BEFORE the "Status" column:
  ```
  {
    key: 'balance' as any,
    header: 'Balance',
    render: (wallet) => {
      const bal = balances.value[wallet.id];
      if (bal === undefined) return <span style={{ color: 'var(--color-text-secondary)' }}>Loading...</span>;
      if (bal === null) return <span style={{ color: 'var(--color-text-secondary)' }}>--</span>;
      return <span>{bal.balance} {bal.symbol}</span>;
    },
  }
  ```

**4. Apply combined filtering to the Table data:**
- Create a `filteredWallets` computed value that applies search + filter criteria to `wallets.value`.
- Pass `filteredWallets` to the `<Table>` component's `data` prop instead of `wallets.value`.

NOTE: Do NOT switch to a custom table. Keep using the existing `<Table>` component for the wallet list (the table does not need row expansion). The walletColumns array should be modified in-place within WalletListContent so it can reference the balances signal.
