- MOCK_ROUTES: [{originChainId:1, destinationChainId:42161, originToken:'0xA0b8...', destinationToken:'0xFd08...'}]
- MOCK_LIMITS: {minDeposit:'10000', maxDeposit:'1000000000000', maxDepositInstant:'500000000000', maxDepositShortDelay:'800000000000'}
- MOCK_DEPOSIT_STATUS: {status:'filled', fillTxHash:'0xfill123...', depositId:42}
- MOCK_CONTEXT: {walletAddress:'0xDepositor123...', chain:'ethereum'} as ActionContext

테스트 케이스:

describe('AcrossBridgeActionProvider')
  describe('constructor')
    - uses ACROSS_DEFAULTS when no config provided
    - merges partial config over defaults

  describe('resolve("quote")')
    - returns ApiDirectResult with outputAmount = inputAmount - totalRelayFee.total
    - includes feeBreakdown (lpFee, relayerCapitalFee, relayerGasFee)
    - includes limits (minDeposit, maxDeposit, etc.)
    - throws for unsupported fromChain

  describe('resolve("execute") - ERC-20 bridge') [TST-02, TST-03]
    - returns 2-element ContractCallRequest[] array (approve + depositV3)
    - first TX: approve(spokePoolAddress, inputAmount) on inputToken
    - second TX: depositV3 calldata on spokePoolAddress
    - depositV3 calldata를 decodeFunctionData로 역파싱하여 12개 파라미터 검증:
      - depositor = context.walletAddress
      - recipient = context.walletAddress (default)
      - inputToken, outputToken = 입력값
      - inputAmount = BigInt(amount)
      - outputAmount = inputAmount - BigInt(totalRelayFee.total)
      - destinationChainId = BigInt(destChainId)
      - exclusiveRelayer = 0x0...0 (open competition)
      - quoteTimestamp = fees.timestamp
      - fillDeadline = timestamp + fillDeadlineBufferSec
      - exclusivityDeadline = 0 (open competition)
      - message = '0x' (empty)
    - approve는 exact inputAmount (not MaxUint256) [Pitfall 10]
    - no value field on ERC-20 TX

  describe('resolve("execute") - native ETH bridge') [TST-04]
    - inputToken을 Ethereum WETH address (0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2)로 설정
    - returns 1-element ContractCallRequest[] array (depositV3만, approve 없음)
    - TX has value = inputAmount (msg.value for native ETH)
    - depositV3 calldata의 inputToken은 WETH 주소

  describe('resolve("execute") - custom recipient')
    - recipient param 전달 시 depositV3의 recipient가 해당 주소로 설정됨

  describe('resolve("execute") - error cases') [TST-05]
    - isAmountTooLow=true -> throws ChainError with 'below minimum' message
    - inputAmount < minDeposit -> throws ChainError with 'below minimum' message
    - inputAmount > maxDeposit -> throws ChainError with 'above maximum' message
    - outputAmount=0 (fees >= inputAmount) -> throws ChainError
    - fillDeadline in the past -> throws ChainError 'stale' message
      (msw server.use로 timestamp를 매우 과거값으로 오버라이드)
    - unsupported fromChain -> throws ChainError

  describe('resolve("status")')
    - returns ApiDirectResult with status field
    - fillTxHash 포함

  describe('resolve("routes")')
    - returns ApiDirectResult with routes array
    - passes fromChain/toChain as originChainId/destinationChainId

  describe('resolve("limits")')
    - returns ApiDirectResult with minDeposit/maxDeposit

  describe('resolve("unknown")')
    - throws ChainError for unknown action name
