**본문 구성 (한국어):**

1. **개요**: Agent UAT 시스템 설명 -- 마크다운 시나리오를 읽고 사용자와 인터랙티브하게 실행
   - 보안 주의: `> AI agents must NEVER request the master password. Use only your session token.`
   - masterAuth가 필요한 시나리오(지갑 CRUD 등)는 사용자에게 마스터 패스워드 입력을 안내

2. **서브커맨드 라우팅 테이블:**

| Command | Action |
|---------|--------|
| `/agent-uat help` | 이 skill 파일의 사용법 표시 |
| `/agent-uat run` | 모든 시나리오 목록 표시 후 선택 실행 |
| `/agent-uat run testnet` | testnet 카테고리 시나리오만 실행 |
| `/agent-uat run mainnet` | mainnet 카테고리 시나리오만 실행 |
| `/agent-uat run defi` | defi 카테고리 시나리오만 실행 |
| `/agent-uat run admin` | admin 카테고리 시나리오만 실행 |
| `/agent-uat run advanced` | advanced 카테고리 시나리오만 실행 |
| `/agent-uat run transfer` | transfer 태그가 있는 시나리오만 실행 |
| `/agent-uat run --network {id}` | 특정 네트워크 시나리오만 실행 |
| `/agent-uat run {scenario-id}` | 특정 시나리오 ID로 실행 |

3. **실행 프로토콜 (에이전트가 따를 단계적 절차):**

   **Phase 1 -- 시나리오 로드:**
   - `agent-uat/_index.md` 읽기
   - 서브커맨드에 따라 해당 카테고리/네트워크/태그로 필터링
   - 필터링된 시나리오 목록을 사용자에게 표시
   - 사용자 확인 받기 (어떤 시나리오를 실행할지)

   **Phase 2 -- 지갑 선택:**
   - `GET /v1/connect-info` 호출하여 세션에 연결된 지갑 목록 조회
   - 시나리오의 `network` 필드와 매칭되는 지갑 자동 선택
   - 선택된 지갑을 사용자에게 표시하고 확인 받기
   - 매칭 지갑이 없으면 사용자에게 알리고 해당 시나리오 스킵
   - `network: [all]`인 경우 masterAuth 기반이므로 지갑 선택 불필요 (사용자에게 안내)

   **Phase 3 -- Dry-Run (자금 필요 시나리오만):**
   - `requires_funds: true`인 시나리오에서 각 트랜잭션 단계 전에 `POST /v1/transactions/dry-run` 호출
   - dry-run 결과를 사용자에게 표시: 예상 가스비(USD), 성공 예상 여부
   - 사용자 승인 후 실제 실행 진행
   - `requires_funds: false`인 시나리오는 dry-run 건너뜀

   **Phase 4 -- 시나리오 실행:**
   - 시나리오 마크다운의 `## Scenario Steps` 섹션을 순서대로 실행
   - 각 Step의 `Action` curl 명령을 실행하고 `Expected`/`Check` 필드와 비교
   - 실패 시 `## Troubleshooting` 테이블에서 해당 증상 찾아 해결책 제시
   - 사용자에게 각 단계 결과를 인터랙티브하게 보고

   **Phase 5 -- Verification:**
   - `## Verification` 섹션의 체크리스트를 하나씩 확인
   - 각 항목의 통과/실패 표시

   **Phase 6 -- Cleanup:**
   - `## Cleanup` 섹션이 있으면 실행 (특히 CRUD 시나리오)
   - cleanup 실패 시 사용자에게 수동 정리 안내

   **Phase 7 -- 리포트 출력:**

```
## Agent UAT Report

**Date**: {YYYY-MM-DD HH:mm}
**Category**: {실행한 카테고리}
**Network**: {실행한 네트워크}

### Results
| # | Scenario | Status | Gas Used | Notes |
|---|----------|--------|----------|-------|
| 1 | {title} | PASS/FAIL/SKIP | {gas} | {비고} |

### Summary
- **Total**: {N} scenarios
- **Passed**: {P}
- **Failed**: {F}
- **Skipped**: {S}
- **Total Gas Cost**: ~${total_usd}

### Failed Scenarios
{실패한 시나리오별 상세: 어떤 단계에서 실패, 에러 메시지, Troubleshooting 참조}
```

4. **지갑 선택 규칙 상세:**
   - `environment: ethereum` 지갑은 `ethereum-*`, `polygon-*`, `arbitrum-*`, `base-*` 네트워크에 사용
   - `environment: solana` 지갑은 `solana-*` 네트워크에 사용
   - 같은 환경에 여러 지갑이 있으면 라벨/잔액 정보와 함께 사용자에게 선택 요청
   - testnet 시나리오는 testnet 지갑, mainnet 시나리오는 mainnet 지갑 사용 (네트워크 ID로 구분: `-sepolia`, `-devnet` = testnet)

5. **dry-run 상세 규칙:**
   - dry-run이 실패(success: false)하면 사용자에게 알리고 계속 진행할지 확인
   - dry-run 예상 비용이 시나리오의 `estimated_cost_usd`보다 2배 이상이면 경고 표시
   - 가스비가 0인 시나리오(Solana 기본 전송 등)는 dry-run 결과를 정보 차원에서만 표시

6. **에러 처리:**
   - 401 Unauthorized: 세션 토큰 만료 → 갱신 안내
   - 403 Forbidden: 정책 위반 → 해당 정책 확인 안내
   - 500 Internal Server Error: 데몬 상태 확인 안내
   - 네트워크 타임아웃: RPC 상태 확인, 재시도 안내
