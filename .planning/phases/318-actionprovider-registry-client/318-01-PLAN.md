    1. **erc8004-registry-client.ts**: viem의 `encodeFunctionData`를 사용하여 3개 레지스트리의 calldata를 인코딩하는 클래스.

    ```typescript
    import { encodeFunctionData, type Hex } from 'viem';
    ```

    클래스 구조:
    - constructor(config: Erc8004Config) -- 레지스트리 주소를 config에서 가져옴
    - getRegistryAddress(registry: 'identity' | 'reputation' | 'validation'): Hex -- validation은 빈 문자열이면 ChainError('INVALID_INSTRUCTION') throw ("Validation Registry not configured")
    - Identity encode 메서드: encodeRegister, encodeRegisterWithMetadata, encodeSetAgentWallet, encodeUnsetAgentWallet, encodeSetAgentURI, encodeSetMetadata
    - Reputation encode 메서드: encodeGiveFeedback, encodeRevokeFeedback
    - Validation encode 메서드: encodeValidationRequest

    모든 encode* 메서드는 `encodeFunctionData({ abi, functionName, args })` 패턴 사용. bigint 변환은 이 레이어에서 수행 (agentId: string -> BigInt(agentId)).

    agentId string -> BigInt 변환 시 예외 처리: try-catch로 감싸고 실패 시 ChainError('INVALID_INPUT', 'ethereum', { message: 'Invalid agentId: must be a numeric string' }).

    setMetadata의 value(string)는 `toHex(value)` (viem)로 bytes 변환. feedbackHash는 그대로 bytes32로 전달 (호출자가 keccak256 결과를 제공).

    읽기 메서드는 이 계획에서 구현하지 않음 -- Phase 319(REST API)와 Phase 320(Cache)에서 PublicClient를 주입받아 구현. 이 계획에서는 calldata 인코딩만 담당.
