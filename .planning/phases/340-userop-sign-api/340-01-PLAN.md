    - Method: POST, path: '/wallets/{id}/userop/sign'
    - Request body: UserOpSignRequestSchema
    - Response 200: UserOpSignResponseSchema
    - Error responses: BUILD_NOT_FOUND, EXPIRED_BUILD, BUILD_ALREADY_USED, CALLDATA_MISMATCH, SENDER_MISMATCH, ACTION_VALIDATION_FAILED, POLICY_DENIED

    **4. Implement sign handler** in the userOpRoutes factory (app.openapi(signUserOpRoute, ...)):

    Step 1: Load wallet, validate smart+evm (same as build handler).

    Step 2: Parse request body (buildId + userOperation).

    Step 3: Look up build record from userop_builds by buildId.
    - Not found -> throw WAIaaSError('BUILD_NOT_FOUND')
    - `build.walletId !== walletId` -> throw WAIaaSError('BUILD_NOT_FOUND') (prevent cross-wallet access)
    - `build.used === 1` -> throw WAIaaSError('BUILD_ALREADY_USED')
    - `build.expiresAt` is a Date -- compare `Math.floor(build.expiresAt.getTime() / 1000) < Math.floor(Date.now() / 1000)` -> throw WAIaaSError('EXPIRED_BUILD')

    Step 4: Validate callData matches (SIGN-04).
    - Compare `userOperation.callData` with `build.callData` (case-insensitive hex comparison using `.toLowerCase()`)
    - Mismatch -> throw WAIaaSError('CALLDATA_MISMATCH')

    Step 5: Validate sender matches wallet address (SIGN-05).
    - Resolve RPC, create SmartAccount (same pattern as build) to get the predicted address
    - Compare `userOperation.sender.toLowerCase()` with `smartAccountInfo.address.toLowerCase()`
    - Mismatch -> throw WAIaaSError('SENDER_MISMATCH')

    Step 6: Policy evaluation (SIGN-06) -- follow sign-only.ts pattern.
    - Use `buildUserOpCalls(request_from_build, wallet.publicKey)` -- BUT we don't store the original request in DB.
    - Alternative: Skip buildUserOpCalls re-parsing. Instead, parse callData using adapter if available, OR skip complex policy re-evaluation and just check basic policies.
    - SIMPLEST APPROACH per SIGN-06 ("INSTANT tier만 허용"): Use the policy engine to evaluate a CONTRACT_CALL param with the callData's target info. Since UserOp callData is SmartAccount.encodeCalls output (not raw tx), we can evaluate as a CONTRACT_CALL to the sender (self-call to smart account).
    - Create a single TransactionParam: `{ type: 'CONTRACT_CALL', amount: '0', toAddress: userOperation.sender, chain: 'ethereum', network, contractAddress: userOperation.sender }`.
    - Call `policyEngine.evaluate(walletId, txParam)`. If denied, throw POLICY_DENIED. If tier is DELAY or APPROVAL, throw POLICY_DENIED with message "UserOp sign does not support {tier} tier".
    - If policyEngine is not provided (no deps.policyEngine), skip policy check (matches existing pattern where deps are optional).

    Step 7: Sign the UserOp (SIGN-07).
    - Decrypt signer key (same pattern as build handler).
    - Create SmartAccount (already done in Step 5, reuse).
    - Call `smartAccountInfo.account.signUserOperation({ ...userOpForSign })` where userOpForSign is the UserOp object with all fields.
    - The viem SmartAccount.signUserOperation() method takes a UserOperation and returns the signature hex.
    - NOTE: viem's signUserOperation takes specific fields. Construct the call properly:
      ```
      const signature = await smartAccountInfo.account.signUserOperation({
        callData: userOperation.callData as Hex,
        callGasLimit: BigInt(userOperation.callGasLimit),
        maxFeePerGas: BigInt(userOperation.maxFeePerGas),
        maxPriorityFeePerGas: BigInt(userOperation.maxPriorityFeePerGas),
        nonce: BigInt(userOperation.nonce),
        preVerificationGas: BigInt(userOperation.preVerificationGas),
        verificationGasLimit: BigInt(userOperation.verificationGasLimit),
        sender: userOperation.sender as Address,
        factory: userOperation.factory as Address | undefined,
        factoryData: userOperation.factoryData as Hex | undefined,
        paymaster: userOperation.paymaster as Address | undefined,
        paymasterData: userOperation.paymasterData as Hex | undefined,
        paymasterPostOpGasLimit: userOperation.paymasterPostOpGasLimit ? BigInt(userOperation.paymasterPostOpGasLimit) : undefined,
        paymasterVerificationGasLimit: userOperation.paymasterVerificationGasLimit ? BigInt(userOperation.paymasterVerificationGasLimit) : undefined,
        signature: '0x' as Hex,  // placeholder, will be replaced
      });
      ```

    Step 8: Release key (SIGN-08) -- in finally block (same pattern as build).

    Step 9: Mark build as used -- `UPDATE userop_builds SET used = 1 WHERE id = buildId`.

    Step 10: Insert transaction record (SIGN-09).
    - `INSERT INTO transactions: id=generateId(), walletId, chain='ethereum', network, type='SIGN', status='SIGNED', sessionId, createdAt, executedAt`

    Step 11: Audit log (SIGN-10).
    - `insertAuditLog(deps.sqlite, { eventType: 'USEROP_SIGNED', actor: 'session', walletId, txId, details: { buildId, sender: userOperation.sender, network }, severity: 'info' })`

    Step 12: Notification (NTFY-03, NTFY-04).
    - `void deps.notificationService?.notify('TX_SUBMITTED', walletId, { txHash: '', amount: '0', to: userOperation.sender, display_amount: '', network }, { txId, signOnly: true })`
    - `deps.eventBus?.emit('wallet:activity', { walletId, activity: 'TX_SUBMITTED', details: { txId, userOpSign: true, buildId }, timestamp: Math.floor(Date.now() / 1000) })`

    Step 13: Return response.
    - `{ signedUserOperation: { ...userOperation, signature: signatureHex }, txId }`

    **5. Wire sign route in server.ts** if not already wired (userOpRoutes already registered, new route auto-included since it's in the same factory).
