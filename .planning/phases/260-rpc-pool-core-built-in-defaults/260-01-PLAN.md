<feature>
  <name>RpcPool -- priority-based RPC URL rotation with cooldown</name>
  <files>packages/core/src/rpc/rpc-pool.ts, packages/core/src/__tests__/rpc-pool.test.ts</files>
  <behavior>
    RpcPool manages multiple RPC URLs per network with priority-based fallback:

    **Registration:**
    - `register(network, urls)` — registers ordered URL list (index 0 = highest priority)
    - `registerAll(entries)` — bulk registration for multiple networks
    - Duplicate URLs for same network are silently ignored

    **URL Resolution:**
    - `getUrl(network)` — returns highest-priority non-cooldown URL
    - If primary is in cooldown, returns next available
    - Throws `AllRpcFailedError` if all URLs for that network are in cooldown
    - Throws if network has no registered URLs

    **Failure Reporting + Cooldown:**
    - `reportFailure(network, url)` — marks URL as failed, applies cooldown
    - First failure: 60s cooldown (configurable via `baseCooldownMs`)
    - Consecutive failures: exponential increase (60s → 120s → 240s)
    - Maximum cooldown: 300s (5 min, configurable via `maxCooldownMs`)
    - Cooldown check uses `Date.now()` comparison (injectable for testing)
    - After cooldown expires, URL automatically available again (no explicit recovery needed)

    **Reset:**
    - `reportSuccess(network, url)` — resets failure count for URL (consecutive count back to 0)
    - `reset(network)` — clears all cooldown state for a network
    - `resetAll()` — clears all state

    **Inspection:**
    - `getStatus(network)` — returns all URLs with their status (available/cooldown, failure count, cooldown remaining)
    - `getNetworks()` — returns list of registered networks
    - `hasNetwork(network)` — boolean check

    **Constructor:**
    - `new RpcPool(options?)` where options: `{ baseCooldownMs?: number, maxCooldownMs?: number, nowFn?: () => number }`
    - Defaults: baseCooldownMs=60_000, maxCooldownMs=300_000, nowFn=Date.now

    **AllRpcFailedError:**
    - Extends Error (not ChainError -- this is infrastructure level, not chain-specific)
    - Properties: `network: string`, `urls: string[]`, `name: 'AllRpcFailedError'`

    Test cases (input -> expected):
    - register('mainnet', ['a','b','c']) → getUrl('mainnet') === 'a'
    - reportFailure('mainnet', 'a') → getUrl('mainnet') === 'b'
    - reportFailure('mainnet', 'a'), reportFailure('mainnet', 'b'), reportFailure('mainnet', 'c') → getUrl throws AllRpcFailedError
    - After cooldown expires → getUrl returns recovered URL
    - 1st failure cooldown=60s, 2nd=120s, 3rd=240s, 4th=300s (capped)
    - reportSuccess('mainnet', 'a') → consecutive count resets to 0
    - register('mainnet', ['a','a','b']) → only ['a','b'] stored (dedup)
    - getUrl('unknown-network') → throws Error
  </behavior>
  <implementation>
    1. Create `packages/core/src/rpc/rpc-pool.ts`:
       - `RpcPoolOptions` interface with baseCooldownMs, maxCooldownMs, nowFn
       - `AllRpcFailedError` class extending Error
       - `RpcEndpointState` internal type: { url, failureCount, cooldownUntil }
       - `RpcPool` class with Map<string, RpcEndpointState[]> keyed by network
       - getUrl: iterate entries in priority order, skip if nowFn() < cooldownUntil, return first available
       - reportFailure: find entry, increment failureCount, compute cooldown = min(base * 2^(failures-1), max), set cooldownUntil = now + cooldown
       - reportSuccess: find entry, reset failureCount to 0, clear cooldownUntil
       - getStatus: map entries to status objects with computed remaining cooldown

    2. Create `packages/core/src/rpc/index.ts`:
       - Barrel export: RpcPool, AllRpcFailedError, RpcPoolOptions, RpcEndpointStatus

    3. Update `packages/core/src/index.ts`:
       - Add `export * from './rpc/index.js';`

    4. Do NOT modify ChainError or error codes — AllRpcFailedError is a standalone Error subclass in the rpc module, not a ChainError.
  </implementation>
