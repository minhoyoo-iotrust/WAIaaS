<task type="auto" tdd="true">
  <name>Task 2: MCP 도구 등록 시 typed schema 적용 + fallback</name>
  <files>packages/mcp/src/tools/action-provider.ts, packages/mcp/src/__tests__/action-provider-schema.test.ts</files>
  <behavior>
    - Test 1: REST API에서 받은 inputSchema JSON이 있으면 MCP 도구의 params에 jsonSchema 기반 Zod 스키마가 사용된다
    - Test 2: inputSchema가 없거나 빈 객체이면 기존 z.record(z.unknown()) fallback이 유지된다
    - Test 3: inputSchema JSON에 properties가 있으면 각 property가 MCP 도구 스키마에 반영된다
  </behavior>
  <action>
    1. `packages/mcp/src/tools/action-provider.ts`의 ActionDefinitionResponse 인터페이스에 `inputSchema?: Record<string, unknown>` 필드를 추가한다.

    2. registerActionProviderTools에서 MCP tool 등록 시:
       - action.inputSchema가 존재하고 properties가 있으면 → JSON Schema의 properties를 개별 z.string()/z.number() 등으로 변환하여 params 대신 직접 사용
       - 변환이 불가능하거나 inputSchema가 없으면 → 기존 `params: z.record(z.unknown())` 유지

       구현 방식: JSON Schema properties를 순회하며 Zod 스키마를 동적으로 구성하는 `jsonSchemaToZodParams()` 헬퍼 함수를 만든다. JSON Schema type -> Zod 타입 매핑:
       - "string" -> z.string()
       - "number"/"integer" -> z.number()
       - "boolean" -> z.boolean()
       - "array" -> z.array(z.unknown())
       - 기타 -> z.unknown()
       각 필드에 description이 있으면 .describe() 적용, required 배열에 없으면 .optional() 적용.

       변환된 params는 `params` wrapper 없이 직접 MCP tool schema에 포함한다. 즉, 기존 `{ params: z.record(), network, wallet_id, gas_condition }` 대신 `{ inputField1, inputField2, ..., network, wallet_id, gas_condition }`으로 등록하되, 핸들러에서는 input fields를 다시 params 객체로 묶어 REST API에 전달한다.

       **중요**: server.tool()의 schema 인자에 inputSchema의 properties를 개별 필드로 풀어넣되, network/wallet_id/gas_condition은 유지한다. 핸들러에서는 network/wallet_id/gas_condition을 제외한 나머지 args를 params 객체로 묶어 POST body에 전달한다.

    3. fallback 안전망: try-catch로 전체 변환 과정을 감싸고, 실패 시 기존 z.record(z.unknown()) 패턴으로 등록한다. console.error로 변환 실패를 로깅한다.

    4. 테스트 파일 신규 생성: `packages/mcp/src/__tests__/action-provider-schema.test.ts`
       - jsonSchemaToZodParams 헬퍼의 단위 테스트 (string/number/boolean/array/optional 매핑)
       - inputSchema 있을 때 typed params 사용 확인
       - inputSchema 없을 때 fallback 확인
  </action>
