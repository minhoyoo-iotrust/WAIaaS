Create `packages/actions/src/providers/aave-v3/index.ts`:

1. Imports:
   ```typescript
   import { z } from 'zod';
   import { ChainError } from '@waiaas/core';
   import type {
     ILendingProvider, ActionProviderMetadata, ActionDefinition, ActionContext,
     ContractCallRequest, LendingPositionSummary, HealthFactor, MarketInfo,
     PositionUpdate, PositionCategory,
   } from '@waiaas/core';
   import type { IPositionProvider } from '@waiaas/core';
   import type { AaveV3Config } from './config.js';
   import { AAVE_V3_DEFAULTS, getAaveAddresses } from './config.js';
   import {
     encodeSupplyCalldata, encodeBorrowCalldata, encodeRepayCalldata,
     encodeWithdrawCalldata, encodeApproveCalldata,
     encodeGetUserAccountDataCalldata, MAX_UINT256,
   } from './aave-contracts.js';
   import {
     type IRpcCaller, decodeGetUserAccountData, simulateHealthFactor,
     decodeGetReserveData, rayToApy, hfToNumber, LIQUIDATION_THRESHOLD_HF,
   } from './aave-rpc.js';
   import { AaveSupplyInputSchema, AaveBorrowInputSchema, AaveRepayInputSchema, AaveWithdrawInputSchema } from './schemas.js';
   ```

2. Define `parseTokenAmount(amount: string, decimals: number = 18): bigint`:
   - Same pattern as Lido parseEthAmount but parameterized decimals
   - Split on '.', compute whole * 10^decimals + fractional part padded/sliced to decimals
   - Throw ChainError if result <= 0n

3. Define and export `AaveV3LendingProvider` class implementing both ILendingProvider and IPositionProvider:

   Constructor: `constructor(config?: Partial<AaveV3Config>, rpcCaller?: IRpcCaller)`
   - Merge defaults: `this.config = { ...AAVE_V3_DEFAULTS, ...config }`
   - Store rpcCaller (optional, used only for query methods)

   `metadata: ActionProviderMetadata`:
   ```typescript
   {
     name: 'aave_v3',
     description: 'Aave V3 DeFi lending protocol for EVM chains: supply, borrow, repay, withdraw',
     version: '1.0.0',
     chains: ['ethereum'],
     mcpExpose: true,
     requiresApiKey: false,
     requiredApis: [],
   }
   ```

   `actions: readonly ActionDefinition[]` -- 4 entries:
   - `{ name: 'aave_supply', description: 'Supply (deposit) an ERC-20 token as collateral to Aave V3 lending pool', chain: 'ethereum', inputSchema: AaveSupplyInputSchema, riskLevel: 'medium', defaultTier: 'DELAY' }`
   - `{ name: 'aave_borrow', description: 'Borrow an asset from Aave V3 lending pool against deposited collateral (variable rate)', chain: 'ethereum', inputSchema: AaveBorrowInputSchema, riskLevel: 'high', defaultTier: 'APPROVAL' }`
   - `{ name: 'aave_repay', description: 'Repay borrowed debt on Aave V3 lending pool. Use amount="max" for full repayment.', chain: 'ethereum', inputSchema: AaveRepayInputSchema, riskLevel: 'medium', defaultTier: 'DELAY' }`
   - `{ name: 'aave_withdraw', description: 'Withdraw supplied collateral from Aave V3 lending pool. Use amount="max" for full withdrawal.', chain: 'ethereum', inputSchema: AaveWithdrawInputSchema, riskLevel: 'high', defaultTier: 'APPROVAL' }`

   NOTE: borrow and withdraw are 'high' risk + 'APPROVAL' tier because they can trigger liquidation.

4. Implement `resolve(actionName: string, params: Record<string, unknown>, context: ActionContext)`:
   - Switch on actionName: 'aave_supply', 'aave_borrow', 'aave_repay', 'aave_withdraw'
   - For unknown action: throw ChainError('INVALID_INSTRUCTION', 'ethereum', { message: `Unknown action: ${actionName}` })

5. Private `resolveSupply(params, context)` -> ContractCallRequest[]:
   - Parse: `AaveSupplyInputSchema.parse(params)`
   - Resolve network: `input.network || 'ethereum-mainnet'`
   - Get addresses: `getAaveAddresses(network)`
   - Parse amount: `parseTokenAmount(input.amount)` (default 18 decimals -- token decimal lookup deferred to Phase 277)
   - Build [approveReq, supplyReq]:
     - approveReq: type='CONTRACT_CALL', to=input.asset, calldata=encodeApproveCalldata(poolAddress, amount), value='0'
     - supplyReq: type='CONTRACT_CALL', to=poolAddress, calldata=encodeSupplyCalldata(asset, amount, walletAddress), value='0'
   - Return array (AAVE-05: multi-step approve + supply)

6. Private `resolveBorrow(params, context)` -> ContractCallRequest:
   - Parse, resolve network, get addresses, parse amount
   - AAVE-09 HF simulation check: If rpcCaller is available, call getUserAccountData and simulateHealthFactor. If simulated HF < LIQUIDATION_THRESHOLD_HF (1.0), throw ChainError('POLICY_VIOLATION', 'ethereum', { message: 'Borrow would cause health factor to drop below liquidation threshold' }).
   - If no rpcCaller (action-only mode), skip simulation (defer to daemon-level policy).
   - Build single ContractCallRequest: type='CONTRACT_CALL', to=poolAddress, calldata=encodeBorrowCalldata(asset, amount, walletAddress), value='0'
   - Return single element (no approve needed for borrow -- Pool releases tokens TO user)

7. Private `resolveRepay(params, context)` -> ContractCallRequest[]:
   - Parse with AaveRepayInputSchema
   - If input.amount === 'max': use MAX_UINT256 for amount
   - Else: parseTokenAmount(input.amount)
   - Build [approveReq, repayReq]:
     - approveReq: to=asset, calldata=encodeApproveCalldata(poolAddress, amount), value='0'
     - repayReq: to=poolAddress, calldata=encodeRepayCalldata(asset, amount, walletAddress), value='0'
   - For 'max' approve, approve MAX_UINT256 too (Pitfall 6: repay MUST have approve)

8. Private `resolveWithdraw(params, context)` -> ContractCallRequest:
   - Parse with AaveWithdrawInputSchema
   - If input.amount === 'max': use MAX_UINT256
   - Else: parseTokenAmount(input.amount)
   - AAVE-09: If rpcCaller available and amount != MAX_UINT256, simulate HF after withdrawal. If simulated HF < LIQUIDATION_THRESHOLD_HF, throw.
   - Build single ContractCallRequest: to=poolAddress, calldata=encodeWithdrawCalldata(asset, amount, walletAddress), value='0'
   - Return single (no approve needed -- Pool releases tokens TO user)

9. ILendingProvider query methods -- implement with rpcCaller:
   - `async getPosition(walletId: string, context: ActionContext): Promise<LendingPositionSummary[]>`:
     - If no rpcCaller, return empty array
     - Determine network from context (use 'ethereum-mainnet' if not resolvable)
     - Call rpcCaller with getUserAccountData calldata targeting Pool address
     - Decode response, build LendingPositionSummary array with collateral and debt entries
   - `async getHealthFactor(walletId: string, context: ActionContext): Promise<HealthFactor>`:
     - If no rpcCaller, return default safe value
     - Call getUserAccountData via rpcCaller
     - Decode, classify status: safe(>=2.0), warning(>=1.5), danger(>=1.2), critical(<1.2)
     - Return HealthFactor object with factor (Number), totalCollateralUsd, totalDebtUsd, currentLtv, status
   - `async getMarkets(chain: string, network?: string): Promise<MarketInfo[]>`:
     - For now, return empty array (market enumeration requires getReservesList + per-reserve queries, deferred to Phase 277 API integration)

10. IPositionProvider methods:
    - `async getPositions(walletId: string): Promise<PositionUpdate[]>`: Return empty array (requires RPC context not available in this interface; PositionTracker will use ILendingProvider.getPosition via adapter in Phase 277)
    - `getProviderName(): string`: return 'aave_v3'
    - `getSupportedCategories(): PositionCategory[]`: return ['LENDING']

IMPORTANT NOTES:
- All resolve methods return ContractCallRequest objects without actionProvider or actionName fields -- the ActionProviderRegistry auto-tags these (per 275-03 summary).
- The provider does NOT sign or submit transactions -- only produces calldata.
- The 'max' literal handling for repay/withdraw is critical (Pitfall 2).
