   - `queue = new Map<string, PositionUpsert>()` where `PositionUpsert` is an internal interface with all defi_positions columns (id, wallet_id, category, provider, chain, network, asset_id, amount, amount_usd, metadata, status, opened_at, closed_at, last_synced_at, created_at, updated_at)
   - `enqueue(update: PositionUpdate): void` method:
     - Composite key: `${update.walletId}:${update.provider}:${update.assetId ?? 'null'}:${update.category}`
     - Convert PositionUpdate to PositionUpsert: generate `id` via `generateId()`, set `last_synced_at` and `created_at` and `updated_at` to `Math.floor(Date.now() / 1000)` (seconds, CLAUDE.md convention), serialize `metadata` to JSON string

     - Return 0 if queue empty
     - Extract up to MAX_BATCH items from queue, delete them from map

ON CONFLICT(wallet_id, provider, asset_id, category) DO UPDATE SET amount=excluded.amount, amount_usd=excluded.amount_usd, metadata=excluded.metadata, status=excluded.status, closed_at=excluded.closed_at, last_synced_at=excluded.last_synced_at, updated_at=excluded.updated_at`
     - Use `sqlite.transaction((items) => { for item: stmt.run(item values) })` with `.immediate()` for SQLITE_BUSY prevention

   ```typescript
   /**
    * PositionTracker: Periodic DeFi position synchronization service.
    *
    * Maintains per-category timers to sync positions from registered
    * IPositionProvider implementations into the defi_positions table
    * via PositionWriteQueue.
    *
    * Features:
    *   - Per-category intervals (LENDING=5min, PERP=1min, STAKING=15min, YIELD=1h)
    *   - Overlap prevention per category (running flag)
    *   - Per-wallet error isolation (one wallet failure does not block others)
    *   - On-demand syncCategory() for urgent refresh (e.g., HealthFactorMonitor)
    *   - Dynamic provider registration/unregistration
    *
    * Design source: m29-00 design doc section 6.2.
    * Pattern: BalanceMonitorService + IncomingTxQueue hybrid.
    */
   import type { Database } from 'better-sqlite3';
   import type { IPositionProvider, PositionCategory } from '@waiaas/core';
   import { POSITION_CATEGORIES } from '@waiaas/core';
   import { PositionWriteQueue } from './position-write-queue.js';
   import type { SettingsService } from '../../infrastructure/settings/settings-service.js';
   ```

   Implementation:
   - `PositionTrackerConfig` exported interface: `{ enabled: boolean }`
   - `DEFAULT_INTERVALS: Record<PositionCategory, number>` = `{ LENDING: 300_000, PERP: 60_000, STAKING: 900_000, YIELD: 3_600_000 }`
   - Constructor takes `{ sqlite: Database, settingsService?: SettingsService }`:
     - Store sqlite and settingsService
     - Create `PositionWriteQueue` instance
     - `providers = new Map<string, IPositionProvider>()`
     - `timers = new Map<PositionCategory, NodeJS.Timeout>()`
     - `running = new Map<PositionCategory, boolean>()`
   - `registerProvider(provider: IPositionProvider): void`:
     - `this.providers.set(provider.getProviderName(), provider)`
   - `unregisterProvider(name: string): void`:
     - `this.providers.delete(name)`
   - `start(): void`:
     - For each category in POSITION_CATEGORIES:
       - `running.set(category, false)`
       - `const intervalMs = DEFAULT_INTERVALS[category]`
       - `const timer = setInterval(() => void this.syncCategory(category), intervalMs)`
       - `timer.unref()`
       - `timers.set(category, timer)`
     - Immediate first sync for LENDING: `void this.syncCategory('LENDING')`
   - `stop(): void`:
     - For each timer: `clearInterval(timer)`
     - `timers.clear()`
     - `running.clear()`
   - `async syncCategory(category: PositionCategory): Promise<void>`:
     - Overlap check: `if (this.running.get(category)) return`
     - `this.running.set(category, true)`
     - try:
       - Get all providers for this category: `this.getProvidersForCategory(category)`
       - For each provider: get active wallets from DB (raw SQL: `SELECT id FROM wallets WHERE status='ACTIVE'` -- simplified, provider doesn't filter by chain yet since that's Phase 276 concern)
       - For each wallet: `try { const positions = await provider.getPositions(walletId); for (const pos of positions) this.writeQueue.enqueue(pos); } catch (err) { console.warn(...); }` -- per-wallet error isolation
       - After all wallets synced: `this.writeQueue.flush(this.sqlite)`
     - finally: `this.running.set(category, false)`
   - `private getProvidersForCategory(category: PositionCategory): IPositionProvider[]`:
     - Filter providers whose `getSupportedCategories()` includes `category`
   - `get providerCount(): number` returns providers.size
   - `get queueSize(): number` returns writeQueue.size
