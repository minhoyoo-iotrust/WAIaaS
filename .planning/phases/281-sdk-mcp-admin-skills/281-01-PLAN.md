# Plan 281-01: SDK + CLI + Python SDK Default Value Reference Removal

## Goal
Remove all default wallet/default network references from the TypeScript SDK (`@waiaas/sdk`), CLI (`@waiaas/cli`), and Python SDK (`python-sdk`). After this plan, no SDK/CLI consumer can use `setDefaultNetwork()`, `defaultWalletId`, or `defaultNetwork` fields.

## Context

Phase 280 deleted the daemon API endpoints (PUT /v1/wallet/default-network, PUT /v1/wallets/:id/default-network) and removed `defaultNetwork`/`isDefault` from all API responses. The SDK/CLI/Python SDK still has client-side types and methods that reference these deleted endpoints and fields. These are now dead code that must be removed.

**Key changes from Phases 279-280:**
- `PUT /v1/wallet/default-network` endpoint deleted (280-02)
- `PUT /v1/wallets/:id/default-network` endpoint deleted (280-02)
- `defaultWalletId` removed from `CreateSessionRequestSchema` (279-01)
- `defaultNetwork`/`isDefault` removed from all API response schemas (280-02)
- `WalletNetworkInfo.isDefault` removed from OpenAPI schemas (280-02)
- Connect-info no longer has `defaultNetwork`/`isDefault` per wallet (280-02)
