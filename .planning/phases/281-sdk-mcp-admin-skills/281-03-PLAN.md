# Plan 281-03: Admin UI Default Wallet/Network UI Removal + Skill File Update

## Goal
Remove all default wallet/default network UI elements from the Admin Web UI and update all skill files to remove references to deleted features and reflect the new explicit-specification model.

## Context

Phase 280 removed the server-side APIs and settings. The Admin UI still renders Default Network display, "Set as Default" buttons, evm_default_network settings form, and session creation with defaultWalletId. These are now non-functional (the API endpoints were deleted) and must be removed from the UI. Additionally, skill files (AI agent documentation) must be updated per CLAUDE.md rule: "When REST API, SDK, or MCP interfaces change, skills/ files must be updated accordingly."

**Key context from Phases 279-280:**
- `PUT /v1/wallets/:id/default-network` endpoint deleted (404)
- `WALLET_DEFAULT_NETWORK` API constant targets a deleted endpoint
- API responses no longer include `isDefault` or `defaultNetwork` fields
- `rpc.evm_default_network` setting key deleted from server
- Session creation no longer accepts `defaultWalletId` parameter
- MCP tools reduced from 25 to 24 (set_default_network deleted in Plan 281-02)
