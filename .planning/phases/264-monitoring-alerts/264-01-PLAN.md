1. In `packages/core/src/rpc/rpc-pool.ts`:

   a. Add a `RpcPoolEvent` type above the class:
   ```typescript
   /** Event emitted by RpcPool when endpoint state changes. */
   export interface RpcPoolEvent {
     type: 'RPC_HEALTH_DEGRADED' | 'RPC_ALL_FAILED' | 'RPC_RECOVERED';
     network: string;
     url: string;
     failureCount: number;
     /** Total registered endpoints for this network. */
     totalEndpoints: number;
   }
   ```

   b. Add `onEvent` to `RpcPoolOptions`:
   ```typescript
   /** Optional callback invoked when endpoint state changes (cooldown/all-failed/recovered). */
   onEvent?: (event: RpcPoolEvent) => void;
   ```

   c. Store `onEvent` as a private field on the class (`private readonly onEvent?`).

   d. In `reportFailure()`, AFTER setting cooldownUntil:
      - Emit `RPC_HEALTH_DEGRADED` event (this endpoint entered cooldown).
      - Then check if ALL endpoints for this network are in cooldown (iterate entries, check `now < entry.cooldownUntil` for all). If so, emit `RPC_ALL_FAILED`.
      - Use `this.onEvent?.({ type, network, url, failureCount, totalEndpoints })`.

   e. In `reportSuccess()`, BEFORE resetting failureCount/cooldownUntil:
      - Check if the entry WAS in cooldown (entry.cooldownUntil > now AND entry.failureCount > 0). If so, emit `RPC_RECOVERED` after resetting.
      - This represents a transition from cooldown -> available.

2. In `packages/core/src/__tests__/rpc-pool.test.ts`:

   Add a new `describe('onEvent callback')` block with tests:

   a. `emits RPC_HEALTH_DEGRADED when endpoint enters cooldown`:
      - Create pool with onEvent spy, register network with 2 URLs, reportFailure on URL A.
      - Verify onEvent called with type 'RPC_HEALTH_DEGRADED', correct network, url, failureCount: 1, totalEndpoints: 2.

   b. `emits RPC_ALL_FAILED when all endpoints are in cooldown`:
      - Create pool with onEvent spy, register network with 2 URLs, reportFailure on both URLs.
      - Verify onEvent called twice for RPC_HEALTH_DEGRADED (one per URL), then once for RPC_ALL_FAILED on the second failure.

   c. `emits RPC_RECOVERED when cooldown endpoint recovers via reportSuccess`:
      - Create pool with onEvent spy, register network with 2 URLs, reportFailure on URL A (enters cooldown), then reportSuccess on URL A.
      - Verify onEvent called with type 'RPC_RECOVERED'.

   d. `does not emit RPC_RECOVERED if endpoint was not in cooldown`:
      - Create pool with onEvent spy, reportSuccess on a healthy URL.
      - Verify onEvent NOT called with type 'RPC_RECOVERED'.

   e. `does not emit events when onEvent not provided`:
      - Create pool without onEvent, reportFailure. Verify no errors (no-op callback).
