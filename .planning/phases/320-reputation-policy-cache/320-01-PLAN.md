  <behavior>
    - getReputation(agentId, tag1, tag2) returns cached score from in-memory Map when cache entry exists and is fresh (cachedAt + TTL > now)
    - getReputation falls back to DB reputation_cache when in-memory cache misses
    - getReputation falls back to RPC (viem readContract getSummary) when both memory and DB miss
    - getReputation returns null when RPC fails or times out (caller handles unrated treatment)
    - RPC call has configurable timeout (default 3000ms), aborted via AbortController
    - Successful RPC result is stored in both in-memory Map and DB reputation_cache
    - DB fallback result is promoted to in-memory Map on read
    - invalidate(agentId) removes all entries for that agentId from both in-memory Map and DB
    - TTL is configurable via SettingsService (actions.erc8004_reputation_cache_ttl_sec)
  </behavior>

    1. Create `packages/daemon/src/services/erc8004/reputation-cache-service.ts`:

    ```typescript
    export interface ReputationScore {
      score: number;        // Normalized 0-100 (summaryValue / 10^decimals scaled)
      rawScore: string;     // Original int128 as string
      decimals: number;     // summaryValueDecimals
      count: number;        // feedback count
      cachedAt: number;     // Unix timestamp (seconds)
    }
    ```

    Class `ReputationCacheService`:
    - Constructor takes: `db: BetterSQLite3Database<typeof schema>`, `settingsService?: SettingsService`
    - Private: `cache: Map<string, ReputationScore>` (in-memory tier)
    - Private helper: `cacheKey(agentId: string, registryAddress: string, tag1: string, tag2: string): string` -> `${agentId}:${registryAddress}:${tag1}:${tag2}`
    - Private helper: `isFresh(entry: ReputationScore): boolean` -> check cachedAt + TTL > now (seconds)
    - Private helper: `getTtl(): number` -> read `actions.erc8004_reputation_cache_ttl_sec` from settings or default 300
    - Private helper: `getRpcTimeout(): number` -> read `actions.erc8004_reputation_rpc_timeout_ms` from settings or default 3000
    - Private helper: `getRegistryAddress(): string` -> read `actions.erc8004_reputation_registry_address` from settings or default
    - Private helper: `normalizeScore(rawScore: bigint, decimals: number): number` -> Convert int128 score to 0-100 float. If decimals=0, clamp to [0,100]. If decimals>0, divide by 10^decimals, clamp to [0,100].

    Method `async getReputation(agentId: string, tag1 = '', tag2 = ''): Promise<ReputationScore | null>`:
    - Step 1: Check in-memory Map with cacheKey
    - Step 2: If miss, check DB reputation_cache table (Drizzle `and(eq(...))`)
    - Step 3: If miss, call RPC with timeout via AbortController
    - Step 4: Store result in both Map and DB on success
    - Step 5: Return null on any RPC failure (timeout, network error, etc.)

    For RPC call:
    - Use `createPublicClient({ transport: http(rpcUrl, { timeout: rpcTimeout }) })` from viem
    - Read `rpc.evm_ethereum_mainnet` from settings for RPC URL (fallback: https://eth.llamarpc.com)
    - Call `readContract({ address, abi: REPUTATION_REGISTRY_ABI, functionName: 'getSummary', args: [BigInt(agentId), [], tag1, tag2] })`
    - Parse result: `[count, summaryValue, summaryValueDecimals]`

    Method `invalidate(agentId: string): void`:
    - Remove all keys starting with `${agentId}:` from in-memory Map
    - DELETE FROM reputation_cache WHERE agent_id = agentId (Drizzle eq)

    Method `invalidateAll(): void`:
    - Clear entire in-memory Map
    - Do NOT clear DB (DB serves as persistent fallback)
