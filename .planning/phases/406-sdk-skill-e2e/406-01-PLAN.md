    - Test 3: sendToken({ to, humanAmount: "1.5", type: "APPROVE", spender, token }) passes validation

    2. **validation.ts**: `validateAmount()` 함수를 `validateAmountOrHumanAmount()` 로 교체:

    **1. transactions.skill.md** — "## 1. Overview" 바로 뒤에 "## Amount Units" 섹션 추가:
    - 규칙: REST API amount는 smallest unit(wei/lamports). humanAmount로 human-readable 형식 사용 가능.
    - humanAmount 우선 예시: `{ "to": "0x...", "humanAmount": "1.5", "type": "TRANSFER" }` (1.5 ETH)
    - amount 대안 예시: `{ "to": "0x...", "amount": "1500000000000000000", "type": "TRANSFER" }` (same 1.5 ETH)
    - XOR 규칙: amount와 humanAmount 동시 지정 불가
    - TOKEN_TRANSFER 예시: humanAmount + token.decimals로 서버가 자동 변환
    - CLOB 예외: Hyperliquid/Drift/Polymarket은 exchange-native units 사용 (humanAmount 미지원)

    **2. actions.skill.md** — "## 1. List Action Providers" 바로 앞에 "## Amount Units for Actions" 섹션 추가:
    - 규칙: Action provider params에서 humanAmount+decimals로 human-readable 금액 지정 가능
    - 예시: Jupiter swap에 `{ "params": { "humanAmount": "1.5", "decimals": 18, ... } }`
    - per-provider 변형 목록: humanAmount(대부분), humanSellAmount(0x), humanAmountIn(Pendle), humanFromAmount(LI.FI)
    - CLOB 예외 동일

    **3. wallet.skill.md** — 파일 상단 설명 근처에 간략한 참조 추가:

    **4. quickstart.skill.md** — 첫 번째 전송 예시 섹션에 humanAmount 가이드 추가:
