    **wc.ts (8건):** `(db as any).session?.client` 8건을 타입 안전한 접근으로 교체한다.
    - Drizzle BetterSQLite3Database는 `.$client` 프로퍼티로 raw better-sqlite3 Database를 노출한다 (v0.38+, 현재 v0.45 사용).
    - 파일 상단에 헬퍼 함수를 만든다: `function getSqliteClient(db: BetterSQLite3Database<typeof schema>): import('better-sqlite3').Database { return db.$client; }`
    - 8곳의 `(db as any).session?.client as import('better-sqlite3').Database`를 `getSqliteClient(db)`로 교체한다.
    - 만약 `.$client`가 타입 에러를 발생시키면, `(db as unknown as { $client: import('better-sqlite3').Database }).$client` 패턴을 사용하되 as any는 절대 사용하지 않는다. 단, 이 경우 주석으로 이유를 명시한다.

    **daemon.ts (4건):**
    - `(server as any).keepAliveTimeout = 600_000` / `(server as any).headersTimeout = 605_000` 2건: Node.js http.Server 타입에는 이 속성이 있다. `import { Server } from 'node:http'`로 타입을 가져오고, `const httpServer = server as Server;` 후 `httpServer.keepAliveTimeout = 600_000;` 식으로 교체한다. (Hono의 serve()가 반환하는 server는 실제로 http.Server이다.)
    - `policyEngine: null as any` 2건: PipelineContext에서 policyEngine이 IPolicyEngine 타입이므로, 해당 위치에서 실제로 policy 평가가 불필요한 맥락 (stages 5-6)에서는 null-object 패턴을 사용한다. `const NULL_POLICY_ENGINE: IPolicyEngine = { evaluate: async () => ({ allowed: true, tier: 'INSTANT' as const }) };` 헬퍼를 만들고 `policyEngine: NULL_POLICY_ENGINE`으로 교체한다. PolicyEvaluation 타입에 맞춰 반환값을 구성한다.

    **wc-session-service.ts (4건):**
    - L27 `(SignClientModule as any).default ?? SignClientModule`: ESM default interop. `const SignClient = (SignClientModule as { default?: typeof SignClientModule }).default ?? SignClientModule;` 패턴으로 교체한다.
    - L109 `storage: this.storage as any`: WalletConnect IKeyValueStorage 호환. `@walletconnect/types`에서 IKeyValueStorage를 import하고, storage 클래스가 이를 implements하도록 하거나 `as IKeyValueStorage`로 좁힌다.
    - L259 `Object.values(session.namespaces)[0] as any`: namespace 구조를 타입으로 좁힌다. `@walletconnect/types`의 `SessionTypes.Namespace`로 단언하거나, 필요한 속성만 optional chaining으로 접근한다.
    - L427 `(this.signClient.core?.relayer?.provider as any)?.disconnect?.()`: WC provider 타입이 disconnect를 노출하지 않는 경우, `as { disconnect?: () => Promise<void> }`로 좁힌다.

    **eip712-signer.ts (3건):**
    - L26-29 `domain as any`, `types as any`, `message: p.value as any`: viem의 `signTypedData` 타입과 매칭시킨다. viem의 `TypedDataDomain`, `TypedDataParameter` 타입을 import하여 명시적 타입 단언 (`as TypedDataDomain`)으로 교체하거나, EIP-712 파라미터 타입을 정의하여 사용한다. viem의 generic이 너무 엄격한 경우 `as const` 또는 satisfies 패턴을 시도한다. 최후의 수단으로 `// @ts-expect-error viem EIP-712 generics require literal types` 주석과 함께 교체 가능.
