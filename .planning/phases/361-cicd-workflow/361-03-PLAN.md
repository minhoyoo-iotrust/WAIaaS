e2e-smoke.yml의 e2e-smoke job 마지막에 실패 시 알림 step을 추가한다.

**Step 추가 (기존 summary step 다음):**

```yaml
      - name: Create issue on failure
        if: failure()
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          VERSION="${{ steps.version.outputs.version }}"
          RUN_URL="${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}"
          BODY="## E2E Smoke Test Failure

          **Version:** ${VERSION}
          **Run:** ${RUN_URL}
          **Triggered by:** ${{ github.event_name }}

          E2E smoke tests failed for version ${VERSION}. Check the workflow run for details.
          "
          # Check for existing open issue to avoid duplicates
          EXISTING=$(gh issue list --label "e2e-failure" --state open --limit 1 --json number -q '.[0].number' 2>/dev/null || echo "")
          if [ -n "$EXISTING" ]; then
            gh issue comment "$EXISTING" --body "E2E failure recurrence for version ${VERSION}. Run: ${RUN_URL}"
          else
            gh issue create \
              --title "E2E Smoke Test Failure: ${VERSION}" \
              --body "$BODY" \
              --label "e2e-failure,bug"
          fi
```

**permissions 추가:** workflow 최상단의 permissions에 `issues: write` 추가 (gh issue create 권한).

**`e2e-failure` label 생성:** 워크플로우에서 자동 생성되지 않으므로, step 앞에 label 확인/생성 step 추가:
```yaml
      - name: Ensure e2e-failure label exists
        if: failure()
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          gh label create "e2e-failure" --color "d73a49" --description "E2E smoke test failure" 2>/dev/null || true
```

**중복 방지 로직:**
- 동일 label의 open issue가 있으면 comment 추가 (새 issue 생성하지 않음)
- 이전 이슈가 close된 상태면 새 issue 생성
