<objective>
Convert WalletNotificationChannel from wallet-instance-based topic routing (waiaas-notify-{wallet.name}) to app-based topic routing (waiaas-notify-{wallet_apps.name}). Notifications are published to each app with alerts_enabled=1, skipping apps with alerts_enabled=0, and skipping entirely when no alert-enabled apps exist.

Purpose: Fulfill NOTI-01/02/03 -- notifications go to app-level topics controlled by the Alerts toggle in Human Wallet Apps, not to per-wallet-instance topics.

Output: Modified WalletNotificationChannel with app-based routing + comprehensive tests.
</objective>

**wallet-notification-channel.ts -- Replace wallet-based routing with app-based routing:**

1. **Replace `resolveTargetWallets` method entirely.** The current method queries `wallets` table for `owner_approval_method = 'sdk_ntfy'`. Replace with a new method `resolveAlertApps()` that queries `wallet_apps` table for `alerts_enabled = 1`:

```typescript
private resolveAlertApps(): string[] {
  const rows = this.sqlite.prepare(
    'SELECT name FROM wallet_apps WHERE alerts_enabled = 1',
  ).all() as Array<{ name: string }>;
  return rows.map((r) => r.name);
}
```

2. **Rewrite `notify()` method's target resolution section.** Replace the existing `resolveTargetWallets(walletId)` call and the per-wallet publish loop. Instead:
   - Call `resolveAlertApps()` to get list of alert-enabled app names
   - If the list is empty, return early (NOTI-03: skip when 0 apps)
   - For each app name, publish to `waiaas-notify-{appName}` topic
   - The `NotificationMessage.walletId` and `NotificationMessage.walletName` fields should use the original `walletId` parameter and the app name respectively. Since notifications are now app-scoped (not wallet-scoped), `walletName` in the message becomes the app name.

3. **Keep all existing gates** (signing_sdk.enabled, notifications_enabled, event filter) unchanged -- they remain as-is above the target resolution.

4. **`publishNotification` method stays the same** -- it already takes a `walletName` parameter used to build the topic `waiaas-notify-{walletName}`. Now it receives the app name instead.

5. **Remove `UUID_REGEX` constant** (no longer needed -- we no longer distinguish UUID vs non-UUID walletId for routing).

6. **Updated `notify()` flow:**
```
Gates (enabled, notifications_enabled, event filter) → resolveAlertApps()
  → empty? return (NOTI-03)
  → for each appName: publishNotification(ntfyServer, appName, message, priority)
```

7. The `NotificationMessage` published for each app should contain:
   - `walletId`: the original walletId parameter (unchanged)
   - `walletName`: the app name (was wallet.name, now app.name)
   - All other fields (eventType, category, title, body, details, timestamp) unchanged

**daemon.ts -- No constructor change needed.** The `WalletNotificationChannel` constructor still takes `{sqlite, settingsService}` -- the sqlite connection is sufficient to query `wallet_apps` directly. No need to inject WalletAppService since we use a simple SQL query. Keep lines 808-814 unchanged.
