1. **hyperliquid.ts (REST routes)**: HyperliquidRouteDeps에 subAccountService를 추가한다. 단, Sub-account 관리는 키 복호화가 필요하므로(createSubAccount, transfer는 signUserSignedAction 호출), REST route에서 keyStore + masterPassword를 통해 privateKey를 복호화해야 한다. 이를 위해 deps에 `keyStore`와 `getMasterPassword` 함수를 추가한다.

   그러나 기존 패턴을 보면 action 도구는 파이프라인(/v1/actions/:provider/:action)을 통해 실행되고 REST query routes는 read-only이다. Sub-account **생성**과 **이체**는 쓰기 작업이므로 파이프라인을 거치는 것이 일관적이다.

   **결정**: Sub-account create/transfer는 HyperliquidSubAccountProvider를 IActionProvider로 등록하여 파이프라인을 거치도록 한다 (기존 perp/spot 패턴과 동일). Query(list, getPositions)만 REST GET endpoint로 추가한다.

   **대안 접근**: Sub-account 생성/이체를 위한 HyperliquidSubAccountProvider를 별도로 만들지 않고, 기존 sub-account-service.ts를 감싸는 thin IActionProvider wrapper를 만든다. 이 Provider는:
   - name: 'hyperliquid_sub'
   - requiresSigningKey: true
   - actions: hl_create_sub_account (riskLevel: medium, defaultTier: DELAY, spending $0), hl_sub_transfer (riskLevel: medium, defaultTier: DELAY, spending = transfer amount)
   - resolve(): context.privateKey로 service 메서드 호출 후 ApiDirectResult 반환

   **그러나** 설계를 재검토하면: HDESIGN-04에서 "Sub-account 작업은 계정 관리" + HDESIGN-07에서 "hl_sub_transfer는 riskLevel=medium, hl_create_sub_account는 riskLevel=medium"으로 정책 적용이 필요하다. 따라서 IActionProvider로 등록하는 것이 올바르다.

   **최종 결정**: 351-01에서 만든 sub-account-service.ts는 핵심 로직만 담당하고, 여기서 HyperliquidSubAccountProvider를 만들어 IActionProvider로 등록한다. 이렇게 하면 기존 /v1/actions/hyperliquid_sub/hl_create_sub_account POST로 파이프라인을 거친다.

   사실 이 접근은 Plan 01에서 처리했어야 하지만 Plan 01은 서비스 로직에만 집중했으므로, 이 Task에서 thin provider wrapper + REST query routes + provider 등록까지 수행한다.

   REST query routes 추가:
   - `GET /v1/wallets/:walletId/hyperliquid/sub-accounts` -> service.listSubAccounts(walletAddress)
   - `GET /v1/wallets/:walletId/hyperliquid/sub-accounts/:subAddress/positions` -> service.getSubAccountPositions(walletAddress, subAddress)

       { name: 'hl_create_sub_account', description: 'Create a new Hyperliquid Sub-account', schema: HlCreateSubAccountInputSchema, riskLevel: 'medium', defaultTier: 'DELAY', spending: () => ({ amount: 0n, asset: 'USDC' }) },
       { name: 'hl_sub_transfer', description: 'Transfer USDC between Master and Sub-account', schema: HlSubTransferInputSchema, riskLevel: 'medium', defaultTier: 'DELAY', spending: (params) => ({ amount: BigInt(Math.ceil(parseFloat(params.amount) * 1e6)), asset: 'USDC' }) },
     ]
   - resolve(actionName, params, context): switch on actionName, call service method with context.privateKey, return ApiDirectResult
   - 파일: packages/actions/src/providers/hyperliquid/sub-account-provider.ts
