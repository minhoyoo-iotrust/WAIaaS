2. `async pollAll(): Promise<{ polled: number; skipped: number; errors: number }>`
   Implement DEFI-04 ASNC-02 pollAll logic exactly:

   a. Query DB for tracking targets:
      ```sql
      SELECT * FROM transactions
      WHERE bridge_status IN ('PENDING', 'BRIDGE_MONITORING')
         OR status = 'GAS_WAITING'
      ```
      Use Drizzle ORM:
      ```typescript
      const targets = this.db.select().from(transactions).where(
        or(
          inArray(transactions.bridgeStatus, ['PENDING', 'BRIDGE_MONITORING']),
          eq(transactions.status, 'GAS_WAITING'),
        ),
      ).all();
      ```

   b. Sequential processing (no Promise.all -- external API rate limit risk):
      ```typescript
      let polled = 0, skipped = 0, errors = 0;
      for (const tx of targets) {
        try {
          // Resolve tracker name
          const trackerName = this.resolveTrackerName(tx);
          const tracker = this.trackers.get(trackerName);
          if (!tracker) { skipped++; continue; }

          // Per-tracker timing check
          const metadata = JSON.parse(tx.bridgeMetadata ?? '{}');
          const lastPolled = (metadata.lastPolledAt as number) ?? 0;
          const now = Date.now();
          if (now - lastPolled < tracker.pollIntervalMs) { skipped++; continue; }

          // MaxAttempts check
          const pollCount = ((metadata.pollCount as number) ?? 0) + 1;
          if (pollCount > tracker.maxAttempts) {
            await this.handleTimeout(tx, tracker, metadata);
            polled++;
            continue;
          }

          // checkStatus call
          const result = await tracker.checkStatus(tx.id, metadata);

          // Process result
          await this.processResult(tx, tracker, result, {
            ...metadata,
            pollCount,
            lastPolledAt: now,
          });
          polled++;
        } catch (err) {
          // Error isolation: log and continue
          console.error(`AsyncPolling error for tx ${tx.id}:`, err);
          errors++;
        }
      }
      return { polled, skipped, errors };
      ```

3. `private resolveTrackerName(tx): string`
   - If `tx.status === 'GAS_WAITING'` return `'gas-condition'`
   - Otherwise parse `tx.bridgeMetadata` JSON and return `metadata.tracker ?? 'bridge'`

4. `private async handleTimeout(tx, tracker, metadata): Promise<void>`
   Based on `tracker.timeoutTransition`:
   - `'BRIDGE_MONITORING'`: Update `bridge_status = 'BRIDGE_MONITORING'` in DB, reset `pollCount = 0` in bridge_metadata (축소 폴링 시작). The BridgeMonitoringTracker (a separate tracker with 5-min interval) will pick it up.
     Actually, per DEFI-04 design: BRIDGE_MONITORING is a *bridge_status* value, and the same BridgeStatusTracker should detect BRIDGE_MONITORING targets and use different timing. But the design says "축소 폴링(5분)" happens after BRIDGE_MONITORING transition.
     Simpler: The AsyncPollingService already does per-tracker timing via lastPolledAt+pollIntervalMs. So for BRIDGE_MONITORING, we could either:
     a. Register a second tracker with name='bridge-monitoring' and pollIntervalMs=300_000, or
     b. Have BridgeStatusTracker return `nextIntervalOverride` to indicate 5-min polling.
     Per the design: "BridgeMonitoringTracker가 5분 간격으로 계속 폴링" suggests a separate tracker concept. But the simpler approach matching the ASNC-02 design is: the same AsyncPollingService detects bridge_status='BRIDGE_MONITORING' transactions and looks for a tracker named by `metadata.tracker` (which would still be 'bridge'). The BridgeStatusTracker handles both PENDING and BRIDGE_MONITORING, but for BRIDGE_MONITORING it has different maxAttempts.
     For v23 (this phase), just implement the timeout transition mechanism. The actual tracker registration is Phase 252's responsibility.
     Implementation: Update DB `bridge_status` to tracker.timeoutTransition value. Update bridge_metadata with `{ ...metadata, pollCount: 0, lastPolledAt: Date.now(), transitionedAt: Date.now() }` to reset the counter for the next stage.

   - `'TIMEOUT'`: Update `bridge_status = 'TIMEOUT'` in DB. No further polling.
   - `'CANCELLED'`: Update `transactions.status = 'CANCELLED'` in DB.

   Use Drizzle update:
   ```typescript
   if (tracker.timeoutTransition === 'CANCELLED') {
     this.db.update(transactions)
       .set({ status: 'CANCELLED', metadata: JSON.stringify({ ...existingMeta, cancelReason: 'timeout' }) })
       .where(eq(transactions.id, tx.id))
       .run();
   } else {
     const newMeta = { ...metadata, pollCount: 0, lastPolledAt: Date.now(), transitionedAt: Date.now() };
     this.db.update(transactions)
       .set({
         bridgeStatus: tracker.timeoutTransition as BridgeStatus,
         bridgeMetadata: JSON.stringify(newMeta),
       })
       .where(eq(transactions.id, tx.id))
       .run();
   }
   ```

5. `private async processResult(tx, tracker, result: AsyncTrackingResult, metadata): Promise<void>`
   Based on `result.state`:
   - `'COMPLETED'`: Update `bridge_status = 'COMPLETED'`, `bridgeMetadata` with final details.
   - `'FAILED'`: Update `bridge_status = 'FAILED'`, `bridgeMetadata` with error details.
   - `'TIMEOUT'`: Delegate to `handleTimeout`.
   - `'PENDING'`: Update `bridgeMetadata` only (pollCount, lastPolledAt incremented).

   For COMPLETED/FAILED: notification and spending limit release will be handled by the specific tracker implementations (Phase 252) via EventBus or direct service calls. AsyncPollingService only handles DB state transitions.

   ```typescript
   const updatedMetadata = {
     ...metadata,
     ...(result.details ?? {}),
   };

   switch (result.state) {
     case 'COMPLETED':
       this.db.update(transactions)
         .set({ bridgeStatus: 'COMPLETED', bridgeMetadata: JSON.stringify(updatedMetadata) })
         .where(eq(transactions.id, tx.id)).run();
       break;
     case 'FAILED':
       this.db.update(transactions)
         .set({ bridgeStatus: 'FAILED', bridgeMetadata: JSON.stringify(updatedMetadata) })
         .where(eq(transactions.id, tx.id)).run();
       break;
     case 'TIMEOUT':
       await this.handleTimeout(tx, tracker, metadata);
       break;
     case 'PENDING':
     default:
       this.db.update(transactions)
         .set({ bridgeMetadata: JSON.stringify(updatedMetadata) })
         .where(eq(transactions.id, tx.id)).run();
       break;
   }
   ```
