**Section 23.5: MarginMonitor ↔ IPerpProvider 데이터 플로우**

Define the 5-stage data flow (same as section 20.1 MaturityMonitor pattern):
1. **DriftProvider.getPositions()** — queries Drift via SDK, returns PositionUpdate[] with PerpMetadata
2. **PositionTracker** — calls getPositions() on PERP polling interval (1 min), writes to defi_positions
3. **defi_positions table** — stores positions with metadata JSON containing margin/leverage/liquidationPrice
4. **MarginMonitor.evaluate()** — reads PERP category positions from DB, checks dual conditions:
   - Margin ratio check: `totalCollateral / usedMargin < maintenanceThreshold` (default 1.15)
   - Liquidation price proximity: `|currentPrice - liquidationPrice| / currentPrice < proximityThreshold` (default 0.10)
5. **DeFiMonitorService** — dispatches MARGIN_WARNING or LIQUIDATION_IMMINENT notification

```
DriftProvider.getPositions() → PositionTracker → defi_positions DB
                                                        ↓
                                    MarginMonitor.evaluate() reads
                                                        ↓
                                    MARGIN_WARNING / LIQUIDATION_IMMINENT
```

**Section 23.6: Drift V2 프로토콜 매핑**

Create mapping tables from 272-RESEARCH.md Drift Protocol Mapping Summary:

**Action mapping table:**

| IPerpProvider Action | Drift SDK Method | Parameters | Notes |
|---------------------|------------------|------------|-------|
| `open_position` | `driftClient.placePerpOrder(orderParams)` | orderType, marketIndex, direction, baseAssetAmount, price, auctionParams | No explicit "open" instruction — all via orders |
| `close_position` | `driftClient.closePosition(marketIndex)` | marketIndex (optional: partial via placePerpOrder with reducing direction) | Convenience method, or manual opposing order |
| `modify_position` | `driftClient.modifyPerpOrder(orderParams, orderId)` | Modified params + orderId. For position resize: place additional/reducing order | Atomic cancel+place |
| `add_margin` | `driftClient.deposit(amount, marketIndex, ata)` | amount (USDC precision), marketIndex=0 (USDC spot), associatedTokenAccount | Deposit USDC as collateral |
| `withdraw_margin` | `driftClient.withdraw(amount, marketIndex, ata)` | Same as deposit, withdrawal from account | Checks free collateral before allowing |

**Query mapping table:**

| IPerpProvider Query | Drift SDK Method | Notes |
|--------------------|------------------|-------|
| `getPosition(walletId)` | `user.getPerpPosition(marketIndex)` per market, loop all | Returns per-market position with baseAssetAmount, quoteAssetAmount, lastCumulativeFundingRate |
| `getMarginInfo(walletId)` | `user.getTotalCollateral()` + `user.getMarginRequirement('Maintenance')` + `user.getFreeCollateral()` + `user.getLeverage()` | All account-level aggregate values |
| `getMarkets(chain)` | `driftClient.getPerpMarketAccounts()` + `driftClient.getOracleDataForPerpMarket(idx)` | Returns all perp markets with oracle prices |

**Drift-specific implementation notes:**
- Precision constants: BASE_PRECISION (1e9), PRICE_PRECISION (1e6), MARGIN_PRECISION (1e4)
- Always use SDK conversion helpers: `convertToPerpPrecision()`, `convertToPricePrecision()`
- Market index 0 = SOL-PERP (mainnet), indices are env-dependent
- Cross-margin by default: all positions share one collateral pool
- Sub-account 0 only for v29-08 (up to 128 supported, future extensibility)
- DriftClient initialization requires Connection + Wallet + env parameter

**Drift order-based model abstraction:**
Document how IPerpProvider abstracts Drift's order-based model into position-based semantics:
- `open_position` → `placePerpOrder()` with target direction
- `close_position` → `closePosition(marketIndex)` (convenience) or opposing order
- `modify_position` → for order modification: `modifyPerpOrder()`. For position resize: additional/reducing order
- The interface hides the order-based model from AI agents — agents think in terms of positions, not orders
