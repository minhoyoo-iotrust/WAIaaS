  <action>
    최소한의 Tauri 2 프로젝트를 생성하여 @reown/appkit QR 페어링을 테스트한다.

    1. **Tauri 프로젝트 초기화**: `packages/desktop-spike/` 디렉토리에 최소 Tauri 2 프로젝트를 생성한다.
       - `pnpm create tauri-app` 대신 수동으로 최소 구조를 만든다 (모노레포 호환):
         - `package.json`: name `@waiaas/desktop-spike`, private true, devDependencies에 `@tauri-apps/cli@^2`, `@tauri-apps/api@^2`, `vite@^6`, `typescript@^5`
         - `vite.config.ts`: 기본 Vite config (host: true for Tauri dev)
         - `index.html`: 최소 HTML (div#app + script src main.ts)
         - `src-tauri/Cargo.toml`: tauri 2.x, tauri-build 2.x
         - `src-tauri/src/lib.rs`: 최소 Tauri run (IPC 커맨드 없음)
         - `src-tauri/tauri.conf.json`: 최소 config

    2. **@reown/appkit 설치 + 초기화**: dependencies에 `@reown/appkit@^1`, `@reown/appkit-adapter-solana@^1`, `@reown/appkit-adapter-wagmi@^1` 추가.
       - `src/main.ts`에서:
         - `createAppKit()` 호출: projectId는 환경변수 `VITE_WC_PROJECT_ID` 또는 하드코딩 placeholder
         - SolanaAdapter + Wagmi adapter 설정 (SIWS + SIWE 양쪽 테스트)
         - metadata: name 'WAIaaS Spike', description, url, icons
         - `<w3m-button>` Web Component를 DOM에 추가하여 모달 트리거 버튼 렌더링
         - 연결 이벤트 리스너 등록: `appKit.subscribeEvents()` 또는 `appKit.subscribeState()` 로 연결 상태, 주소, chainId 로그 출력
         - SIWS/SIWE 서명 요청 버튼: 연결 후 `signMessage()` 호출하여 "WAIaaS Owner Verification" 메시지 서명 테스트

    3. **CSP 설정** (doc-39 섹션 8.3 기반, per WCON-01):
       `tauri.conf.json`의 `app.security.csp`:
       ```
       default-src 'self'; script-src 'self' 'wasm-unsafe-eval'; style-src 'self' 'unsafe-inline'; connect-src 'self' wss://relay.walletconnect.com https://rpc.walletconnect.com https://pulse.walletconnect.org https://*.infura.io https://*.alchemy.com; img-src 'self' data: https:; font-src 'self' data:; frame-src https://secure.walletconnect.com
       ```

    4. **Capabilities 설정**: `src-tauri/capabilities/default.json`에 최소 permissions 설정.

    5. **중요 주의사항**:
       - @reown/appkit은 `lit` 기반 Web Components를 사용하므로 프레임워크 불필요 (vanilla TS로 충분)
       - Tauri 2의 `app.security.csp`가 WebView CSP를 주입하므로, meta 태그 CSP는 불필요
       - `app.windows[0].url`은 Vite dev server URL이 기본 (Tauri CLI가 자동 설정)
       - Reown Cloud에서 projectId 발급 필요: https://cloud.reown.com/ (무료 플랜)

    6. **projectId 처리**: 사용자가 Reown Cloud에서 projectId를 발급받아야 한다. `src/main.ts`에서 `VITE_WC_PROJECT_ID` 환경변수로 읽되, 없으면 콘솔에 안내 메시지를 출력하고 빈 문자열로 초기화 시도한다 (AppKit이 projectId 없이도 UI는 렌더링되므로 CSP/렌더링 검증은 가능).

  <how-to-verify>
    **사전 준비:**
    1. Reown Cloud(https://cloud.reown.com/)에서 프로젝트 생성 후 projectId를 발급받는다
    2. `packages/desktop-spike/.env` 파일에 `VITE_WC_PROJECT_ID=your_project_id` 설정

    1. `cd packages/desktop-spike && pnpm tauri dev` 실행하여 Tauri 앱 시작
    2. WebView에서 `<w3m-button>`(Connect Wallet 버튼)이 렌더링되는지 확인

    4. 브라우저 DevTools(Tauri가 열어주는 inspector)에서:
       - Console에 CSP 위반 에러가 없는지 확인
       - Network 탭에서 `wss://relay.walletconnect.com` WebSocket 연결이 성공하는지 확인
    5. 모바일 지갑(Phantom 또는 MetaMask)으로 QR 스캔하여 페어링 완료
    6. 페어링 성공 후, "Sign Message" 버튼 클릭하여 서명 요청이 지갑에 전달되는지 확인
    7. 서명 승인 후 서명값이 콘솔에 출력되는지 확인

    **실패 시 확인 사항:**
    - CSP 에러: tauri.conf.json CSP 수정 필요 (어떤 도메인이 차단되는지 기록)
    - WebSocket 연결 실패: WebKit/WebView2 WebSocket 제한 확인
    - QR 렌더링 실패: Web Component 등록 문제, lit 호환성 확인
    - 서명 실패: adapter 호환성, 에러 메시지 기록
  </how-to-verify>
