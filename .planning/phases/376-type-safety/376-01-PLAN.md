wallets.ts에서 Drizzle `select().from(wallets)` 결과를 사용하는 곳의 `as any` 캐스팅을 모두 제거한다. Drizzle가 schema로부터 타입을 자동 추론하므로 `wallet.accountType`, `wallet.signerKey`, `wallet.deployed`, `wallet.aaProvider`, `wallet.aaPaymasterUrl`, `wallet.factoryAddress`로 직접 접근 가능하다.

변경 대상 (24곳):
1. **listWalletsRoute** (line ~382-388): `(a as any).accountType` -> `a.accountType`, `(a as any).signerKey` -> `a.signerKey`, `(a as any).deployed` -> `a.deployed`, `(a as any).factoryAddress` -> `a.factoryAddress`, `(a as any).aaProvider` -> `a.aaProvider`, `(a as any).aaPaymasterUrl` -> `a.aaPaymasterUrl`
2. **walletDetailRoute** (line ~421-422, 466-473): `(wallet as any).accountType` -> `wallet.accountType`, `(wallet as any).factoryAddress` -> `wallet.factoryAddress`, `(wallet as any).signerKey` -> `wallet.signerKey`, `(wallet as any).deployed` -> `wallet.deployed`, `(wallet as any).aaProvider` -> `wallet.aaProvider`, `(wallet as any).aaPaymasterUrl` -> `wallet.aaPaymasterUrl`
3. **updateWalletRoute** (line ~738-743): 동일 패턴 제거
4. **providerRoute** (line ~1432): `(wallet as any).accountType` -> `wallet.accountType`

또한 **createWalletRoute** (line ~493-500)에서 `(parsed as any).accountType` 등의 패턴도 제거한다. `c.req.valid('json')`의 반환 타입이 OpenAPIHono에서 `CreateWalletRequestSchema`에 기반하므로 `accountType`, `aaProvider` 등이 타입에 포함되어 있다. 만약 OpenAPIHono의 타입 추론이 superRefine 때문에 해당 필드를 노출하지 않는 경우, `CreateWalletRequest` 타입으로 assertion 한다: `const parsed = c.req.valid('json') as CreateWalletRequest`. 이는 `as any`보다 훨씬 타입 안전하다.

주의사항:
- `?? 'eoa'`, `?? null`, `?? true` 등의 fallback은 그대로 유지 (runtime safety)
- `wallet.chain as ChainType` 등 기존 타입 narrowing은 변경하지 않음
- import 문 변경 불필요 (Drizzle 타입은 자동 추론)
