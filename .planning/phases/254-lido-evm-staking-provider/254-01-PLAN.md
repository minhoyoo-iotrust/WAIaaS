Create `packages/actions/src/providers/lido-staking/lido-contract.ts`:

1. `encodeSubmitCalldata(referral: string = '0x0000000000000000000000000000000000000000'): string`
   - Lido `submit(address _referral)` — function selector `0xa1903eab`
   - Encode referral as 32-byte padded address
   - Returns `0x` prefixed hex string

2. `encodeRequestWithdrawalsCalldata(amounts: bigint[], owner: string): string`
   - Lido WithdrawalQueue `requestWithdrawals(uint256[] _amounts, address _owner)` — function selector `0xd669a4e2`
   - Use manual ABI encoding:
     - offset to amounts array (64 bytes from start = 0x40)
     - owner address (32-byte padded)
     - array length
     - each amount as 32-byte uint256
   - Returns `0x` prefixed hex string

3. `encodeApproveCalldata(spender: string, amount: bigint): string`
   - Standard ERC-20 `approve(address,uint256)` — function selector `0x095ea7b3`
   - Same pattern as zerox-swap's encodeApproveCalldata but taking bigint
   - Returns `0x` prefixed hex string

All ABI encoding is manual (no viem dependency needed at the provider level). Use BigInt.toString(16).padStart(64, '0') for uint256 encoding. Function selectors are hardcoded hex constants derived from keccak256 of the function signatures.

1. Define input Zod schemas inline (following zerox-swap pattern):
   ```typescript
   const LidoStakeInputSchema = z.object({
     amount: z.string().min(1, 'amount is required (ETH, e.g. "1.0")'),
   });

   const LidoUnstakeInputSchema = z.object({
     amount: z.string().min(1, 'amount is required (stETH, e.g. "1.0")'),
   });
   ```

2. Implement `LidoStakingActionProvider implements IActionProvider`:
   - Constructor takes optional `Partial<LidoStakingConfig>`
   - `metadata`: name='lido_staking', description (Lido liquid staking ETH->stETH), version='1.0.0', chains=['ethereum'], mcpExpose=true, requiresApiKey=false, requiredApis=[]
   - `actions`: two ActionDefinition entries:
     - `stake`: "Stake ETH to receive stETH via Lido protocol (submit). Immediate, no lock-up.", chain='ethereum', inputSchema=LidoStakeInputSchema, riskLevel='medium', defaultTier='DELAY'
     - `unstake`: "Request stETH to ETH withdrawal via Lido Withdrawal Queue. Takes 1-5 days to finalize.", chain='ethereum', inputSchema=LidoUnstakeInputSchema, riskLevel='medium', defaultTier='DELAY'

3. `resolve()` method:
   - **stake**: Parse amount string, convert to wei (parseFloat(amount) * 1e18, handle as BigInt). Build single ContractCallRequest:
     ```typescript
     {
       type: 'CONTRACT_CALL',
       to: this.config.stethAddress,
       calldata: encodeSubmitCalldata(),  // submit(address(0)) referral
       value: amountWei.toString(),       // ETH value sent with tx
     }
     ```
     Return as single element (not array — registry normalizes).

   - **unstake**: Parse amount string, convert to wei. Build two ContractCallRequest elements (approve + requestWithdrawals):
     1. Approve stETH to WithdrawalQueue:
        ```typescript
        { type: 'CONTRACT_CALL', to: config.stethAddress, calldata: encodeApproveCalldata(config.withdrawalQueueAddress, amountWei), value: '0' }
        ```
     2. Request withdrawal:
        ```typescript
        { type: 'CONTRACT_CALL', to: config.withdrawalQueueAddress, calldata: encodeRequestWithdrawalsCalldata([amountWei], context.walletAddress), value: '0' }
        ```
     Return as array [approve, requestWithdrawals].

   - Unknown action: throw `new ChainError('INVALID_INSTRUCTION', 'ethereum', { message: ... })`

   - Amount parsing: use `parseEthAmount(amount: string): bigint` helper that does:
     ```typescript
     function parseEthAmount(amount: string): bigint {
       const parts = amount.split('.');
       const whole = BigInt(parts[0] || '0');
       const decimals = (parts[1] || '').padEnd(18, '0').slice(0, 18);
       return whole * 10n ** 18n + BigInt(decimals);
     }
     ```
     Validate that result > 0n, otherwise throw `new ChainError('INVALID_INSTRUCTION', 'ethereum', { message: 'Amount must be greater than 0' })`.
