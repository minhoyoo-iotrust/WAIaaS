    **Step 2: Design the extraction pattern**

    Since DaemonLifecycle is a single class with `this` state, use the "extract method to standalone function" pattern:
    - Each extracted function takes `self: DaemonLifecycle` (or the specific fields it needs) as parameter
    - DaemonLifecycle exposes internal state via a `DaemonInternals` type/interface that the extracted modules can access
    - OR: Use a simpler approach -- define the extracted functions as module-level functions that accept the DaemonLifecycle fields they need as a context object

    **Recommended approach**: Create a `DaemonState` interface in daemon.ts that exposes all the private fields as a mutable record. The startup/shutdown/pipeline functions accept `DaemonState` as first arg. daemon.ts class methods call through to the extracted functions passing `this` fields.

    ```typescript
    }
    ```

    The DaemonLifecycle class implements this interface internally and passes `this` to extracted functions.
