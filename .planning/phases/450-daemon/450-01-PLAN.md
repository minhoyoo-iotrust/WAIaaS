    1. Create `push-relay-signing-channel.ts` with class `PushRelaySigningChannel`:
       - Constructor takes `PushRelaySigningChannelOpts { signRequestBuilder, signResponseHandler, settingsService }`
       - `sendRequest(params: SendRequestParams)` method:
         a. Call `signRequestBuilder.buildRequest(params)` to get request + universalLinkUrl + requestTopic
         b. Call `signResponseHandler.registerRequest(request)`
         c. Extract `pushRelayUrl` from `request.responseChannel` (must be type 'push_relay')
         d. POST to `${pushRelayUrl}/v1/push` with body `{ subscriptionToken: requestTopic, category: 'sign_request', payload: { request: base64url(JSON.stringify(request)), universalLinkUrl } }`, header `X-Api-Key` from settings `signing_sdk.push_relay_api_key`
         e. If POST fails: log error with `console.error('[PushRelaySigningChannel] Failed to send request:', err)` but do NOT throw (transaction stays PENDING_APPROVAL per ERR-01)
         f. Start long-polling in background (non-blocking): `this.pollForResponse(pushRelayUrl, request.requestId, request.expiresAt)`
         g. Return `{ requestId, requestTopic, responseTopic: '' }`
       - `pollForResponse(pushRelayUrl, requestId, expiresAt)` private async method:
         a. Track with AbortController in `activeSubscriptions` Map (same pattern as NtfySigningChannel)
         b. Set expiration timer from expiresAt (unref'd)
         c. Loop up to MAX_POLL_ATTEMPTS (3):
           - GET `${pushRelayUrl}/v1/sign-response/${requestId}?timeout=30`
           - On 200: parse response body `{ response }`, base64url-decode, parse as SignResponse via SignResponseSchema, call `signResponseHandler.handle()`, cleanup subscription, return
           - On 204 (no response yet): continue polling (next iteration)
           - On error/non-2xx: exponential backoff delay (1s * 2^attempt), retry
         d. After max retries: cleanup subscription (request expires naturally via expiration timer)
       - `cancelSubscription(requestId)` and `shutdown()`: same pattern as NtfySigningChannel
       - Constants: `LONG_POLL_TIMEOUT_S = 30`, `MAX_POLL_ATTEMPTS = 3`, `BASE_BACKOFF_MS = 1000`
       - Note: The long-polling loop should continue polling as long as a 204 is received (no response yet). The MAX_POLL_ATTEMPTS (3) limit applies only to ERROR retries, not to 204 "try again" responses. On 204, immediately re-poll. On error, apply exponential backoff and retry up to 3 times.
