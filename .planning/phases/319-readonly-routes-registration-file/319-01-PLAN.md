    - GET /v1/erc8004/registration-file/:walletId for unregistered wallet returns basic file without registrations field
    - GET /v1/erc8004/validation/:requestHash returns 200 with {requestHash, validator, agentId, response, responseHash, tag, lastUpdate}
    - GET /v1/erc8004/validation/:requestHash with unconfigured Validation Registry returns 503 feature-gated error
    - All endpoints reject without session token (401)
  </behavior>
  <action>
    1. Add 4 Zod response schemas to `openapi-schemas.ts`:
       - `Erc8004AgentInfoResponseSchema`: agentId (string), wallet (string, address), uri (string), metadata (Record), registryAddress (string), chainId (number)
       - `Erc8004ReputationResponseSchema`: agentId (string), count (number), score (string, stringified int128), decimals (number), tag1 (string), tag2 (string)
       - `Erc8004RegistrationFileResponseSchema`: z.record(z.unknown()) (free-form JSON per ERC-8004 spec)
       - `Erc8004ValidationResponseSchema`: requestHash (string), validator (string), agentId (string), response (number, 0-255 uint8), responseHash (string), tag (string), lastUpdate (number)

    2. Create `packages/daemon/src/api/routes/erc8004.ts`:
       - `Erc8004RouteDeps` interface: db, settingsService, adapterPool (optional, for RPC URL resolution)
       - Helper `getErc8004Config(settingsService)`: reads 6 `actions.erc8004_*` settings, returns Erc8004Config. Use same pattern as ActionProvider config reading.
       - Helper `createRpcClient(settingsService)`: creates viem publicClient using RPC URL from settings (`rpc.evm_ethereum_mainnet` or similar). Use `createPublicClient({ transport: http(rpcUrl) })` same pattern as tokens.ts.

       Route 1: `GET /erc8004/agent/:agentId` (tags: ['ERC-8004'])
       - Read agentId from path param, convert to BigInt
       - Call `readContract` on Identity Registry: `tokenURI(agentId)` for URI, `getAgentWallet(agentId)` for wallet address
       - Also check DB `agent_identities` for local metadata (status, chainId)
       - Return merged on-chain + local data

       Route 2: `GET /erc8004/agent/:agentId/reputation` (tags: ['ERC-8004'])
       - Read agentId from path, optional query params tag1, tag2 (default '')
       - Call `readContract` on Reputation Registry: `getSummary(agentId, [], tag1, tag2)` (empty clientAddresses array for global summary)
       - Return {agentId, count: Number(count), score: String(summaryValue), decimals: Number(summaryValueDecimals), tag1, tag2}

       Route 3: `GET /erc8004/registration-file/:walletId` (tags: ['ERC-8004'])
       - Read walletId from path param
       - Query DB: `wallets` table for wallet name/chain, `agent_identities` for registration status
       - Build registration file using `buildRegistrationFile()` from @waiaas/actions with:
         - name from wallet.name
         - baseUrl from request Host header (protocol://host) or config registrationFileBaseUrl
         - If registered: include agentId, registryAddress, chainId from agent_identities
       - Return JSON directly (Content-Type: application/json)

       Route 4: `GET /erc8004/validation/:requestHash` (tags: ['ERC-8004'])
       - Check validationRegistryAddress from config -- if empty, return 503 with WAIaaSError('SERVICE_UNAVAILABLE', 'Validation Registry not configured')
       - Call `readContract` on Validation Registry: `getValidationStatus(requestHash)`
       - Return {requestHash, validator, agentId: String(agentId), response: Number(response), responseHash, tag, lastUpdate: Number(lastUpdate)}

       Route factory: `erc8004Routes(deps: Erc8004RouteDeps): OpenAPIHono`

    3. Add export to `index.ts`: `export { erc8004Routes, type Erc8004RouteDeps } from './erc8004.js';`

    4. Register in `server.ts`:
       - Add `sessionAuth` for `/v1/erc8004/*` paths (same block as other sessionAuth registrations)
       - Add `app.route('/v1', erc8004Routes({...}))` with db, settingsService deps
       - Place after connect-info registration, before OpenAPI spec endpoint

    Note: Use viem `createPublicClient` with `http()` transport for on-chain reads, same pattern as `packages/daemon/src/api/routes/tokens.ts`. RPC URL resolved from settingsService `rpc.evm_ethereum_mainnet` setting. Wrap all readContract calls in try/catch, throw WAIaaSError('CHAIN_ERROR') on RPC failure. Use `fromHex` to decode bytes metadata values from on-chain.
