# Plan 282-02: Existing Test Fix + Full Test Pass Verification

## Goal
Fix all existing test files that still reference removed default wallet/default network concepts (`defaultNetwork`, `isDefault`, `is_default`, `default_network`, `evm_default_network`), then verify the entire test suite passes.

## Context

Phases 279-281 removed:
- `defaultNetwork` from wallets Drizzle schema
- `isDefault` from session_wallets Drizzle schema (and the column itself)
- `default_network` from wallets DB column (migration v27)
- `is_default` from session_wallets DB column (migration v27)
- `wallet.defaultNetwork` from PipelineContext type (now `{ publicKey, chain, environment }`)
- `rpc.evm_default_network` from settings

52 test files still contain references to these removed concepts. Most are:
1. PipelineContext wallet objects with `defaultNetwork` field
2. Drizzle inserts with `defaultNetwork` field
3. Manual SQL CREATE TABLE with old column definitions
4. Extension test helpers accepting `defaultNetwork` parameter
5. Mock API responses with `isDefault` field

**Key distinction:** Migration chain tests (`migration-chain.test.ts`, `migration-v6-v8.test.ts`) and schema-compatibility tests that test **pre-migration** state MUST KEEP `default_network`/`is_default` references because they test the migration path FROM the old schema. Only tests for **current** schema behavior need updating.
