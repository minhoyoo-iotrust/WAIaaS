  <action>
    **wizard-store.ts** (per WIZA-01, WIZA-02, WIZA-03):
    Create Preact signals-based wizard state management:
    - `wizardStep` signal (1-5): current step number
    - `wizardComplete` signal (boolean): set true when wizard finishes, persisted to localStorage key `waiaas_setup_complete`
    - `isFirstRun()` function: returns `true` if (a) `isDesktop()` is true AND (b) localStorage `waiaas_setup_complete` is NOT `'true'`. This is the gate for showing the wizard.
    - `wizardData` signal: object accumulating user choices `{ password: string, chain: 'solana' | 'ethereum' | 'base' | 'polygon' | 'arbitrum', walletName: string, walletId: string | null, skipOwner: boolean }`
    - `nextStep()`: increment wizardStep (max 5)
    - `prevStep()`: decrement wizardStep (min 1)
    - `completeWizard()`: set localStorage `waiaas_setup_complete` = `'true'`, set `wizardComplete.value = true`, call `login(wizardData.value.password)` to auto-login after wizard
    - `skipOwnerStep()`: set `wizardData.value.skipOwner = true`, call `nextStep()` (per WIZA-02)

    **password-step.tsx** (Step 1):
    - Two password inputs (password + confirm) with match validation
    - Calls `PUT /v1/admin/master-password` to set password via fetch with the password in `X-Master-Password` header and body `{ password: <value> }`. On success, stores password in `wizardData.value.password` and calls `nextStep()`.
    - Min 8 chars validation. Show error if mismatch or too short.
    - Use project CSS variable styling (var(--color-*), var(--space-*), var(--radius-*)).

    **chain-step.tsx** (Step 2):
    - Radio button grid showing supported chains: Solana, Ethereum, Base, Polygon, Arbitrum
    - Each with chain icon/emoji and name. Pre-select Ethereum.
    - On "Next", store selected chain in `wizardData.value.chain` and call `nextStep()`.

    **wallet-step.tsx** (Step 3):
    - Text input for wallet name (default: "My Wallet")
    - On "Create Wallet", calls `POST /v1/wallets` with `{ name, chain }` using the typed API client (import `api` from `../../api/typed-client`). Must include `X-Master-Password` header from `wizardData.value.password`.
    - On success, stores `walletId` in `wizardData.value.walletId` and calls `nextStep()`.
    - Shows loading state during creation.

    **owner-step.tsx** (Step 4):
    - Title: "Connect Owner Wallet (Optional)"
    - Description explaining that Owner registration enables transaction approval from an external wallet
    - "Connect via WalletConnect" button (placeholder -- will be wired in Plan 03)
    - "Skip for now" link button that calls `skipOwnerStep()` (per WIZA-02)
    - Export a `setOwnerConnector` function that Plan 03 will call to inject the WC connect handler

    **complete-step.tsx** (Step 5):
    - Success checkmark icon and "Setup Complete!" message
    - Summary of what was configured (chain, wallet name)
    - "Go to Dashboard" button that calls `completeWizard()`

    All step components receive no props -- they read/write wizard-store signals directly. Use consistent card-style layout: centered card with max-width 480px, step indicator (dots or "Step N of 5"), prev/next navigation.
