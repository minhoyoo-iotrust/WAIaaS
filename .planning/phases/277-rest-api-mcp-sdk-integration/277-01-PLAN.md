**Step D: IRpcCaller injection in daemon lifecycle**

**Research conclusion (Blocker 3 resolved):**
- EvmAdapter has a private `_client: PublicClient` (viem) with no public accessor -- cannot use it directly
- AdapterPool.resolve() returns IChainAdapter (no raw RPC methods)
- RpcPool has `getUrl(network)` returning the highest-priority non-cooldown URL
- **Best approach:** Create `RpcPoolRpcCaller` that uses `RpcPool.getUrl()` to resolve the EVM RPC URL, then makes a direct `fetch` eth_call. This correctly leverages RPC Pool's priority-based rotation and cooldown, without needing to access EvmAdapter's private client.

```typescript
// Create IRpcCaller for Aave V3 using RpcPool eth_call
// RpcPool.getUrl(network) provides priority-based URL rotation with cooldown.
// Default network: 'ethereum-mainnet' (Aave V3 primary deployment).
import type { IRpcCaller } from '@waiaas/actions';

const rpcCaller: IRpcCaller | undefined = this.rpcPool ? (() => {
  const pool = this.rpcPool!;
  return {
    call: async (params: { to: string; data: string; chainId?: number }): Promise<string> => {
      // Map chainId to RpcPool network name
      // Default to ethereum-mainnet for Aave V3 primary chain
      const networkMap: Record<number, string> = {
        1: 'ethereum-mainnet',
        42161: 'arbitrum-mainnet',
        10: 'optimism-mainnet',
        137: 'polygon-mainnet',
        8453: 'base-mainnet',
      };
      const network = params.chainId ? (networkMap[params.chainId] ?? 'ethereum-mainnet') : 'ethereum-mainnet';
      const rpcUrl = pool.getUrl(network);

      const resp = await fetch(rpcUrl, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          jsonrpc: '2.0',
          id: 1,
          method: 'eth_call',
          params: [{ to: params.to, data: params.data }, 'latest'],
        }),
      });
      const json = await resp.json() as { result?: string; error?: { message: string } };
      if (json.error) throw new Error(json.error.message);
      return json.result ?? '0x';
    },
  };
})() : undefined;
```

Then pass it to registerBuiltInProviders:
```typescript
const builtIn = registerBuiltInProviders(this.actionProviderRegistry, this._settingsService!, { rpcCaller });
```

**Why this approach (not alternatives):**
- Cannot use EvmAdapter.client -- private field, no public accessor
- Cannot use AdapterPool.resolve() -- returns IChainAdapter which has no raw eth_call method
- Direct fetch + RpcPool.getUrl() correctly uses priority rotation, cooldown, and multi-endpoint failover
- The chainId-to-network mapping handles Aave V3's 5 supported chains
