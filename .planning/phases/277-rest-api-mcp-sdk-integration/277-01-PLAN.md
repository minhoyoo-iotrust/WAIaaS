<objective>
REST API endpoints for DeFi position queries and health factor, plus IRpcCaller injection into AaveV3LendingProvider.

Purpose: Enable AI agents and Admin UI to query DeFi lending positions and health factor through the daemon REST API. The IRpcCaller injection is essential -- without it, AaveV3LendingProvider returns empty/default values for all query methods.

Output:
- `defi-positions.ts` -- Two OpenAPIHono routes: GET /v1/wallet/positions (reads from defi_positions DB cache) and GET /v1/wallet/health-factor (live ILendingProvider query)
- OpenAPI Zod response schemas in openapi-schemas.ts
- IRpcCaller adapter in daemon lifecycle that delegates eth_call to AdapterPool EVM adapters
- Registration in server.ts (sessionAuth via /v1/wallet/* wildcard)
- Unit tests for both endpoints
</objective>
