## DeFi Positions

### Get DeFi Positions
- **Endpoint:** `GET /v1/wallet/positions`
- **Auth:** Session token (Bearer)
- **Query params:**
  - `wallet_id` (optional): Target wallet ID. Omit to use the default wallet.
- **Response:** List of DeFi positions with USD amounts
  - `walletId`: Wallet identifier
  - `positions[]`: Array of positions
    - `id`: Position ID
    - `category`: LENDING / YIELD / PERP / STAKING
    - `provider`: Provider name (e.g., "aave_v3")
    - `chain`: Blockchain (e.g., "ethereum")
    - `amount`: Raw amount string
    - `amountUsd`: USD equivalent (null if price unavailable)
    - `status`: ACTIVE / CLOSED / LIQUIDATED
  - `totalValueUsd`: Sum of all position USD values
- **MCP tool:** `waiaas_get_defi_positions`
- **SDK (TS):** `client.getPositions()`
- **SDK (Python):** `await client.get_positions()`

### Get Health Factor
- **Endpoint:** `GET /v1/wallet/health-factor`
- **Auth:** Session token (Bearer)
- **Query params:**
  - `wallet_id` (optional): Target wallet ID
  - `network` (optional): Target network for health factor query
- **Response:** Lending health factor with severity classification
  - `walletId`: Wallet identifier
  - `factor`: Health factor as decimal (e.g., 2.5)
  - `totalCollateralUsd`: Total collateral value in USD
  - `totalDebtUsd`: Total debt value in USD
  - `currentLtv`: Current loan-to-value ratio
  - `status`: `safe` / `warning` / `danger` / `critical`
- **MCP tool:** `waiaas_get_health_factor`
- **SDK (TS):** `client.getHealthFactor()`
- **SDK (Python):** `await client.get_health_factor()`
```

## Aave V3 Lending

Aave V3 DeFi lending provider for EVM chains. Supports supply, borrow, repay, and withdraw actions.

### Supported Chains
- Ethereum, Arbitrum, Optimism, Polygon, Base

### Enable
Set `actions.aave_v3_enabled` to `true` in Admin Settings or config.toml.

### Actions

#### Supply (aave_supply)
- **MCP tool:** `action_aave_v3_aave_supply`
- **SDK:** `client.executeAction('aave_v3', 'aave_supply', { params: { asset, amount } })`
- **Params:**
  - `asset`: Token address to supply
  - `amount`: Amount in human-readable format (e.g., "100.5")
  - `network` (optional): Target network

#### Borrow (aave_borrow)
- **MCP tool:** `action_aave_v3_aave_borrow`
- **SDK:** `client.executeAction('aave_v3', 'aave_borrow', { params: { asset, amount } })`
- **Params:**
  - `asset`: Token address to borrow
  - `amount`: Amount in human-readable format

#### Repay (aave_repay)
- **MCP tool:** `action_aave_v3_aave_repay`
- **SDK:** `client.executeAction('aave_v3', 'aave_repay', { params: { asset, amount } })`
- **Params:**
  - `asset`: Token address to repay
  - `amount`: Amount in human-readable format, or `"max"` for full repayment

#### Withdraw (aave_withdraw)
- **MCP tool:** `action_aave_v3_aave_withdraw`
- **SDK:** `client.executeAction('aave_v3', 'aave_withdraw', { params: { asset, amount } })`
- **Params:**
  - `asset`: Token address to withdraw
  - `amount`: Amount in human-readable format, or `"max"` for full withdrawal

### Safety
- Borrow/withdraw actions include health factor simulation to prevent self-liquidation
- Supply and repay are classified as non-spending (not counted toward spending limits)
- ERC-20 approve is automatically included for supply/repay actions
```
