    **config.ts changes:**
    Add `rpcUrl?: string` field to PendleConfig interface. No default value needed (optional).

    **index.ts changes — getPositions() implementation:**
    Replace the stub `return []` in `getPositions(walletId)` with:

    1. If `!this.config.rpcUrl`, return `[]` early (consistent with Lido pattern).
    2. Wrap in try/catch, return `[]` on error (resilient — position tracking should not crash daemon).
    3. For each supported chain in PENDLE_CHAIN_ID_MAP:
       a. Create PendleApiClient, call `getMarkets()` to get active markets.
       b. For each market, read PT and YT ERC-20 balanceOf via raw `fetch()` eth_call to `this.config.rpcUrl`.
       c. Skip zero-balance tokens.
       d. For non-zero balances, build PositionUpdate:
          - `category: 'YIELD'`
          - `provider: 'pendle'`
          - `chain: 'ethereum'` (derive from chainId)
          - `network`: derive from chain ID reverse lookup (e.g. `ethereum-mainnet`)
          - `assetId`: `formatCaip19('eip155:{chainId}', 'erc20', tokenAddress)`
          - `amount`: formatted from wei (18 decimals, same formatWei helper as Lido)
          - `amountUsd: null` (no simple on-chain oracle for Pendle PT/YT pricing)
          - `metadata`: `{ tokenType: 'PT'|'YT', maturity: unixTimestamp, underlyingAsset: symbol, impliedApy: number, marketAddress: string }`
          - `status`: if `maturity < now` then `'MATURED'`, else `'ACTIVE'`
          - `openedAt: now`

    **Important implementation notes:**
    - Use `encodeBalanceOfCalldata` pattern from lido-contract.ts. Since Pendle provider has no contract helpers yet, add a local `encodeBalanceOfCalldata(address)` and `decodeUint256Result(hex)` helper (same 4-byte selector `0x70a08231` + padded address). OR import from a shared location if one exists — check `@waiaas/core` exports first.
    - Only query `ethereum-mainnet` chain by default (Pendle's primary chain). Other chains (arbitrum, base) can query the same rpcUrl only if the config supports multi-chain RPC — for simplicity, start with ethereum-mainnet only per the existing `chains: ['ethereum']` in metadata.
    - Use `getPendleChainId('ethereum-mainnet')` to get chainId=1 for CAIP-19.
    - The PendleApiClient already has `getMarkets()` which returns all market data including PT/YT addresses, expiry, impliedApy, underlyingAsset.

    **Test file changes:**
    Add a new `describe('getPositions')` block to the existing test file. Mock both Pendle API (`/v1/markets/all`) and EVM RPC (eth_call for balanceOf) using MSW.

    For RPC mocking: MSW can intercept POST to an RPC URL. Use a mock RPC URL like `http://localhost:9999` and add MSW handler for POST requests that return balanceOf results based on the calldata.
  </action>
