1. OS Detection (DL-02):
   ```js
   function detectOS() {
     // Try modern API first
     if (navigator.userAgentData?.platform) {
       const p = navigator.userAgentData.platform.toLowerCase();
       if (p.includes('mac')) return 'macos';
       if (p.includes('win')) return 'windows';
       if (p.includes('linux')) return 'linux';
     }
     // Fallback to userAgent
     const ua = navigator.userAgent.toLowerCase();
     if (ua.includes('mac')) return 'macos';
     if (ua.includes('win')) return 'windows';
     if (ua.includes('linux')) return 'linux';
     return 'unknown';
   }
   ```

2. GitHub Releases API with 5-min TTL cache (DL-03/DL-04/DL-06):
   ```js
   const CACHE_KEY = 'waiaas_desktop_release';
   const CACHE_TTL = 5 * 60 * 1000; // 5 minutes

   async function fetchLatestRelease() {
     // Check cache first
     const cached = localStorage.getItem(CACHE_KEY);
     if (cached) {
       const { data, timestamp } = JSON.parse(cached);
       if (Date.now() - timestamp < CACHE_TTL) return data;
     }
     // Fetch from GitHub API -- filter desktop-v* releases
     const res = await fetch('https://api.github.com/repos/minhoyoo-iotrust/WAIaaS/releases');
     if (!res.ok) throw new Error(`GitHub API ${res.status}`);
     const releases = await res.json();
     const desktopRelease = releases.find(r => r.tag_name.startsWith('desktop-v') && !r.prerelease && !r.draft);
     if (!desktopRelease) throw new Error('No desktop release found');
     // Cache it
     localStorage.setItem(CACHE_KEY, JSON.stringify({ data: desktopRelease, timestamp: Date.now() }));
     return desktopRelease;
   }
   ```

3. Asset matching -- map OS to asset file patterns:
   - macos: prefer `_aarch64.dmg` (Apple Silicon primary), also list `_x64.dmg` (Intel)
   - windows: prefer `_x64-setup.exe`, also list `_x64_en-US.msi`
   - linux: prefer `_amd64.AppImage`, also list `_amd64.deb`

4. Render logic:
   - On load: detectOS() -> set primary CTA to detected OS download
   - fetchLatestRelease() -> populate version (tag_name minus "desktop-v" prefix), release date (published_at formatted as YYYY-MM-DD), all download links with file sizes (asset.size formatted as MB)
   - On API failure (DL-05): hide `.download-primary` and `.download-secondary`, show `.download-fallback` with text "GitHub Releases에서 직접 다운로드" linking to https://github.com/minhoyoo-iotrust/WAIaaS/releases?q=desktop-v

**No-JS fallback:** The `.download-fallback` section with GitHub Releases link is visible by default in HTML. JavaScript hides it and shows the dynamic content. This way, users with JS disabled still get a working page.
